7 はじめに

これまでに t検定 と MANOVA を扱ってきました。いずれも説明変数がカテゴリカル中心でしたが、実データでは 連続変数（例：語彙サイズ、適性）を含むことが一般的です。本章では、Notionノートの内容を基に、重回帰分析（線形モデル）と Simple Slope Analysis（連続×カテゴリカルの交互作用の下位検定）を、実行可能なRコードとともに示します。

ノート

想定データ：2つの学習方法（Glossing / Writing）、2つのテスト（Recall / Recognition）、2つの時点（Immediate / Delayed）、共変量として Proficiency（語彙サイズ）, Aptitude（記憶力/適性）。

8 データの準備

コードを表示 
# パッケージ
library(tidyverse)
library(performance)
library(effectsize)
library(emmeans)
library(sjPlot)
library(broom)
library(MASS)      # step() の周辺や今後の拡張で使用可
theme_set(theme_bw())

set.seed(123)

# パラメータ
n_participants <- 50
n_items        <- 30
treatments     <- c("Glossing", "Writing")
tests          <- c("Recall", "Recognition")
times          <- c("Immediate", "Delayed")

# 完全要因の展開
dat <- expand.grid(
  Participant = 1:n_participants,
  Item        = 1:n_items,
  Treatment   = treatments,
  Test        = tests,
  Time        = times,
  KEEP.OUT.ATTRS = FALSE
)

# アイテム割当（Glossing:1-15, Writing:16-30）
dat <- dat[
  (dat$Treatment == "Glossing" & dat$Item <= 15) |
  (dat$Treatment == "Writing"  & dat$Item >  15), ]

# 0/1反応（Writing > Glossing, Recognition > Recall）
dat$Score <- ifelse(
  dat$Treatment == "Glossing",
  rbinom(nrow(dat), 1, ifelse(dat$Test == "Recognition", 0.5, 0.2)),
  rbinom(nrow(dat), 1, ifelse(dat$Test == "Recognition", 0.9, 0.5))
)

# Immediate 効果を少し上げる（上限1でクリップ）
dat$Score <- ifelse(dat$Time == "Immediate",
                    pmin(dat$Score + sample(0:1, nrow(dat), TRUE), 1),
                    dat$Score)

# 参加者×条件に合計（0-15点）
dat_summarized <- dat |>
  mutate(across(c(Participant, Item, Treatment, Test, Time), as.factor)) |>
  group_by(Participant, Treatment, Test, Time) |>
  summarise(Score = sum(Score), .groups = "drop")

# Wide（多変量）→ 本章では Score_Recall を主目的変数に利用
wide_scores <- dat_summarized |>
  pivot_wider(names_from = Test, values_from = Score, names_prefix = "Score_") |>
  mutate(across(c(Participant, Treatment, Time), as.factor))

# 参加者レベルの共変量を付与（Proficiency ~ N(2500, 500), Aptitude ~ N(10, 10)）
participants       <- unique(wide_scores$Participant)
proficiency_values <- rnorm(length(participants), mean = 2500, sd = sqrt(500))
aptitude_values    <- rnorm(length(participants), mean = 10,   sd = sqrt(10))

wide_scores <- wide_scores |>
  left_join(tibble(Participant = participants, Proficiency = proficiency_values), by = "Participant") |>
  left_join(tibble(Participant = participants, Aptitude    = aptitude_values),   by = "Participant")

9 説明変数の変換

  • z変換（連続変数）
  • コントラスト（カテゴリカル）
  • 補足：係数の読み方

連続変数は単位を揃えるために z 変換します。これにより係数は「1 SDの変化で従属変数が何単位変わるか」の解釈になり、交互作用の比較や可視化がしやすくなります。

コードを表示 
wide_scores <- wide_scores |>
  mutate(
    Proficiency = as.numeric(scale(Proficiency)),
    Aptitude    = as.numeric(scale(Aptitude))
  )

交互作用を解釈しやすい sum coding（水準和=0）を用います。2水準の場合、レベルは {-1, +1} に再符号化され、主効果は両群の平均差、交互作用は傾きの差として読みやすくなります。

コードを表示 
contrasts(wide_scores$Treatment) <- contr.sum(2)
contrasts(wide_scores$Time)      <- contr.sum(2)
  • Proficiency と Aptitude は標準化済みなので、係数は標準化回帰係数に近い解釈になります。
  • Treatment と Time は sum coding のため、切片は水準平均、主効果は平均差を意味します。
  • 交互作用（例：Treatment:Proficiency）は、Proficiencyの傾きが群でどれだけ異なるかを表します。

ノート

モデル選択の考え方
- AIC は汎化を意識した情報量基準。小さいほど良い（過学習を一定程度抑制）。
- BIC はパラメータ数への罰則が強く、より保守的。
- 可能なら 交差検証（CV） を併用して、予測性能も確認しましょう。

11 モデルの前提の確認

  • 線形性・外れ値
  • 等分散性（ホモスケダスティシティ）
  • 正規性（誤差項）
  • 独立性（系列相関）
  • 多重共線性（VIF）

残差 vs 予測値の散布図で系統的な曲線がないかを確認します。強い曲線や極端な外れ値は非線形や影響点の可能性。

コードを表示 
check_model(mra_step)  # 総合ダイアグノスティクス

残差の分散が予測値に依存して増減しないかをチェックします。ファン形は違反のサイン。

コードを表示 
performance::check_heteroscedasticity(mra_step)
OK: Error variance appears to be homoscedastic (p = 0.811).

Q-Qプロットで直線から大きく外れないことを確認。サンプルが大きい場合は軽微な逸脱に過敏にならないこと。

コードを表示 
performance::check_normality(mra_step)
OK: residuals appear as normally distributed (p = 0.957).

時系列や縦断データの場合は自己相関に注意。Durbin–Watson などで補助確認。

コードを表示 
performance::check_autocorrelation(mra_step)
OK: Residuals appear to be independent and not autocorrelated (p = 0.726).

VIF が 10超で注意、5超でも文脈により検討。相関が高い説明変数は解釈を不安定にします。

14 下位検定・Simple Slope Analysis

連続変数を含む交互作用の下位検定は、どの値で比較するかを明示する必要があります。ここでは emmeans の ref_grid() を使って、関心の z 値（例：-2, -1, 0, 1, 2）で条件付き平均と群差を評価します。

  • 概念
  • 条件付きの群差（Timeごと）
  • Simple Slope（Proficiencyの範囲で）
  • Simple slope：連続変数の特定値で「群差の傾き」を比較
  • 注意：offset 引数は回帰の切片ずらし用途で、連続モデレータの下位検定には不向き。ref_grid() を使います。
  • 解釈：例えば Treatment:Proficiency が有意なら、「Proficiency が 1 SD 増えたときの傾き差」が有意。

15 まとめ

  • 連続変数を含む線形モデルは lm() で実装し、z変換＋sum coding を使うと交互作用の解釈が明瞭。
  • AIC によるモデル選択で過度な複雑さを回避しつつ汎化性能を確保。
  • performance と emmeans で 前提確認 と 下位検定 を系統立てて実施。
  • Simple Slope はあくまで補助。可視化（plot_model / emmip）と併せて解釈。

# はじめに

これまでに **t検定** と **MANOVA** を扱ってきました。いずれも説明変数がカテゴリカル中心でしたが、実データでは **連続変数**（例：語彙サイズ、適性）を含むことが一般的です。本章では、Notionノートの内容を基に、**重回帰分析**（線形モデル）と **Simple Slope Analysis**（連続×カテゴリカルの交互作用の下位検定）を、**実行可能なRコード**とともに示します。

::: {.callout-note}
**想定データ**：2つの学習方法（Glossing / Writing）、2つのテスト（Recall / Recognition）、2つの時点（Immediate / Delayed）、共変量として Proficiency（語彙サイズ）, Aptitude（記憶力/適性）。
:::

# データの準備

```{r}
# パッケージ
library(tidyverse)
library(performance)
library(effectsize)
library(emmeans)
library(sjPlot)
library(broom)
library(MASS)      # step() の周辺や今後の拡張で使用可
theme_set(theme_bw())

set.seed(123)

# パラメータ
n_participants <- 50
n_items        <- 30
treatments     <- c("Glossing", "Writing")
tests          <- c("Recall", "Recognition")
times          <- c("Immediate", "Delayed")

# 完全要因の展開
dat <- expand.grid(
  Participant = 1:n_participants,
  Item        = 1:n_items,
  Treatment   = treatments,
  Test        = tests,
  Time        = times,
  KEEP.OUT.ATTRS = FALSE
)

# アイテム割当（Glossing:1-15, Writing:16-30）
dat <- dat[
  (dat$Treatment == "Glossing" & dat$Item <= 15) |
  (dat$Treatment == "Writing"  & dat$Item >  15), ]

# 0/1反応（Writing > Glossing, Recognition > Recall）
dat$Score <- ifelse(
  dat$Treatment == "Glossing",
  rbinom(nrow(dat), 1, ifelse(dat$Test == "Recognition", 0.5, 0.2)),
  rbinom(nrow(dat), 1, ifelse(dat$Test == "Recognition", 0.9, 0.5))
)

# Immediate 効果を少し上げる（上限1でクリップ）
dat$Score <- ifelse(dat$Time == "Immediate",
                    pmin(dat$Score + sample(0:1, nrow(dat), TRUE), 1),
                    dat$Score)

# 参加者×条件に合計（0-15点）
dat_summarized <- dat |>
  mutate(across(c(Participant, Item, Treatment, Test, Time), as.factor)) |>
  group_by(Participant, Treatment, Test, Time) |>
  summarise(Score = sum(Score), .groups = "drop")

# Wide（多変量）→ 本章では Score_Recall を主目的変数に利用
wide_scores <- dat_summarized |>
  pivot_wider(names_from = Test, values_from = Score, names_prefix = "Score_") |>
  mutate(across(c(Participant, Treatment, Time), as.factor))

# 参加者レベルの共変量を付与（Proficiency ~ N(2500, 500), Aptitude ~ N(10, 10)）
participants       <- unique(wide_scores$Participant)
proficiency_values <- rnorm(length(participants), mean = 2500, sd = sqrt(500))
aptitude_values    <- rnorm(length(participants), mean = 10,   sd = sqrt(10))

wide_scores <- wide_scores |>
  left_join(tibble(Participant = participants, Proficiency = proficiency_values), by = "Participant") |>
  left_join(tibble(Participant = participants, Aptitude    = aptitude_values),   by = "Participant")

head(wide_scores)
```

# 説明変数の変換

::: {.panel-tabset}

### z変換（連続変数）

連続変数は**単位を揃える**ために z 変換します。これにより係数は「**1 SDの変化で従属変数が何単位変わるか**」の解釈になり、**交互作用の比較**や可視化がしやすくなります。

```{r}
wide_scores <- wide_scores |>
  mutate(
    Proficiency = as.numeric(scale(Proficiency)),
    Aptitude    = as.numeric(scale(Aptitude))
  )
```

### コントラスト（カテゴリカル）

交互作用を解釈しやすい **sum coding**（水準和=0）を用います。2水準の場合、レベルは \{-1, +1\} に再符号化され、**主効果は両群の平均差**、**交互作用は傾きの差**として読みやすくなります。

```{r}
contrasts(wide_scores$Treatment) <- contr.sum(2)
contrasts(wide_scores$Time)      <- contr.sum(2)
```

### 補足：係数の読み方

- `Proficiency` と `Aptitude` は**標準化済み**なので、係数は**標準化回帰係数**に近い解釈になります。  
- `Treatment` と `Time` は **sum coding** のため、切片は**水準平均**、主効果は**平均差**を意味します。  
- 交互作用（例：`Treatment:Proficiency`）は、**Proficiencyの傾きが群でどれだけ異なるか**を表します。
:::

# 重回帰分析の実施

## フルモデルの当てはめ

```{r}
# 4要因＋全交互作用のフルモデル
mra_full <- lm(Score_Recall ~ Treatment * Time * Proficiency * Aptitude, data = wide_scores)
summary(mra_full)
```

## モデルの作成と選択（AIC）

```{r}
# ステップワイズ（両方向）：AIC最小化
mra_step <- step(mra_full, trace = 0)
mra_step$call
summary(mra_step)
```

::: {.callout-note}
**モデル選択の考え方**  
- **AIC** は汎化を意識した情報量基準。**小さいほど良い**（過学習を一定程度抑制）。  
- **BIC** はパラメータ数への罰則が強く、より**保守的**。  
- 可能なら **交差検証（CV）** を併用して、予測性能も確認しましょう。
:::

# モデルの前提の確認

::: {.panel-tabset}

### 線形性・外れ値

残差 vs 予測値の散布図で**系統的な曲線**がないかを確認します。強い曲線や極端な外れ値は**非線形**や**影響点**の可能性。

```{r}
check_model(mra_step)  # 総合ダイアグノスティクス
```

### 等分散性（ホモスケダスティシティ）

残差の分散が予測値に依存して増減しないかをチェックします。**ファン形**は違反のサイン。

```{r}
performance::check_heteroscedasticity(mra_step)
```

### 正規性（誤差項）

Q-Qプロットで**直線から大きく外れない**ことを確認。サンプルが大きい場合は軽微な逸脱に過敏にならないこと。

```{r}
performance::check_normality(mra_step)
```

### 独立性（系列相関）

時系列や縦断データの場合は**自己相関**に注意。Durbin–Watson などで補助確認。

```{r}
performance::check_autocorrelation(mra_step)
```

### 多重共線性（VIF）

VIF が **10超**で注意、**5超**でも文脈により検討。相関が高い説明変数は解釈を不安定にします。

```{r}
performance::check_collinearity(mra_step)
model_performance(mra_step)
```

:::

# 下位検定・Simple Slope Analysis

連続変数を含む交互作用の下位検定は、**どの値で比較するか**を明示する必要があります。ここでは `emmeans` の **`ref_grid()`** を使って、関心の z 値（例：-2, -1, 0, 1, 2）で条件付き平均と群差を評価します。

::: {.panel-tabset}

### 概念

- **Simple slope**：連続変数の特定値で「群差の傾き」を比較  
- **注意**：`offset` 引数は回帰の切片ずらし用途で、**連続モデレータの下位検定には不向き**。`ref_grid()` を使います。  
- **解釈**：例えば `Treatment:Proficiency` が有意なら、「Proficiency が 1 SD 増えたときの傾き差」が有意。

### 条件付きの群差（Timeごと）

```{r}
emmeans(mra_step, pairwise ~ Treatment | Time)
```

### Simple Slope（Proficiencyの範囲で）

```{r}
rg <- ref_grid(mra_step, at = list(Proficiency = c(-2, -1, 0, 1, 2)))
emmeans(rg, pairwise ~ Treatment | Proficiency)
```

:::

## 交互作用の可視化

```{r}
# Timeを平均化し、Proficiency上の傾きを図示（点区切り）
emmip(rg, Treatment ~ Proficiency, style = "factor", CIs = TRUE,
      linearg = list(), dotarg = list(size = 2), CIarg = list(alpha = 1))
```

# まとめ

- 連続変数を含む線形モデルは **lm()** で実装し、**z変換**＋**sum coding** を使うと交互作用の解釈が明瞭。
- **AIC によるモデル選択**で過度な複雑さを回避しつつ汎化性能を確保。
- **performance** と **emmeans** で **前提確認** と **下位検定** を系統立てて実施。
- Simple Slope はあくまで補助。**可視化**（`plot_model` / `emmip`）と併せて解釈。
