11 はじめに

この章の主役はツリーモデルです。具体的には、

  • 決定木（Decision Tree）
  • ランダムフォレスト（Random Forest）
  • XGBoost

を扱います。

そして、これらのモデルの精度を評価する方法の1つとして、ROC曲線とAUCを導入します。

英語教育研究では、たとえば以下のような予測課題が実務で登場します。

  • 語彙テスト項目の正誤（正解/不正解）を予測する
  • 学習者が次の小テストで基準を超えるかを予測する
  • CEFR到達判定の補助指標を構築する

この章では、こうした課題に近い架空データ（英語語彙学習）を用いて、モデル比較から評価までを体系的に実装します。

Conventions
  • Figure labels are in English.
  • All code chunks are R.
  • 数式は最小限にし、実装と解釈を重視します。
本章の到達目標
  1. ツリーモデル3種を同じデータで比較できる。
  2. ROC-AUCだけでなくPR-AUC・閾値・キャリブレーションも確認できる。
  3. 「精度が高い」で終わらず、教育実務で使える判断に落とし込める。
本章の要点
  • 分析課題: 「どの学習者・どの語彙項目が正答しやすいか」を予測する。
  • 使うモデル: 単純（決定木）から高精度（RF, XGBoost）へ段階的に比較する。
  • 最終判断: ROC-AUCだけでなく、閾値・PR・キャリブレーションまで見て運用設計する。
読み方ガイド
  1. まず「ケース設定」まで読み、どんな教育データを想定しているかをつかむ。
  2. 次に「決定木 → RF → XGBoost」を比較し、モデルごとの強みを理解する。
  3. 最後に「ROC/PR・閾値・キャリブレーション」で運用判断の流れを確認する。

11.1 英語教育分野の先行研究（ツリーモデル）

この章の設計は、英語教育・応用言語学での最近の機械学習研究に合わせています。代表例は次の通りです。

  1. CEFR難易度分類
    Zhang & Lu (2025) は、L2英語学習者向けテキストをCEFRレベルに分類するためにRandom Forestを使用し、複数レベルで実用的な精度を報告しました。
  2. L2アカデミックライティング評価
    Mau & Feng (2025) は、自動作文評価でXGBoost分類器を採用し、move分析情報を統合した評価モデルを報告しました。
  3. 英語教育評価へのRF応用（会議報告）
    Niu (2025) は、英語教育評価の精度改善に向けたRandom Forest活用を報告しています。

この章で「単木 → RF → XGBoost」を順に扱うのは、こうした先行研究の流れを、大学院レベルで再現可能な形に落とし込むためです。

本章の目的は、先行研究の読解にとどまらず、同一の分析発想を再実装できることにあります。

※ なお、会議論文は査読誌より方法報告が簡潔な場合があるため、再現時は必ず評価設計（分割・指標・閾値）を明示します。

11.2.1 論文の手順をこの章で再現するミニガイド

  1. Dominance（Budescu, Azen系）
    まず回帰モデルを固定し、全サブモデルで「その変数を追加したときの改善量」を平均して重要度を出します。
    Azen & Traxel (2009) はロジスティック回帰でこの考え方を拡張し、擬似R2とブートストラップで安定性を検討しました。
    本章では glm-dominance-analysis で平均寄与、glm-dominance-level でモデルサイズ別寄与を確認します。
  2. SHAP（Lundberg系）
    1件ごとの予測を「ベースライン + 特徴寄与」に分解し、そのlocal説明を多数集約してglobal理解へ進みます。
    Lundberg et al. (2020) はこの流れで、希少だが高リスクな要因、相互作用、性能劣化要因を抽出しました。
    本章では xgb-shap-local（個別）と xgb-shap-global（全体）で同じ読み方を再現します。
  3. 教育データでの読み替え例
    たとえばある学習者のlocal SHAPで test_time_delayed が負、proficiency が正なら、
    「遅延テスト条件は不利だが、習熟度がそれを一部補っている」と解釈できます。
    さらにglobal SHAPで condition や word_freq が上位なら、授業介入と語彙難度の設計に優先順位をつけられます。

12.2 評価設計（最初に決める）

最初に評価設計を固定します。ここを曖昧にすると、後から都合のよい指標だけを見る「後出し」になりがちです。

コード 
metric_tbl <- metric_set(

)

metric_tbl
A metric set, consisting of:
- `roc_auc()`, a probability metric     | direction: maximize
- `pr_auc()`, a probability metric      | direction: maximize
- `accuracy()`, a class metric          | direction: maximize
- `sens()`, a class metric              | direction: maximize
- `spec()`, a class metric              | direction: maximize
- `brier_class()`, a probability metric | direction: minimize

この章では、次のように役割分担して読みます。

  • ROC_AUC: モデル全体の識別性能を比較
  • PR_AUC: 陽性クラス（ここでは correct）を重視した比較
  • Sensitivity / Specificity: 閾値を決めた後の実運用評価
  • Brier: 予測確率の信頼性（キャリブレーション）
なぜROCだけでなくPRも見るのか
  • ROC-AUCは閾値に依存しない便利な比較指標。
  • ただし陽性が少ないデータではPR曲線の方が差が見えやすい場合がある（Saito & Rehmsmeier, 2015）。

12.3 混同行列の確認（最小例）

ここで一度だけ、ROCの土台になる混同行列を小さな例で確認します。

コード 
# 以降の章でも使える、閾値ごとの基本指標関数
evaluate_threshold <- function(truth, prob, threshold) {
  pred <- factor(
    if_else(prob >= threshold, "correct", "incorrect"),
    levels = levels(truth)
  )

  tp <- sum(pred == "correct" & truth == "correct")
  fp <- sum(pred == "correct" & truth == "incorrect")
  tn <- sum(pred == "incorrect" & truth == "incorrect")
  fn <- sum(pred == "incorrect" & truth == "correct")

  tibble(
    threshold = threshold,
    TP = tp, FP = fp, TN = tn, FN = fn,
    Sensitivity = tp / (tp + fn),
    Specificity = tn / (tn + fp),
    Precision = tp / (tp + fp)
  )
}

mini_dat <- tibble(
  truth = factor(
    c("correct", "incorrect", "correct", "incorrect", "correct", "incorrect"),
    levels = c("incorrect", "correct")
  ),
  pred_prob = c(0.90, 0.80, 0.63, 0.56, 0.42, 0.10)
)

bind_rows(
  evaluate_threshold(mini_dat$truth, mini_dat$pred_prob, 0.40),
  evaluate_threshold(mini_dat$truth, mini_dat$pred_prob, 0.60)
) |>
  mutate(across(where(is.numeric), ~ round(., 3))) |>
  gt() |>
  tab_header(
    title = "Mini Threshold Example",
    subtitle = "Lower threshold often improves sensitivity but increases false positives"
  )

13 ケース設定：英語語彙学習データ（架空）

この節の目的は、モデル比較の基盤となるデータ構造を明確にすることです。

13.1 研究場面の想定

  • 対象: 大学生のL2語彙学習
  • 目的変数: 項目正答（correct / incorrect）
  • 説明変数: 学習者要因 + 語彙項目要因 + 学習条件

回帰分析の知識がある方は、次のように捉えると理解しやすいです。

  • ロジスティック回帰: 係数で線形に説明しやすい
  • ツリーモデル: 非線形・相互作用を自然に拾いやすい
変数を教育文脈で言い換える
  • proficiency: 習熟度（高いほど正答しやすい想定）
  • weekly_study: 週学習時間（行動指標）
  • anxiety: テスト不安（高いほど不利な想定）
  • word_freq / word_length: 語彙項目の難しさ
  • condition: 指導法（glossing / retrieval / sentence）

13.2 データ生成

コード 
n_learners <- 220
n_items <- 120

learner_tbl <- tibble(
  learner_id = factor(1:n_learners),
  proficiency = rnorm(n_learners, 0, 1),
  weekly_study = pmax(rnorm(n_learners, 4, 1.5), 0.5),
  anxiety = rnorm(n_learners, 0, 1),
  l1_distance = runif(n_learners, 0, 1),
  u_learner = rnorm(n_learners, 0, 0.55)
)

item_tbl <- tibble(
  item_id = factor(1:n_items),
  word_freq = rlnorm(n_items, 4.3, 0.8),
  word_length = sample(3:12, n_items, replace = TRUE),
  cognate = factor(rbinom(n_items, 1, 0.25), labels = c("no", "yes")),
  pos = factor(sample(c("noun", "verb", "adjective"), n_items, replace = TRUE,
                      prob = c(0.45, 0.35, 0.20))),
  u_item = rnorm(n_items, 0, 0.45)
)

# learner-item の全組み合わせから一部を観測
obs <- crossing(learner_id = learner_tbl$learner_id,
                item_id = item_tbl$item_id) |>
  slice_sample(n = 3500)

df <- obs |>
  left_join(learner_tbl, by = "learner_id") |>
  left_join(item_tbl, by = "item_id") |>
  mutate(
    condition = factor(sample(c("glossing", "retrieval", "sentence"), n(), replace = TRUE,
                              prob = c(0.40, 0.35, 0.25))),
    test_time = factor(sample(c("immediate", "delayed"), n(), replace = TRUE,
                              prob = c(0.62, 0.38))),
    # 最小限の式で「それらしい」難易度構造を生成
    eta = -1.45 +
      0.95 * proficiency +
      0.10 * weekly_study -
      0.35 * anxiety -
      0.30 * l1_distance +
      0.38 * log10(word_freq) -
      0.12 * word_length +
      0.55 * (cognate == "yes") +
      0.42 * (condition == "retrieval") +
      0.20 * (condition == "sentence") -
      0.33 * (test_time == "delayed") +
      0.25 * proficiency * (condition == "retrieval") -
      0.18 * (condition == "retrieval") * (test_time == "delayed") +
      u_learner + u_item,
    p_correct = plogis(eta),
    correct = factor(
      if_else(rbinom(n(), 1, p_correct) == 1, "correct", "incorrect"),
      levels = c("incorrect", "correct")
    )
  ) |>
  select(correct, proficiency, weekly_study, anxiety, l1_distance,
         word_freq, word_length, cognate, pos, condition, test_time)

15.1 補助解析：Dominance Analysis（GLM）

係数表は「方向と有意性」を示すのに有用ですが、
説明変数が複数あると「相対的重要性」は読み取りにくくなります。
そこで、GLMに対して dominance analysis を追加します（Budescu, 1993; Azen & Budescu, 2003; Azen & Traxel, 2009）。

文献での活用ポイント（具体）:

  • Azen & Traxel (2009) は、ロジスティック回帰で dominance analysis を使うために、
    R2類似指標の選定とブートストラップ推論を組み合わせて、重要度の再現性を検討しました。
  • Mizumoto (2022) は、応用言語学データの再解析で dominance analysis を併用し、
    「標準化βだけでは見えにくい相対的重要性」を示しました。
  • 本章での具体例としては、proficiency と condition の relative share を比較し、
    「学習者要因と授業デザイン要因のどちらが予測改善に効いているか」を同じ尺度で読む、という使い方になります。

16 決定木（Decision Tree）

この節の目的は、解釈可能性の高い木モデルを構築し、分岐規則を確認することです。

決定木の基本構造
  • 決定木は、データを「条件分岐」で繰り返し分けるモデルです。
  • 分岐点（ノード）では、例: proficiency > 0.4 のような質問をします。
  • 最後の葉（leaf）で、正答確率やクラス（correct / incorrect）を出します。
  • 良い点: ルールが可視化でき、教育的解釈がしやすい。
  • 注意点: 1本の木はデータの小さな揺れに影響されやすい。

英語教育の文脈では、「どの条件の学習者が正答しやすいか」を
if-then ルールで説明できる点が、決定木の大きな利点です。

17 ランダムフォレスト（Random Forest）

この節の目的は、単木の弱点（不安定性）をアンサンブルで補うことです。

ランダムフォレストの基本原理
  • ランダムフォレストは、たくさんの決定木の「多数決」で予測します。
  • 各木は、別々のブートストラップ標本で学習します（bagging）。
  • 各分岐では、全変数でなく一部の変数だけを候補にします。
  • これにより木どうしの似すぎを防ぎ、過学習を抑えやすくなります。
  • 代償として、単木より説明はやや複雑になります。

英語教育データでは、学習者要因と語彙要因の相互作用が多く、
単木よりもランダムフォレストの方が安定して高い再現性能を得やすい場面が多くあります。

18.2 SHAP（全体と個別の解釈）

ここまでの可視化でも比較は可能ですが、大学院レベルの読み物としては、
「なぜその予測になったか」を説明できるSHAPを入れておくと実務との接続が強くなります（Lundberg & Lee, 2017; Lundberg et al., 2020）。

文献での活用ポイント（具体）:

  • Lundberg & Lee (2017) は、SHAPを「加法的特徴寄与」の統一枠組みとして定義し、
    1件予測の寄与分解を理論的に一意化しました。
  • Lundberg et al. (2020) は、医療データに対して local説明を多数集約し、
    低頻度だが高リスクな要因や、特徴間の非線形相互作用、運用中モデルの劣化要因を特定しました。
  • Štrumbelj & Kononenko (2013) は、個別予測の寄与説明がモデル理解を高めることを実験的に示しました。
  • 本章ではこれを教育データに置き換え、
    xgb-shap-global で全体傾向、xgb-shap-local で個別学習者ケースの説明を行います。

22 英語教育研究としての解釈

この節は、分析結果を「論文の文章」に変換するためのテンプレートです。

22.1 何を報告すると説得力が出るか

最低限、次をセットで報告します。

  1. データ分割（train/test または CV設計）
  2. 主要指標（ROC-AUC, PR-AUC, Sens/Spec, Brier）
  3. 閾値の決定根拠（Youden, あるいはコスト設計）
  4. 解釈（重要特徴・教育的意味）
  5. 限界（外的妥当性、バイアス、公平性）

22.2 教育的含意の例（この章のケース）

  • retrieval条件はglossingより正答率を押し上げる傾向。
  • ただし遅延テストで効果が減衰する可能性がある（交互作用）。
  • 学習時間だけでなく語彙特性（頻度・長さ・同根語）も予測に寄与。
結果記述テンプレート

「XGBoostはテストデータでROC-AUC = XX、PR-AUC = XXを示した。
閾値を0.XX（Youden最適）に設定した場合、Sensitivity = XX、Specificity = XXであった。
教育実務では見逃しコストを考慮し、閾値を0.XXに下げる運用も妥当である。」

23 方法論上の留意点

留意点1: AUC単独でモデル選択を行わない
  • 対応: PR-AUC、閾値依存指標（Sens/Spec/Precision）も併記する。
留意点2: 目的変数のイベント水準を明示する
  • 対応: event_level = "second" などを明示し、再現可能性を確保する。
留意点3: 前処理リークを防止する
  • 対応: recipeの学習は学習データのみに限定し、workflowで一体化する。
留意点4: 複雑モデルを無条件に採用しない
  • 対応: 単純モデル（GLM/単木）を必ず比較対象に含める。

24 まとめ

本章の主要な結論は次の3点である。

  1. ツリーモデルは、英語教育データで起こりやすい非線形・相互作用を扱いやすい。
  2. ROCは有用だが、ROCだけでは不十分。
  3. 実務では、PR・閾値・キャリブレーションまで見て初めて意思決定に使える。

# はじめに

この章の主役は**ツリーモデル**です。具体的には、

- 決定木（Decision Tree）
- ランダムフォレスト（Random Forest）
- XGBoost

を扱います。

そして、これらのモデルの精度を評価する方法の1つとして、**ROC曲線とAUC**を導入します。

英語教育研究では、たとえば以下のような予測課題が実務で登場します。

- 語彙テスト項目の正誤（正解/不正解）を予測する
- 学習者が次の小テストで基準を超えるかを予測する
- CEFR到達判定の補助指標を構築する

この章では、こうした課題に近い**架空データ（英語語彙学習）**を用いて、モデル比較から評価までを体系的に実装します。

### 論文の手順をこの章で再現するミニガイド

1. **Dominance（Budescu, Azen系）**  
   まず回帰モデルを固定し、全サブモデルで「その変数を追加したときの改善量」を平均して重要度を出します。  
   Azen & Traxel (2009) はロジスティック回帰でこの考え方を拡張し、擬似R2とブートストラップで安定性を検討しました。  
   本章では `glm-dominance-analysis` で平均寄与、`glm-dominance-level` でモデルサイズ別寄与を確認します。  
2. **SHAP（Lundberg系）**  
   1件ごとの予測を「ベースライン + 特徴寄与」に分解し、そのlocal説明を多数集約してglobal理解へ進みます。  
   Lundberg et al. (2020) はこの流れで、希少だが高リスクな要因、相互作用、性能劣化要因を抽出しました。  
   本章では `xgb-shap-local`（個別）と `xgb-shap-global`（全体）で同じ読み方を再現します。  
3. **教育データでの読み替え例**  
   たとえばある学習者のlocal SHAPで `test_time_delayed` が負、`proficiency` が正なら、  
   「遅延テスト条件は不利だが、習熟度がそれを一部補っている」と解釈できます。  
   さらにglobal SHAPで `condition` や `word_freq` が上位なら、授業介入と語彙難度の設計に優先順位をつけられます。

## 混同行列の確認（最小例）

ここで一度だけ、ROCの土台になる混同行列を小さな例で確認します。

```{r}
#| label: mini-confusion-threshold

# 以降の章でも使える、閾値ごとの基本指標関数
evaluate_threshold <- function(truth, prob, threshold) {
  pred <- factor(
    if_else(prob >= threshold, "correct", "incorrect"),
    levels = levels(truth)
  )

  tp <- sum(pred == "correct" & truth == "correct")
  fp <- sum(pred == "correct" & truth == "incorrect")
  tn <- sum(pred == "incorrect" & truth == "incorrect")
  fn <- sum(pred == "incorrect" & truth == "correct")

  tibble(
    threshold = threshold,
    TP = tp, FP = fp, TN = tn, FN = fn,
    Sensitivity = tp / (tp + fn),
    Specificity = tn / (tn + fp),
    Precision = tp / (tp + fp)
  )
}

mini_dat <- tibble(
  truth = factor(
    c("correct", "incorrect", "correct", "incorrect", "correct", "incorrect"),
    levels = c("incorrect", "correct")
  ),
  pred_prob = c(0.90, 0.80, 0.63, 0.56, 0.42, 0.10)
)

bind_rows(
  evaluate_threshold(mini_dat$truth, mini_dat$pred_prob, 0.40),
  evaluate_threshold(mini_dat$truth, mini_dat$pred_prob, 0.60)
) |>
  mutate(across(where(is.numeric), ~ round(., 3))) |>
  gt() |>
  tab_header(
    title = "Mini Threshold Example",
    subtitle = "Lower threshold often improves sensitivity but increases false positives"
  )
```

この表の読み方:

- 閾値を下げる（0.60 → 0.40）と、拾える正解者（Sensitivity）は上がりやすい。
- その代わり、誤って正解判定する人数（FP）が増えることがある。

- 説明変数: 学習者要因 + 語彙項目要因 + 学習条件

- ツリーモデル: 非線形・相互作用を自然に拾いやすい

- `weekly_study`: 週学習時間（行動指標）
- `anxiety`: テスト不安（高いほど不利な想定）
- `word_freq` / `word_length`: 語彙項目の難しさ
- `condition`: 指導法（glossing / retrieval / sentence）

## データ生成

```{r}
#| label: simulate-edu-data

n_learners <- 220
n_items <- 120

learner_tbl <- tibble(
  learner_id = factor(1:n_learners),
  proficiency = rnorm(n_learners, 0, 1),
  weekly_study = pmax(rnorm(n_learners, 4, 1.5), 0.5),
  anxiety = rnorm(n_learners, 0, 1),
  l1_distance = runif(n_learners, 0, 1),
  u_learner = rnorm(n_learners, 0, 0.55)
)

item_tbl <- tibble(
  item_id = factor(1:n_items),
  word_freq = rlnorm(n_items, 4.3, 0.8),
  word_length = sample(3:12, n_items, replace = TRUE),
  cognate = factor(rbinom(n_items, 1, 0.25), labels = c("no", "yes")),
  pos = factor(sample(c("noun", "verb", "adjective"), n_items, replace = TRUE,
                      prob = c(0.45, 0.35, 0.20))),
  u_item = rnorm(n_items, 0, 0.45)
)

# learner-item の全組み合わせから一部を観測
obs <- crossing(learner_id = learner_tbl$learner_id,
                item_id = item_tbl$item_id) |>
  slice_sample(n = 3500)

df <- obs |>
  left_join(learner_tbl, by = "learner_id") |>
  left_join(item_tbl, by = "item_id") |>
  mutate(
    condition = factor(sample(c("glossing", "retrieval", "sentence"), n(), replace = TRUE,
                              prob = c(0.40, 0.35, 0.25))),
    test_time = factor(sample(c("immediate", "delayed"), n(), replace = TRUE,
                              prob = c(0.62, 0.38))),
    # 最小限の式で「それらしい」難易度構造を生成
    eta = -1.45 +
      0.95 * proficiency +
      0.10 * weekly_study -
      0.35 * anxiety -
      0.30 * l1_distance +
      0.38 * log10(word_freq) -
      0.12 * word_length +
      0.55 * (cognate == "yes") +
      0.42 * (condition == "retrieval") +
      0.20 * (condition == "sentence") -
      0.33 * (test_time == "delayed") +
      0.25 * proficiency * (condition == "retrieval") -
      0.18 * (condition == "retrieval") * (test_time == "delayed") +
      u_learner + u_item,
    p_correct = plogis(eta),
    correct = factor(
      if_else(rbinom(n(), 1, p_correct) == 1, "correct", "incorrect"),
      levels = c("incorrect", "correct")
    )
  ) |>
  select(correct, proficiency, weekly_study, anxiety, l1_distance,
         word_freq, word_length, cognate, pos, condition, test_time)

glimpse(df)
```

## 補助解析：Dominance Analysis（GLM）

係数表は「方向と有意性」を示すのに有用ですが、  
説明変数が複数あると「相対的重要性」は読み取りにくくなります。  
そこで、GLMに対して dominance analysis を追加します（Budescu, 1993; Azen & Budescu, 2003; Azen & Traxel, 2009）。

文献での活用ポイント（具体）:

- Azen & Traxel (2009) は、ロジスティック回帰で dominance analysis を使うために、  
  R2類似指標の選定とブートストラップ推論を組み合わせて、重要度の再現性を検討しました。  
- Mizumoto (2022) は、応用言語学データの再解析で dominance analysis を併用し、  
  「標準化βだけでは見えにくい相対的重要性」を示しました。  
- 本章での具体例としては、`proficiency` と `condition` の relative share を比較し、  
  「学習者要因と授業デザイン要因のどちらが予測改善に効いているか」を同じ尺度で読む、という使い方になります。  

```{r}
#| label: glm-dominance-analysis

# dominance analysis は通常の glm オブジェクトで実行する
glm_dom <- glm(
  correct ~ proficiency + weekly_study + anxiety + l1_distance +
    log10(word_freq + 1) + word_length + cognate + pos + condition + test_time,
  data = train_data,
  family = binomial()
)

da_glm <- dominanceanalysis::dominanceAnalysis(glm_dom)
da_avg <- dominanceanalysis::averageContribution(da_glm)

  Fit_Index = .y,
  Predictor = names(.x),
  Average_Contribution = as.numeric(.x)
)) |>
  group_by(Fit_Index) |>
  mutate(
    Relative = Average_Contribution / sum(Average_Contribution),
    Rank = dense_rank(desc(Average_Contribution))
  ) |>
  ungroup()

# 章内では Nagelkerke R2 ベース（r2.n）を主に表示
da_tbl |>
  filter(Fit_Index == "r2.n") |>

  mutate(across(c(Average_Contribution, Relative), ~ round(., 4))) |>
  gt() |>
  tab_header(
    title = "Dominance Analysis (GLM)",
    subtitle = "Relative importance based on Nagelkerke R2 (r2.n)"
  ) |>
  cols_label(
    Fit_Index = "Fit Index",
    Predictor = "Predictor",
    Average_Contribution = "Average Contribution",
    Relative = "Relative Share",
    Rank = "Rank"
  )
```

# 決定木（Decision Tree）

この節の目的は、解釈可能性の高い木モデルを構築し、分岐規則を確認することです。

::: {.callout-note}
## 決定木の基本構造
- 決定木は、データを「条件分岐」で繰り返し分けるモデルです。  
- 分岐点（ノード）では、例: `proficiency > 0.4` のような質問をします。  
- 最後の葉（leaf）で、正答確率やクラス（correct / incorrect）を出します。  
- 良い点: ルールが可視化でき、教育的解釈がしやすい。  
- 注意点: 1本の木はデータの小さな揺れに影響されやすい。  
:::

英語教育の文脈では、「どの条件の学習者が正答しやすいか」を  
if-then ルールで説明できる点が、決定木の大きな利点です。

## モデルとチューニング

まずは「深すぎる木を避ける」ことを意識して、単純な探索から始めます。


# ランダムフォレスト（Random Forest）

この節の目的は、単木の弱点（不安定性）をアンサンブルで補うことです。

::: {.callout-note}
## ランダムフォレストの基本原理
- ランダムフォレストは、たくさんの決定木の「多数決」で予測します。  
- 各木は、別々のブートストラップ標本で学習します（bagging）。  
- 各分岐では、全変数でなく一部の変数だけを候補にします。  
- これにより木どうしの似すぎを防ぎ、過学習を抑えやすくなります。  
- 代償として、単木より説明はやや複雑になります。  
:::

英語教育データでは、学習者要因と語彙要因の相互作用が多く、  
単木よりもランダムフォレストの方が安定して高い再現性能を得やすい場面が多くあります。

## SHAP（全体と個別の解釈）

ここまでの可視化でも比較は可能ですが、大学院レベルの読み物としては、  
「なぜその予測になったか」を説明できるSHAPを入れておくと実務との接続が強くなります（Lundberg & Lee, 2017; Lundberg et al., 2020）。

文献での活用ポイント（具体）:

- Lundberg & Lee (2017) は、SHAPを「加法的特徴寄与」の統一枠組みとして定義し、  
  1件予測の寄与分解を理論的に一意化しました。  
- Lundberg et al. (2020) は、医療データに対して local説明を多数集約し、  
  低頻度だが高リスクな要因や、特徴間の非線形相互作用、運用中モデルの劣化要因を特定しました。  
- Štrumbelj & Kononenko (2013) は、個別予測の寄与説明がモデル理解を高めることを実験的に示しました。  
- 本章ではこれを教育データに置き換え、  
  `xgb-shap-global` で全体傾向、`xgb-shap-local` で個別学習者ケースの説明を行います。  
- 具体的には、ある学習者で `anxiety` が負寄与、`weekly_study` が正寄与と出れば、  
  「不安による不利を学習時間がどこまで補っているか」をケース単位で説明できます。  

### 全体: どの特徴が平均的に効いたか

# 閾値をどう決めるか（ROCを運用に接続）

AUCは「ランク付けの上手さ」を示しますが、現場では最終的に**閾値**が必要です。

ここが「研究結果を実務判断へ変換する」一番重要な節です。

## 例: YoudenのJで閾値を選ぶ

ここでは例として、最上位モデル（ここでは`XGBoost`を想定）でYoudenのJを使います。

```{r}
#| label: threshold-selection

xgb_pred <- pred_all |> filter(model == "XGBoost")

roc_tbl_xgb <- xgb_pred |>
  roc_curve(truth, .pred_correct, event_level = "second") |>
  mutate(youden_j = sensitivity + specificity - 1)

best_threshold <- roc_tbl_xgb |>
  slice_max(youden_j, n = 1, with_ties = FALSE) |>
  pull(.threshold)

best_threshold
```

```{r}
#| label: confusion-at-best-threshold

xgb_threshold_eval <- xgb_pred |>
  mutate(
    .pred_class_j = factor(
      if_else(.pred_correct >= best_threshold, "correct", "incorrect"),
      levels = levels(truth)
    )
  )

conf_mat(xgb_threshold_eval, truth = truth, estimate = .pred_class_j)
```

## 閾値を変えると何が変わるか（実データで比較）

```{r}
#| label: threshold-candidate-comparison

threshold_cmp <- bind_rows(
  evaluate_threshold(xgb_pred$truth, xgb_pred$.pred_correct, 0.30),
  evaluate_threshold(xgb_pred$truth, xgb_pred$.pred_correct, best_threshold),
  evaluate_threshold(xgb_pred$truth, xgb_pred$.pred_correct, 0.70)
) |>
  mutate(Setting = c("Recall重視", "Youden最適", "Precision重視")) |>
  select(Setting, threshold, TP, FP, TN, FN, Sensitivity, Specificity, Precision)

threshold_cmp |>
  mutate(across(where(is.numeric), ~ round(., 3))) |>
  gt() |>
  tab_header(
    title = "Threshold Trade-off Table (XGBoost)",
    subtitle = "Lower threshold generally increases recall but also false positives"
  )
```

上から順に「取りこぼしを減らす設定」→「バランス設定」→「誤警報を減らす設定」と読めます。

::: {.callout-tip}
## 実務ではコストベースで閾値を決める
- 早期介入モデルなら「見逃し（FN）コスト」が高い。
- その場合、Youden最適より低めの閾値にしてRecallを優先する設計がありえます。
:::

# キャリブレーション確認（確率をそのまま信じてよいか）

高いAUCでも、確率が歪んでいる場合があります。教育現場で「0.8の確率で正答」のように使うなら、校正の確認が必要です。

特に、学習支援の優先順位づけに確率を使う場合は、この確認を省略しないのが安全です。

```{r}
xgb_calib |>
```

- 点が対角線より**下**: 実際の正答率が予測より低い（過大評価気味）。
:::

# 英語教育研究としての解釈
