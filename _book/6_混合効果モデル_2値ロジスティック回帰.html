9 はじめに

一般化線形混合効果モデル（GLMM）は、2値・頻度など非正規の目的変数に対して、固定効果とランダム効果を同時に扱う枠組みです。本章では、語彙学習の架空データを用いて、2値ロジスティックGLMMを段階的に実装します。

10 データの作成

コードを表示 
library(tidyverse)
theme_set(theme_bw())

set.seed(123)

n_participants <- 50
n_items        <- 30
treatments     <- c("Glossing","Writing")
tests          <- c("Recall","Recognition")
times          <- c("Immediate","Delayed")

dat <- expand.grid(
  Participant = 1:n_participants,
  Item        = 1:n_items,
  Treatment   = treatments,
  Test        = tests,
  Time        = times,
  KEEP.OUT.ATTRS = FALSE
)

# アイテム割当（Glossing:1-15, Writing:16-30）
dat <- dat[
  (dat$Treatment == "Glossing" & dat$Item <= 15) |
  (dat$Treatment == "Writing"  & dat$Item >  15), ]

# 0/1反応生成：Writing > Glossing, Recognition > Recall, Immediate > Delayed
dat$Score <- ifelse(
  dat$Treatment == "Glossing",
  rbinom(nrow(dat), 1, ifelse(dat$Test == "Recognition", 0.50, 0.20)),
  rbinom(nrow(dat), 1, ifelse(dat$Test == "Recognition", 0.90, 0.50))
)
dat$Score <- ifelse(dat$Time == "Immediate",
                    pmin(dat$Score + sample(0:1, nrow(dat), TRUE), 1),
                    dat$Score)

11 共変量の追加

コードを表示 
participants       <- unique(dat$Participant)
proficiency_values <- rnorm(length(participants), mean = 2500, sd = sqrt(500))
aptitude_values    <- rnorm(length(participants), mean = 10,   sd = sqrt(10))

participant_data <- tibble(
  Participant = participants,
  Proficiency = proficiency_values,
  Aptitude    = aptitude_values
)

dat <- dat |>
  left_join(participant_data, by = "Participant") |>
  mutate(
    Participant = factor(Participant),
    Item        = factor(Item),
    Treatment   = factor(Treatment),
    Time        = factor(Time),
    Proficiency = as.numeric(scale(Proficiency)),
    Aptitude    = as.numeric(scale(Aptitude))
  )

13 コーディング

  • 理論解説
  • コード例
  • 結果解釈

カテゴリカル変数は sum coding（和=0） を用います。これにより、切片は水準平均、主効果は両群の平均差、交互作用は傾きの差として解釈できます。2水準ではレベルが {−1, +1} に再符号化され、単純主効果の符号と大きさが読み取りやすくなります。

コードを表示 
# 2水準のカテゴリカルは sum coding（和=0）
contrasts(dat_recall$Treatment) <- contr.sum(2)
contrasts(dat_recall$Time)      <- contr.sum(2)

contrasts(dat_recognition$Treatment) <- contr.sum(2)
contrasts(dat_recognition$Time)      <- contr.sum(2)

sum coding 下では、係数は両群平均からの偏差として現れます。可視化（emmip/plot_model）と合わせると交互作用の向きを直感的に確認できます。

14 モデル仕様（lme4）

コードを表示 
library(lme4)
library(lmerTest)   # p値の補助（連続目的のとき）。GLMMでは主に推定補助。
library(emmeans)
library(MuMIn)
library(performance)
library(DHARMa)
library(sjPlot)

# 例：Recall の最大モデル（固定：Treatment*Time + 共変量、ランダム：Itemに傾き、Participantに切片）
Full_Model_Recall <- glmer(
  Score ~ Treatment * Time + Proficiency + Aptitude +
    (1 + Proficiency + Aptitude | Item) +
    (1 | Participant),
  data    = dat_recall,
  family  = binomial(),
  control = glmerControl(optimizer = "bobyqa")
)

16 ランダム効果の選択

  • 理論解説
  • コード例
  • 結果解釈

(1|群) はランダム切片、(0+傾き|群) はランダム傾き、|| は相関を外す指定です。singular fit は過度な複雑さや情報不足を示唆します。

コードを表示 
# 固定効果は Best_Fixed_Recall の式を流用し、ランダム効果を段階比較
form_fixed <- formula(Best_Fixed_Recall)
form_fixed
Score ~ Time + Treatment + (1 + Proficiency + Aptitude | Item) + 
    (1 | Participant) + Time:Treatment
コードを表示 
fit_r1 <- update(Best_Fixed_Recall, . ~ . - (1 + Proficiency + Aptitude | Item) + (1 | Item) + (1 | Participant))
fit_r2 <- update(Best_Fixed_Recall, . ~ . - (1 + Proficiency + Aptitude | Item) + (0 + Proficiency | Item) + (1 | Participant))
fit_r3 <- update(Best_Fixed_Recall, . ~ . - (1 + Proficiency + Aptitude | Item) + (0 + Aptitude    | Item) + (1 | Participant))
fit_r4 <- update(Best_Fixed_Recall, . ~ . - (1 + Proficiency + Aptitude | Item) + (1 + Proficiency || Item) + (1 | Participant))
fit_r5 <- update(Best_Fixed_Recall, . ~ . - (1 + Proficiency + Aptitude | Item) + (1 + Aptitude    || Item) + (1 | Participant))
fit_r6 <- update(Best_Fixed_Recall, . ~ . - (1 + Proficiency + Aptitude | Item) + (1 | Item) + (1 | Participant))

# はじめに

一般化線形混合効果モデル（GLMM）は、2値・頻度など**非正規**の目的変数に対して、**固定効果**と**ランダム効果**を同時に扱う枠組みです。本章では、語彙学習の架空データを用いて、**2値ロジスティックGLMM**を段階的に実装します。

# データの作成

```{r}
library(tidyverse)
theme_set(theme_bw())

set.seed(123)

n_participants <- 50
n_items        <- 30
treatments     <- c("Glossing","Writing")
tests          <- c("Recall","Recognition")
times          <- c("Immediate","Delayed")

dat <- expand.grid(
  Participant = 1:n_participants,
  Item        = 1:n_items,
  Treatment   = treatments,
  Test        = tests,
  Time        = times,
  KEEP.OUT.ATTRS = FALSE
)

# アイテム割当（Glossing:1-15, Writing:16-30）
dat <- dat[
  (dat$Treatment == "Glossing" & dat$Item <= 15) |
  (dat$Treatment == "Writing"  & dat$Item >  15), ]

# 0/1反応生成：Writing > Glossing, Recognition > Recall, Immediate > Delayed
dat$Score <- ifelse(
  dat$Treatment == "Glossing",
  rbinom(nrow(dat), 1, ifelse(dat$Test == "Recognition", 0.50, 0.20)),
  rbinom(nrow(dat), 1, ifelse(dat$Test == "Recognition", 0.90, 0.50))
)
dat$Score <- ifelse(dat$Time == "Immediate",
                    pmin(dat$Score + sample(0:1, nrow(dat), TRUE), 1),
                    dat$Score)
```

# 共変量の追加

```{r}
participants       <- unique(dat$Participant)
proficiency_values <- rnorm(length(participants), mean = 2500, sd = sqrt(500))
aptitude_values    <- rnorm(length(participants), mean = 10,   sd = sqrt(10))

participant_data <- tibble(
  Participant = participants,
  Proficiency = proficiency_values,
  Aptitude    = aptitude_values
)

dat <- dat |>
  left_join(participant_data, by = "Participant") |>
  mutate(
    Participant = factor(Participant),
    Item        = factor(Item),
    Treatment   = factor(Treatment),
    Time        = factor(Time),
    Proficiency = as.numeric(scale(Proficiency)),
    Aptitude    = as.numeric(scale(Aptitude))
  )
```

# データ整形

dat_recall       <- dplyr::filter(dat, Test == "Recall")
dat_recognition  <- dplyr::filter(dat, Test == "Recognition")

# 確認
dplyr::count(dat_recall, Treatment, Time) |> arrange(Treatment, Time)
dplyr::count(dat_recognition, Treatment, Time) |> arrange(Treatment, Time)

# コーディング

::: {.panel-tabset}

### 理論解説

カテゴリカル変数は **sum coding（和=0）** を用います。これにより、切片は**水準平均**、主効果は**両群の平均差**、交互作用は**傾きの差**として解釈できます。2水準ではレベルが {−1, +1} に再符号化され、**単純主効果の符号と大きさ**が読み取りやすくなります。

### コード例

```{r}
# 2水準のカテゴリカルは sum coding（和=0）
contrasts(dat_recall$Treatment) <- contr.sum(2)
contrasts(dat_recall$Time)      <- contr.sum(2)

contrasts(dat_recognition$Treatment) <- contr.sum(2)
contrasts(dat_recognition$Time)      <- contr.sum(2)
```

### 結果解釈

sum coding 下では、係数は**両群平均からの偏差**として現れます。可視化（`emmip`/`plot_model`）と合わせると交互作用の向きを直感的に確認できます。

:::

# モデル仕様（lme4）

```{r}
library(lme4)
library(lmerTest)   # p値の補助（連続目的のとき）。GLMMでは主に推定補助。
library(emmeans)
library(MuMIn)
library(performance)
library(DHARMa)
library(sjPlot)

# 例：Recall の最大モデル（固定：Treatment*Time + 共変量、ランダム：Itemに傾き、Participantに切片）
Full_Model_Recall <- glmer(
  Score ~ Treatment * Time + Proficiency + Aptitude +
    (1 + Proficiency + Aptitude | Item) +
    (1 | Participant),
  data    = dat_recall,
  family  = binomial(),
  control = glmerControl(optimizer = "bobyqa")
)
```

# 固定効果の選択（AICc）

::: {.panel-tabset}

### 理論解説

`MuMIn::dredge()` による AICc 最小化で固定効果候補を比較します。`options(na.action = "na.fail")` を設定し、**同一データ**でモデル群を評価します。AICc が近いモデルは**解釈の一貫性**や**理論的妥当性**で選びます。

### コード例

```{r}
options(na.action = "na.fail")
selection_recall <- MuMIn::dredge(Full_Model_Recall)   # AICc で順位付け
head(selection_recall)
plot(selection_recall, col.mode = "value", labAsExpr = TRUE)

Best_Fixed_Recall <- MuMIn::get.models(selection_recall, 1)[[1]]
summary(Best_Fixed_Recall)
```

### 結果解釈

`Best_Fixed_Recall` は AICc が最小の固定効果仕様です。以降のランダム効果選択では、この**固定効果式**を基礎にします。

:::

# ランダム効果の選択

::: {.panel-tabset}

### 理論解説

`(1|群)` はランダム切片、`(0+傾き|群)` はランダム傾き、`||` は**相関を外す**指定です。**singular fit** は過度な複雑さや情報不足を示唆します。

### コード例

```{r}
# 固定効果は Best_Fixed_Recall の式を流用し、ランダム効果を段階比較
form_fixed <- formula(Best_Fixed_Recall)
form_fixed

fit_r1 <- update(Best_Fixed_Recall, . ~ . - (1 + Proficiency + Aptitude | Item) + (1 | Item) + (1 | Participant))
fit_r2 <- update(Best_Fixed_Recall, . ~ . - (1 + Proficiency + Aptitude | Item) + (0 + Proficiency | Item) + (1 | Participant))
fit_r3 <- update(Best_Fixed_Recall, . ~ . - (1 + Proficiency + Aptitude | Item) + (0 + Aptitude    | Item) + (1 | Participant))
fit_r4 <- update(Best_Fixed_Recall, . ~ . - (1 + Proficiency + Aptitude | Item) + (1 + Proficiency || Item) + (1 | Participant))
fit_r5 <- update(Best_Fixed_Recall, . ~ . - (1 + Proficiency + Aptitude | Item) + (1 + Aptitude    || Item) + (1 | Participant))
fit_r6 <- update(Best_Fixed_Recall, . ~ . - (1 + Proficiency + Aptitude | Item) + (1 | Item) + (1 | Participant))

anova(fit_r1, fit_r2, fit_r3, fit_r4, fit_r5, fit_r6, test = "Chisq")
```

### 結果解釈

情報量基準と対数尤度のバランスで、**過度な相関付き傾き**を避けつつ、必要十分な構造を選びます。

:::

### 結果解釈
