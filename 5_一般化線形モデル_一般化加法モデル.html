8 はじめに

線形回帰分析（lm）は誤差が正規分布に従うことを前提にしています。しかし現実のデータは、2値（正解/不正解）、頻度（0,1,2,…）、歪みの大きい正の連続値など、正規分布から外れることが少なくありません。一般化線形モデル（GLM; glm）は、リンク関数と分布族（誤差分布）を指定して、これらを統一的に扱います。さらに、一般化加法モデル（GAM; mgcv::gam）は、説明変数ごとに平滑関数を用いて非線形を柔軟にモデリングできます。

8.1 GLM の要点

  • 線形予測子：\(\eta = \mathbf{X}\beta\)
  • リンク関数：\(g(\mu) = \eta\)（\(\mu = \mathbb{E}[Y]\)）
  • 分布族：指数型分布族（正規、二項、ポアソン、ガンマ、逆正規 など）

代表的な分布とリンク関数（既定）：

分布 (family) \(g(\mu)\) Rでの指定
正規 (gaussian) \(\mu\) family = gaussian(link = "identity")
二項 (binomial) \(\log\frac{\mu}{1-\mu}\) family = binomial(link = "logit")
ポアソン (poisson) \(\log(\mu)\) family = poisson(link = "log")
ガンマ (Gamma) \(1/\mu\) family = Gamma(link = "inverse")
逆正規 (Inverse.gaussian) \(1/\mu^2\) family = inverse.gaussian(link = "1/mu^2")
ノート

GLMの理論メモ（重要ポイント） - 正準リンク（canonical link）：各分布に「数式的に扱いやすい既定リンク」があります（例：二項→logit、ポアソン→log）。正準リンクは推定・漸近特性に利点がある一方、実務上は解釈の容易さで他のリンクを選ぶこともあります。
- 分散関数と分散パラメータ：\(\mathrm{Var}(Y)=\phi\,V(\mu)\)。二項・ポアソンの\(\phi\)（分散パラメータ）は既定で1。過分散がある場合はquasibinomialやquasipoisson、あるいは別分布（負の二項など）を検討します。
- 逸脱度（Deviance）：飽和モデルとの対数尤度差に基づく当てはまりの指標。AICは\(-2\log L + 2k\)。GAMでは平滑化パラメータも複雑さとして考慮されます。
- 係数の解釈：リンク関数の世界で線形和を作り、リンクの逆関数で平均\(\mu\)に戻して解釈します（例：ロジット→オッズ比、ポアソン→IRR）。

glm(formula, family, data) で、formula は lm と同様に y ~ x1 + x2 形式です。違いは family を明示する点です。

9 平滑化回帰

平滑化回帰は、近傍点に重みを付けて局所的に回帰する手法（例：loess）で、複雑な曲線関係を柔らかく捉えます。まず、非線形を含むデータを作り、多項式回帰と比べてみます。

コードを表示 
x1 <- seq(-10, 10, 0.1)
y1 <- 50 * sin(x1) + x1^2 + 10 * rnorm(length(x1), 0, 1)

plot_data <- tibble(x1 = x1, y1 = y1)

# 多項式回帰（2次/5次/8次）
lm_p2 <- lm(y1 ~ poly(x1, 2))
lm_p5 <- lm(y1 ~ poly(x1, 5))
lm_p8 <- lm(y1 ~ poly(x1, 8))

plot_data <- plot_data |>
  mutate(
    fit_p2 = fitted(lm_p2),
    fit_p5 = fitted(lm_p5),
    fit_p8 = fitted(lm_p8))

ggplot(plot_data, aes(x = x1)) +
  geom_point(aes(y = y1)) +
  geom_line(aes(y = fit_p2, colour = "2次多項式")) +
  geom_line(aes(y = fit_p5, colour = "5次多項式")) +
  geom_line(aes(y = fit_p8, colour = "8次多項式")) +
  labs(y = "y1", title = "多項式によるフィットの比較", colour = "モデルタイプ")

多項式次数を上げると当てはまりは改善する一方、外挿や端点の挙動が不安定になりがちです。次に、loess で平滑化回帰を行います。

summaryの読み方（要点）
- Equivalent Number of Parameters：複雑さの目安（大きいほど複雑）。
- Residual Standard Error：予測と実測のズレ。小さいほど当てはまり良好。
- Trace of smoother matrix：平滑化の強さの指標。

  • loess の主なパラメータ
  • 多項式回帰の注意点
  • loess vs 情報量規準
  • span：平滑度。小さいほど曲線は細かく追従、大きいほどなめらか。
  • degree：局所多項式の次数（通常 1 or 2）。
  • family："gaussian"（既定）/"symmetric"（外れ値にやや頑健）。
  • 次数を上げると境界で暴れる（ランウェイ効果/端点不安定）。
  • 係数の解釈が高次ほど困難。説明より予測用途に向きがち。
  • loessは最尤推定に基づかないため、AICは一般に使いません。
  • モデル選択はCVや外部検証、RSE、視覚的評価で行います。

10 Generalized Additive Model

GAM は \[ y = \alpha + f_1(x_1) + f_2(x_2) + \cdots + \varepsilon \] のように、各説明変数に平滑関数 \(f_j(\cdot)\) を割り当てる枠組みです。lm や glm は \(f_j(x)=\beta_j x\) という線形の特別な場合と見なせます。R では mgcv::gam() を用い、連続変数を s() で包むのが基本です。

10.1 GAMの要点と診断

  • 平滑基底とEDF
  • 平滑化パラメータの推定
  • 診断（必須）
  • s(x) はスプライン基底の線形結合。EDF（有効自由度）が曲線の複雑さ。
  • k は基底次元。小さすぎるとパターンを捉えきれず、大きすぎると計算負荷↑。
  • 既定のmethod = "REML"（推奨）か"GCV.Cp"で過剰適合を抑制。
  • select = TRUEで不要な平滑の自動縮退（ゼロ効果の罰則）を有効化。
  • gam.check(fit)：残差/QQ/スケール・kの適否（k'検定）
  • concurvity(fit)：曲線同士の依存（多重共線性のGAM版）
  • plot(fit, pages = 1, shade = TRUE, rug = TRUE, residuals = TRUE)：平滑の形状を確認

13 2値ロジスティック回帰と一般化加法モデル

2値データ（成功/失敗, 正解/不正解）には 二項分布を仮定し、glm(..., family = binomial()) を用います。非線形が疑われる場合は gam(..., family = binomial()) で logit スケール上の平滑化を導入できます。

コードを表示 
library(mgcv)
library(emmeans)
library(tidyverse)
library(performance)
library(sjPlot)

theme_set(theme_bw())

set.seed(123)
n <- 100
cat_var1 <- factor(sample(1:3, n, replace = TRUE))
cat_var2 <- factor(sample(0:1, n, replace = TRUE))
cont_var <- rnorm(n, mean = 60, sd = sqrt(20))

cat_effect  <- ifelse(cat_var1 == 1, 1, ifelse(cat_var1 == 2, 2, -1))
cont_effect <- sin(cont_var / 10) + cont_var^2 / 400

# ロジット線形予測子 → 確率
logit_p <- 0.5 * cat_effect + 0.05 * cont_effect
logit_p <- logit_p - mean(logit_p)  # シフトして平均0付近に
prob <- plogis(logit_p)

response <- rbinom(n, 1, prob)
data <- tibble(cat_var1, cat_var2, cont_var, response)

# はじめに

線形回帰分析（`lm`）は**誤差が正規分布**に従うことを前提にしています。しかし現実のデータは、2値（正解/不正解）、頻度（0,1,2,…）、歪みの大きい正の連続値など、正規分布から外れることが少なくありません。**一般化線形モデル（GLM; `glm`）**は、**リンク関数**と**分布族（誤差分布）**を指定して、これらを統一的に扱います。さらに、**一般化加法モデル（GAM; `mgcv::gam`）**は、説明変数ごとに**平滑関数**を用いて**非線形**を柔軟にモデリングできます。

## GLM の要点

- **線形予測子**：$\eta = \mathbf{X}\beta$  
- **リンク関数**：$g(\mu) = \eta$（$\mu = \mathbb{E}[Y]$）  
- **分布族**：指数型分布族（正規、二項、ポアソン、ガンマ、逆正規 など）

代表的な分布とリンク関数（既定）：

| 分布 (family) | $g(\mu)$ | Rでの指定 |
|---|---|---|
| 正規 (gaussian) | $\mu$ | `family = gaussian(link = "identity")` |
| 二項 (binomial) | $\log\frac{\mu}{1-\mu}$ | `family = binomial(link = "logit")` |
| ポアソン (poisson) | $\log(\mu)$ | `family = poisson(link = "log")` |
| ガンマ (Gamma) | $1/\mu$ | `family = Gamma(link = "inverse")` |
| 逆正規 (Inverse.gaussian) | $1/\mu^2$ | `family = inverse.gaussian(link = "1/mu^2")` |

::: {.callout-note}
**GLMの理論メモ（重要ポイント）**
- **正準リンク（canonical link）**：各分布に「数式的に扱いやすい既定リンク」があります（例：二項→logit、ポアソン→log）。正準リンクは推定・漸近特性に利点がある一方、実務上は解釈の容易さで他のリンクを選ぶこともあります。  
- **分散関数と分散パラメータ**：$\mathrm{Var}(Y)=\phi\,V(\mu)$。二項・ポアソンの**$\phi$（分散パラメータ）は既定で1**。**過分散**がある場合は`quasibinomial`や`quasipoisson`、あるいは別分布（負の二項など）を検討します。  
- **逸脱度（Deviance）**：飽和モデルとの対数尤度差に基づく**当てはまりの指標**。**AIC**は$-2\log L + 2k$。GAMでは平滑化パラメータも複雑さとして考慮されます。  
- **係数の解釈**：リンク関数の世界で線形和を作り、**リンクの逆関数**で平均$\mu$に戻して解釈します（例：ロジット→**オッズ比**、ポアソン→**IRR**）。  
:::

> `glm(formula, family, data)` で、`formula` は `lm` と同様に `y ~ x1 + x2` 形式です。**違いは `family`** を明示する点です。

# 平滑化回帰

**平滑化回帰**は、近傍点に重みを付けて局所的に回帰する手法（例：`loess`）で、複雑な曲線関係を柔らかく捉えます。まず、非線形を含むデータを作り、多項式回帰と比べてみます。

```{r}
x1 <- seq(-10, 10, 0.1)
y1 <- 50 * sin(x1) + x1^2 + 10 * rnorm(length(x1), 0, 1)

plot_data <- tibble(x1 = x1, y1 = y1)

# 多項式回帰（2次/5次/8次）
lm_p2 <- lm(y1 ~ poly(x1, 2))
lm_p5 <- lm(y1 ~ poly(x1, 5))
lm_p8 <- lm(y1 ~ poly(x1, 8))

plot_data <- plot_data |>
  mutate(
    fit_p2 = fitted(lm_p2),
    fit_p5 = fitted(lm_p5),
    fit_p8 = fitted(lm_p8))

ggplot(plot_data, aes(x = x1)) +
  geom_point(aes(y = y1)) +
  geom_line(aes(y = fit_p2, colour = "2次多項式")) +
  geom_line(aes(y = fit_p5, colour = "5次多項式")) +
  geom_line(aes(y = fit_p8, colour = "8次多項式")) +
  labs(y = "y1", title = "多項式によるフィットの比較", colour = "モデルタイプ")
```

多項式次数を上げると当てはまりは改善する一方、**外挿**や**端点の挙動**が不安定になりがちです。次に、`loess` で平滑化回帰を行います。

```{r}
# 平滑化回帰（loess）
loess_fit <- loess(y1 ~ x1, span = 0.3)  # span: 平滑度
summary(loess_fit)

plot_data$loess <- predict(loess_fit, newdata = data.frame(x1 = x1))

ggplot(plot_data, aes(x = x1)) +
  geom_point(aes(y = y1)) +
  geom_line(aes(y = loess), linewidth = 0.8) +
  labs(y = "y1", title = "平滑化回帰（loess）")
```

**summaryの読み方**（要点）  
- *Equivalent Number of Parameters*：複雑さの目安（大きいほど複雑）。  
- *Residual Standard Error*：予測と実測のズレ。小さいほど当てはまり良好。  
- *Trace of smoother matrix*：平滑化の強さの指標。  

::: {.panel-tabset}

### loess の主なパラメータ
- `span`：平滑度。**小さい**ほど曲線は細かく追従、**大きい**ほどなめらか。  
- `degree`：局所多項式の次数（通常 1 or 2）。  
- `family`：`"gaussian"`（既定）/`"symmetric"`（外れ値にやや頑健）。  

### 多項式回帰の注意点
- 次数を上げると境界で**暴れる**（ランウェイ効果/端点不安定）。  
- 係数の解釈が**高次**ほど困難。**説明より予測**用途に向きがち。  

### loess vs 情報量規準
- `loess`は**最尤推定に基づかない**ため、AICは一般に使いません。  
- モデル選択は**CV**や**外部検証**、**RSE**、**視覚的評価**で行います。  

:::

### ロバスト回帰との比較（参考）

外れ値に影響されにくい**ロバスト回帰（`MASS::rlm`）**と通常の最小二乗法（`lm`）を、`stackloss` データで比較します。

```{r}
data(stackloss)

fit_linear  <- lm(stack.loss ~ Air.Flow + Water.Temp + Acid.Conc., data = stackloss)
fit_robust  <- rlm(stack.loss ~ Air.Flow + Water.Temp + Acid.Conc., data = stackloss)

tab_model(fit_linear, show.aic = TRUE, show.aicc = TRUE, show.stat = TRUE, show.est = TRUE)
tab_model(fit_robust, show.aic = TRUE, show.aicc = TRUE, show.stat = TRUE, show.est = TRUE)

plot_models(fit_linear, fit_robust, show.values = TRUE, m.labels = c("Linear Model", "Robust Regression Model")) +
  theme_bw()
```

`performance::model_performance()` で適合度を一括算出します（AIC/AICc/BIC, $R^2$, RMSE等）。

```{r}
library(performance)
model_performance(fit_linear)
model_performance(fit_robust)
```

> **注意**：`loess` は**最尤推定**の枠組みではないため AIC は一般に定義しません。非線形の柔軟性と**情報量規準**によるモデル選択の両立には、**GAM** が便利です。

# Generalized Additive Model

**GAM** は
$$
y = \alpha + f_1(x_1) + f_2(x_2) + \cdots + \varepsilon
$$
のように、各説明変数に**平滑関数 $f_j(\cdot)$** を割り当てる枠組みです。`lm` や `glm` は $f_j(x)=\beta_j x$ という**線形**の特別な場合と見なせます。R では `mgcv::gam()` を用い、**連続変数を `s()` で包む**のが基本です。

## GAMの要点と診断

::: {.panel-tabset}

### 平滑基底とEDF
- `s(x)` はスプライン基底の線形結合。**EDF（有効自由度）**が**曲線の複雑さ**。  
- `k` は基底次元。**小さすぎ**るとパターンを捉えきれず、**大きすぎ**ると計算負荷↑。  

### 平滑化パラメータの推定
- 既定の`method = "REML"`（推奨）か`"GCV.Cp"`で**過剰適合を抑制**。  
- `select = TRUE`で**不要な平滑の自動縮退**（ゼロ効果の罰則）を有効化。  

### 診断（必須）
- `gam.check(fit)`：残差/QQ/スケール・`k`の適否（`k'`検定）  
- `concurvity(fit)`：**曲線同士の依存**（多重共線性のGAM版）  
- `plot(fit, pages = 1, shade = TRUE, rug = TRUE, residuals = TRUE)`：平滑の形状を確認  

```{r}

```
:::

# 重回帰分析と一般化加法モデル

ここでは、**連続×カテゴリカル**を含むデータを合成し、**重回帰（LM）**と**GAM**を比較します。

```{r}
library(mgcv)
library(emmeans)
library(tidyverse)
library(performance)
library(sjPlot)

set.seed(123)
n <- 100
cat_var1 <- factor(sample(1:3, n, replace = TRUE))  # 3水準
cat_var2 <- factor(sample(0:1, n, replace = TRUE))  # 2水準
cont_var <- rnorm(n, mean = 60, sd = sqrt(20))      # 連続

# 効果の設計
cat_effect  <- ifelse(cat_var1 == 1, 1, ifelse(cat_var1 == 2, 2, -1))
cont_effect <- 2 * sin(cont_var / 5) + cont_var^2 / 200

mu <- 60 + 5 * cat_effect + 5 * cont_effect
response <- rnorm(n, mean = mu, sd = 5)

data <- tibble(cat_var1, cat_var2, cont_var, response)

# 可視化
ggplot(data, aes(x = cont_var, y = response, color = cat_var1, shape = cat_var2)) +
  geom_point() +
  labs(x = "Continuous Variable", y = "Response", title = "Scatter Plot with Categorical Variables")
```

## LM と GAM の当てはめ

```{r}
# LM
linear_model <- lm(response ~ cat_var1 + cat_var2 + cont_var, data = data)

# GAM（連続変数を s() で平滑化）
gam_model <- gam(response ~ cat_var1 + cat_var2 + s(cont_var), data = data)

model_performance(linear_model)
model_performance(gam_model)
```

# 2値ロジスティック回帰と一般化加法モデル

**2値データ**（成功/失敗, 正解/不正解）には **二項分布**を仮定し、`glm(..., family = binomial())` を用います。非線形が疑われる場合は `gam(..., family = binomial())` で **logit** スケール上の**平滑化**を導入できます。

```{r}
library(mgcv)
library(emmeans)
library(tidyverse)
library(performance)
library(sjPlot)

theme_set(theme_bw())

set.seed(123)
n <- 100
cat_var1 <- factor(sample(1:3, n, replace = TRUE))
cat_var2 <- factor(sample(0:1, n, replace = TRUE))
cont_var <- rnorm(n, mean = 60, sd = sqrt(20))

cat_effect  <- ifelse(cat_var1 == 1, 1, ifelse(cat_var1 == 2, 2, -1))
cont_effect <- sin(cont_var / 10) + cont_var^2 / 400

# ロジット線形予測子 → 確率
logit_p <- 0.5 * cat_effect + 0.05 * cont_effect
logit_p <- logit_p - mean(logit_p)  # シフトして平均0付近に
prob <- plogis(logit_p)

response <- rbinom(n, 1, prob)
data <- tibble(cat_var1, cat_var2, cont_var, response)
```

## GLM（ロジスティック）と GAM（ロジスティック）

```{r}
# GLM
logistic_model <- glm(response ~ cat_var1 + cat_var2 + cont_var, family = binomial(), data = data)

summary(logistic_model)
tab_model(logistic_model, show.aic = TRUE, show.aicc = TRUE, show.stat = TRUE, show.est = TRUE)

plot_model(logistic_model, type = "pred", terms = c("cont_var", "cat_var1", "cat_var2"), show.data = TRUE)

model_performance(logistic_model)

(pw_logit <- emmeans(logistic_model, pairwise ~ cat_var1))
plot(pw_logit, comparisons = TRUE)
emmip(logistic_model, ~ cat_var1, CIs = TRUE)

# GAM（ロジスティック）
gam_logistic_model <- gam(response ~ cat_var1 + cat_var2 + s(cont_var), family = binomial(), data = data)

summary(gam_logistic_model)
tab_model(gam_logistic_model, show.aic = TRUE, show.aicc = TRUE, show.stat = TRUE, show.est = TRUE)

plot_model(gam_logistic_model, type = "pred", terms = c("cont_var", "cat_var1", "cat_var2"), show.data = TRUE)

model_performance(gam_logistic_model)

(pw_gam_logit <- emmeans(gam_logistic_model, pairwise ~ cat_var1))
plot(pw_gam_logit, comparisons = TRUE)
emmip(gam_logistic_model, ~ cat_var1, CIs = TRUE)
```

::: {.panel-tabset}

### オッズ比の解釈
- 係数の**指数変換**がオッズ比。**1**を跨ぐ95%CI→差がない可能性。  
- 連続変数は「1単位増でオッズが何倍か」。標準化すれば**1SD増**の解釈に。

### ROC/AUC としきい値
```{r}
performance::performance_roc(logistic_model)
performance::performance_roc(gam_logistic_model)
```
- AUCは**1に近いほど**識別性能が高い。しきい値0.5固定は最適でない場合がある。

### キャリブレーション（較正）
```{r}
check_model(logistic_model)       # ロジスティックの診断（残差/QQ/影響点）
check_model(gam_logistic_model)
```
- 予測確率が**過大/過小**に寄っていないかを視覚的に確認。

:::

> 出力の**オッズ比**は係数の指数変換です。95%CIに **1** を含むかどうかが差の判定の目安になります。

# ポアソン回帰分析と一般化加法モデル

**頻度データ**には **ポアソン回帰**を使います（`glm(..., family = poisson())`）。平均と分散が等しい仮定に反する場合は**過分散**に注意し、負の二項（`MASS::glm.nb`）や準ポアソンを検討します。非線形は `gam(..., family = poisson())` で扱えます。

```{r}
set.seed(123)
n <- 100
cat_var1 <- factor(sample(1:3, n, replace = TRUE))
cat_var2 <- factor(sample(0:1, n, replace = TRUE))
cont_var <- rnorm(n, mean = 60, sd = sqrt(20))

nonlinear_effect <- 0.1 * sin(cont_var / 10) + 0.001 * cont_var^2
lambda <- exp(0.1 * as.numeric(cat_var1) - 0.05 * as.numeric(cat_var2) + nonlinear_effect)
counts <- rpois(n, lambda)

data <- tibble(cat_var1, cat_var2, cont_var, counts)
```

## GLM（ポアソン）と GAM（ポアソン）

```{r}
# GLM
poisson_model <- glm(counts ~ cat_var1 + cat_var2 + cont_var, family = poisson(), data = data)
summary(poisson_model)
tab_model(poisson_model, show.aic = TRUE, show.aicc = TRUE, show.stat = TRUE, show.est = TRUE)

plot_model(poisson_model, type = "pred", terms = c("cont_var", "cat_var1", "cat_var2"), show.data = TRUE)
model_performance(poisson_model)

(pw_pois <- emmeans(poisson_model, pairwise ~ cat_var1))
plot(pw_pois, comparisons = TRUE)
emmip(poisson_model, ~ cat_var1, CIs = TRUE)

# GAM
gam_poisson_model <- gam(counts ~ cat_var1 + cat_var2 + s(cont_var), family = poisson(), data = data)
summary(gam_poisson_model)
tab_model(gam_poisson_model, show.aic = TRUE, show.aicc = TRUE, show.stat = TRUE, show.est = TRUE)

plot_model(gam_poisson_model, type = "pred", terms = c("cont_var", "cat_var1", "cat_var2"), show.data = TRUE)
model_performance(gam_poisson_model)

(pw_gam_pois <- emmeans(gam_poisson_model, pairwise ~ cat_var1))
plot(pw_gam_pois, comparisons = TRUE)
emmip(gam_poisson_model, ~ cat_var1, CIs = TRUE)
```

::: {.panel-tabset}

### 過分散チェック
```{r}
performance::check_overdispersion(poisson_model)
performance::check_overdispersion(gam_poisson_model)
```
- **過分散**（$\phi \gg 1$）なら `quasipoisson` や **負の二項**を検討。

### 負の二項（代替）
```{r}
MASS::glm.nb(counts ~ cat_var1 + cat_var2 + cont_var, data = data)
```

### オフセット（曝露量）
同じ「**時間・人数・面積**」あたりの発生率を比較するには、`offset(log(exposure))`を使います。
```{r}
set.seed(1)
data$exposure <- runif(nrow(data), min = 0.5, max = 2)  # 例：観測時間（時間）
glm(counts ~ cat_var1 + cat_var2 + cont_var + offset(log(exposure)),
    family = poisson(), data = data) |> tab_model()
```
:::

> `tab_model()` の **IRR（Incidence Rate Ratio）** は係数の指数変換。95%CIが **1** を跨ぐと差がない可能性を示唆します。

# その他

`glm` の `family` を変えることで、さまざまな目的変数に対応できます。代表例：

| 回帰 | 目的変数 | Rの関数 | 備考 |
|---|---|---|---|
| ベータ回帰 | 0–1 の比率 | `betareg::betareg()` | `link = "probit"` でプロビット可 |
| ガンマ回帰 | 正の連続 | `glm(family = Gamma())` | 既定リンクは inverse |
| 負の二項回帰 | 過分散カウント | `MASS::glm.nb()` | GLMM は `lme4::glmer.nb()` |
| ゼロ切断 | 0 を含まないカウント | `VGAM::vglm(family = pospoisson)` | Poisson/NegBin 両系統あり |
| ゼロ過剰 | 0 が過多のカウント | `pscl::zeroinfl()` | Poisson/NegBin 両系統あり |
| 多値ロジスティック | 名義多水準 | `nnet::multinom()` | `mgcv` にも `multinom`（名称衝突注意） |
| 順序ロジスティック | 順序カテゴリ | `MASS::polr()`, `ordinal::clm()` | `VGAM::vglm(family = acat())` は隣接法 |

::: {.callout-note}
**実務ヒント**
- **ゼロ過剰**：発生メカニズムが「**発生する/しない**」と「**何回起きる**」の**二段階**であるときに有効。  
- **準二項/準ポアソン**：分散推定を**自由化**（$\phi$推定）して**標準誤差**を補正。係数自体は（家族を変えない限り）同じでも**有意性が変わる**ことがある。  
- **リンク選択**：解釈（オッズ比/IRR）・安定性・予測精度の**三点**で評価。  
:::

> **プロビット**はロジットに近い結果を返します（係数は概ねロジット係数/1.7）。選択は好みと解釈の便宜によります。
