8 はじめに

英語教育研究では、従属変数が正規分布を仮定しにくい場面が多くあります。典型例は、 正誤（0/1）、相互作用回数（count）、正の連続量です。これらを 通常の線形回帰で処理すると、分布仮定と解釈がずれやすくなります。

この問題に対する標準的な枠組みが 一般化線形モデル（GLM）(Nelder & Wedderburn, 1972) であり、 非線形効果を柔軟に扱う拡張が 一般化加法モデル（GAM）(Hastie & Tibshirani, 1990; Wood, 2017, 2020) です。

本章は、理論説明を最小限にし、実データでそのまま再利用できる実務手順に重点を置きます。

5章と6章の使い分け
状況 使う章
観測が独立（反復測定なし） 本章（GLM / GAM）
同じ学習者から複数観測がある 6章（GLMM）
独立性 + 非線形の両方が問題 GAMM（6章末尾で概説）

迷ったら、まず本章で GLM/GAM を試し、章末の「6章への接続」で GLMM との比較を確認してください。

この章の到達目標
  • 分布とリンク関数を、研究目的に応じて選べる。
  • GLM と GAM を同じ研究課題で比較し、採用理由を説明できる。
  • 係数を OR / IRR / 倍率に変換し、応答スケール予測で可視化できる。
  • 章内コードをそのままテンプレートとして使える。

8.1 分析判断の早見表

9 理論はここだけ押さえる

GLM を一言でまとめると、「3ステップのレシピ」です。

  1. アウトカムの形を決める（2値なら binomial、カウントなら Poisson、正の連続量なら Gamma）
  2. リンク関数という橋を選ぶ（アウトカムの期待値と線形式をつなぐ変換）
  3. 最尤法で推定する（最小二乗法の代わりに、データの「起こりやすさ」を最大化するパラメータを探す）

この3要素を数式で書くと以下のようになります(Nelder & Wedderburn, 1972)。

  • 線形予測子: \(\eta_i = \mathbf{x}_i^\top\beta\)
  • リンク関数: \(g(\mu_i)=\eta_i\)
  • 分布: \(Y_i\) の分布（binomial, Poisson, Gamma など）

GAM は GLM の線形項を平滑関数 \(f(\cdot)\) に置き換え、 平滑化パラメータを REML 等で推定します(Wood, 2011, 2017)。

\[ g(\mu_i)=\beta_0 + f_1(x_{i1}) + \cdots + f_p(x_{ip}) \]

実装仕様は stats::glm / stats::family / mgcv の公式ドキュメントを参照してください (R Core Team, 2026b, 2026a; Wood, 2026)。

9.1 モデル比較のための尤度（必須）

モデル比較を説明するには、尤度を明示しておく必要があります。

\[ L(\theta \mid y) = \prod_{i=1}^{n} f(y_i \mid \theta), \quad \ell(\theta) = \log L(\theta \mid y) \]

  • 尤度 \(L\): 「このモデルが観測データをどれだけ起こしやすいか」
  • 対数尤度 \(\ell\): 尤度を足し算で扱える形にしたもの

実務でよく使う比較指標は以下です(Burnham & Anderson, 2002)。

\[ \mathrm{AIC} = -2\ell + 2k,\quad \mathrm{BIC} = -2\ell + k\log n \]

  • \(k\): 推定パラメータ数
  • 小さい AIC/BIC は、適合と複雑さのバランスがよいことを示す
実務での AIC/BIC の読み方

2つのモデルの AIC を比較するとき、差（ΔAIC）の大きさが判断材料になります(Burnham & Anderson, 2002)。

ΔAIC 解釈
0〜2 ほぼ同等。どちらを選んでも大きな差はない
4〜7 中程度の証拠。AIC が小さいモデルがやや優勢
10 以上 強い証拠。AIC が大きいモデルは支持されにくい

BIC はサンプルサイズが大きいとき AIC よりも複雑なモデルにペナルティを課すため、 AIC と BIC が同じ方向を向いていれば判断は明快です。

入れ子モデルなら尤度比検定（LRT）も使えます。

\[ 2\{\ell(\text{full})-\ell(\text{reduced})\} \sim \chi^2_{\Delta df} \]

LRT を使うときの条件
  • 同じ分布族・同じリンク関数・同じデータで比較する。
  • reduced と full が入れ子であることを確認する。
  • GAM では平滑化推定が入るため、GLMのLRTと同じ感覚で乱用しない。 本章では GAM 比較は主に AIC / compare_performance() / gam.check() で判断する。
LRT と AIC/BIC の使い分け
  • LRT: 入れ子モデル（一方が他方の特殊ケース）でのみ使用可能。p 値を返す。
  • AIC/BIC: 入れ子でなくても使用可能。異なる分布族の比較にも使える場合がある（ただし同じデータ・同じ尤度の定義が前提）。
  • 両方が同じ結論を指すなら、判断は明快です。食い違う場合は、理論的な整合性と診断結果を加味して最終判断してください。

9.2 モデル前提チェックリスト（GLM/GAM）

最もよくある間違い

「アウトカムの正規性」を Shapiro-Wilk 検定で確認する方が多いですが、 GLM ではアウトカムの正規性は仮定していません。 分布は family 引数で指定するものであり、2値データが正規分布に従わないのは当然です。 確認すべきは「family がアウトカムの型と整合しているか」と「残差に大きな問題がないか」です。

GLM/GAM の前提は「正規性」だけではありません。実務では次を最低限確認します。

  1. outcome と family/link が整合しているか（binary/count/positive）。
  2. 観測の独立性が成り立つか（反復・階層があるなら GLMM/GAMM へ）。
  3. GLM の線形項がリンクスケールで妥当か（不自然なら GAM を検討）。
  4. count で過分散がないか（Poisson 過信を避ける）。
  5. GAM の平滑項が過不足ないか（gam.check() の k-index / edf）。
  6. 共線性が強すぎないか（推定不安定の回避）。
前提確認の実装対応（本章）
  • 形状と平滑: gam.check(), concurvity()
  • 分布妥当性: check_overdispersion(), check_model()
  • 共線性: check_collinearity()
  • 階層性: 章末の「6章への接続」で GLMM 比較を実演

9.3 2値データのリンク関数ガイド（詳細）

2値アウトカムでは、同じ説明変数でもリンク関数の選び方で 「係数の意味」「尾部の当てはまり」「予測確率の形」が変わります。 stats::binomial() では logit, probit, cloglog, cauchit, log, identity が指定できます(R Core Team, 2026a, 2026c)。

Link \(g(p)\) \(g^{-1}(\\eta)\) 係数の主な解釈 実務メモ
logit \(\\log\\{p/(1-p)\\}\) \(\\mathrm{logit}^{-1}(\\eta)\) \(\\exp(\\beta)\) はオッズ比（OR） 最も標準的。解釈がしやすい
probit \(\\Phi^{-1}(p)\) \(\\Phi(\\eta)\) 潜在正規尺度での変化量 logit と予測は近いが係数尺度は異なる
cloglog \(\\log\\{-\\log(1-p)\\}\) \(1-\\exp\\{-\\exp(\\eta)\\}\) 非対称リンク 稀少イベント/離散時間ハザードで有用(Prentice & Gloeckler, 1978)
log-log \(-\\log\\{-\\log(p)\\}\) \(\\exp\\{-\\exp(-\\eta)\\}\) cloglog の鏡像 事象定義を反転したい場合に有用(Yee, 2026)
cauchit \(\\tan\\{\\pi(p-1/2)\\}\) \(\\arctan(\\eta)/\\pi + 1/2\) 重尾リンク 外れ値的な線形予測子に比較的頑健
log \(\\log(p)\) \(\\exp(\\eta)\) \(\\exp(\\beta)\) はリスク比に近い解釈 確率>1の制約に注意
identity \(p\) \(\\eta\) リスク差に近い [0,1] 制約を満たしにくく通常は非推奨
logit と probit の違い（誤解しやすい点）
  • 予測確率の形はしばしば似るが、係数尺度は同一ではない。
  • 係数比較では「どちらが有意か」より、応答スケール予測で比較する方が安全。
  • exp(beta) を OR として解釈できるのは logit のときのみ。
log-log について

log-log は cloglog の反転関係にあり、VGAM 文書でも 「0/1の定義を反転すると実質的に loglog が適用される」ことが示されています (Yee, 2026)。stats::binomial() には loglog 名が直接はないため、 必要なら VGAM か目的変数の符号反転で対応します。

12 ケース1: 助言関係の有無（Binomial GLM/GAM）

目的: 2人組の特徴（同クラスか、習熟度差、ベースライン差）で、 助言関係（有/無）を予測します。

12.1 モデル用データの作成（実データ）

ここでは、助言ネットワーク（30名）の全有向ペア（sender → receiver）を作成します。 各行が1つの有向ペアに対応し、アウトカム tie は実際に助言関係が観測されたら 1、そうでなければ 0 です。 予測子には、ペアレベルの特徴量を含めます。

  • same_class: 同じクラスに所属していれば 1
  • prof_gap: 習熟度の絶対差（0, 1, 2）
  • score_from_sc / score_to_sc: 送信者・受信者のベースライン得点（標準化済み）
  • score_gap: 得点の絶対差

出力は n_rows（全有向ペア数）と n_ties（実際に助言関係が観測されたペア数）を示します。 \(30 \times 29 = 870\) の有向ペアのうち、助言関係が観測されたのは一部です。 このベースレート（tie の割合）が低い場合、モデルは少数の 1 を多数の 0 から区別する必要があります。 これは2値データの典型的な特性であり、通常の線形回帰（OLS）ではなく GLM を使う理由の一つです。

ベースレートが極端に低い場合の注意

ベースレートが 5% 未満になると、完全分離（complete separation）や準完全分離（quasi-complete separation）が起きやすくなります。 これが生じると、一部の係数が極端に大きくなり、標準誤差が無限大に近づきます。 モデル当てはめ時に警告が出た場合は、この問題を疑ってください。

12.2 GLM と GAM

ここでは同じ研究課題に対し、4種類のリンク関数による GLM と 1つの GAM を当てはめます。 複数のリンク関数を試す目的は、有意な結果を「探す」ためではなく、 実質的な結論がリンク関数の選択に依存しないかを確認するためです。

各モデルの設定を理解しておきましょう。

  • family = binomial(link = "logit"): アウトカムが 0/1 であることを R に伝え、logit リンクを使う
  • family = binomial(link = "probit"): 同じ2値アウトカムだが、リンク関数を正規分布の累積分布関数に変更
  • family = binomial(link = "cloglog"): 非対称リンク。稀少イベントや離散時間ハザードモデルで有用
  • family = binomial(link = "cauchit"): 重い裾をもつリンク。外れ値的な線形予測子に頑健

GAM では、連続予測子 score_from_sc と score_to_sc を平滑関数 s() で扱い、 非線形関係を自動的に検出させます。

  • method = "REML": 平滑化パラメータの推定に制限付き最尤法を使用（推奨設定）
  • select = TRUE: 不要な平滑項をゼロに縮退させることを許可（自動変数選択に近い効果）
  • k = 8: 平滑項の基底次元の上限。実際に使われる自由度（edf）はデータから推定される

12.4.1 OR テーブルの読み方（worked interpretation）

各予測子の OR を以下のように解釈します。

  • same_class: OR が 1 より大きければ、同クラスのペアは異クラスに比べてその倍数だけ助言関係のオッズが高い。たとえば OR = 2.5 なら「同クラスは異クラスの 2.5 倍のオッズで助言関係をもつ」ことを意味します。
  • prof_gap: OR が 1 より大きければ、習熟度差が 1 段階大きくなるごとにオッズがその倍率で増加します。
  • score_from_sc: 送信者のベースライン得点が 1SD 高くなるごとのオッズの変化を示します。
  • score_to_sc: 受信者のベースライン得点が 1SD 高くなるごとのオッズの変化です。
  • score_gap: 送信者と受信者の得点差が 1SD 大きくなるごとのオッズの変化です。
  • prof_from / prof_to: カテゴリカル変数のため、参照水準（A2）と比較した各水準の OR です。
論文報告で OR を使うときの4点セット

OR を報告する際は、以下の4つを必ずセットで提示してください。

  1. 効果の方向（正 or 負）
  2. OR と 95% 信頼区間（CI が 1 をまたがなければ有意）
  3. p 値
  4. 応答スケールでの予測確率（下のセクションで算出）

OR だけでは「実際にどれくらい確率が変わるか」が伝わりにくいため、 予測確率の差を併記することで読者の理解が格段に向上します。

12.5 応答スケール予測（予測確率）

OR はオッズの比であり、確率そのものではありません。 同じ OR = 2 でも、ベースラインの確率が 5% のときと 50% のときでは、確率の変化幅が大きく異なります。 そのため、応答スケール（確率スケール）での予測を可視化し、 モデルの実質的な含意を直感的に伝えることが重要です。

ここでは、送信者のベースライン得点（score_from_sc）を −2 から +2 まで動かし、 同クラス / 異クラスの条件ごとに予測確率を算出します。 GLM と GAM の予測を重ねることで、「非線形を許すとどれだけ予測が変わるか」も確認できます。

13.2 Poisson から開始し、過分散を確認

カウントデータの自然な出発点は Poisson GLM です。 Poisson 分布は「平均 = 分散」を仮定しますが、 実データではしばしば分散が平均を上回ります（過分散）。 まず Poisson を当て、直後に過分散を検定します。

コード

  weight ~ same_class + prof_gap + baseline_mean_sc,
  family = poisson(link = "log"),
  data = dat_count
)

  • dispersion ratio: この値が 1 に近ければ Poisson の仮定は妥当。1 より大きく、かつ p 値が有意であれば過分散が存在します。
  • 過分散がある場合、Poisson の標準誤差は過小評価されます。その結果、本来有意でない効果が有意と判定されるリスク（偽陽性の膨張）が生じます。

Poisson 回帰で過分散を無視すると、標準誤差が実際より小さく推定され、Type I エラー率が膨張します。 過分散が検出された場合は、次のステップで負の二項（NB）モデルに切り替えます。

13.3 NB-GLM と NB-GAM

  • MASS::glm.nb(): NB-GLM を当てはめます。内部で theta を推定します。
  • mgcv::gam(..., family = nb()): NB-GAM。連続予測子の非線形を許容します。

13.3.1 NB-GLM 出力の読み方

NB-GLM の summary() 出力は Poisson GLM と構造が似ていますが、以下の点が異なります。

  • 係数は log スケール: Poisson と同様に、係数を指数変換すると IRR（Incidence Rate Ratio） になります。
  • Theta: 出力の末尾に表示される分散パラメータです。theta が小さいほど過分散が強いことを意味します。theta が十分大きい場合（例：数百以上）、NB は Poisson とほぼ同じ結果を返します。
  • AIC: Poisson と NB の AIC を比較することで、NB への切り替えが正当化されるかを確認できます（NB の AIC が小さければ、過分散を考慮する利得がある）。

14 ケース3: 協働強度（Positive Continuous: Gamma GLM/GAM）

目的: 学習者ごとの協働強度（重み総和）を、 ベースライン得点・クラス・習熟度で説明します。

14.1 GLM/GAM の当てはめ

正の連続量（協働強度: 重み総和）には Gamma 分布 + log リンク が自然な選択です。

  • Gamma 分布: 正の値のみをとり、右裾が長い分布に適合します。分散が平均に比例して大きくなるデータ（分散が一定でない場合）に向いています。
  • log リンク: 予測値が常に正になることを保証し、係数を乗法的（何倍か）に解釈できます。

比較のために、通常の線形回帰（OLS: 正規分布仮定）も当てはめます。 OLS は「加法的効果」（何単位増えるか）、Gamma GLM は「乗法的効果」（何倍になるか）を推定する点が根本的に異なります。

m_pos_gamma_glm <- glm(
  collab_strength ~ baseline_score_sc + class + proficiency,
  family = Gamma(link = "log"),

)

  collab_strength ~ s(baseline_score_sc, k = 8) + class + proficiency,

  method = "REML",

14.1.1 OLS vs Gamma GLM の出力比較

OLS（m_pos_lm）と Gamma GLM（m_pos_gamma_glm）の係数が異なるのは当然です。

  • OLS の係数: 「ベースライン得点が 1SD 上がると、協働強度が β 単位増える」（加法）
  • Gamma GLM の係数（log スケール）: 「ベースライン得点が 1SD 上がると、協働強度が exp(β) 倍になる」（乗法）

どちらが適切かは、データの性質に依存します。 分散が平均に比例する（得点が高い人ほどばらつきも大きい）場合は Gamma が適しています。 モデル比較（AIC/BIC）と check_model() の診断で判断します。

Gamma GLM の注意点

Gamma 分布は厳密に正の値のみをモデル化できます。 collab_strength に 0 が含まれる場合、Gamma GLM はエラーを返すか、不安定な推定になります。 0 を含むデータでは、小さな定数（例: 0.001）を加えるか、Tweedie 分布を検討してください。

15 6章への接続: モデル比較とランダム効果選択（最短実装）

この節は分析パイプラインで最も重要な判断を含む

5章で GLM/GAM の適合と分布選択が決まった後、データ構造が独立性を崩しているかを判断します。 助言ネットワークデータでは、同じ学習者が送信者として 29 回、受信者として 29 回登場します。 この依存構造を無視したまま GLM の結果を報告すると、標準誤差が過小になり偽陽性が増えます。

5章で GLM/GAM の判断ができたら、次に確認すべきは 「観測の独立性が崩れていないか」です。学習者・項目の反復がある場合は、 GLMM を候補に含めて比較します(Baayen ほか, 2008; Barr ほか, 2013; Jaeger, 2008; Meteyard & Davies, 2020)。

実務では次の順序が安定です。

  1. まず GLM（固定効果のみ）を作る。
  2. ランダム切片（例: from / to）を追加した GLMM を作る。
  3. 必要ならランダム傾きを試す。
  4. compare_performance()、anova(..., test="Chisq")、isSingular() を併用して採否を決める。

以下で5つの候補モデルを当てはめます。

  • m_bridge_glm: ランダム効果なし（5章の基準モデル）
  • m_bridge_from: 送信者のランダム切片のみ（送信者ごとの助言傾向の違いを許容）
  • m_bridge_to: 受信者のランダム切片のみ（受信者ごとの「求められやすさ」の違いを許容）
  • m_bridge_ri: 送信者 + 受信者の交差ランダム切片（両方を同時に考慮）
  • m_bridge_rs: 送信者のランダム傾き + 受信者のランダム切片（same_class 効果が送信者ごとに異なることを許容）

15.0.1 この出力をどう判断するか

compare_performance() テーブル:

  • GLM に比べて GLMM（ランダム切片あり）の AIC が 4 以上小さければ、依存構造の考慮が重要。
  • ランダム切片モデル同士（from のみ vs from + to）の差が小さければ、一方で十分かもしれません。

anova() の LRT:

  • m_bridge_from → m_bridge_ri の LRT が有意なら、受信者のランダム効果にも意味がある。
  • m_bridge_ri → m_bridge_rs が非有意なら、ランダム傾きの追加利得は小さい。

isSingular():

  • FALSE なら構造的に問題なし。
  • TRUE ならそのモデルのランダ効果は過剰。シンプルなモデルに戻してください。
5章での判断を6章へつなぐポイント
  • 非線形が主課題なら GAM、依存構造が主課題なら GLMM。
  • どちらも疑われる場合は、まず GLMM で依存構造を処理してから GAMM を検討する。
  • モデル選択は単一指標ではなく、適合指標・特異適合・理論整合性の3点で判断する。

17 トラブルシューティング（実務で詰まりやすい点）

family の不一致

症状: モデルは収束するが、診断プロットが全面的に破綻。予測値が理論上ありえない範囲に出る。

原因: outcome の型（0/1, count, positive continuous）と family が合っていない。

修正: 以下の対応表を確認してください。

Outcome 正しい family よくある間違い
0/1 binomial gaussian（OLS）
0以上の整数 poisson or nb() gaussian
正の連続量 Gamma(link="log") gaussian
過分散の見落とし

症状: Poisson GLM の係数が有意だが、NB に切り替えると非有意になる。

原因: Poisson は「平均 = 分散」を仮定するため、実際の分散が大きい場合に SE が過小評価される。

修正: count データでは必ず check_overdispersion() を最初に実行。dispersion ratio > 1 かつ有意なら NB に切り替え。

GAM の k が不足

症状: gam.check() で k-index が 0.8 未満、p 値が有意。

原因: 基底次元が小さすぎて、データの非線形パターンを捉えきれていない。

修正: k を倍に増やして再当てはめ（例: k = 8 → k = 16）。k-index ≈ 1 になるまで調整。ただし k をデータ数以上にはしない。

係数の解釈を混同する

症状: 「OR = 2 だから確率が2倍」と誤解する。

原因: OR はオッズの比であり、確率の比ではない。ベースラインの確率によって、同じ OR でも確率の変化幅は大きく異なる。

修正: 必ず応答スケール（予測確率 / 期待カウント / 期待値）を併記する。OR / IRR / 倍率だけで報告しない。

完全分離・準完全分離（binary GLM）

症状: 一部の係数が極端に大きい（例: 15〜20以上）。SE が非常に大きいか Inf。収束警告が出る。

原因: ある予測子の値でアウトカムが完全に 0 または 1 に分かれてしまう。

修正: (1) 該当する予測子のカテゴリを統合、(2) Firth のペナルティ付き尤度（brglm2 パッケージ）を検討、(3) サンプルサイズを増やす。

Gamma GLM でゼロ値エラー

症状: Error in eval(family$initialize): y values must be > 0

原因: Gamma 分布は厳密に正の値のみをモデル化するため、0 を含むデータでは使えない。

修正: (1) 小さな定数を加える（例: collab_strength + 0.001）、(2) Tweedie 分布（tweedie() in mgcv）を検討、(3) 0 を含む場合はハードルモデルを検討。

GAM の edf が k-1 に張り付く

症状: gam.check() で edf がほぼ k-1 と一致。

原因: 基底次元の上限まで平滑が使われており、これ以上の柔軟性が必要だが k が不足している。

修正: k を大きくして再当てはめ。edf が k-1 に張り付かなくなるまで増やす。

# はじめに

英語教育研究では、従属変数が正規分布を仮定しにくい場面が多くあります。典型例は、
**正誤（0/1）**、**相互作用回数（count）**、**正の連続量**です。これらを
通常の線形回帰で処理すると、分布仮定と解釈がずれやすくなります。

この問題に対する標準的な枠組みが
**一般化線形モデル（GLM）**[@nelder1972] であり、
非線形効果を柔軟に扱う拡張が
**一般化加法モデル（GAM）**[@hastie1990; @wood2017; @wood2020] です。

本章は、理論説明を最小限にし、**実データでそのまま再利用できる実務手順**に重点を置きます。

::: {.callout-important}

# 理論はここだけ押さえる

GLM を一言でまとめると、**「3ステップのレシピ」**です。

1. **アウトカムの形を決める**（2値なら binomial、カウントなら Poisson、正の連続量なら Gamma）
2. **リンク関数という橋を選ぶ**（アウトカムの期待値と線形式をつなぐ変換）
3. **最尤法で推定する**（最小二乗法の代わりに、データの「起こりやすさ」を最大化するパラメータを探す）

この3要素を数式で書くと以下のようになります[@nelder1972]。

- 線形予測子: $\eta_i = \mathbf{x}_i^\top\beta$
- リンク関数: $g(\mu_i)=\eta_i$
- 分布: $Y_i$ の分布（binomial, Poisson, Gamma など）

GAM は GLM の線形項を平滑関数 $f(\cdot)$ に置き換え、
平滑化パラメータを REML 等で推定します[@wood2011; @wood2017]。

$$
g(\mu_i)=\beta_0 + f_1(x_{i1}) + \cdots + f_p(x_{ip})
$$

実装仕様は `stats::glm` / `stats::family` / `mgcv` の公式ドキュメントを参照してください
[@rstatsglm; @rstatsfamily; @mgcvpkg]。

## モデル比較のための尤度（必須）

モデル比較を説明するには、尤度を明示しておく必要があります。

$$
L(\theta \mid y) = \prod_{i=1}^{n} f(y_i \mid \theta), \quad
\ell(\theta) = \log L(\theta \mid y)
$$

- 尤度 $L$: 「このモデルが観測データをどれだけ起こしやすいか」
- 対数尤度 $\ell$: 尤度を足し算で扱える形にしたもの

実務でよく使う比較指標は以下です[@burnham2002]。

$$
\mathrm{AIC} = -2\ell + 2k,\quad
\mathrm{BIC} = -2\ell + k\log n
$$

- $k$: 推定パラメータ数
- 小さい AIC/BIC は、適合と複雑さのバランスがよいことを示す

::: {.callout-tip}
### 実務での AIC/BIC の読み方

2つのモデルの AIC を比較するとき、差（ΔAIC）の大きさが判断材料になります[@burnham2002]。

| ΔAIC | 解釈 |
|---|---|
| 0〜2 | ほぼ同等。どちらを選んでも大きな差はない |
| 4〜7 | 中程度の証拠。AIC が小さいモデルがやや優勢 |
| 10 以上 | 強い証拠。AIC が大きいモデルは支持されにくい |

BIC はサンプルサイズが大きいとき AIC よりも複雑なモデルにペナルティを課すため、
AIC と BIC が同じ方向を向いていれば判断は明快です。
:::

入れ子モデルなら尤度比検定（LRT）も使えます。

$$
2\{\ell(\text{full})-\ell(\text{reduced})\}
\sim \chi^2_{\Delta df}
$$

::: {.callout-note}
### LRT を使うときの条件

- 同じ分布族・同じリンク関数・同じデータで比較する。
- reduced と full が入れ子であることを確認する。
- GAM では平滑化推定が入るため、GLMのLRTと同じ感覚で乱用しない。
  本章では GAM 比較は主に AIC / `compare_performance()` / `gam.check()` で判断する。
:::

::: {.callout-note}
### LRT と AIC/BIC の使い分け

- **LRT**: 入れ子モデル（一方が他方の特殊ケース）でのみ使用可能。p 値を返す。
- **AIC/BIC**: 入れ子でなくても使用可能。異なる分布族の比較にも使える場合がある（ただし同じデータ・同じ尤度の定義が前提）。
- 両方が同じ結論を指すなら、判断は明快です。食い違う場合は、理論的な整合性と診断結果を加味して最終判断してください。
:::

## モデル前提チェックリスト（GLM/GAM）

::: {.callout-important}
### 最もよくある間違い

「アウトカムの正規性」を Shapiro-Wilk 検定で確認する方が多いですが、
GLM では**アウトカムの正規性は仮定していません**。
分布は `family` 引数で指定するものであり、2値データが正規分布に従わないのは当然です。
確認すべきは「`family` がアウトカムの型と整合しているか」と「残差に大きな問題がないか」です。
:::

GLM/GAM の前提は「正規性」だけではありません。実務では次を最低限確認します。

1. `outcome` と `family/link` が整合しているか（binary/count/positive）。
2. 観測の独立性が成り立つか（反復・階層があるなら GLMM/GAMM へ）。
3. GLM の線形項がリンクスケールで妥当か（不自然なら GAM を検討）。
4. count で過分散がないか（Poisson 過信を避ける）。
5. GAM の平滑項が過不足ないか（`gam.check()` の k-index / edf）。
6. 共線性が強すぎないか（推定不安定の回避）。

::: {.callout-tip}
### 前提確認の実装対応（本章）

- 形状と平滑: `gam.check()`, `concurvity()`
- 分布妥当性: `check_overdispersion()`, `check_model()`
- 共線性: `check_collinearity()`
- 階層性: 章末の「6章への接続」で GLMM 比較を実演
:::

## 2値データのリンク関数ガイド（詳細）

2値アウトカムでは、同じ説明変数でもリンク関数の選び方で
「係数の意味」「尾部の当てはまり」「予測確率の形」が変わります。
`stats::binomial()` では `logit`, `probit`, `cloglog`, `cauchit`, `log`, `identity`
が指定できます[@rstatsfamily; @rmakelink]。

| Link | $g(p)$ | $g^{-1}(\\eta)$ | 係数の主な解釈 | 実務メモ |
|---|---|---|---|---|
| logit | $\\log\\{p/(1-p)\\}$ | $\\mathrm{logit}^{-1}(\\eta)$ | $\\exp(\\beta)$ はオッズ比（OR） | 最も標準的。解釈がしやすい |
| probit | $\\Phi^{-1}(p)$ | $\\Phi(\\eta)$ | 潜在正規尺度での変化量 | logit と予測は近いが係数尺度は異なる |
| cloglog | $\\log\\{-\\log(1-p)\\}$ | $1-\\exp\\{-\\exp(\\eta)\\}$ | 非対称リンク | 稀少イベント/離散時間ハザードで有用[@prentice1978] |
| log-log | $-\\log\\{-\\log(p)\\}$ | $\\exp\\{-\\exp(-\\eta)\\}$ | cloglog の鏡像 | 事象定義を反転したい場合に有用[@vgamcloglog] |
| cauchit | $\\tan\\{\\pi(p-1/2)\\}$ | $\\arctan(\\eta)/\\pi + 1/2$ | 重尾リンク | 外れ値的な線形予測子に比較的頑健 |
| log | $\\log(p)$ | $\\exp(\\eta)$ | $\\exp(\\beta)$ はリスク比に近い解釈 | 確率>1の制約に注意 |
| identity | $p$ | $\\eta$ | リスク差に近い | [0,1] 制約を満たしにくく通常は非推奨 |

::: {.callout-note}
### logit と probit の違い（誤解しやすい点）

- 予測確率の形はしばしば似るが、係数尺度は同一ではない。
- 係数比較では「どちらが有意か」より、**応答スケール予測**で比較する方が安全。
- `exp(beta)` を OR として解釈できるのは logit のときのみ。
:::

::: {.callout-note}
### log-log について

`log-log` は `cloglog` の反転関係にあり、`VGAM` 文書でも
「0/1の定義を反転すると実質的に loglog が適用される」ことが示されています
[@vgamcloglog]。`stats::binomial()` には `loglog` 名が直接はないため、
必要なら `VGAM` か目的変数の符号反転で対応します。
:::

# ケース1: 助言関係の有無（Binomial GLM/GAM）

**目的**: 2人組の特徴（同クラスか、習熟度差、ベースライン差）で、
助言関係（有/無）を予測します。

## モデル用データの作成（実データ）

ここでは、助言ネットワーク（30名）の全有向ペア（sender → receiver）を作成します。
各行が1つの有向ペアに対応し、アウトカム `tie` は実際に助言関係が観測されたら 1、そうでなければ 0 です。
予測子には、ペアレベルの特徴量を含めます。

- `same_class`: 同じクラスに所属していれば 1
- `prof_gap`: 習熟度の絶対差（0, 1, 2）
- `score_from_sc` / `score_to_sc`: 送信者・受信者のベースライン得点（標準化済み）
- `score_gap`: 得点の絶対差

```{r}
#| label: make-binary-data

# advice nodes (30名) に baseline score を結合
nodes_advice <- nodes_advice |>
  left_join(
    nodes_collab |>
      dplyr::select(id, baseline_score, baseline_score_sc),
    by = "id"
  ) |>
  mutate(
    class = factor(class),
    proficiency = factor(proficiency, levels = prof_levels),
    prof_num = unname(prof_map[as.character(proficiency)])
  )

# 全有向ペア（from != to）を作成
dat_bin <- tidyr::crossing(from = nodes_advice$id, to = nodes_advice$id) |>
  filter(from != to) |>
  left_join(
    nodes_advice |>
      dplyr::select(
        from = id,
        class_from = class,
        prof_from = proficiency,
        prof_num_from = prof_num,
        score_from_sc = baseline_score_sc
      ),
    by = "from"
  ) |>
  left_join(
    nodes_advice |>
      dplyr::select(
        to = id,
        class_to = class,
        prof_to = proficiency,
        prof_num_to = prof_num,
        score_to_sc = baseline_score_sc
      ),
    by = "to"
  ) |>
  mutate(
    same_class = as.integer(class_from == class_to),
    prof_gap = abs(prof_num_from - prof_num_to),
    score_gap = abs(score_from_sc - score_to_sc),
    tie = as.integer(paste(from, to) %in% paste(edges_advice$from, edges_advice$to))
  )

c(n_rows = nrow(dat_bin), n_ties = sum(dat_bin$tie))
```

出力は `n_rows`（全有向ペア数）と `n_ties`（実際に助言関係が観測されたペア数）を示します。
$30 \times 29 = 870$ の有向ペアのうち、助言関係が観測されたのは一部です。
この**ベースレート（tie の割合）**が低い場合、モデルは少数の 1 を多数の 0 から区別する必要があります。
これは2値データの典型的な特性であり、通常の線形回帰（OLS）ではなく GLM を使う理由の一つです。

::: {.callout-note}
### ベースレートが極端に低い場合の注意

ベースレートが 5% 未満になると、**完全分離**（complete separation）や**準完全分離**（quasi-complete separation）が起きやすくなります。
これが生じると、一部の係数が極端に大きくなり、標準誤差が無限大に近づきます。
モデル当てはめ時に警告が出た場合は、この問題を疑ってください。
:::

## GLM と GAM

ここでは同じ研究課題に対し、**4種類のリンク関数による GLM** と **1つの GAM** を当てはめます。
複数のリンク関数を試す目的は、有意な結果を「探す」ためではなく、
**実質的な結論がリンク関数の選択に依存しないか**を確認するためです。

各モデルの設定を理解しておきましょう。

- `family = binomial(link = "logit")`: アウトカムが 0/1 であることを R に伝え、logit リンクを使う
- `family = binomial(link = "probit")`: 同じ2値アウトカムだが、リンク関数を正規分布の累積分布関数に変更
- `family = binomial(link = "cloglog")`: 非対称リンク。稀少イベントや離散時間ハザードモデルで有用
- `family = binomial(link = "cauchit")`: 重い裾をもつリンク。外れ値的な線形予測子に頑健

GAM では、連続予測子 `score_from_sc` と `score_to_sc` を平滑関数 `s()` で扱い、
非線形関係を自動的に検出させます。

- `method = "REML"`: 平滑化パラメータの推定に制限付き最尤法を使用（推奨設定）
- `select = TRUE`: 不要な平滑項をゼロに縮退させることを許可（自動変数選択に近い効果）
- `k = 8`: 平滑項の基底次元の上限。実際に使われる自由度（edf）はデータから推定される

```{r}
#| label: fit-binary

f_bin <- tie ~ same_class + prof_gap + score_from_sc + score_to_sc + score_gap +
  prof_from + prof_to

m_bin_glm <- glm(
  f_bin,
  family = binomial(link = "logit"),
  data = dat_bin
)

m_bin_probit <- glm(
  f_bin,
  family = binomial(link = "probit"),
  data = dat_bin
)

m_bin_cloglog <- glm(
  f_bin,
  family = binomial(link = "cloglog"),
  data = dat_bin
)

m_bin_cauchit <- glm(
  f_bin,
  family = binomial(link = "cauchit"),
  data = dat_bin
)

m_bin_gam <- gam(
  tie ~ same_class + prof_gap +
    s(score_from_sc, k = 8) +
    s(score_to_sc, k = 8) +
    score_gap +
    prof_from + prof_to,
  family = binomial(link = "logit"),
  data = dat_bin,
  method = "REML",
  select = TRUE
)

summary(m_bin_glm)
summary(m_bin_gam)
```

### `summary(m_bin_glm)` の読み方（ロジスティック GLM）

出力の **Coefficients テーブル** を以下の手順で読みます。

| 列 | 意味 | 実務上の読み方 |
|---|---|---|
| `Estimate` | **ロジットスケール**の係数（確率ではない） | 正なら対数オッズが増加＝助言関係の可能性が高まる |
| `Std. Error` | 推定の不確実さ | 小さいほど精度が高い |
| `z value` | Wald 検定統計量（Estimate / SE） | 絶対値が約 2 以上なら有意に近い |
| `Pr(>|z|)` | p 値 | < .05 で「偶然では説明しにくい」と判断 |

- **Null deviance vs Residual deviance**: Null deviance は切片のみモデルの逸脱度、Residual deviance は当てはめ後の逸脱度です。差が大きいほど、予測子全体がモデルの適合を改善しています。
- **AIC**: モデル比較に使います（値が小さいほど、適合と複雑さのバランスがよい）。

::: {.callout-tip}
### 係数の符号を読む（ケース1）

- `same_class` の推定値が正 → 同クラスのペアは助言関係をもちやすい
- `prof_gap` の推定値が正 → 習熟度差が大きいペアほど助言関係をもちやすい
- `score_from_sc` の推定値が正 → ベースライン得点が高い送信者ほど助言を求めやすい

ただし、これらはすべて**ロジットスケール**です。実質的な意味を解釈するには、後述の OR（オッズ比）への変換が必要です。
:::

### `summary(m_bin_gam)` の読み方（GAM 固有の出力）

GAM の出力には GLM にはない **Approximate significance of smooth terms** セクションがあります。

| 項目 | 意味 | 判断基準 |
|---|---|---|
| `edf`（有効自由度） | 平滑項が実質的に使っている自由度 | edf ≈ 1 なら線形と同等（GAM の利点なし）。edf > 2 なら意味のある非線形 |
| `Ref.df` | 検定に使用される参照自由度 | 技術的な値（通常は解釈不要） |
| `Chi.sq` / `p-value` | 平滑項全体の有意性検定 | p < .05 なら「直線では捉えられない関係がある」 |

- edf が **0 に近い**場合（`select = TRUE` 使用時）：その平滑項はペナルティで完全に縮退しており、予測に寄与していません。
- **R-sq.(adj)**: 逸脱度ベースの近似決定係数。GLM の値と比較して GAM の追加利得を評価できます。

## リンク関数の比較（Case 1: 2値）

4つのリンク関数（logit, probit, cloglog, cauchit）が同じデータでどの程度異なる予測を生むかを確認します。
ここでの核心的な問いは「リンク関数の選択で結論が変わるか？」です。

まず `compare_performance()` で適合度を数値比較し、
その後、予測確率の曲線を視覚的に比較します。

```{r}
#| label: compare-link-binary

performance::compare_performance(
  m_bin_glm,
  m_bin_probit,
  m_bin_cloglog,
  m_bin_cauchit
) |>
  gt() |>
  tab_header(title = "Case 1: Link Function Comparison (GLM)")
```

上の表で **AIC 列**を比較します。4つの AIC が 2〜3 以内に収まっていれば、
このデータではリンク関数間に実質的な差はないと判断できます。
その場合は、**解釈のしやすさ（OR として報告可能）** から logit を採用するのが実務上の標準です。

逆に、1つのリンク関数だけ AIC が 4 以上小さければ、そのリンクがデータの裾の挙動をよりよく捉えている可能性があります。

次に、各リンク関数の予測確率を可視化して比較します。


**読み方のポイント**:

- 4本の曲線が中央付近でほぼ重なっている場合、リンク関数の選択は実質的に結果に影響しません。
- 差が生じるのは主に**裾（極端な得点域）**です。cloglog は上裾の立ち上がりが速く、cauchit は裾が重い（ゆっくり 0/1 に近づく）傾向があります。
- 曲線が大きく重なっていれば、**logit を採用して OR で報告する**のが最も通りがよい選択です。

## モデル比較と OR

GLM と GAM のどちらを最終モデルにするかを判断します。
判断基準は「GAM にすることで、GLM では捉えられなかった非線形パターンが改善されるか？」です。

まず情報量規準（AIC / BIC）で数値比較し、
次に尤度比検定（LRT）で入れ子モデルを統計的に比較し、
最後に `compare_performance()` で多指標を一括確認します。

```{r}
#| label: compare-binary

make_ic_table(
  "Binomial GLM" = m_bin_glm,
  "Binomial GAM" = m_bin_gam
) |>
  gt() |>
  tab_header(title = "Case 1: Model Comparison (Binary)")
```

AIC / BIC / deviance を比較します。GAM の AIC が GLM より 4 以上小さければ非線形の追加利得がある可能性があります。
差が小さければ、より解釈しやすい GLM を採用するのが実務上の安全策です。

次に、GLM 同士の入れ子比較（尤度比検定）を実演します。
ここでは `score_gap` を除いた reduced モデルと full モデルを比較し、
「この予測子を追加することで適合が統計的に改善するか」を検定します。

```{r}
#| label: likelihood-binary

# 尤度ベース比較（入れ子 GLM）
m_bin_glm_reduced <- update(m_bin_glm, . ~ . - score_gap)

tibble(
  model = c("Reduced GLM", "Full GLM"),
  logLik = c(as.numeric(logLik(m_bin_glm_reduced)), as.numeric(logLik(m_bin_glm))),
  AIC = c(AIC(m_bin_glm_reduced), AIC(m_bin_glm)),
  BIC = c(BIC(m_bin_glm_reduced), BIC(m_bin_glm))
) |>
  gt() |>
  tab_header(title = "Case 1: Likelihood-based Comparison (Nested GLM)")

anova(m_bin_glm_reduced, m_bin_glm, test = "Chisq")
```

`anova()` の出力で `Pr(>Chi)` が有意（< .05）であれば、`score_gap` の追加が適合を有意に改善したと判断できます。
非有意であれば、`score_gap` を除いたシンプルなモデルで十分です。

`compare_performance()` は AIC / BIC に加え、Tjur's R²（2値モデル専用の擬似決定係数）なども一括で比較でき、
単一指標への依存を避けたいときに便利です。

```{r}
#| label: perf-compare-binary

performance::compare_performance(m_bin_glm, m_bin_gam) |>
  gt() |>
  tab_header(title = "Case 1: compare_performance()")
```

`sjPlot::tab_model()` は、論文に直接貼れる形式の回帰結果テーブルを HTML で生成します。
OR（オッズ比）、信頼区間、p 値が自動計算されるため、報告用テーブルの作成に便利です。

```{r}
#| label: sjplot-binary-table

sjPlot::tab_model(
  m_bin_glm,
  show.aic = TRUE,
  show.ci = TRUE,
  show.se = TRUE,
  dv.labels = "Case 1 Binomial GLM"
)
```

ロジスティック回帰の係数をそのまま読んでも直感的ではありません（ロジットスケールのため）。
そこで、係数を**指数変換**して**オッズ比（OR: Odds Ratio）**に変換します。

- OR = 1: 効果なし
- OR > 1: その予測子が増えると、アウトカム（助言関係）のオッズが増加
- OR < 1: オッズが減少

`broom::tidy(..., exponentiate = TRUE)` がこの変換を自動で行います。

```{r}
#| label: binary-or

broom::tidy(m_bin_glm, conf.int = TRUE, exponentiate = TRUE) |>
  gt() |>
  tab_header(title = "Case 1: Odds Ratios (Binomial GLM)") |>
  cols_label(
    term = "Term",
    estimate = "OR",
    conf.low = "2.5%",
    conf.high = "97.5%",
    p.value = "p"
  )
```

### OR テーブルの読み方（worked interpretation）

各予測子の OR を以下のように解釈します。

- **same_class**: OR が 1 より大きければ、同クラスのペアは異クラスに比べてその倍数だけ助言関係のオッズが高い。たとえば OR = 2.5 なら「同クラスは異クラスの 2.5 倍のオッズで助言関係をもつ」ことを意味します。
- **prof_gap**: OR が 1 より大きければ、習熟度差が 1 段階大きくなるごとにオッズがその倍率で増加します。
- **score_from_sc**: 送信者のベースライン得点が 1SD 高くなるごとのオッズの変化を示します。
- **score_to_sc**: 受信者のベースライン得点が 1SD 高くなるごとのオッズの変化です。
- **score_gap**: 送信者と受信者の得点差が 1SD 大きくなるごとのオッズの変化です。
- **prof_from / prof_to**: カテゴリカル変数のため、参照水準（A2）と比較した各水準の OR です。

::: {.callout-tip}
### 論文報告で OR を使うときの4点セット

OR を報告する際は、以下の4つを必ずセットで提示してください。

1. **効果の方向**（正 or 負）
2. **OR と 95% 信頼区間**（CI が 1 をまたがなければ有意）
3. **p 値**
4. **応答スケールでの予測確率**（下のセクションで算出）

OR だけでは「実際にどれくらい確率が変わるか」が伝わりにくいため、
予測確率の差を併記することで読者の理解が格段に向上します。
:::

## 応答スケール予測（予測確率）

OR はオッズの比であり、確率そのものではありません。
同じ OR = 2 でも、ベースラインの確率が 5% のときと 50% のときでは、確率の変化幅が大きく異なります。
そのため、**応答スケール（確率スケール）での予測**を可視化し、
モデルの実質的な含意を直感的に伝えることが重要です。

ここでは、送信者のベースライン得点（`score_from_sc`）を −2 から +2 まで動かし、
同クラス / 異クラスの条件ごとに予測確率を算出します。
GLM と GAM の予測を重ねることで、「非線形を許すとどれだけ予測が変わるか」も確認できます。

```{r}
#| label: pred-binary

new_bin <- tidyr::expand_grid(
  same_class = c(0L, 1L),
  prof_gap = 1,
  score_from_sc = seq(-2, 2, length.out = 120),
  score_to_sc = 0,
  prof_from = factor("B1", levels = levels(dat_bin$prof_from)),
  prof_to = factor("B1", levels = levels(dat_bin$prof_to))
) |>
  mutate(score_gap = abs(score_from_sc - score_to_sc))

new_bin$prob_glm <- predict(m_bin_glm, newdata = new_bin, type = "response")
new_bin$prob_gam <- predict(m_bin_gam, newdata = new_bin, type = "response")

plot_bin <- new_bin |>

    model = dplyr::recode(model, prob_glm = "GLM", prob_gam = "GAM"),

  )

ggplot(plot_bin, aes(x = score_from_sc, y = prob, color = model, linetype = same_class)) +
  geom_line(linewidth = 1) +
  labs(
    x = "Sender Baseline Score (z)",
    y = "Predicted Probability of Advice Tie",
    title = "Case 1: Response-Scale Prediction"

```

**図の読み方**:

- **y 軸**: 助言関係の予測確率（0〜1）。読者にとって最も直感的なスケールです。
- **GLM と GAM の曲線がほぼ一致**: ロジットスケールでの線形仮定が妥当であり、より単純な GLM で十分。
- **GAM 曲線に明確な屈曲がある**: 非線形関係があり、GAM がより適切。
- **同クラスと異クラスの曲線の間隔**: `same_class` 効果の実質的な大きさを確率スケールで示します。間隔が大きいほど、クラスの同一性が助言関係に大きく影響しています。

## 下位検定（Case 1: 送信者/受信者習熟度）

GLM の係数テーブルは「習熟度が有意か」を教えてくれますが、
「具体的にどの習熟度レベル間で差があるのか」は教えてくれません（3水準以上のカテゴリ変数の場合）。

`emmeans`（estimated marginal means）を使うと、各水準における予測確率を算出し、
さらにペアごとの差を検定できます。
`type = "response"` を指定することで、ロジットスケールではなく**確率スケール**で結果が得られます。
Holm 法による多重比較補正を適用しています。

```{r}
#| label: posthoc-binary

emm_from <- emmeans(m_bin_glm, ~ prof_from, type = "response")
emm_to <- emmeans(m_bin_glm, ~ prof_to, type = "response")

as.data.frame(emm_from) |>
  gt() |>
  tab_header(title = "Case 1: EMMs for Sender Proficiency (response scale)")

pairs(emm_from, adjust = "holm") |>
  as.data.frame() |>
  gt() |>
  tab_header(title = "Case 1: Pairwise (Sender Proficiency, Holm)")

as.data.frame(emm_to) |>
  gt() |>
  tab_header(title = "Case 1: EMMs for Receiver Proficiency (response scale)")

pairs(emm_to, adjust = "holm") |>
  as.data.frame() |>
  gt() |>
  tab_header(title = "Case 1: Pairwise (Receiver Proficiency, Holm)")
```

### EMMs テーブルの読み方

- **EMMs テーブル**: 各習熟度水準（A2, B1, B2）の予測確率を示します。他の予測子は平均値に固定されています。「B2 の送信者は平均的に X% の確率で助言関係をもつ」のように読みます。
- **Pairwise テーブル**: 各ペア（A2-B1, A2-B2, B1-B2）の対数オッズ差と p 値を示します。`p.value` が Holm 補正後に .05 未満であれば、そのペア間に有意な差があります。
- 報告では、EMMs の確率値をペア間の差と合わせて提示すると、読者に伝わりやすくなります（例：「B2 送信者の予測確率は X% であり、A2 送信者の Y% と比べて有意に高かった, p = .ZZZ」）。

## 診断

モデルを報告する前に、主要な仮定が破綻していないかを確認します。
以下のタブで、それぞれ異なる側面を点検します。

::: {.panel-tabset}

### GAM 診断（gam.check）

`gam.check()` は GAM 固有の診断で、4つのプロットと基底次元の妥当性検定を出力します。

```{r}
#| label: diag-binary-gamcheck

gam.check(m_bin_gam)
```

**出力の読み方**:

- **左上（QQ プロット）**: 残差が対角線に沿っていれば分布仮定は妥当。裾で大きくずれていれば分布の不適合を示唆。
- **右上（残差 vs 線形予測子）**: 系統的なパターンがなければ OK。漏斗状の広がりがあれば不等分散を示唆。
- **k-index テーブル**: k-index が 1 に近ければ現在の基底次元（`k`）で十分。0.8 未満かつ p 値が有意なら `k` を増やすべき。

::: {.callout-important}
### k の判断ルール

`gam.check()` で k-index < 0.8 かつ p < .05 の場合、GAM を `k` を倍にして再当てはめしてください（例：`k = 8` → `k = 16`）。
k-index ≈ 1.0 であれば現在の設定で問題ありません。
:::

### Concurvity（平滑項間の冗長性）

`concurvity()` は、平滑項同士が冗長でないか（多重共線性の平滑版）を確認します。

```{r}
#| label: diag-binary-concurvity

concurvity(m_bin_gam, full = FALSE)
```

**判断基準**:

- 値が 0 に近い: 冗長性なし（問題なし）
- 0.3〜0.8: 中程度の冗長性（注意が必要だが致命的ではない）
- 0.8 以上: 強い冗長性。平滑項の統合または一方の削除を検討

### 過分散・共線性

GLM 側の診断として、過分散と共線性を確認します。

```{r}
#| label: diag-binary-overdispersion

performance::check_overdispersion(m_bin_glm)
performance::check_collinearity(m_bin_glm)
```

**過分散の読み方**:

- dispersion ratio が 1 に近ければ問題なし。2値データ（各行が 0/1）でグループ化していない場合、過分散は通常問題になりません。
- count データ（ケース2）ではこのチェックが特に重要です。

**共線性（VIF）の読み方**:

- VIF < 3: 問題なし
- VIF 3〜5: 中程度の共線性。推定は可能だが注意
- VIF > 5: 高い共線性。予測子の削除・統合を検討
- カテゴリカル変数の VIF は構造的にやや高くなりやすい点に留意

### 総合診断（check_model）

`check_model()` はモデル仮定の視覚的な一括チェックを提供します。

```{r}
#| label: diag-binary-checkmodel

performance::check_model(m_bin_glm)
performance::check_model(m_bin_gam)
```

**出力の読み方**:

- **Posterior predictive check**: モデルから生成したデータが実データに似ていれば、モデルの仮定は妥当。
- **Binned residuals**: 2値データでは通常の残差プロットが使えないため、残差をビンに分けて平均残差を確認します。ほとんどのビンがゼロ近辺にあれば OK。
- **Influential observations**: クック距離が大きい観測値がないかを確認します。

:::

::: {.callout-note}
### 実務メモ

- GAM の平滑項が縮退（edf がほぼ 0 または 1）した場合、
  「非線形の追加利得は小さい」と判断できる。
- 最終報告は OR と予測確率図を併記すると読み手に伝わりやすい。
- 診断で深刻な問題がなければ、論文には「主要な仮定違反は認められなかった」と1文で記述し、必要に応じて図を supplementary materials に掲載します。
:::

# ケース2: 協働回数（Count: Poisson/NB/GAM）

**目的**: 学習者ペアの協働回数（0を含む）を、
同クラスかどうか、習熟度差、ベースライン平均で説明します。

## モデル用データの作成（実データ）

協働ネットワーク（60名）の全無向ペアを作成します。
アウトカム `weight` は、各ペアの協働回数（0を含む整数値）です。
0 のペアが多いのはネットワークデータの典型的な特徴です。

予測子は以下のとおりです。

- `same_class`: 同クラスなら 1
- `prof_gap`: 習熟度の絶対差
- `baseline_mean_sc`: ペア両者のベースライン得点の平均（標準化済み）

```{r}
#| label: make-count-data

# 全ペア（無向）を作成
all_pairs <- tidyr::crossing(id1 = nodes_collab$id, id2 = nodes_collab$id) |>
  mutate(
    n1 = as.integer(gsub("S", "", id1)),
    n2 = as.integer(gsub("S", "", id2))
  ) |>
  filter(n1 < n2) |>
  dplyr::select(-n1, -n2)

# 観測されたエッジ重みを無向ペアへ統合
w_obs <- edges_collab |>
  mutate(
    a = pmin(from, to),
    b = pmax(from, to)
  ) |>
  group_by(a, b) |>
  summarise(weight = sum(weight), .groups = "drop")

# 全ペアに観測重みを結合（非観測は0）
dat_count <- all_pairs |>
  mutate(a = pmin(id1, id2), b = pmax(id1, id2)) |>
  left_join(w_obs, by = c("a", "b")) |>
  mutate(weight = tidyr::replace_na(weight, 0L)) |>
  left_join(
    nodes_collab |>
      dplyr::select(id1 = id, class1 = class, profn1 = prof_num, base1_sc = baseline_score_sc),
    by = "id1"
  ) |>
  left_join(
    nodes_collab |>
      dplyr::select(id2 = id, class2 = class, profn2 = prof_num, base2_sc = baseline_score_sc),
    by = "id2"
  ) |>
  mutate(
    same_class = as.integer(class1 == class2),
    prof_gap = abs(profn1 - profn2),
    baseline_mean_sc = (base1_sc + base2_sc) / 2
  )

summary(dat_count$weight)
sum(dat_count$weight > 0)
```

`summary()` で `weight` の分布を確認します。0 が多い分布は右裾が長くなる傾向があり、
正規分布ベースの OLS には不向きです。
`sum(dat_count$weight > 0)` は協働が実際に観測されたペア数を示します。

## Poisson から開始し、過分散を確認

カウントデータの自然な出発点は **Poisson GLM** です。
Poisson 分布は「平均 = 分散」を仮定しますが、
実データではしばしば分散が平均を上回ります（**過分散**）。
まず Poisson を当て、直後に過分散を検定します。

```{r}
#| label: fit-poisson-count

m_count_pois <- glm(
  weight ~ same_class + prof_gap + baseline_mean_sc,
  family = poisson(link = "log"),
  data = dat_count
)

performance::check_overdispersion(m_count_pois)
```

**過分散テストの読み方**:

- **dispersion ratio**: この値が 1 に近ければ Poisson の仮定は妥当。1 より大きく、かつ p 値が有意であれば過分散が存在します。
- 過分散がある場合、Poisson の標準誤差は**過小評価**されます。その結果、本来有意でない効果が有意と判定されるリスク（偽陽性の膨張）が生じます。

::: {.callout-warning}
### カウントデータでは過分散チェックを絶対に飛ばさない

Poisson 回帰で過分散を無視すると、標準誤差が実際より小さく推定され、Type I エラー率が膨張します。
過分散が検出された場合は、次のステップで負の二項（NB）モデルに切り替えます。
:::

## NB-GLM と NB-GAM

過分散が検出されたため、**負の二項（Negative Binomial, NB）分布**に切り替えます。
NB は Poisson に**分散パラメータ（theta）**を追加したモデルで、平均と分散を独立に扱えます。

- `MASS::glm.nb()`: NB-GLM を当てはめます。内部で theta を推定します。
- `mgcv::gam(..., family = nb())`: NB-GAM。連続予測子の非線形を許容します。

```{r}
#| label: fit-nb-count

m_count_nb_glm <- MASS::glm.nb(
  weight ~ same_class + prof_gap + baseline_mean_sc,
  data = dat_count
)

m_count_nb_gam <- gam(
  weight ~ same_class + prof_gap + s(baseline_mean_sc, k = 10),
  family = nb(),
  data = dat_count,
  method = "REML",
  select = TRUE
)

summary(m_count_nb_glm)
summary(m_count_nb_gam)
```

### NB-GLM 出力の読み方

NB-GLM の `summary()` 出力は Poisson GLM と構造が似ていますが、以下の点が異なります。

- **係数は log スケール**: Poisson と同様に、係数を指数変換すると **IRR（Incidence Rate Ratio）** になります。
- **Theta**: 出力の末尾に表示される分散パラメータです。theta が小さいほど過分散が強いことを意味します。theta が十分大きい場合（例：数百以上）、NB は Poisson とほぼ同じ結果を返します。
- **AIC**: Poisson と NB の AIC を比較することで、NB への切り替えが正当化されるかを確認できます（NB の AIC が小さければ、過分散を考慮する利得がある）。

### NB-GAM 出力の読み方

GAM 固有の出力（edf, k-index）の読み方はケース1と同じです。
edf ≈ 1 であれば、`baseline_mean_sc` の非線形効果は小さく、NB-GLM で十分です。

## モデル比較と IRR

Poisson, NB-GLM, NB-GAM の3モデルを情報量規準で比較します。
通常、過分散がある場合は Poisson の AIC が最も大きく（最も悪い適合）なります。

:::

# ケース3: 協働強度（Positive Continuous: Gamma GLM/GAM）

**目的**: 学習者ごとの協働強度（重み総和）を、
ベースライン得点・クラス・習熟度で説明します。

## GLM/GAM の当てはめ

正の連続量（協働強度: 重み総和）には **Gamma 分布 + log リンク** が自然な選択です。

- **Gamma 分布**: 正の値のみをとり、右裾が長い分布に適合します。分散が平均に比例して大きくなるデータ（分散が一定でない場合）に向いています。
- **log リンク**: 予測値が常に正になることを保証し、係数を**乗法的**（何倍か）に解釈できます。

比較のために、通常の線形回帰（OLS: 正規分布仮定）も当てはめます。
OLS は「加法的効果」（何単位増えるか）、Gamma GLM は「乗法的効果」（何倍になるか）を推定する点が根本的に異なります。

```{r}
#| label: fit-positive

m_pos_lm <- lm(
  collab_strength ~ baseline_score_sc + class + proficiency,
  data = dat_pos
)

m_pos_gamma_glm <- glm(
  collab_strength ~ baseline_score_sc + class + proficiency,
  family = Gamma(link = "log"),
  data = dat_pos
)

m_pos_gamma_gam <- gam(
  collab_strength ~ s(baseline_score_sc, k = 8) + class + proficiency,
  family = Gamma(link = "log"),
  data = dat_pos,
  method = "REML",
  select = TRUE
)

summary(m_pos_lm)
summary(m_pos_gamma_glm)
summary(m_pos_gamma_gam)
```

### OLS vs Gamma GLM の出力比較

OLS（`m_pos_lm`）と Gamma GLM（`m_pos_gamma_glm`）の係数が異なるのは当然です。

- **OLS の係数**: 「ベースライン得点が 1SD 上がると、協働強度が β 単位増える」（加法）
- **Gamma GLM の係数（log スケール）**: 「ベースライン得点が 1SD 上がると、協働強度が exp(β) 倍になる」（乗法）

どちらが適切かは、データの性質に依存します。
分散が平均に比例する（得点が高い人ほどばらつきも大きい）場合は Gamma が適しています。
モデル比較（AIC/BIC）と `check_model()` の診断で判断します。

::: {.callout-note}
### Gamma GLM の注意点

Gamma 分布は**厳密に正の値**のみをモデル化できます。
`collab_strength` に 0 が含まれる場合、Gamma GLM はエラーを返すか、不安定な推定になります。
0 を含むデータでは、小さな定数（例: 0.001）を加えるか、Tweedie 分布を検討してください。
:::

:::

# 6章への接続: モデル比較とランダム効果選択（最短実装）

::: {.callout-important}
### この節は分析パイプラインで最も重要な判断を含む

5章で GLM/GAM の適合と分布選択が決まった後、**データ構造が独立性を崩しているか**を判断します。
助言ネットワークデータでは、同じ学習者が送信者として 29 回、受信者として 29 回登場します。
この依存構造を無視したまま GLM の結果を報告すると、標準誤差が過小になり偽陽性が増えます。
:::

5章で GLM/GAM の判断ができたら、次に確認すべきは
「観測の独立性が崩れていないか」です。学習者・項目の反復がある場合は、
GLMM を候補に含めて比較します[@jaeger2008; @baayen2008; @barr2013; @meteyard2020]。

実務では次の順序が安定です。

1. まず GLM（固定効果のみ）を作る。
2. ランダム切片（例: `from` / `to`）を追加した GLMM を作る。
3. 必要ならランダム傾きを試す。
4. `compare_performance()`、`anova(..., test="Chisq")`、`isSingular()` を併用して採否を決める。

以下で5つの候補モデルを当てはめます。

- `m_bridge_glm`: ランダム効果なし（5章の基準モデル）
- `m_bridge_from`: 送信者のランダム切片のみ（送信者ごとの助言傾向の違いを許容）
- `m_bridge_to`: 受信者のランダム切片のみ（受信者ごとの「求められやすさ」の違いを許容）
- `m_bridge_ri`: 送信者 + 受信者の交差ランダム切片（両方を同時に考慮）
- `m_bridge_rs`: 送信者のランダム傾き + 受信者のランダム切片（`same_class` 効果が送信者ごとに異なることを許容）

```{r}
#| label: bridge-glm-to-glmm

f_bridge <- tie ~ same_class + prof_gap + score_from_sc + score_to_sc + score_gap

m_bridge_glm <- glm(
  f_bridge,
  family = binomial(link = "logit"),
  data = dat_bin
)

m_bridge_from <- lme4::glmer(
  update(f_bridge, . ~ . + (1 | from)),
  family = binomial(link = "logit"),
  data = dat_bin,
  control = lme4::glmerControl(
    optimizer = "bobyqa",
    optCtrl = list(maxfun = 2e5)
  )
)

m_bridge_to <- lme4::glmer(
  update(f_bridge, . ~ . + (1 | to)),
  family = binomial(link = "logit"),
  data = dat_bin,
  control = lme4::glmerControl(
    optimizer = "bobyqa",
    optCtrl = list(maxfun = 2e5)
  )
)

m_bridge_ri <- lme4::glmer(
  update(f_bridge, . ~ . + (1 | from) + (1 | to)),
  family = binomial(link = "logit"),
  data = dat_bin,
  control = lme4::glmerControl(
    optimizer = "bobyqa",
    optCtrl = list(maxfun = 2e5)
  )
)

m_bridge_rs <- lme4::glmer(
  update(f_bridge, . ~ . + (1 + same_class | from) + (1 | to)),
  family = binomial(link = "logit"),
  data = dat_bin,
  control = lme4::glmerControl(
    optimizer = "bobyqa",
    optCtrl = list(maxfun = 2e5)
  )
)

performance::compare_performance(
  m_bridge_glm,
  m_bridge_from,
  m_bridge_to,
  m_bridge_ri,
  m_bridge_rs
) |>
  gt() |>
  tab_header(title = "Bridge: GLM vs GLMM Candidates")

anova(m_bridge_from, m_bridge_ri, test = "Chisq")
anova(m_bridge_to, m_bridge_ri, test = "Chisq")
anova(m_bridge_ri, m_bridge_rs, test = "Chisq")

lme4::isSingular(m_bridge_ri)
lme4::isSingular(m_bridge_rs)
performance::check_model(m_bridge_ri)
```

### この出力をどう判断するか

**`compare_performance()` テーブル**:

- GLM に比べて GLMM（ランダム切片あり）の AIC が 4 以上小さければ、依存構造の考慮が重要。
- ランダム切片モデル同士（from のみ vs from + to）の差が小さければ、一方で十分かもしれません。

**`anova()` の LRT**:

- `m_bridge_from` → `m_bridge_ri` の LRT が有意なら、受信者のランダム効果にも意味がある。
- `m_bridge_ri` → `m_bridge_rs` が非有意なら、ランダム傾きの追加利得は小さい。

**`isSingular()`**:

- `FALSE` なら構造的に問題なし。
- `TRUE` ならそのモデルのランダ効果は過剰。シンプルなモデルに戻してください。

::: {.callout-tip}
### 5章での判断を6章へつなぐポイント

- 非線形が主課題なら GAM、依存構造が主課題なら GLMM。
- どちらも疑われる場合は、まず GLMM で依存構造を処理してから GAMM を検討する。
- モデル選択は単一指標ではなく、適合指標・特異適合・理論整合性の3点で判断する。
:::

# トラブルシューティング（実務で詰まりやすい点）

::: {.callout-warning}
### `family` の不一致

**症状**: モデルは収束するが、診断プロットが全面的に破綻。予測値が理論上ありえない範囲に出る。

**原因**: outcome の型（0/1, count, positive continuous）と `family` が合っていない。

**修正**: 以下の対応表を確認してください。

| Outcome | 正しい family | よくある間違い |
|---|---|---|
| 0/1 | `binomial` | `gaussian`（OLS） |
| 0以上の整数 | `poisson` or `nb()` | `gaussian` |
| 正の連続量 | `Gamma(link="log")` | `gaussian` |
:::

::: {.callout-warning}
### 過分散の見落とし

**症状**: Poisson GLM の係数が有意だが、NB に切り替えると非有意になる。

**原因**: Poisson は「平均 = 分散」を仮定するため、実際の分散が大きい場合に SE が過小評価される。

**修正**: count データでは必ず `check_overdispersion()` を最初に実行。dispersion ratio > 1 かつ有意なら NB に切り替え。
:::

::: {.callout-warning}
### GAM の `k` が不足

**症状**: `gam.check()` で k-index が 0.8 未満、p 値が有意。

**原因**: 基底次元が小さすぎて、データの非線形パターンを捉えきれていない。

**修正**: `k` を倍に増やして再当てはめ（例: `k = 8` → `k = 16`）。k-index ≈ 1 になるまで調整。ただし k をデータ数以上にはしない。
:::

::: {.callout-warning}
### 係数の解釈を混同する

**症状**: 「OR = 2 だから確率が2倍」と誤解する。

**原因**: OR はオッズの比であり、確率の比ではない。ベースラインの確率によって、同じ OR でも確率の変化幅は大きく異なる。

**修正**: 必ず応答スケール（予測確率 / 期待カウント / 期待値）を併記する。OR / IRR / 倍率だけで報告しない。
:::

::: {.callout-warning}
### 完全分離・準完全分離（binary GLM）

**症状**: 一部の係数が極端に大きい（例: 15〜20以上）。SE が非常に大きいか `Inf`。収束警告が出る。

**原因**: ある予測子の値でアウトカムが完全に 0 または 1 に分かれてしまう。

**修正**: (1) 該当する予測子のカテゴリを統合、(2) Firth のペナルティ付き尤度（`brglm2` パッケージ）を検討、(3) サンプルサイズを増やす。
:::

::: {.callout-warning}
### Gamma GLM でゼロ値エラー

**症状**: `Error in eval(family$initialize): y values must be > 0`

**原因**: Gamma 分布は厳密に正の値のみをモデル化するため、0 を含むデータでは使えない。

**修正**: (1) 小さな定数を加える（例: `collab_strength + 0.001`）、(2) Tweedie 分布（`tweedie()` in mgcv）を検討、(3) 0 を含む場合はハードルモデルを検討。
:::

::: {.callout-warning}
### GAM の edf が k-1 に張り付く

**症状**: `gam.check()` で edf がほぼ k-1 と一致。

**原因**: 基底次元の上限まで平滑が使われており、これ以上の柔軟性が必要だが k が不足している。

**修正**: `k` を大きくして再当てはめ。edf が k-1 に張り付かなくなるまで増やす。
:::
