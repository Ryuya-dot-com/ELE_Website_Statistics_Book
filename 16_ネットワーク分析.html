18.1 イントロダクション

ネットワーク分析（social network analysis; SNA）は、「個体の属性」ではなく「個体間の関係」を分析単位に置く方法論である。従来の回帰分析や分散分析は個々の学習者を独立した観測単位として扱うが、ネットワーク分析は学習者間の相互作用の構造そのものを対象とする。この視点の転換により、従来の変数中心アプローチ（variable-centered approach）では捉えきれなかった社会的文脈の影響を定量的に分析できるようになる(Kennedy Terry & Bayley, 2024; Saqr ほか, 2024)。

英語教育研究では、ネットワーク分析によって以下のような問いに直接的に答えることが可能である。

  • 協働学習において、誰がグループのハブ（情報結節点）として機能しているか
  • クラス内の交流パターンがクラス外へどの程度広がっているか
  • 介入前後で学習者コミュニティの分断が弱まったか、あるいは強まったか
  • 留学先での社会的ネットワーク構造がL2習熟度の伸びにどう関連するか(Isbell, 2024; Strawbridge, 2023)

本章は、英語教育専攻の大学院生（回帰分析の基礎既習）を対象に、理論と実装の両面からネットワーク分析を体系的に解説する。具体的には、以下の4部構成で進める。

  1. 理論基礎: グラフ理論の数学的定義と、SLA研究における意義
  2. 記述分析: igraph / tidygraph / ggraph による可視化・要約・中心性分析
  3. 統計的推論: ERGM（指数ランダムグラフモデル）とQAP回帰による仮説検定
  4. 応用と報告: SLA研究への具体的な応用例とAPA報告テンプレート
この章の到達目標
  • グラフ理論の基本概念（ノード、エッジ、経路、次数）を定義し、説明できる。
  • igraph / tidygraph / ggraph を用いてネットワークの構築・可視化・要約ができる。
  • 中心性指標の違いを理解し、研究目的に合った指標を選択できる。
  • コミュニティ検出を実行し、外部変数と照合して解釈できる。
  • ERGMの基本構造を理解し、ネットワーク形成メカニズムを検定できる。
  • QAP回帰でネットワーク相関・回帰を実行できる。
  • ネットワーク分析の結果をAPA形式で報告できる。
読む順序（迷ったとき）
  • 最短ルート（30分）: イントロ → セットアップ → データ構築 → 可視化 → 中心性 → 報告テンプレート
  • 標準ルート（60分）: 上記 + グラフ理論基礎 + コミュニティ検出 + 有向ネットワーク + 回帰接続
  • 拡張ルート（90分〜）: 上記 + ERGM + QAP回帰 + SLA応用事例 + 感度分析
この章の立場
  • 本章は「協働ネットワーク（学習者間）」に焦点を絞る。語彙共起ネットワークや空間ネットワークは別の分析フレームワークに属するため、ここでは扱わない。
  • 目的は「多機能なコード集」ではなく、研究で使える分析思考を身につけることである。

18.2 グラフ理論の基礎

ネットワーク分析の数学的基盤はグラフ理論（graph theory）である。ここでは、分析に必要な最小限の数学的定義を、SLA研究の文脈に即して導入する。

18.2.1 グラフの数学的定義

グラフ \(G\) は、ノード（頂点）の集合 \(V\) とエッジ（辺）の集合 \(E\) の組として定義される。

\[G = (V, E)\]

ここで \(V = \{v_1, v_2, \ldots, v_n\}\) はネットワークに含まれるノード（例: 学習者）の集合であり、\(E \subseteq V \times V\) はノード間の関係（例: 協働経験の有無）の集合である。

グラフの構造は隣接行列（adjacency matrix）\(\mathbf{A}\) によって表現できる。\(n \times n\) の行列 \(\mathbf{A}\) の各要素は次のように定義される。

\[A_{ij} = \begin{cases} 1 & \text{if } (v_i, v_j) \in E \\ 0 & \text{otherwise} \end{cases}\]

無向グラフ（undirected graph）では \(A_{ij} = A_{ji}\)（対称行列）であり、有向グラフ（directed graph）では非対称となりうる。英語教育研究では、「協働作業を行った」という関係は無向（双方向的）であるのに対し、「相談をした」という関係は有向（一方向的）である場合が多い。

重み付きグラフ（weighted graph）では、隣接行列の要素がエッジの重み \(w_{ij}\)（例: 協働回数）を取る。

\[W_{ij} = \begin{cases} w_{ij} & \text{if } (v_i, v_j) \in E \\ 0 & \text{otherwise} \end{cases}\]

18.2.2 主要な指標の数学的定義

次数と強度

次数（degree）は、あるノードが持つエッジの数である。

\[k_i = \sum_{j=1}^{n} A_{ij}\]

重み付きグラフでは、強度（strength）がエッジの重みの合計として定義される。

\[s_i = \sum_{j=1}^{n} W_{ij}\]

次数は「何人と関わったか」を、強度は「どの程度の量の関わりがあったか」を表す。留学研究において、ネットワークの次数（接触人数）よりも強度（接触頻度）の方がL2習熟度の伸びをよく予測するという知見がある(Isbell, 2024)。

経路と測地距離

ノード \(v_i\) から \(v_j\) への経路（path）は、エッジを辿ってたどり着くまでの一連のノード列である。最短経路の長さを測地距離（geodesic distance）\(d(v_i, v_j)\) と呼ぶ。測地距離が短いほど、情報や資源が素早く伝達されうることを意味する。

密度

密度（density）は、理論上可能なエッジの総数に対する実際のエッジ数の比率であり、ネットワーク全体の「密な」度合いを示す。

\[D = \frac{2|E|}{|V|(|V| - 1)} \quad \text{（無向グラフの場合）}\]

密度 \(D = 1\) はすべてのノードが互いに接続された完全グラフ、\(D = 0\) はエッジが一つもない状態である。教室ネットワークでは、密度は「クラス全体の交流の活発さ」と解釈できる。

推移性（クラスタリング係数）

推移性（transitivity）またはクラスタリング係数は、「友人の友人もまた友人である」という三者関係（triad closure）の傾向を測定する。ノード \(v_i\) の局所クラスタリング係数は次のように定義される。

\[C_i = \frac{2 t_i}{k_i(k_i - 1)}\]

ここで \(t_i\) はノード \(v_i\) の隣接ノード間に実際に存在するエッジの数である。推移性が高いネットワークでは、小グループ（クリーク）が形成されやすい傾向にある。

18.2.3 SLAにおけるネットワーク分析の意義

Kennedy Terry & Bayley (2024) は、ネットワーク分析がSLA研究に提供する独自の視座を3つに整理している。

  1. 社会関係資本（social capital）: 学習者がネットワーク内の位置によってアクセスできる資源。中心的な位置にある学習者は、より多くの言語的インプットや修正フィードバックを得やすい。
  2. 仲介性（brokerage）: 異なるグループ間を橋渡しする役割。留学先で異なるL1背景を持つ学習者同士をつなぐブローカーは、L2使用の機会を拡大しうる(Paradowski ほか, 2024)。
  3. 同類選好（homophily）: 類似した属性（L1、熟達度、クラスなど）を持つ個体同士が結びつきやすい傾向。同類選好が強すぎると、L2使用の機会が制限される可能性がある。

これらの概念は、本章の後半で具体的な分析手法（中心性、コミュニティ検出、ERGM）と結びつく。

18.3 回帰分析との対応で理解する

回帰分析との対応関係を明確にしておくと、ネットワーク分析の発想をより速く理解できる(Saqr ほか, 2024)。

回帰分析の発想 ネットワーク分析での対応
観測単位は個体（学生） 観測単位は関係（学生 \(i\) – 学生 \(j\)）
説明変数は属性（性別・得点など） 説明変数は関係特性（同クラスか、熟達差など）
目的変数は個体のアウトカム 目的変数はリンク有無・重み、またはネットワーク指標
独立性の仮定が重要 相互依存が本質（独立性が崩れやすい）

この違いにより、ネットワーク分析では次の設計判断が実装前に必要となる。

  • 境界（boundary）: どこまでをネットワークに含めるか
  • 関係定義（tie definition）: 何をエッジとみなすか
  • 方向（directed / undirected）: 相談「する/される」を区別するか
  • 重み（weighted / unweighted）: 回数・強度を保持するか

18.4 研究設計チェックリスト（実装前）

最低限の設計項目
  1. ノード集合: 誰を含めるか（授業履修者のみか、TAを含むか）
  2. エッジ定義: 例「共同課題を一緒に行った回数」
  3. 観測窓: いつの関係か（第1–4週、学期全体など）
  4. 欠測処理: 未回答者・欠席者をどう扱うか
  5. 解釈範囲: 構造記述か、因果推論か
英語教育研究での境界設定の例
  • 留学プログラム: ホストファミリー、クラスメイト、現地の友人をどこまで含めるか(Isbell, 2024)
  • クラスルーム: TA・教員をノードに含めるか、学習者のみか
  • オンライン環境: 同期（Zoom）と非同期（LMS掲示板）を一つのネットワークにまとめるか
よくある誤り
  • レイアウト図のノード間距離を、実際の心理的距離や地理的距離と誤解する
  • 中心性の高低を、因果的な「能力の高さ」と直結させる
  • しきい値の変更で結論が変わる可能性を報告しない

パイプ演算子について

本章では magrittr パッケージ由来のパイプ演算子 %>% を使用する。R 4.1 以降で導入されたネイティブパイプ |> も同様に利用できるが、tidygraph や ggraph のパイプライン内でプレースホルダー（%>% では .、|> では _）の挙動が異なるケースがある。tidygraph::as_tbl_graph() を起点とするパイプチェーンでは %>% が安定して動作するため、本章ではこれに統一した。自身のプロジェクトでは、どちらのパイプを使ってもよいが、一つのスクリプト内では混在させないことを推奨する。

18.6 例題データ: 協働ネットワークのシミュレーション

以下のシナリオを想定してデータを生成する。

  • 3クラス（A / B / C）、各20名、計60名
  • エッジは「共同作業を行った」関係を表す
  • 同クラス・熟達度が近いほど関係が形成されやすい（同類選好）
  • エッジ重みは「共同作業の回数」
コード 
# 学習者ノード
n_per_class <- 20
classes <- rep(c("A", "B", "C"), each = n_per_class)
ids <- sprintf("S%02d", seq_along(classes))

nodes <- tibble(
  id = ids,
  class = classes,
  proficiency = sample(c("A2", "B1", "B2"), size = length(ids), replace = TRUE, prob = c(0.25, 0.5, 0.25)),
  baseline_score = round(rnorm(length(ids), mean = 60, sd = 8), 1)
)

pairs <- tibble(from = pair_mat[, 1], to = pair_mat[, 2]) %>%
  left_join(nodes %>% select(id, class, prof_num, baseline_score), by = c("from" = "id")) %>%
  rename(class_from = class, prof_from = prof_num, baseline_from = baseline_score) %>%
  left_join(nodes %>% select(id, class, prof_num, baseline_score), by = c("to" = "id")) %>%
  rename(class_to = class, prof_to = prof_num, baseline_to = baseline_score) %>%
  mutate(
    same_class = as.integer(class_from == class_to),
    prof_gap = abs(prof_from - prof_to),
    baseline_gap = abs(baseline_from - baseline_to),
    # ロジット: 同クラスで増加、ギャップで減少
    p_link = plogis(-2.8 + 2.0 * same_class - 0.8 * prof_gap - 0.08 * baseline_gap),
    has_link = rbinom(n(), size = 1, prob = p_link),
    # 重み: 同クラスほど共同回数が増えやすい
    weight = ifelse(
      has_link == 1,
      1 + rpois(n(), lambda = exp(0.2 + 0.7 * same_class - 0.2 * prof_gap)),
      NA_real_
    )
  )

edges <- pairs %>%
  filter(has_link == 1) %>%
  transmute(from, to, weight = as.numeric(weight))

18.10 ノード中心性を計算する

中心性（centrality）は「ネットワーク内で何を重要とみなすか」の操作的定義である(Freeman, 1979)。目的に応じて使い分ける必要がある。

18.10.1 中心性の数学的定義

次数中心性（Degree Centrality）

\[C_D(v_i) = k_i = \sum_{j} A_{ij}\]

最も直感的な中心性であり、直接接続の数を数える。「何人の学習者と協働したか」を測定する。

強度中心性（Strength Centrality）

\[C_S(v_i) = s_i = \sum_{j} W_{ij}\]

重み付きの次数であり、接続の量（頻度・強さ）を考慮する。「どの程度の量の協働を行ったか」を測定する。

媒介中心性（Betweenness Centrality）

\[C_B(v_i) = \sum_{s \neq i \neq t} \frac{\sigma_{st}(v_i)}{\sigma_{st}}\]

ここで \(\sigma_{st}\) はノード \(s\) から \(t\) への最短経路の総数、\(\sigma_{st}(v_i)\) はそのうちノード \(v_i\) を通過するものの数である。媒介中心性が高いノードは、異なるグループ間の「橋渡し」役を担っている。留学研究では、異なる言語コミュニティ間をつなぐブローカー的な学習者を特定するのに有用である(Paradowski ほか, 2024)。

近接中心性（Closeness Centrality）

\[C_C(v_i) = \frac{n - 1}{\sum_{j \neq i} d(v_i, v_j)}\]

全ノードへの平均距離の逆数であり、「他の全員にどの程度素早く到達できるか」を示す。非連結グラフでは不安定になるため、本章では調和中心性（harmonic centrality）を優先する(igraph developers, 2024)。

\[C_H(v_i) = \frac{1}{n-1} \sum_{j \neq i} \frac{1}{d(v_i, v_j)}\]

固有ベクトル中心性（Eigenvector Centrality）

\[C_E(v_i) = \frac{1}{\lambda} \sum_{j} A_{ij} \cdot C_E(v_j)\]

「重要なノードとつながっているノードは、それ自体も重要である」という再帰的な定義に基づく。単に多くの接続を持つだけでなく、影響力のある学習者とつながっている学習者を高く評価する。

18.11 コミュニティ検出

コミュニティ検出は、ネットワークを「内部で密・外部で疎」なサブグループに分割する手法である。教室ネットワークにおいて、検出されたコミュニティがクラス編成とどの程度一致するかを確認することで、クラスを超えた交流の程度を評価できる。

18.11.1 モジュラリティの定義

コミュニティ分割の質はモジュラリティ \(Q\) で評価される(Blondel ほか, 2008; Newman, 2004)。

\[Q = \frac{1}{2m}\sum_{ij}\left[A_{ij} - \frac{k_i k_j}{2m}\right]\delta(c_i, c_j)\]

ここで \(m\) はエッジの総数、\(k_i\) はノード \(i\) の次数、\(c_i\) はノード \(i\) が属するコミュニティ、\(\delta\) はクロネッカーのデルタ関数（同じコミュニティなら1、異なれば0）である。\(Q\) が高いほど、コミュニティ内のエッジ密度がランダムな期待値よりも高いことを意味する。一般に \(Q > 0.3\) で有意なコミュニティ構造があるとみなされる。

18.13 ERGM: 指数ランダムグラフモデル

18.13.1 なぜERGMが必要か

ここまでの分析は記述的（descriptive）である。密度や中心性は構造を要約するが、「同じクラスの学習者同士は有意に協働しやすいか」「熟達度が近い学習者同士のリンク形成確率は高いか」といった推論的な問いには答えられない。

通常の回帰分析でダイアド（ペア）データを分析する場合、観測の独立性が仮定されるが、ネットワークデータでは同一ノードが複数のダイアドに関与するため、この仮定が構造的に破れる。指数ランダムグラフモデル（Exponential Random Graph Model; ERGM）は、この問題に対処するためのモデルである(Lusher ほか, 2013; Robins ほか, 2007)。

ERGMの直感的理解

ERGMは「ロジスティック回帰 + ネットワーク依存項」と考えるとわかりやすい。通常のロジスティック回帰がダイアドの属性からリンクの有無を予測するのに対し、ERGMはそれに加えて「三角形の閉鎖」や「互恵性」といったネットワーク構造の効果もモデルに組み込む。

18.13.2 数学的基礎

ERGMは、観測されたネットワーク \(y\) が生じる確率を以下のように定義する。

\[P(Y = y \mid \theta) = \frac{\exp\left(\theta^{\top} g(y)\right)}{c(\theta)}\]

ここで:

  • \(g(y)\): ネットワーク \(y\) の十分統計量のベクトル（例: エッジ数、同クラスペア数、三角形数）
  • \(\theta\): 各統計量に対応するパラメータ（log-odds として解釈される）
  • \(c(\theta)\): 正規化定数（すべての可能なネットワーク上の和。直接計算は不可能なため、MCMC-MLE で推定する）

各パラメータ \(\theta_k\) は、「他の条件が同じとき、統計量 \(g_k(y)\) が1単位増えた場合のリンク形成のlog-odds変化」と解釈できる。これは、ロジスティック回帰における係数の解釈と全く同じである。

ERGMのよくある問題
  • モデル退化（degeneracy）: triangle 項を直接投入するとMCMLEが収束しないことがある。代わりに gwesp や gwdsp を使用する。
  • 計算時間: ノード数100以上では数分～数十分かかる場合がある。control.ergm() で反復回数を調整する。

  • 解釈の範囲: ERGMはネットワーク形成メカニズムのモデルであり、因果関係を直接示すものではない。

18.14 QAP回帰: ネットワークの非独立性に対処する

18.14.1 なぜQAPが必要か

ネットワークデータにおいて、ノード \(i\) が関与するすべてのダイアド \((i, j_1), (i, j_2), \ldots\) は独立ではない。通常のOLS回帰やロジスティック回帰をダイアドデータに適用すると、標準誤差が過小推定され、Type I error が増大する(Dekker ほか, 2007; Krackhardt, 1988)。

Quadratic Assignment Procedure（QAP）は、ネットワーク行列の行と列を同時に置換する（permutation）ことで、ネットワーク構造を保持したまま帰無分布を構成する非パラメトリック検定法である。

18.14.3 MRQAP回帰

Multiple Regression QAP（MRQAP）は、ダイアド特性（同クラスか否か、熟達度の差など）から関係の強さ（協働回数）を予測する回帰分析である(Dekker ほか, 2007)。帰無分布の構成にはDouble Semi-Partialing（DSP）法を用いる。

コード 
# ダイアド変数を行列として構築
n_nodes <- length(common_ids)
same_class_mat <- matrix(0, n_nodes, n_nodes, dimnames = list(common_ids, common_ids))
prof_gap_mat <- matrix(0, n_nodes, n_nodes, dimnames = list(common_ids, common_ids))

for (i in seq_len(n_nodes)) {
  for (j in seq_len(n_nodes)) {
    if (i == j) next
    ni <- which(nodes$id == common_ids[i])
    nj <- which(nodes$id == common_ids[j])
    same_class_mat[i, j] <- as.integer(nodes$class[ni] == nodes$class[nj])
    prof_gap_mat[i, j] <- abs(nodes$prof_num[ni] - nodes$prof_num[nj])
  }
}

# MRQAP回帰（DSP法）
set.seed(2024)
mrqap_fit <- sna::netlm(
  adj_collab_aligned,
  list(same_class_mat, prof_gap_mat),
  reps = 2000,

通常のOLS回帰では、ダイアド間の非独立性により標準誤差が過小推定される。QAP回帰は行列の行と列を同時に置換することでこの問題に対処し、より保守的（正確な）p値を提供する(Dekker ほか, 2007)。

ERGMとQAPの使い分け
  • ERGM: ネットワーク形成の生成モデル。同類選好・三角形閉鎖などのメカニズムを同時に検定できる。構造的な依存をモデルの中に組み込む。
  • QAP: ネットワーク行列間の関連検定。置換により非独立性に対処する。計算が軽く、探索的分析に向いている。
  • 両者は補完的に使用することが推奨される(Saqr ほか, 2024)。

18.15 ネットワーク指標を回帰へ接続する

18.16.1 留学ネットワークと熟達度の伸び

留学研究では、学習者のソーシャルネットワーク構造がL2習熟度の伸びに関連することが報告されている(Dewey ほか, 2013; Isbell, 2024; Strawbridge, 2023)。ここでは、エゴネットワーク（ego-network: ある個人を中心とするネットワーク）データを想定したシミュレーションを示す。

set.seed(2024)
n_pa <- 30

  id = sprintf("PA%02d", 1:n_pa),

  n_reviews <- sample(3:5, 1)

18.19 FAQ（よくある質問）

Q: ネットワーク分析に最低何名の参加者が必要か？

明確な下限はないが、一般にノード数20名未満では中心性指標の安定性が低下する。ERGMの推定にはノード数50以上が推奨される(Lusher ほか, 2013)。小規模データではQAPやエゴネットワーク分析を検討すると良い。

Q: 縦断データ（複数時点）はどう扱うか？

各時点のネットワークを個別に分析し、指標の変化を追跡する方法が最も単純である。時点間のネットワーク変化を統計的にモデル化するには、TERGM（temporal ERGM）やREM（relational event model）が適しているが、本章の範囲外である。

Q: 二部ネットワーク（学習者 × 活動）はどう扱うか？

igraph の bipartite 属性を使用して二部グラフとして表現する。分析時には、一方のモードへの射影（projection）を行い、「同じ活動に参加した学習者同士」のネットワークに変換してから中心性やコミュニティ検出を適用するのが一般的である。

Q: igraph と statnet は同じ結果を返すか？

基本的な指標（次数、密度、推移性など）は同じ結果を返す。ただし、ERGMは statnet / ergm パッケージ専用であるため、ERGMを使う場合は intergraph::asNetwork() で変換が必要である(Handcock ほか, 2008)。

Q: 中心性が高い学習者 = 優秀な学習者か？

いいえ。中心性は構造的位置の指標であり、能力や成績の指標ではない。betweennessが高い学習者は異なるグループ間の橋渡し役であるが、それが学習成果に正の影響を与えるかは別の検証が必要である。因果推論には実験デザインや適切な統計モデリングが求められる。

Q: ERGMが収束しない場合はどうするか？
  1. triangle を gwesp に置き換える、(2) control.ergm() で反復回数やバーンイン期間を増やす、(3) モデルを簡略化する（項を減らす）、の順に対処する。ノード数が少なすぎる場合（< 30）は、ERGMの適用自体を再検討する。

## イントロダクション

ネットワーク分析（social network analysis; SNA）は、**「個体の属性」ではなく「個体間の関係」**を分析単位に置く方法論である。従来の回帰分析や分散分析は個々の学習者を独立した観測単位として扱うが、ネットワーク分析は学習者間の相互作用の**構造**そのものを対象とする。この視点の転換により、従来の変数中心アプローチ（variable-centered approach）では捉えきれなかった社会的文脈の影響を定量的に分析できるようになる[@kennedyterry2024; @saqr2024]。

英語教育研究では、ネットワーク分析によって以下のような問いに直接的に答えることが可能である。

- 協働学習において、誰がグループのハブ（情報結節点）として機能しているか
- クラス内の交流パターンがクラス外へどの程度広がっているか
- 介入前後で学習者コミュニティの分断が弱まったか、あるいは強まったか
- 留学先での社会的ネットワーク構造がL2習熟度の伸びにどう関連するか[@isbell2024; @strawbridge2023]

本章は、英語教育専攻の大学院生（回帰分析の基礎既習）を対象に、理論と実装の両面からネットワーク分析を体系的に解説する。具体的には、以下の4部構成で進める。

1. **理論基礎**: グラフ理論の数学的定義と、SLA研究における意義
2. **記述分析**: `igraph` / `tidygraph` / `ggraph` による可視化・要約・中心性分析
3. **統計的推論**: ERGM（指数ランダムグラフモデル）とQAP回帰による仮説検定
4. **応用と報告**: SLA研究への具体的な応用例とAPA報告テンプレート

## グラフ理論の基礎

ネットワーク分析の数学的基盤はグラフ理論（graph theory）である。ここでは、分析に必要な最小限の数学的定義を、SLA研究の文脈に即して導入する。

### グラフの数学的定義

グラフ $G$ は、ノード（頂点）の集合 $V$ とエッジ（辺）の集合 $E$ の組として定義される。

$$G = (V, E)$$

ここで $V = \{v_1, v_2, \ldots, v_n\}$ はネットワークに含まれるノード（例: 学習者）の集合であり、$E \subseteq V \times V$ はノード間の関係（例: 協働経験の有無）の集合である。

グラフの構造は**隣接行列**（adjacency matrix）$\mathbf{A}$ によって表現できる。$n \times n$ の行列 $\mathbf{A}$ の各要素は次のように定義される。

$$A_{ij} = \begin{cases} 1 & \text{if } (v_i, v_j) \in E \\ 0 & \text{otherwise} \end{cases}$$

**無向グラフ**（undirected graph）では $A_{ij} = A_{ji}$（対称行列）であり、**有向グラフ**（directed graph）では非対称となりうる。英語教育研究では、「協働作業を行った」という関係は無向（双方向的）であるのに対し、「相談をした」という関係は有向（一方向的）である場合が多い。

**重み付きグラフ**（weighted graph）では、隣接行列の要素がエッジの重み $w_{ij}$（例: 協働回数）を取る。

$$W_{ij} = \begin{cases} w_{ij} & \text{if } (v_i, v_j) \in E \\ 0 & \text{otherwise} \end{cases}$$

### 主要な指標の数学的定義

#### 次数と強度

**次数**（degree）は、あるノードが持つエッジの数である。

$$k_i = \sum_{j=1}^{n} A_{ij}$$

重み付きグラフでは、**強度**（strength）がエッジの重みの合計として定義される。

$$s_i = \sum_{j=1}^{n} W_{ij}$$

次数は「何人と関わったか」を、強度は「どの程度の量の関わりがあったか」を表す。留学研究において、ネットワークの次数（接触人数）よりも強度（接触頻度）の方がL2習熟度の伸びをよく予測するという知見がある[@isbell2024]。

#### 経路と測地距離

ノード $v_i$ から $v_j$ への**経路**（path）は、エッジを辿ってたどり着くまでの一連のノード列である。最短経路の長さを**測地距離**（geodesic distance）$d(v_i, v_j)$ と呼ぶ。測地距離が短いほど、情報や資源が素早く伝達されうることを意味する。

#### 密度

**密度**（density）は、理論上可能なエッジの総数に対する実際のエッジ数の比率であり、ネットワーク全体の「密な」度合いを示す。

$$D = \frac{2|E|}{|V|(|V| - 1)} \quad \text{（無向グラフの場合）}$$

密度 $D = 1$ はすべてのノードが互いに接続された完全グラフ、$D = 0$ はエッジが一つもない状態である。教室ネットワークでは、密度は「クラス全体の交流の活発さ」と解釈できる。

#### 推移性（クラスタリング係数）

**推移性**（transitivity）または**クラスタリング係数**は、「友人の友人もまた友人である」という三者関係（triad closure）の傾向を測定する。ノード $v_i$ の局所クラスタリング係数は次のように定義される。

$$C_i = \frac{2 t_i}{k_i(k_i - 1)}$$

ここで $t_i$ はノード $v_i$ の隣接ノード間に実際に存在するエッジの数である。推移性が高いネットワークでは、小グループ（クリーク）が形成されやすい傾向にある。

### SLAにおけるネットワーク分析の意義

@kennedyterry2024 は、ネットワーク分析がSLA研究に提供する独自の視座を3つに整理している。

1. **社会関係資本**（social capital）: 学習者がネットワーク内の位置によってアクセスできる資源。中心的な位置にある学習者は、より多くの言語的インプットや修正フィードバックを得やすい。
2. **仲介性**（brokerage）: 異なるグループ間を橋渡しする役割。留学先で異なるL1背景を持つ学習者同士をつなぐブローカーは、L2使用の機会を拡大しうる[@paradowski2024]。
3. **同類選好**（homophily）: 類似した属性（L1、熟達度、クラスなど）を持つ個体同士が結びつきやすい傾向。同類選好が強すぎると、L2使用の機会が制限される可能性がある。

これらの概念は、本章の後半で具体的な分析手法（中心性、コミュニティ検出、ERGM）と結びつく。

| ノード | node / vertex | ネットワークの構成単位（学習者、語彙項目など） |
| エッジ | edge / tie | ノード間の関係（協働、相談、共起など） |
| 次数 | degree | あるノードが持つエッジの数 |
| 強度 | strength | 重み付きエッジの合計値 |
| 経路 | path | ノード間を結ぶエッジの連鎖 |
| 測地距離 | geodesic distance | 最短経路の長さ |
| 媒介中心性 | betweenness centrality | 最短経路上に位置する頻度の指標 |
| 同類選好 | homophily | 類似属性のノード同士が結びつきやすい傾向 |
| モジュラリティ | modularity | コミュニティ分割の質を測る指標 |
:::

## 回帰分析との対応で理解する

回帰分析との対応関係を明確にしておくと、ネットワーク分析の発想をより速く理解できる[@saqr2024]。

| 回帰分析の発想 | ネットワーク分析での対応 |
|---|---|
| 観測単位は個体（学生） | 観測単位は関係（学生 $i$ -- 学生 $j$） |
| 説明変数は属性（性別・得点など） | 説明変数は関係特性（同クラスか、熟達差など） |
| 目的変数は個体のアウトカム | 目的変数はリンク有無・重み、またはネットワーク指標 |
| 独立性の仮定が重要 | 相互依存が本質（独立性が崩れやすい） |

この違いにより、ネットワーク分析では次の設計判断が実装前に必要となる。

- **境界（boundary）**: どこまでをネットワークに含めるか
- **関係定義（tie definition）**: 何をエッジとみなすか
- **方向（directed / undirected）**: 相談「する/される」を区別するか
- **重み（weighted / unweighted）**: 回数・強度を保持するか

## 研究設計チェックリスト（実装前）

::: {.callout-note}
### 最低限の設計項目
1. **ノード集合**: 誰を含めるか（授業履修者のみか、TAを含むか）
2. **エッジ定義**: 例「共同課題を一緒に行った回数」
3. **観測窓**: いつの関係か（第1--4週、学期全体など）
4. **欠測処理**: 未回答者・欠席者をどう扱うか
5. **解釈範囲**: 構造記述か、因果推論か
:::

::: {.callout-tip}
### 英語教育研究での境界設定の例
- **留学プログラム**: ホストファミリー、クラスメイト、現地の友人をどこまで含めるか[@isbell2024]
- **クラスルーム**: TA・教員をノードに含めるか、学習者のみか
- **オンライン環境**: 同期（Zoom）と非同期（LMS掲示板）を一つのネットワークにまとめるか
:::

::: {.callout-warning}
### よくある誤り
- レイアウト図のノード間距離を、実際の心理的距離や地理的距離と誤解する
- 中心性の高低を、因果的な「能力の高さ」と直結させる
- しきい値の変更で結論が変わる可能性を報告しない
:::

:::

## 例題データ: 協働ネットワークのシミュレーション

以下のシナリオを想定してデータを生成する。

- 3クラス（A / B / C）、各20名、計60名
- エッジは「共同作業を行った」関係を表す
- 同クラス・熟達度が近いほど関係が形成されやすい（同類選好）
- エッジ重みは「共同作業の回数」

```{r}
#| label: simulate-nodes-edges
# 学習者ノード
n_per_class <- 20
classes <- rep(c("A", "B", "C"), each = n_per_class)
ids <- sprintf("S%02d", seq_along(classes))

nodes <- tibble(
  id = ids,
  class = classes,
  proficiency = sample(c("A2", "B1", "B2"), size = length(ids), replace = TRUE, prob = c(0.25, 0.5, 0.25)),
  baseline_score = round(rnorm(length(ids), mean = 60, sd = 8), 1)
)

prof_map <- c(A2 = 1, B1 = 2, B2 = 3)
nodes <- nodes %>% mutate(prof_num = unname(prof_map[proficiency]))

# 全ペア（i < j）を作成し、リンク生成確率を計算
pair_mat <- t(utils::combn(nodes$id, 2))
pairs <- tibble(from = pair_mat[, 1], to = pair_mat[, 2]) %>%
  left_join(nodes %>% select(id, class, prof_num, baseline_score), by = c("from" = "id")) %>%
  rename(class_from = class, prof_from = prof_num, baseline_from = baseline_score) %>%
  left_join(nodes %>% select(id, class, prof_num, baseline_score), by = c("to" = "id")) %>%
  rename(class_to = class, prof_to = prof_num, baseline_to = baseline_score) %>%
  mutate(
    same_class = as.integer(class_from == class_to),
    prof_gap = abs(prof_from - prof_to),
    baseline_gap = abs(baseline_from - baseline_to),
    # ロジット: 同クラスで増加、ギャップで減少
    p_link = plogis(-2.8 + 2.0 * same_class - 0.8 * prof_gap - 0.08 * baseline_gap),
    has_link = rbinom(n(), size = 1, prob = p_link),
    # 重み: 同クラスほど共同回数が増えやすい
    weight = ifelse(
      has_link == 1,
      1 + rpois(n(), lambda = exp(0.2 + 0.7 * same_class - 0.2 * prof_gap)),
      NA_real_
    )
  )

edges <- pairs %>%
  filter(has_link == 1) %>%
  transmute(from, to, weight = as.numeric(weight))

# 保存（再利用用）
dir.create("data/network", recursive = TRUE, showWarnings = FALSE)
write.csv(nodes %>% select(-prof_num), "data/network/nodes_collaboration.csv", row.names = FALSE)
write.csv(edges, "data/network/edges_collaboration.csv", row.names = FALSE)
```

```{r}
edges %>%
```

- `edges`: `from`, `to`, `weight`（回数や強度）

### レイアウトの比較

ネットワーク可視化では、レイアウトアルゴリズムの選択が図の印象を大きく左右する。以下の4種類を並列して比較する。

```{r}

layout_names <- c("fr", "kk", "stress", "circle")
layout_labels <- c(
  "Fruchterman-Reingold", "Kamada-Kawai",
  "Stress Minimization", "Circle"
)

plots <- lapply(seq_along(layout_names), function(i) {
  set.seed(2024)
  ggraph(g_tbl, layout = layout_names[i]) +
    geom_edge_link(color = "grey70", alpha = 0.3) +
    geom_node_point(aes(color = class), size = 3, alpha = 0.9) +
    scale_color_manual(values = class_palette) +
    labs(title = layout_labels[i]) +
    theme_graph(base_family = "sans") +
    theme(legend.position = "none", plot.title = element_text(size = 11, hjust = 0.5))
})

(plots[[1]] + plots[[2]]) / (plots[[3]] + plots[[4]])
```

- **Fruchterman-Reingold（FR）**: 力学モデルに基づく代表的なレイアウト。ランダム性があるため `set.seed()` が必要。
- **Kamada-Kawai（KK）**: グラフ理論的距離を空間距離に近づける。比較的決定論的。
- **Stress Minimization**: 測地距離を忠実に再現する。**特にこだわりがなければ推奨**[@pedersen2024ggraph]。
- **Circle**: すべてのノードを円上に配置。コミュニティ構造を見るには不向きだが、次数の分布が一覧しやすい。

## ネットワーク全体の指標

## ノード中心性を計算する

中心性（centrality）は「ネットワーク内で何を重要とみなすか」の操作的定義である[@freeman1979]。目的に応じて使い分ける必要がある。

### 中心性の数学的定義

#### 次数中心性（Degree Centrality）

$$C_D(v_i) = k_i = \sum_{j} A_{ij}$$

最も直感的な中心性であり、直接接続の数を数える。「何人の学習者と協働したか」を測定する。

#### 強度中心性（Strength Centrality）

$$C_S(v_i) = s_i = \sum_{j} W_{ij}$$

重み付きの次数であり、接続の量（頻度・強さ）を考慮する。「どの程度の量の協働を行ったか」を測定する。

#### 媒介中心性（Betweenness Centrality）

$$C_B(v_i) = \sum_{s \neq i \neq t} \frac{\sigma_{st}(v_i)}{\sigma_{st}}$$

ここで $\sigma_{st}$ はノード $s$ から $t$ への最短経路の総数、$\sigma_{st}(v_i)$ はそのうちノード $v_i$ を通過するものの数である。媒介中心性が高いノードは、異なるグループ間の「橋渡し」役を担っている。留学研究では、異なる言語コミュニティ間をつなぐブローカー的な学習者を特定するのに有用である[@paradowski2024]。

#### 近接中心性（Closeness Centrality）

$$C_C(v_i) = \frac{n - 1}{\sum_{j \neq i} d(v_i, v_j)}$$

全ノードへの平均距離の逆数であり、「他の全員にどの程度素早く到達できるか」を示す。非連結グラフでは不安定になるため、本章では**調和中心性**（harmonic centrality）を優先する[@igraph_closeness]。

$$C_H(v_i) = \frac{1}{n-1} \sum_{j \neq i} \frac{1}{d(v_i, v_j)}$$

#### 固有ベクトル中心性（Eigenvector Centrality）

$$C_E(v_i) = \frac{1}{\lambda} \sum_{j} A_{ij} \cdot C_E(v_j)$$

「重要なノードとつながっているノードは、それ自体も重要である」という再帰的な定義に基づく。単に多くの接続を持つだけでなく、影響力のある学習者とつながっている学習者を高く評価する。

## コミュニティ検出

コミュニティ検出は、ネットワークを「内部で密・外部で疎」なサブグループに分割する手法である。教室ネットワークにおいて、検出されたコミュニティがクラス編成とどの程度一致するかを確認することで、クラスを超えた交流の程度を評価できる。

### モジュラリティの定義

コミュニティ分割の質は**モジュラリティ** $Q$ で評価される[@blondel2008; @newman2004]。

$$Q = \frac{1}{2m}\sum_{ij}\left[A_{ij} - \frac{k_i k_j}{2m}\right]\delta(c_i, c_j)$$

ここで $m$ はエッジの総数、$k_i$ はノード $i$ の次数、$c_i$ はノード $i$ が属するコミュニティ、$\delta$ はクロネッカーのデルタ関数（同じコミュニティなら1、異なれば0）である。$Q$ が高いほど、コミュニティ内のエッジ密度がランダムな期待値よりも高いことを意味する。一般に $Q > 0.3$ で有意なコミュニティ構造があるとみなされる。

### Louvain法によるコミュニティ検出

### 外部妥当性の検証（クラスラベルとの照合）

検出されたコミュニティが既知のクラス編成（A / B / C）とどの程度一致するかを、クロス集計と**Adjusted Rand Index（ARI）**で評価する。ARIは $[-1, 1]$ の範囲を取り、1が完全一致、0がランダムな一致水準を示す。

::: {.callout-note}
### コミュニティ検出の解釈
- 高モジュラリティ（$Q > 0.3$）は「内部で密・外部で疎」の傾向を示す。
- ARI が高い（例: $> 0.5$）場合、コミュニティはクラス編成を概ね反映している。
- ARI が低い場合、クラスを超えた交流グループが形成されている可能性がある。
- コミュニティ検出はアルゴリズム依存であるため、単一の結果を絶対視せず、外部情報（クラス編成、活動設計）と照合すること[@blondel2008; @traag2019]。
:::

## ERGM: 指数ランダムグラフモデル

### なぜERGMが必要か

ここまでの分析は記述的（descriptive）である。密度や中心性は構造を要約するが、「同じクラスの学習者同士は有意に協働しやすいか」「熟達度が近い学習者同士のリンク形成確率は高いか」といった**推論的**な問いには答えられない。

通常の回帰分析でダイアド（ペア）データを分析する場合、観測の独立性が仮定されるが、ネットワークデータでは同一ノードが複数のダイアドに関与するため、この仮定が構造的に破れる。**指数ランダムグラフモデル**（Exponential Random Graph Model; ERGM）は、この問題に対処するためのモデルである[@robins2007; @lusher2013]。

::: {.callout-tip}
### ERGMの直感的理解
ERGMは「ロジスティック回帰 + ネットワーク依存項」と考えるとわかりやすい。通常のロジスティック回帰がダイアドの属性からリンクの有無を予測するのに対し、ERGMはそれに加えて「三角形の閉鎖」や「互恵性」といったネットワーク構造の効果もモデルに組み込む。
:::

### 数学的基礎

ERGMは、観測されたネットワーク $y$ が生じる確率を以下のように定義する。

$$P(Y = y \mid \theta) = \frac{\exp\left(\theta^{\top} g(y)\right)}{c(\theta)}$$

ここで:

- $g(y)$: ネットワーク $y$ の**十分統計量**のベクトル（例: エッジ数、同クラスペア数、三角形数）
- $\theta$: 各統計量に対応するパラメータ（log-odds として解釈される）
- $c(\theta)$: 正規化定数（すべての可能なネットワーク上の和。直接計算は不可能なため、MCMC-MLE で推定する）

各パラメータ $\theta_k$ は、「他の条件が同じとき、統計量 $g_k(y)$ が1単位増えた場合のリンク形成のlog-odds変化」と解釈できる。これは、ロジスティック回帰における係数の解釈と全く同じである。

### statnet による実装

ERGM の推定には `statnet` / `ergm` パッケージを使用する[@hunter2008; @handcock2008]。`igraph` オブジェクトを `network` オブジェクトに変換してからモデルを推定する。

ERGMの係数はlog-oddsとして解釈する。以下のガイドを参照されたい。

| ERGM項 | 意味 | SLA的解釈 |
|---|---|---|
| `edges` | ベースラインのリンク形成log-odds | ネットワーク密度の傾向 |
| `nodematch("class")` | 同クラスの同類選好 | 同じクラスの学習者同士が協働しやすいか |
| `nodematch("proficiency")` | 同熟達度の同類選好 | 近い熟達度の学習者同士が結びつきやすいか |
| `gwesp(0.5)` | 三角形の閉鎖傾向（GWESP） | 「友人の友人とも協働する」傾向の強さ |

### オッズ比の読み方
例えば `nodematch("class")` のオッズ比が 5.0 であれば、「同じクラスに属するペアは、異なるクラスのペアと比較して、協働関係を持つオッズが5倍高い」と解釈する。信頼区間が1を含まなければ統計的に有意である。
:::

::: {.callout-warning}
### ERGMのよくある問題
- **モデル退化**（degeneracy）: `triangle` 項を直接投入するとMCMLEが収束しないことがある。代わりに `gwesp` や `gwdsp` を使用する。
- **計算時間**: ノード数100以上では数分～数十分かかる場合がある。`control.ergm()` で反復回数を調整する。
- **有向/無向の一致**: グラフの方向性とモデル項（`mutual` は有向のみ）を一致させる。
- **解釈の範囲**: ERGMはネットワーク形成メカニズムのモデルであり、因果関係を直接示すものではない。
:::

## QAP回帰: ネットワークの非独立性に対処する

### なぜQAPが必要か

ネットワークデータにおいて、ノード $i$ が関与するすべてのダイアド $(i, j_1), (i, j_2), \ldots$ は独立ではない。通常のOLS回帰やロジスティック回帰をダイアドデータに適用すると、標準誤差が過小推定され、Type I error が増大する[@krackhardt1988; @dekker2007]。

**Quadratic Assignment Procedure（QAP）**は、ネットワーク行列の行と列を同時に置換する（permutation）ことで、ネットワーク構造を保持したまま帰無分布を構成する非パラメトリック検定法である。

### QAP相関

協働ネットワークと相談ネットワークの間に関連があるかを検定する。

```{r}
#| label: qap-correlation
#| cache: true
# igraph → 隣接行列
adj_collab <- as_adjacency_matrix(g, attr = "weight", sparse = FALSE)
adj_advice <- as_adjacency_matrix(g_dir, sparse = FALSE)

# ノード順序を一致させる
common_ids <- intersect(rownames(adj_collab), rownames(adj_advice))
adj_collab_aligned <- adj_collab[common_ids, common_ids]
adj_advice_aligned <- adj_advice[common_ids, common_ids]

# QAP検定
set.seed(2024)
qap_result <- sna::qaptest(
  list(adj_collab_aligned, adj_advice_aligned),
  sna::gcor, g1 = 1, g2 = 2, reps = 2000
)

cat("QAP相関:\n")
cat("観測相関:", round(qap_result$testval, 3), "\n")
cat("p値:", round(qap_result$pgreq, 4), "\n")
```

### MRQAP回帰

**Multiple Regression QAP（MRQAP）**は、ダイアド特性（同クラスか否か、熟達度の差など）から関係の強さ（協働回数）を予測する回帰分析である[@dekker2007]。帰無分布の構成にはDouble Semi-Partialing（DSP）法を用いる。

```{r}
#| label: mrqap-regression
#| cache: true
# ダイアド変数を行列として構築
n_nodes <- length(common_ids)
same_class_mat <- matrix(0, n_nodes, n_nodes, dimnames = list(common_ids, common_ids))
prof_gap_mat <- matrix(0, n_nodes, n_nodes, dimnames = list(common_ids, common_ids))

for (i in seq_len(n_nodes)) {
  for (j in seq_len(n_nodes)) {
    if (i == j) next
    ni <- which(nodes$id == common_ids[i])
    nj <- which(nodes$id == common_ids[j])
    same_class_mat[i, j] <- as.integer(nodes$class[ni] == nodes$class[nj])
    prof_gap_mat[i, j] <- abs(nodes$prof_num[ni] - nodes$prof_num[nj])
  }
}

# MRQAP回帰（DSP法）
set.seed(2024)
mrqap_fit <- sna::netlm(
  adj_collab_aligned,
  list(same_class_mat, prof_gap_mat),
  reps = 2000,
  nullhyp = "qap"
)

cat("MRQAP回帰結果:\n")
summary(mrqap_fit)
```

```{r}
           x = obs_coef_class, y = Inf, vjust = 1.5, hjust = -0.1,
           label = paste0("Observed = ", round(obs_coef_class, 3)),
           color = "#d62728", size = 3.8, fontface = "bold",
           family = "sans") +
  labs(
    title = "MRQAP Permutation Distribution: Same Class",
    x = "Permuted Coefficient", y = "Density"
  ) +
  theme_bw(base_family = "sans")
```

通常のOLS回帰では、ダイアド間の非独立性により標準誤差が過小推定される。QAP回帰は行列の行と列を同時に置換することでこの問題に対処し、より保守的（正確な）p値を提供する[@dekker2007]。

::: {.callout-note}
### ERGMとQAPの使い分け
- **ERGM**: ネットワーク形成の**生成モデル**。同類選好・三角形閉鎖などのメカニズムを同時に検定できる。構造的な依存をモデルの中に組み込む。
- **QAP**: ネットワーク行列間の**関連検定**。置換により非独立性に対処する。計算が軽く、探索的分析に向いている。
- 両者は補完的に使用することが推奨される[@saqr2024]。
:::

## ネットワーク指標を回帰へ接続する

回帰分析既習者向けに、ネットワーク中心性を個人レベルのアウトカムの予測子として用いる最小例を示す。ここでは、中心性が「学期中の伸び（gain）」と関連するかを確認する。

::: {.callout-important}
### 重要な留意事項
これは**方法理解のためのシミュレーション**であり、因果推論を行う例ではない。ネットワークデータは相互依存を持つため、通常の回帰仮定（独立性）が崩れやすい。厳密な推論には、前節で紹介したERGMやQAPが必要である[@krackhardt1988]。
:::

### 留学ネットワークと熟達度の伸び

留学研究では、学習者のソーシャルネットワーク構造がL2習熟度の伸びに関連することが報告されている[@isbell2024; @dewey2013; @strawbridge2023]。ここでは、エゴネットワーク（ego-network: ある個人を中心とするネットワーク）データを想定したシミュレーションを示す。

```{r}
#| label: sla-study-abroad
set.seed(2024)
n_learners <- 40

ego_data <- tibble(
  learner_id = sprintf("SA%02d", 1:n_learners),
  # エゴネットワーク指標
  network_size = rpois(n_learners, lambda = 8),
  prop_l2_contacts = rbeta(n_learners, shape1 = 3, shape2 = 4),
  network_density = rbeta(n_learners, shape1 = 2, shape2 = 5),
  # 熟達度の伸び（L2接触割合が高いほど伸びやすい設計）
  pre_score = round(rnorm(n_learners, mean = 55, sd = 10), 1),
  gain = round(
    3 + 4 * prop_l2_contacts + 0.5 * scale(network_size) +
    -1.5 * network_density + rnorm(n_learners, sd = 2), 1
  )
)

m_abroad <- lm(gain ~ pre_score + network_size + prop_l2_contacts + network_density, data = ego_data)

ego_data %>%
  slice_head(n = 8) %>%
  gt() %>%
  tab_header(title = "留学ネットワークデータ（シミュレーション）の先頭8行") %>%
  fmt_number(columns = c(prop_l2_contacts, network_density, gain), decimals = 2)
```

```{r}
#| label: sla-study-abroad-results
summary(m_abroad)
```

このシミュレーションでは、L2話者との接触割合（`prop_l2_contacts`）が熟達度の伸びの主要な予測子となっている。これは @isbell2024 の知見――ネットワークの次数よりもL2使用の質（特にL2話者との接触比率）が重要である――と整合的な結果である。

### ピア評価ネットワーク

ライティング授業などで実施されるピア・アセスメント（peer assessment）は、「誰が誰を評価したか」という**有向重み付きネットワーク**として自然にモデル化できる。ここでは、30名の学習者が互いの作文を評価する場面をシミュレーションし、ネットワーク指標とライティング成績の関連を検討する。

```{r}
#| label: sla-peer-assessment-data
set.seed(2024)
n_pa <- 30

# 学習者属性
pa_nodes <- tibble(
  id = sprintf("PA%02d", 1:n_pa),
  proficiency = sample(c("A2", "B1", "B2"), n_pa, replace = TRUE, prob = c(0.2, 0.5, 0.3)),
  writing_pre = round(rnorm(n_pa, mean = 60, sd = 10), 1)
)

# ピア評価のエッジ生成（有向: i が j を評価）
# 各学習者は3-5名を評価する設計
pa_edges_list <- lapply(1:n_pa, function(i) {
  n_reviews <- sample(3:5, 1)
  targets <- sample(setdiff(1:n_pa, i), n_reviews)
  tibble(
    from = pa_nodes$id[i],
    to = pa_nodes$id[targets],
    # 評価スコア（1-5点）: 被評価者の能力が高いほど高得点を受けやすい
    score = pmin(5, pmax(1, round(
      3 + 0.3 * scale(pa_nodes$writing_pre[targets]) +
      rnorm(n_reviews, sd = 0.8)
    )))
  )
})
pa_edges <- bind_rows(pa_edges_list)

# 有向グラフの構築
g_pa <- graph_from_data_frame(pa_edges, directed = TRUE, vertices = pa_nodes)

cat("ピア評価ネットワーク:\n")
cat("  ノード数:", vcount(g_pa), "\n")
cat("  エッジ数（評価件数）:", ecount(g_pa), "\n")
cat("  互恵性（reciprocity）:", round(reciprocity(g_pa), 3), "\n")
```

互恵性（reciprocity）は、「AがBを評価し、かつBもAを評価している」ペアの割合を示す。ランダムな割り当てでは互恵性は低く、自発的な選択では高くなる傾向がある。

```{r}
#| label: sla-peer-assessment-centrality
# Hub（良い評価者）とAuthority（良い被評価者）の計算
pa_stats <- tibble(
  id = V(g_pa)$name,
  proficiency = V(g_pa)$proficiency,
  writing_pre = as.numeric(V(g_pa)$writing_pre),
  outdegree = igraph::degree(g_pa, mode = "out"),
  indegree = igraph::degree(g_pa, mode = "in"),
  mean_score_received = sapply(V(g_pa)$name, function(v) {
    in_edges <- E(g_pa)[.to(v)]
    if (length(in_edges) == 0) return(NA_real_)
    mean(in_edges$score)
  }),
  hub = hub.score(g_pa)$vector,
  authority = authority.score(g_pa)$vector
)

pa_stats %>%
  arrange(desc(authority)) %>%
  slice_head(n = 10) %>%
  gt() %>%
  tab_header(title = "ピア評価ネットワーク: Authority上位10名") %>%
  fmt_number(columns = c(writing_pre, mean_score_received, hub, authority), decimals = 3)
```

## FAQ（よくある質問）

::: {.callout-note collapse="true"}
#### Q: ネットワーク分析に最低何名の参加者が必要か？

明確な下限はないが、一般にノード数20名未満では中心性指標の安定性が低下する。ERGMの推定にはノード数50以上が推奨される[@lusher2013]。小規模データではQAPやエゴネットワーク分析を検討すると良い。
:::

::: {.callout-note collapse="true"}
#### Q: 縦断データ（複数時点）はどう扱うか？

各時点のネットワークを個別に分析し、指標の変化を追跡する方法が最も単純である。時点間のネットワーク変化を統計的にモデル化するには、TERGM（temporal ERGM）やREM（relational event model）が適しているが、本章の範囲外である。
:::

::: {.callout-note collapse="true"}
#### Q: 二部ネットワーク（学習者 × 活動）はどう扱うか？

`igraph` の `bipartite` 属性を使用して二部グラフとして表現する。分析時には、一方のモードへの射影（projection）を行い、「同じ活動に参加した学習者同士」のネットワークに変換してから中心性やコミュニティ検出を適用するのが一般的である。
:::

::: {.callout-note collapse="true"}
#### Q: igraph と statnet は同じ結果を返すか？

基本的な指標（次数、密度、推移性など）は同じ結果を返す。ただし、ERGMは `statnet` / `ergm` パッケージ専用であるため、ERGMを使う場合は `intergraph::asNetwork()` で変換が必要である[@handcock2008]。
:::

::: {.callout-note collapse="true"}
#### Q: 中心性が高い学習者 = 優秀な学習者か？

いいえ。中心性は構造的位置の指標であり、能力や成績の指標ではない。betweennessが高い学習者は異なるグループ間の橋渡し役であるが、それが学習成果に正の影響を与えるかは別の検証が必要である。因果推論には実験デザインや適切な統計モデリングが求められる。
:::

::: {.callout-note collapse="true"}
#### Q: ERGMが収束しない場合はどうするか？

(1) `triangle` を `gwesp` に置き換える、(2) `control.ergm()` で反復回数やバーンイン期間を増やす、(3) モデルを簡略化する（項を減らす）、の順に対処する。ノード数が少なすぎる場合（< 30）は、ERGMの適用自体を再検討する。
:::

- [ ] 有向/無向、重み有/無の選択理由を説明できる
- [ ] 境界設定（誰を含め、何をエッジとみなすか）が明確である
- [ ] 指標の意味を混同していない（degree と betweenness など）
- [ ] 可視化の美しさではなく、解釈可能性を優先している
- [ ] コミュニティ検出では複数のアルゴリズムを比較している
- [ ] ERGMを使用する場合、モデル退化を回避し適合度を確認している

- [ ] しきい値や欠測に対する感度分析を実施している
- [ ] 結果を因果主張に飛躍させていない
