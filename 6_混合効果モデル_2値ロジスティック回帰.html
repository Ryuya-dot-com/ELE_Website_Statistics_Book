8.1 はじめに

5章では GLM/GAM の使い分けを扱いました。次の実務課題は、 データの独立性が崩れる場面です。

英語教育データでは「同じ学習者が複数のペアに登場する」「同じテスト項目に複数の学習者が回答する」 といった繰り返し構造がほぼ必ず存在します。 このとき GLM を使うと、実際には独立でない観測を独立と見なすことになり、 標準誤差が過小評価され、p 値が楽観的に出てしまいます(Baayen et al., 2008; Meteyard & Davies, 2020)。

この問題に対する標準解が 一般化線形混合効果モデル（GLMM）です。 GLMM は、報告したい効果（固定効果）と個人差・項目差（ランダム効果）を 同時にモデル化することで、標準誤差を正直に推定します(Bates, Machler, et al., 2015)。 SLA（第二言語習得）分野でも GLMM は標準ツールとなっており、 Cunnings (2012) と Linck & Cunnings (2015) が分野に特化したチュートリアルを提供しています。 2値言語データに対するロジット混合モデルの有効性も、 複数の方法論研究で一貫して示されてきました(Jaeger, 2008; Quené & Bergh, 2008)。

本章では、実データを用いて 2値ロジスティック GLMM を 「最小モデルから診断・報告まで」一気通貫で実装します。

この章の到達目標
  • GLM と GLMM を同一データで比較し、ランダム効果の必要性を説明できる。
  • 交差分類ランダム効果（sender / receiver）を実装できる。
  • ICC とデザインエフェクトを用いて、独立性仮定の破れを説明できる。
  • OR と予測確率を併記して、読み手に伝わる結果提示ができる。
  • 特異適合・過分散・残差の点検を、最小限の手順で実行できる。

8.2 GLMM が必要になる理由（最短版）

GLM と GLMM の核心的な違い

GLM は推定値そのもの（β）を大きく変えるわけではありません。GLMM の本質は標準誤差を正直にすることにあります。

  • GLM: 全ペアが独立と仮定 → SE が過小 → p 値が楽観的
  • GLMM: sender/receiver の繰り返しを考慮 → SE が適切 → p 値が正直

つまり、GLMM は「効果を見つける」ためではなく、「効果の不確かさを正しく評価する」ために使います。

8.2.1 GLMM の数学的表現

2値 GLMM は次で表されます(Bates, Machler, et al., 2015; Nelder & Wedderburn, 1972)。

\[ g(p_{ij}) = \mathbf{x}_{ij}^\top\beta + \mathbf{z}_{ij}^\top b, \quad b \sim \mathcal{N}(0, \Sigma) \]

この式を日本語で読み下すと：

ペア \(ij\) の助言確率 \(p_{ij}\) をリンク関数 \(g()\) で変換した値は、「固定効果（全員に共通の平均的効果 \(\beta\)）」と「ランダム効果（個人ごとの偏り \(b\)）」の和で表される。ランダム効果 \(b\) は平均ゼロの正規分布から生成される。

もう少し直感的に言えば：

  • 固定効果 \(\beta\)（報告したい効果）: 「同クラスだと助言しやすい」「スコアが高いと助言しやすい」など、研究で検証したい平均的な効果
  • ランダム効果 \(b\)（統制したいばらつき）: 「田中さんは全般に助言をよくする」「鈴木さんは助言をあまり受けない」など、個人ごとの偏り
この式がピンとこない場合

数式よりもコードで理解したいタイプの方には、以下の入門チュートリアルを先に読むことをお勧めします。

  1. Winter (2013) — 言語学データの例で、線形モデルから混合モデルへの拡張を段階的に解説。
  2. Brown (2021) — 心理学データを使い、lme4 での実装を一から追える R チュートリアル。
  3. DeBruine & Barr (2021) — 「自分でデータを生成して、ランダム効果が何をしているか体感する」という独自のアプローチ。数式の意味が腑に落ちます。

いずれも本章の前提知識として有用ですが、本章だけでも完結する設計になっています。

交差分類 vs 入れ子
  • 入れ子（nested）: 生徒がクラスにのみ属する形（例：生徒 1-10 はクラス A、11-20 はクラス B）。(1 | class/student) で実装。
  • 交差分類（crossed）: 本データでは sender と receiver が同じ 30 人のプールから出ている。sender 1 は receiver 2, 3, … 29 と全員のペアを作る。(1 | sender) + (1 | receiver) で実装。

英語教育では「学習者差」「項目差」が同時に入る交差分類が多いです(Baayen et al., 2008; Barr et al., 2013; Quené & Bergh, 2008)。

8.2.2 GLMM でもリンク関数は選べる

本章の主モデルは logit ですが、GLMM でも probit や cloglog を選択できます。 実装上は glmer(..., family = binomial(link = \"...\")) を変えるだけです。

  • logit: OR 解釈が可能（実務上もっとも扱いやすい）
  • probit: 潜在正規尺度での解釈（確率予測は logit と近いことが多い）
  • cloglog: 非対称リンク。離散時間ハザードの文脈と整合的(Prentice & Gloeckler, 1978)

リンクの候補は stats::family / stats::make.link を参照してください (R Core Team, 2026a, 2026b)。また、log-log は cloglog の反転関係として扱えます (Yee, 2026)。

8.3.1 モデル前提チェックリスト（GLMM）

GLMM の前提は「正規残差」ではなく、次の構造的条件を確認することです (B. Bolker et al., 2025; B. M. Bolker et al., 2009; Meteyard & Davies, 2020)。

  1. outcome と family/link が整合しているか（本章は 2値 + logit）。
  2. 群化構造（sender/receiver）が研究設計に一致しているか。
  3. ランダム効果構造が過剰でないか（isSingular(), rePCA()）。
  4. 過分散・残差の大きな破綻がないか（DHARMa, check_overdispersion()）。
  5. 固定効果の推定が不安定でないか（check_collinearity()）。
よくある誤解：「残差が正規分布でないからダメ」

GLM/GLMM では アウトカムの正規性は仮定していません。family（本章では binomial）がアウトカムの型と整合していれば、残差の正規性はそもそも問題になりません。上の 5 項目が GLMM の本当のチェックポイントです。

なお、混合モデルは分布仮定からの軽度な逸脱に対して比較的頑健であることが、大規模シミュレーション研究で確認されています(Schielzeth et al., 2020)。

本章での対応関数
  • 構造比較: compare_performance(), anova(..., test="Chisq")
  • 特異適合: isSingular(), performance::check_singularity(), rePCA()
  • 残差診断: DHARMa::simulateResiduals(), testDispersion(), testUniformity()
  • 補助診断: check_overdispersion(), check_collinearity(), check_model()

8.6 分析データ作成（有向ペア単位）

sender != receiver の全有向ペアを作り、 各ペアが実際に助言エッジとして観測されたか（tie）を 0/1 で定義します。

変数名のポイント: sender/receiver という命名にしておくと、後の glmer() 式で (1 | sender) + (1 | receiver) と書くだけで交差分類ランダム効果が実装でき、コードの可読性が上がります。また、factor() への変換は lme4 がランダム効果のグループ変数を正しく認識するために必須です。

コード 
# ── 全有向ペアを生成（自分自身は除外）──
dat_bin <- tidyr::crossing(
  sender   = nodes$id,
  receiver = nodes$id
) |>
  filter(sender != receiver) |>

  # sender 側の属性を結合
  left_join(
    nodes |>
      dplyr::select(sender = id, class_sender = class,
                    prof_sender = proficiency, prof_sender_num = prof_num,
                    score_sender_sc = baseline_score_sc),
    by = "sender"
  ) |>

  # receiver 側の属性を結合
  left_join(
    nodes |>
      dplyr::select(receiver = id, class_receiver = class,
                    prof_receiver = proficiency, prof_receiver_num = prof_num,
                    score_receiver_sc = baseline_score_sc),
    by = "receiver"
  ) |>

  mutate(
    # ── 予測子の作成 ──
    same_class = as.integer(class_sender == class_receiver),  # 同クラスなら 1
    prof_gap   = abs(prof_sender_num - prof_receiver_num),    # 熟達度差（0, 1, 2）
    score_gap  = abs(score_sender_sc - score_receiver_sc),    # スコア差（連続量）

    # ── アウトカム：助言エッジがあるか ──
    tie = as.integer(
      paste(sender, receiver) %in% paste(edges_advice$from, edges_advice$to)
    ),

    # ── glmer のランダム効果に必要なファクター化 ──
    sender   = factor(sender),
    receiver = factor(receiver)
  )

# ── データの概要を確認 ──
tibble(
  `全ペア数`     = nrow(dat_bin),
  `助言あり`     = sum(dat_bin$tie),
  `ベースレート` = mean(dat_bin$tie) |> round(3)
)

tie_rate が低いということは、大半のペアで tie = 0 であることを意味します。このようなスパースなデータでは、各学習者が複数のペアに繰り返し出現するという依存構造（sender が 29 回、receiver が 29 回出現）が推定に大きく影響します。GLM はこの依存を無視するため、標準誤差が過小評価される典型的な場面です。

8.7 階層性の確認: ICC とデザインエフェクト

このデータは「有向ペア」が観測単位ですが、観測は sender / receiver に跨って 繰り返し出現します。したがって独立性仮定は破れやすく、GLM 単独では 標準誤差が過小になる可能性があります(Hox et al., 2018; Snijders & Bosker, 2012)。

8.7.1 交差分類データ構造（本章データ）

  • Level 1: 有向ペア（sender → receiver）
  • 高次構造: sender と receiver の交差分類ランダム効果
  • 重要点: 厳密な単純階層（生徒がクラスにのみ入る形）ではないため、 ICC/DEFF は「構造把握の近似指標」として使う

8.7.2 null GLMM で ICC を推定

ICC（級内相関係数）を推定するために、まず固定効果を含まない空モデル（null model）を当てはめます。

コード 
# ── Step 1: 空モデルを当てはめる ──
# tie ~ 1 は「切片だけ」（固定効果なし）
# (1 | sender) + (1 | receiver) は交差分類ランダム切片
m_null <- glmer(
  tie ~ 1 + (1 | sender) + (1 | receiver),
  family = binomial(link = "logit"),
  data = dat_bin,
  control = glmerControl(
    optimizer = "bobyqa",           # 収束しやすい最適化手法
    optCtrl = list(maxfun = 2e5)    # 最大反復回数を増やす
  )
)

方法 A: performance::icc() で一発計算（実務向け）

実務では performance::icc() を使うのが最もシンプルです。 内部でロジスティック分布の残差分散（\(\pi^2/3\)）を自動的に考慮してくれます。

コード

    Adjusted ICC: 0.122
  Unadjusted ICC: 0.122

performance::icc() は adjusted ICC（条件付き）と unadjusted ICC（周辺）を返します(Luedeke, Ben-Shachar, Patil, & Makowski, 2026c)。 null モデルでは固定効果がないため両者は実質同じ値です。

方法 B: 手計算で原理を理解する

performance::icc() の中身を理解するために、ICC を手計算してみましょう。 ロジスティック GLMM では、Level 1 の残差分散を \(\pi^2/3 \approx 3.29\) と近似する 「潜在変数法」が標準的です(Hox et al., 2018; Snijders & Bosker, 2012)。

\[ \mathrm{ICC}_{\mathrm{sender}} = \frac{\sigma^2_{\mathrm{sender}}} {\sigma^2_{\mathrm{sender}} + \sigma^2_{\mathrm{receiver}} + \pi^2/3} \]

コード 
# ランダム効果の分散を取り出す
vc <- as.data.frame(VarCorr(m_null))
var_sender   <- vc |> filter(grp == "sender")   |> pull(vcov)
var_receiver <- vc |> filter(grp == "receiver") |> pull(vcov)
var_logistic <- (pi^2) / 3   # ≈ 3.29（ロジスティック分布の残差分散）

# ICC を手計算
icc_sender   <- var_sender   / (var_sender + var_receiver + var_logistic)
icc_receiver <- var_receiver / (var_sender + var_receiver + var_logistic)
icc_total    <- (var_sender + var_receiver) /
                (var_sender + var_receiver + var_logistic)

) |>

8.7.3 デザインエフェクト（DEFF）の近似

DEFF（デザインエフェクト）は、「独立性仮定が破れることで、実効サンプルサイズがどれだけ縮小するか」を示す指標です。DEFF = 2 なら、見かけのサンプルサイズの半分しか情報がないことを意味します。以下では ICC と平均クラスターサイズから DEFF を近似計算します。

コード 
# 各 sender / receiver が何ペアに登場するか（= 平均クラスターサイズ）
m_sender   <- dat_bin |> count(sender)   |> summarise(mean_n = mean(n)) |> pull()
m_receiver <- dat_bin |> count(receiver) |> summarise(mean_n = mean(n)) |> pull()

# DEFF = 1 + (クラスターサイズ - 1) × ICC
deff_sender   <- 1 + (m_sender - 1)   * icc_sender
deff_receiver <- 1 + (m_receiver - 1) * icc_receiver
deff_total    <- 1 + (mean(c(m_sender, m_receiver)) - 1) * icc_total

# 実効サンプルサイズ = 観測数 / DEFF
n_obs       <- nrow(dat_bin)
n_effective <- n_obs / deff_total

単純階層データでは \(\mathrm{DEFF}=1+(m-1)\mathrm{ICC}\) が標準式です(Kish, 1965)。 本章は交差分類なので厳密適用ではありませんが、独立性違反の強さを概観する 目的には有用です。DEFF が 1 から離れるほど、GLM の独立性仮定は危うくなります。

実務での読み方（ICC/DEFF）
  • ICC が 0 に近い: 群内相関は小さく、GLM と GLMM の差は小さいことが多い。
  • ICC が中程度以上: GLMM を優先し、標準誤差の過小評価を避ける。
  • DEFF が 1 を大きく超える: 独立性仮定の破れが大きい可能性がある。
  • 交差分類では ICC/DEFF は「採用判断の補助」であり、最終判断は compare_performance() と診断を併用する。

8.8 ステップ1: GLM（基準モデル）

まずランダム効果なしのロジスティック回帰で基準線を作ります。この GLM は「全ペアが独立」と仮定しており、sender/receiver の繰り返し構造を無視しています。目的は GLMM との比較基準を作ることであり、このモデル自体を最終モデルとして採用するわけではありません。

8.9 ステップ2: GLMM（ランダム切片）

次に sender / receiver のランダム切片を追加します。(1 | sender) + (1 | receiver) は「各送り手と各受け手が、全体平均からどれだけ偏るか」を推定する項です。

  • (1 | sender): 「助言をしやすい人/しにくい人」の個人差
  • (1 | receiver): 「助言を受けやすい人/受けにくい人」の個人差
  • この 2 つは 交差分類（同じ 30 人のプールから sender と receiver が出ている）

control = glmerControl(optimizer = "bobyqa", optCtrl = list(maxfun = 2e5)) は、デフォルトの最適化アルゴリズムでは収束しにくい場合の標準対処です。bobyqa（制約付き最適化）と最大反復回数の増加を組み合わせています。

コード 
# GLMM: 交差分類ランダム切片（sender + receiver）
m_glmm_ri <- glmer(
  tie ~ same_class + prof_gap + score_sender_sc + score_receiver_sc + score_gap +
    (1 | sender) +    # sender のランダム切片（助言をしやすい/しにくい個人差）
    (1 | receiver),   # receiver のランダム切片（助言を受けやすい/受けにくい個人差）
  family = binomial(link = "logit"),
  data = dat_bin,
  control = glmerControl(
    optimizer = "bobyqa",         # 収束しやすい最適化手法
    optCtrl = list(maxfun = 2e5)  # 最大反復回数
  )
)

8.10 ステップ3: ランダム傾き候補と特異適合

ランダム切片モデルは「全体平均からの偏り」だけを推定しますが、ランダム傾きモデルは「固定効果の強さ自体が個人ごとに異なるか」を検討します。

ここでは (1 + same_class | sender) を試します。これは「same_class の効果が sender ごとに異なるか」——つまり「同クラスを好む傾向に送り手間の個人差があるか」を意味します。ただし、ランダム構造が過剰だとモデルが 特異適合（singular fit） を起こし、分散成分が 0 に張り付くことがあります。

コード 
# GLMM: ランダム傾き追加（same_class の効果が sender ごとに異なるか？）
m_glmm_rs <- glmer(
  tie ~ same_class + prof_gap + score_sender_sc + score_receiver_sc + score_gap +
    (1 + same_class | sender) +  # 切片 + same_class の傾きを sender ごとに推定
    (1 | receiver),
  family = binomial(link = "logit"),
  data = dat_bin,
  control = glmerControl(
    optimizer = "bobyqa",
    optCtrl = list(maxfun = 2e5)
  )
)

# 特異適合の確認：FALSE なら安全、TRUE なら過剰パラメータ化
isSingular(m_glmm_ri)  # ランダム切片モデル
[1] FALSE
コード 
isSingular(m_glmm_rs)  # ランダム傾き追加モデル
[1] TRUE

isSingular() の読み方：

  • FALSE → 特異適合ではない。ランダム効果構造は推定可能。
  • TRUE → 特異適合が発生。分散成分の一部が 0 に張り付いている。
特異適合が起きたときの対処法
  1. 無視しない: 特異適合はモデルの過剰パラメータ化の徴候であり、推定結果の信頼性に影響する。
  2. ランダム構造を簡素化: ランダム傾きを落とし、ランダム切片モデルに戻す。
  3. 試みたことを報告: 論文では「ランダム傾きを試みたが特異適合が生じたため、ランダム切片モデルを採用した」と書くことが推奨される(Bates, Kliegl, et al., 2015; Matuschek et al., 2017)。SLA 分野での具体的な判断例は Linck & Cunnings (2015) を参照。

本データでは、m_glmm_ri（ランダム切片のみ）が FALSE であれば安全に使用でき、m_glmm_rs（ランダム傾き追加）が TRUE であれば、ランダム切片モデルに留めるのが実務上の正解です。

score_sender_sc 送り手のスコアが 1 SD 高いごとに、助言のオッズが OR 倍に変化
score_receiver_sc 受け手のスコアが 1 SD 高いごとに、助言のオッズが OR 倍に変化
score_gap スコア差が 1 SD 広がるごとに、助言のオッズが OR 倍に変化
OR 報告の 4 点セット
  1. 方向: OR > 1（促進）か OR < 1（抑制）か
  2. OR と 95% CI: OR = X.XX, 95% CI [LL, UL]
  3. p 値: 統計的有意性
  4. 確率差への変換（応答スケール）: 次のセクションで emmeans を使って報告
ノート

OR は確率差そのものではありません。例えば OR = 2.0 は「オッズが 2 倍」であり、確率が 2 倍になるわけではありません。確率差を知るには、次のセクションの応答スケール予測が必要です。

8.14.1 marginal R² と conditional R²（GLMMで必須）

混合効果モデルの \(R^2\) は、単一の値ではなく marginal / conditional を分けて報告するのが標準です (Johnson, 2014; Nakagawa et al., 2017; Nakagawa & Schielzeth, 2013)。

\[ R^2_{\mathrm{marginal}} = \frac{\mathrm{Var}(X\beta)} {\mathrm{Var}(X\beta)+\mathrm{Var}(Zu)+\mathrm{Var}_{\mathrm{dist}}} \]

\[ R^2_{\mathrm{conditional}} = \frac{\mathrm{Var}(X\beta)+\mathrm{Var}(Zu)} {\mathrm{Var}(X\beta)+\mathrm{Var}(Zu)+\mathrm{Var}_{\mathrm{dist}}} \]

  • marginal R²: 固定効果だけで説明される割合
  • conditional R²: 固定 + ランダム効果で説明される割合
  • 差分 conditional - marginal: ランダム効果が担う説明分の目安
コード 
# Nakagawa の R² を計算（GLMM 専用の分散分解）
r2_glmm <- performance::r2_nakagawa(m_glmm_ri)
r2_df   <- as.data.frame(r2_glmm)

# 見やすい表にまとめる
r2_tbl <- tibble(
  metric = c(
    "Marginal R²（固定効果のみ）",
    "Conditional R²（固定 + ランダム効果）",
    "ランダム効果の寄与（差分）"
  ),
  value = c(
    r2_df$R2_marginal[1],
    r2_df$R2_conditional[1],
    r2_df$R2_conditional[1] - r2_df$R2_marginal[1]  # この差がランダム効果の説明分
  )
)

  • R²の大小だけで採否を決めず、compare_performance() と残差診断を併用する。

8.15 5章からの接続: GLMM と GAMM の使い分け

  • GLMM: 非線形よりも、階層・反復構造の扱いが主課題のとき。
  • GAMM: 階層構造に加えて連続予測子の非線形も重要なとき(Pedersen et al., 2019; Wood, 2017, 2020)。

次の判断が有効です。

  1. まず GLMM（線形効果）
  2. 残差/理論から非線形が必要なら GAMM へ拡張
  3. 複雑化で解釈が落ちる場合は GLMM に戻して報告
アウトカムが 2 値以外の場合

本章は 2 値（binomial）ですが、glmer() の family 引数を変えるだけで同じ枠組みを他のデータ型に適用できます。

  5. R² の併記: marginal R² と conditional R² の両方を報告したか
  6. 診断結果: DHARMa（dispersion + uniformity）、VIF、残差プロットの結果を記載したか
  7. オプティマイザ設定: bobyqa、maxfun の値を記載したか（再現性のため）

  • 統計記号はイタリック: β, SE, z, p, OR, R2
  • p 値は p = .032、非常に小さい場合は p < .001
  • 区間推定は 95% CI [LL, UL]

8.17 トラブルシューティング（よくあるつまずき）

収束警告（convergence warning）が出る

症状: Model failed to converge や max|grad| = ... といった警告メッセージ。

原因: 最適化アルゴリズムが最大反復回数内に収束しなかった。変数のスケールが大きく異なる場合や、ランダム効果構造が複雑すぎる場合に起こりやすい。

段階的な修正手順:

  1. 変数の標準化: 連続予測子を z スコア化する（scale()）。これが最も効果的。
  2. オプティマイザ変更: glmerControl(optimizer = "bobyqa") に変更。
  3. 反復回数増加: optCtrl = list(maxfun = 2e5) または 5e5。
  4. 別のオプティマイザ: nloptwrap や optimx パッケージの nlminb を試す。
  5. ランダム構造簡素化: ランダム傾きを落とす。
# 段階的対処の実装例
control_step1 <- glmerControl(optimizer = "bobyqa", optCtrl = list(maxfun = 2e5))
control_step2 <- glmerControl(optimizer = "bobyqa", optCtrl = list(maxfun = 5e5))
特異適合（singular fit）

症状: isSingular(model) が TRUE を返す。boundary (singular) fit という警告。

原因: ランダム効果の分散成分が 0（またはその境界）に推定された。モデルの複雑さに対してデータが不足している。

修正手順:

  1. VarCorr(model) で分散が 0 に近い項を特定。
  2. その項を含むランダム効果を除外してモデルを再推定。
  3. 論文には「ランダム傾きを試みたが特異適合が生じたため、ランダム切片モデルを採用した」と記載。
OR の誤解

症状: OR = 2.0 を「確率が 2 倍」と誤って解釈。

原因: OR（オッズ比）と確率比（リスク比）の混同。

正しい理解: OR = 2.0 は「オッズが 2 倍」。ベースライン確率が低い場合（例：p = .05）は確率もほぼ 2 倍に近いが、ベースラインが高い場合（例：p = .50）はオッズ 2 倍でも確率は .50 → .67 程度の変化にとどまる。

対処: OR だけでなく、emmeans() による応答スケール予測（確率差）を必ず併記する。

re.form の混同

症状: 予測値が観測値と大きくずれる、または個人ごとの予測が全員同じになる。

原因: predict() の re.form 引数の誤用。

正しい使い分け:

引数 意味 用途
re.form = NA ランダム効果を含まない（母平均的予測） 論文の主要結果図、一般的な効果の報告
re.form = NULL ランダム効果を含む（条件付き予測） 個人レベルの予測、モデルの適合度確認
re.form = ~0 NA と同じ 同上
DHARMa の警告が出た

症状: testDispersion や testUniformity が有意（p < .05）。

原因と対処:

  • testDispersion が有意: 過分散の可能性。2値データでは観測レベルのランダム効果（(1 | obs_id)）を追加するか、MASS::glmmPQL を検討。ただし、2値単独観測では過分散は定義上起こらないため、群レベルの集計をしていないか確認。
  • testUniformity が有意: モデル構造のミスフィット。非線形効果の追加（GAMM への拡張）や、外れ値の確認を検討。
  • 有意だが視覚的に問題がない場合: サンプルサイズが大きいと微小な逸脱でも有意になることがあります。QQ プロットと残差プロットを視覚的に確認し、系統的パターンがなければ実務上は問題ないと判断できます。混合モデルは軽度の分布仮定の逸脱に対して頑健であることがシミュレーションで確認されているため(Schielzeth et al., 2020)、検定が有意でも視覚的に問題がなければ過度に心配する必要はありません。
完全分離（complete separation）

症状: glmer が収束しない、または係数が極端に大きい（|β| > 10）。

原因: ある予測子の値によってアウトカムが完全に予測される（例：same_class = 1 のとき tie が常に 1）。

修正手順:

  1. クロス表で完全分離の有無を確認: table(dat_bin$same_class, dat_bin$tie)
  2. 完全分離がある場合、blme::bglmer() で弱い事前分布を追加するか、Firth 補正を検討。
  3. カテゴリの統合や、問題のある予測子の除外も選択肢。
グループ数が少ない

症状: ランダム効果の分散推定が不安定。推定値が極端に大きいまたは小さい。

原因: ランダム効果のグループ数（例：sender 数）が少ない（目安として < 10）と、分散の推定に十分な情報がない。

対処:

  • グループ数が 5 未満の場合、ランダム効果ではなく固定効果として投入することを検討。
  • グループ数が 10–20 の場合、ランダム切片のみに留め、ランダム傾きは避ける。
  • ベイズ推定（brms パッケージ）は少グループでもより安定した推定を行える。

## はじめに

5章では GLM/GAM の使い分けを扱いました。次の実務課題は、
**データの独立性が崩れる場面**です。

英語教育データでは「同じ学習者が複数のペアに登場する」「同じテスト項目に複数の学習者が回答する」
といった**繰り返し構造**がほぼ必ず存在します。
このとき GLM を使うと、実際には独立でない観測を独立と見なすことになり、
標準誤差が過小評価され、p 値が楽観的に出てしまいます[@baayen2008; @meteyard2020]。

この問題に対する標準解が
**一般化線形混合効果モデル（GLMM）**です。
GLMM は、報告したい効果（固定効果）と個人差・項目差（ランダム効果）を
同時にモデル化することで、標準誤差を**正直に**推定します[@bates2015]。
SLA（第二言語習得）分野でも GLMM は標準ツールとなっており、
@cunnings2012 と @linck2015 が分野に特化したチュートリアルを提供しています。
2値言語データに対するロジット混合モデルの有効性も、
複数の方法論研究で一貫して示されてきました[@jaeger2008; @quene2008]。

本章では、実データを用いて 2値ロジスティック GLMM を
「最小モデルから診断・報告まで」一気通貫で実装します。

::: {.callout-tip}
### この章の到達目標

- GLM と GLMM を同一データで比較し、ランダム効果の必要性を説明できる。
- 交差分類ランダム効果（sender / receiver）を実装できる。
- ICC とデザインエフェクトを用いて、独立性仮定の破れを説明できる。
- OR と予測確率を併記して、読み手に伝わる結果提示ができる。
- 特異適合・過分散・残差の点検を、最小限の手順で実行できる。
:::

## GLMM が必要になる理由（最短版）

::: {.callout-important}
#### GLM と GLMM の核心的な違い

GLM は**推定値そのもの**（β）を大きく変えるわけではありません。GLMM の本質は**標準誤差を正直にする**ことにあります。

- **GLM**: 全ペアが独立と仮定 → SE が過小 → p 値が楽観的
- **GLMM**: sender/receiver の繰り返しを考慮 → SE が適切 → p 値が正直

つまり、GLMM は「効果を見つける」ためではなく、「効果の不確かさを正しく評価する」ために使います。
:::

### GLMM の数学的表現

2値 GLMM は次で表されます[@nelder1972; @bates2015]。

$$
g(p_{ij}) = \mathbf{x}_{ij}^\top\beta + \mathbf{z}_{ij}^\top b,
\quad b \sim \mathcal{N}(0, \Sigma)
$$

この式を日本語で読み下すと：

> **ペア $ij$ の助言確率 $p_{ij}$ をリンク関数 $g()$ で変換した値は、「固定効果（全員に共通の平均的効果 $\beta$）」と「ランダム効果（個人ごとの偏り $b$）」の和で表される。ランダム効果 $b$ は平均ゼロの正規分布から生成される。**

もう少し直感的に言えば：

- **固定効果 $\beta$**（報告したい効果）: 「同クラスだと助言しやすい」「スコアが高いと助言しやすい」など、研究で検証したい平均的な効果
- **ランダム効果 $b$**（統制したいばらつき）: 「田中さんは全般に助言をよくする」「鈴木さんは助言をあまり受けない」など、個人ごとの偏り

::: {.callout-tip}
#### この式がピンとこない場合

数式よりもコードで理解したいタイプの方には、以下の入門チュートリアルを先に読むことをお勧めします。

1. @winter2013 — 言語学データの例で、線形モデルから混合モデルへの拡張を段階的に解説。
2. @brown2021 — 心理学データを使い、`lme4` での実装を一から追える R チュートリアル。
3. @debruine2021 — 「自分でデータを生成して、ランダム効果が何をしているか体感する」という独自のアプローチ。数式の意味が腑に落ちます。

いずれも本章の前提知識として有用ですが、**本章だけでも完結する設計**になっています。
:::

::: {.callout-note}
#### 交差分類 vs 入れ子

- **入れ子（nested）**: 生徒がクラスにのみ属する形（例：生徒 1-10 はクラス A、11-20 はクラス B）。`(1 | class/student)` で実装。
- **交差分類（crossed）**: 本データでは sender と receiver が同じ 30 人のプールから出ている。sender 1 は receiver 2, 3, ... 29 と全員のペアを作る。`(1 | sender) + (1 | receiver)` で実装。

英語教育では「学習者差」「項目差」が同時に入る交差分類が多いです[@baayen2008; @barr2013; @quene2008]。
:::

### GLMM でもリンク関数は選べる

本章の主モデルは `logit` ですが、GLMM でも `probit` や `cloglog` を選択できます。
実装上は `glmer(..., family = binomial(link = \"...\"))` を変えるだけです。

- `logit`: OR 解釈が可能（実務上もっとも扱いやすい）
- `probit`: 潜在正規尺度での解釈（確率予測は logit と近いことが多い）
- `cloglog`: 非対称リンク。離散時間ハザードの文脈と整合的[@prentice1978]

リンクの候補は `stats::family` / `stats::make.link` を参照してください
[@rstatsfamily; @rmakelink]。また、`log-log` は `cloglog` の反転関係として扱えます
[@vgamcloglog]。

### モデル前提チェックリスト（GLMM）

GLMM の前提は「正規残差」ではなく、次の**構造的条件**を確認することです
[@bolker2009; @meteyard2020; @glmmfaq]。

1. `outcome` と `family/link` が整合しているか（本章は 2値 + logit）。
2. 群化構造（sender/receiver）が研究設計に一致しているか。
3. ランダム効果構造が過剰でないか（`isSingular()`, `rePCA()`）。
4. 過分散・残差の大きな破綻がないか（`DHARMa`, `check_overdispersion()`）。
5. 固定効果の推定が不安定でないか（`check_collinearity()`）。

::: {.callout-important}
#### よくある誤解：「残差が正規分布でないからダメ」

GLM/GLMM では **アウトカムの正規性は仮定していません**。family（本章では `binomial`）がアウトカムの型と整合していれば、残差の正規性はそもそも問題になりません。上の 5 項目が GLMM の本当のチェックポイントです。

なお、混合モデルは分布仮定からの軽度な逸脱に対して比較的頑健であることが、大規模シミュレーション研究で確認されています[@schielzeth2020]。
:::

::: {.callout-tip}
#### 本章での対応関数

- 構造比較: `compare_performance()`, `anova(..., test="Chisq")`
- 特異適合: `isSingular()`, `performance::check_singularity()`, `rePCA()`
- 残差診断: `DHARMa::simulateResiduals()`, `testDispersion()`, `testUniformity()`
- 補助診断: `check_overdispersion()`, `check_collinearity()`, `check_model()`
:::

## データ準備（実データ）

本章は `data/network` の実データを使います。

- `nodes.csv`: 助言ネットワークの学習者属性（`id`, `class`, `proficiency`）
- `edges_directed_advice.csv`: 有向助言関係（from → to）
- `nodes_collaboration.csv`: ベースライン得点（共変量に使用）

以下のコードでは、ノード属性と助言エッジを読み込み、`proficiency` を順序つきファクター（A2 < B1 < B2）に変換し、ベースラインスコアを標準化（z スコア）します。`factor()` への変換は、後に `glmer()` がグループ変数を正しく認識するために必要です。

```{r}
#| label: data-load

# ── CSV ファイルの読み込み ──
nodes_advice <- readr::read_csv("data/network/nodes.csv",
                                show_col_types = FALSE)

nodes_collab <- readr::read_csv("data/network/nodes_collaboration.csv",
                                show_col_types = FALSE) |>
  dplyr::select(id, baseline_score)  # 必要な列だけ選択

edges_advice <- readr::read_csv("data/network/edges_directed_advice.csv",
                                show_col_types = FALSE)

# ── 変数の前処理 ──
prof_levels <- c("A2", "B1", "B2")          # 順序を定義
prof_map    <- c(A2 = 1, B1 = 2, B2 = 3)   # 数値に変換するマップ

nodes <- nodes_advice |>
  left_join(nodes_collab, by = "id") |>     # ベースラインスコアを結合
  mutate(
    class = factor(class),                                        # ファクター化
    proficiency = factor(proficiency, levels = prof_levels),       # 順序付きファクター
    prof_num = unname(prof_map[as.character(proficiency)]),        # 数値化（差の計算用）
    baseline_score_sc = as.numeric(scale(baseline_score))          # z 標準化
  )

head(nodes, 8) |>
  gt() |>
  tab_header(title = "Node Data Overview: First 8 Rows")
```

`summary()` の出力で確認すべき点：

- **class**: 各クラスの人数バランス。極端な偏りがあるとランダム効果の推定が不安定になる。
- **proficiency**: A2/B1/B2 の分布。3水準が揃っているか。
- **baseline_score_sc**: 平均≈0、SD≈1 になっていれば標準化は正常。

## 分析データ作成（有向ペア単位）

`sender != receiver` の全有向ペアを作り、
各ペアが実際に助言エッジとして観測されたか（`tie`）を 0/1 で定義します。

**変数名のポイント**: `sender`/`receiver` という命名にしておくと、後の `glmer()` 式で `(1 | sender) + (1 | receiver)` と書くだけで交差分類ランダム効果が実装でき、コードの可読性が上がります。また、`factor()` への変換は `lme4` がランダム効果のグループ変数を正しく認識するために必須です。

```{r}
#| label: make-analysis-data

# ── 全有向ペアを生成（自分自身は除外）──
dat_bin <- tidyr::crossing(
  sender   = nodes$id,
  receiver = nodes$id
) |>
  filter(sender != receiver) |>

  # sender 側の属性を結合
  left_join(
    nodes |>
      dplyr::select(sender = id, class_sender = class,
                    prof_sender = proficiency, prof_sender_num = prof_num,
                    score_sender_sc = baseline_score_sc),
    by = "sender"
  ) |>

  # receiver 側の属性を結合
  left_join(
    nodes |>
      dplyr::select(receiver = id, class_receiver = class,
                    prof_receiver = proficiency, prof_receiver_num = prof_num,
                    score_receiver_sc = baseline_score_sc),
    by = "receiver"
  ) |>

  mutate(
    # ── 予測子の作成 ──
    same_class = as.integer(class_sender == class_receiver),  # 同クラスなら 1
    prof_gap   = abs(prof_sender_num - prof_receiver_num),    # 熟達度差（0, 1, 2）
    score_gap  = abs(score_sender_sc - score_receiver_sc),    # スコア差（連続量）

    # ── アウトカム：助言エッジがあるか ──
    tie = as.integer(
      paste(sender, receiver) %in% paste(edges_advice$from, edges_advice$to)
    ),

    # ── glmer のランダム効果に必要なファクター化 ──
    sender   = factor(sender),
    receiver = factor(receiver)
  )

# ── データの概要を確認 ──
tibble(
  `全ペア数`     = nrow(dat_bin),
  `助言あり`     = sum(dat_bin$tie),
  `ベースレート` = mean(dat_bin$tie) |> round(3)
)
```

出力の読み方：

| 指標 | 確認ポイント |
|------|------------|
| `全ペア数` | 全有向ペア数 = $n \times (n-1)$（n は学習者数）。30人なら 870 行 |
| `助言あり` | 実際に助言関係があったペア数 |
| `ベースレート` | 助言関係の割合。**極端に低い**（< 0.05）場合は完全分離のリスクあり |

::: {.callout-important}
#### ベースレートと依存構造

`tie_rate` が低いということは、大半のペアで `tie = 0` であることを意味します。このようなスパースなデータでは、各学習者が複数のペアに繰り返し出現するという依存構造（sender が 29 回、receiver が 29 回出現）が推定に大きく影響します。GLM はこの依存を無視するため、標準誤差が過小評価される典型的な場面です。
:::

## 階層性の確認: ICC とデザインエフェクト

このデータは「有向ペア」が観測単位ですが、観測は sender / receiver に跨って
繰り返し出現します。したがって独立性仮定は破れやすく、GLM 単独では
標準誤差が過小になる可能性があります[@snijders2012; @hox2018]。

### 交差分類データ構造（本章データ）

- Level 1: 有向ペア（sender → receiver）
- 高次構造: sender と receiver の**交差分類ランダム効果**
- 重要点: 厳密な単純階層（生徒がクラスにのみ入る形）ではないため、
  ICC/DEFF は「構造把握の近似指標」として使う

### null GLMM で ICC を推定

ICC（級内相関係数）を推定するために、まず固定効果を含まない**空モデル（null model）**を当てはめます。

```{r}
#| label: icc-null-glmm

# ── Step 1: 空モデルを当てはめる ──
# tie ~ 1 は「切片だけ」（固定効果なし）
# (1 | sender) + (1 | receiver) は交差分類ランダム切片
m_null <- glmer(
  tie ~ 1 + (1 | sender) + (1 | receiver),
  family = binomial(link = "logit"),
  data = dat_bin,
  control = glmerControl(
    optimizer = "bobyqa",           # 収束しやすい最適化手法
    optCtrl = list(maxfun = 2e5)    # 最大反復回数を増やす
  )
)
```

#### 方法 A: `performance::icc()` で一発計算（実務向け）

実務では `performance::icc()` を使うのが最もシンプルです。
内部でロジスティック分布の残差分散（$\pi^2/3$）を自動的に考慮してくれます。

```{r}
#| label: icc-performance

performance::icc(m_null)
```

`performance::icc()` は adjusted ICC（条件付き）と unadjusted ICC（周辺）を返します[@performance_icc]。
null モデルでは固定効果がないため両者は実質同じ値です。

#### 方法 B: 手計算で原理を理解する

`performance::icc()` の中身を理解するために、ICC を手計算してみましょう。
ロジスティック GLMM では、Level 1 の残差分散を $\pi^2/3 \approx 3.29$ と近似する
「潜在変数法」が標準的です[@snijders2012; @hox2018]。

$$
\mathrm{ICC}_{\mathrm{sender}}
= \frac{\sigma^2_{\mathrm{sender}}}
       {\sigma^2_{\mathrm{sender}} + \sigma^2_{\mathrm{receiver}} + \pi^2/3}
$$

```{r}
#| label: icc-manual

# ランダム効果の分散を取り出す
vc <- as.data.frame(VarCorr(m_null))
var_sender   <- vc |> filter(grp == "sender")   |> pull(vcov)
var_receiver <- vc |> filter(grp == "receiver") |> pull(vcov)
var_logistic <- (pi^2) / 3   # ≈ 3.29（ロジスティック分布の残差分散）

# ICC を手計算
icc_sender   <- var_sender   / (var_sender + var_receiver + var_logistic)
icc_receiver <- var_receiver / (var_sender + var_receiver + var_logistic)
icc_total    <- (var_sender + var_receiver) /
                (var_sender + var_receiver + var_logistic)

tibble(
  component = c("sender ICC", "receiver ICC", "total clustered share"),
  estimate  = c(icc_sender, icc_receiver, icc_total)
) |>
  gt() |>
  fmt_number(columns = estimate, decimals = 3) |>
  tab_header(title = "ICC（手計算：潜在変数近似）")
```

::: {.callout-tip}
#### ICC の読み方

| 値 | 意味 |
|----|------|
| **sender ICC** | 助言行動のばらつきのうち、「誰が送り手か」で説明される割合 |
| **receiver ICC** | 「誰が受け手か」で説明される割合 |
| **total** | 両方を合わせた群内相関。GLM → GLMM の必要性の主要な根拠 |

方法 A と方法 B の値がほぼ一致することを確認してください。一致していれば、`performance::icc()` が正しく潜在変数近似を使っていることがわかります。
:::

### デザインエフェクト（DEFF）の近似

DEFF（デザインエフェクト）は、「独立性仮定が破れることで、実効サンプルサイズがどれだけ縮小するか」を示す指標です。DEFF = 2 なら、見かけのサンプルサイズの半分しか情報がないことを意味します。以下では ICC と平均クラスターサイズから DEFF を近似計算します。

```{r}
#| label: design-effect-approx

# 各 sender / receiver が何ペアに登場するか（= 平均クラスターサイズ）
m_sender   <- dat_bin |> count(sender)   |> summarise(mean_n = mean(n)) |> pull()
m_receiver <- dat_bin |> count(receiver) |> summarise(mean_n = mean(n)) |> pull()

# DEFF = 1 + (クラスターサイズ - 1) × ICC
deff_sender   <- 1 + (m_sender - 1)   * icc_sender
deff_receiver <- 1 + (m_receiver - 1) * icc_receiver
deff_total    <- 1 + (mean(c(m_sender, m_receiver)) - 1) * icc_total

# 実効サンプルサイズ = 観測数 / DEFF
n_obs       <- nrow(dat_bin)
n_effective <- n_obs / deff_total

tibble(
  quantity = c(
    "観測数（ペア数）",
    "平均クラスターサイズ（sender）",
    "平均クラスターサイズ（receiver）",
    "DEFF sender", "DEFF receiver", "DEFF total",
    "実効サンプルサイズ ≈ 観測数 / DEFF total"
  ),
  value = c(n_obs, m_sender, m_receiver,
            deff_sender, deff_receiver, deff_total,
            n_effective)
) |>
  gt() |>
  fmt_number(columns = value, decimals = 1) |>
  tab_header(title = "Design Effect（DEFF）の近似計算")
```

**DEFF 表の読み方**：

| 指標 | 解釈 |
|------|------|
| 平均クラスターサイズ | 各 sender/receiver が関与するペア数の平均（n-1 に近い）|
| DEFF sender / receiver | 繰り返し構造による情報の目減り。**1 なら独立と同等**、大きいほど深刻 |
| DEFF total | 両方を合わせた概算 |
| 実効サンプルサイズ | **表の最終行が実効 N** です。観測数との差が大きいほど、GLM の SE は楽観的すぎる |

単純階層データでは $\mathrm{DEFF}=1+(m-1)\mathrm{ICC}$ が標準式です[@kish1965]。
本章は交差分類なので厳密適用ではありませんが、**独立性違反の強さを概観する**
目的には有用です。DEFF が 1 から離れるほど、GLM の独立性仮定は危うくなります。

::: {.callout-note}
#### 実務での読み方（ICC/DEFF）

- ICC が 0 に近い: 群内相関は小さく、GLM と GLMM の差は小さいことが多い。
- ICC が中程度以上: GLMM を優先し、標準誤差の過小評価を避ける。
- DEFF が 1 を大きく超える: 独立性仮定の破れが大きい可能性がある。
- 交差分類では ICC/DEFF は「採用判断の補助」であり、最終判断は
  `compare_performance()` と診断を併用する。
:::

## ステップ1: GLM（基準モデル）

まずランダム効果なしのロジスティック回帰で基準線を作ります。この GLM は「全ペアが独立」と仮定しており、sender/receiver の繰り返し構造を無視しています。**目的は GLMM との比較基準を作ること**であり、このモデル自体を最終モデルとして採用するわけではありません。

固定効果の意味：

| 予測子 | 意味 |
|--------|------|
| `same_class` | 同じクラスに所属するペアか（0/1）|
| `prof_gap` | 熟達度水準の差の絶対値（0, 1, 2）|
| `score_sender_sc` | 送り手のベースラインスコア（z 標準化）|
| `score_receiver_sc` | 受け手のベースラインスコア（z 標準化）|
| `score_gap` | スコア差の絶対値 |

```{r}
#| label: fit-glm

# ランダム効果なしの標準ロジスティック回帰（比較基準）
m_glm <- glm(
  tie ~ same_class + prof_gap + score_sender_sc + score_receiver_sc + score_gap,
  family = binomial(link = "logit"),  # 2値 → ロジスティック
  data = dat_bin
)

broom::tidy(m_glm, conf.int = TRUE) |>
  gt() |>
  tab_header(title = "Binomial GLM: Logistic Regression") |>
  fmt_number(columns = where(is.numeric), decimals = 3)
```

**`summary(m_glm)` の読み方**：

| 出力項目 | 読み方 |
|---------|--------|
| `Estimate` | logit スケールでの係数。正なら助言関係の **オッズが増加** する方向 |
| `Std. Error` | 係数の標準誤差。**GLM はペアの独立性を仮定しているため、この SE は過小評価の可能性が高い** |
| `z value` | Estimate / SE。Wald 検定統計量 |
| `Pr(>\|z\|)` | 帰無仮説（係数 = 0）の p 値 |
| `AIC` | モデルの相対的な当てはまり。後で GLMM の AIC と比較する |

::: {.callout-warning}
#### この GLM の SE をそのまま信用しない

ICC/DEFF の分析で見たとおり、このデータには sender/receiver による依存構造があります。GLM はこの構造を無視するため、標準誤差が過小に見積もられ、**p 値が本来より小さく出ている可能性**があります。次のステップで GLMM に移行した際に、SE がどのように変化するか（通常は大きくなる）を確認してください。
:::

## ステップ2: GLMM（ランダム切片）

次に sender / receiver のランダム切片を追加します。`(1 | sender) + (1 | receiver)` は「各送り手と各受け手が、全体平均からどれだけ偏るか」を推定する項です。

- `(1 | sender)`: 「助言をしやすい人/しにくい人」の個人差
- `(1 | receiver)`: 「助言を受けやすい人/受けにくい人」の個人差
- この 2 つは **交差分類**（同じ 30 人のプールから sender と receiver が出ている）

`control = glmerControl(optimizer = "bobyqa", optCtrl = list(maxfun = 2e5))` は、デフォルトの最適化アルゴリズムでは収束しにくい場合の標準対処です。`bobyqa`（制約付き最適化）と最大反復回数の増加を組み合わせています。

```{r}
#| label: fit-glmm-ri

# GLMM: 交差分類ランダム切片（sender + receiver）
m_glmm_ri <- glmer(
  tie ~ same_class + prof_gap + score_sender_sc + score_receiver_sc + score_gap +
    (1 | sender) +    # sender のランダム切片（助言をしやすい/しにくい個人差）
    (1 | receiver),   # receiver のランダム切片（助言を受けやすい/受けにくい個人差）
  family = binomial(link = "logit"),
  data = dat_bin,
  control = glmerControl(
    optimizer = "bobyqa",         # 収束しやすい最適化手法
    optCtrl = list(maxfun = 2e5)  # 最大反復回数
  )
)

broom.mixed::tidy(m_glmm_ri, conf.int = TRUE) |>
  gt() |>
  tab_header(title = "GLMM: Random Intercept Model") |>
  fmt_number(columns = where(is.numeric), decimals = 3)
```

**`summary(m_glmm_ri)` の読み方**：

GLMM の出力は GLM と構造が似ていますが、**ランダム効果の分散成分**が追加されています。

**ランダム効果セクション**：

| 出力項目 | 読み方 |
|---------|--------|
| `Groups` / `Name` | ランダム効果のグループ変数と項（ここでは sender と receiver の切片）|
| `Variance` | そのグループの分散。**大きいほど個人差が大きい** |
| `Std.Dev.` | 分散の平方根（logit スケール）。0.5 以上なら実質的な個人差がある |
| `Number of obs` / `groups` | 観測数とグループ数 |

**固定効果セクション**：GLM と同じ列構成ですが、**SE が GLM より大きくなっている**はずです。これが GLMM の本質的な効果——依存構造を考慮することで SE が「正直に」なるのです。

::: {.callout-important}
#### GLM と GLMM の SE を比較する

`summary(m_glm)` と `summary(m_glmm_ri)` の SE を並べて見てください。GLMM の SE が GLM より大きくなっていれば、GLM が独立性仮定のもとで SE を過小評価していた証拠です。**推定値（Estimate）自体はそれほど変わらないが、SE が変わる**——これが GLMM の核心です。「GLMM は推定値を変えるのではなく、精度の主張を正直にする」と理解してください。
:::

分散成分の大きさを直感的に解釈するには、ロジスティック分布の残差分散 $\pi^2/3 \approx 3.29$ と比較します。sender 分散が 0.5、receiver 分散が 0.3 なら、残差（$\pi^2/3$）に対する比率はそれぞれ約 15%、9% であり、sender 側の個人差がより大きいことがわかります。

## ステップ3: ランダム傾き候補と特異適合

ランダム切片モデルは「全体平均からの偏り」だけを推定しますが、**ランダム傾きモデル**は「固定効果の強さ自体が個人ごとに異なるか」を検討します。

ここでは `(1 + same_class | sender)` を試します。これは「same_class の効果が sender ごとに異なるか」——つまり「同クラスを好む傾向に送り手間の個人差があるか」を意味します。ただし、ランダム構造が過剰だとモデルが **特異適合（singular fit）** を起こし、分散成分が 0 に張り付くことがあります。

```{r}
#| label: fit-glmm-rs

# GLMM: ランダム傾き追加（same_class の効果が sender ごとに異なるか？）
m_glmm_rs <- glmer(
  tie ~ same_class + prof_gap + score_sender_sc + score_receiver_sc + score_gap +
    (1 + same_class | sender) +  # 切片 + same_class の傾きを sender ごとに推定
    (1 | receiver),
  family = binomial(link = "logit"),
  data = dat_bin,
  control = glmerControl(
    optimizer = "bobyqa",
    optCtrl = list(maxfun = 2e5)
  )
)

# 特異適合の確認：FALSE なら安全、TRUE なら過剰パラメータ化
isSingular(m_glmm_ri)  # ランダム切片モデル
isSingular(m_glmm_rs)  # ランダム傾き追加モデル
```

**`isSingular()` の読み方**：

- `FALSE` → 特異適合ではない。ランダム効果構造は推定可能。
- `TRUE` → **特異適合が発生**。分散成分の一部が 0 に張り付いている。

::: {.callout-warning}
#### 特異適合が起きたときの対処法

1. **無視しない**: 特異適合はモデルの過剰パラメータ化の徴候であり、推定結果の信頼性に影響する。
2. **ランダム構造を簡素化**: ランダム傾きを落とし、ランダム切片モデルに戻す。
3. **試みたことを報告**: 論文では「ランダム傾きを試みたが特異適合が生じたため、ランダム切片モデルを採用した」と書くことが推奨される[@bates2015parsimonious; @matuschek2017]。SLA 分野での具体的な判断例は @linck2015 を参照。

本データでは、`m_glmm_ri`（ランダム切片のみ）が `FALSE` であれば安全に使用でき、`m_glmm_rs`（ランダム傾き追加）が `TRUE` であれば、ランダム切片モデルに留めるのが実務上の正解です。
:::

## モデル比較ワークフロー

ステップ 1–3 でモデルの候補がそろいました。ここからはモデル比較に入ります。GLMM のモデル比較は **3 段階**で進めるのが定石です[@bolker2009; @barr2013; @bates2015parsimonious; @matuschek2017; @meteyard2020; @glmmfaq]。

```{mermaid}

flowchart TD
    A["① ランダム効果構造の比較\n（固定効果は固定のまま）"] --> B["② リンク関数の選択\n（logit / probit / cloglog）"]
    B --> C["③ 固定効果の比較\n（ランダム構造は固定のまま）"]
    C --> D["最終モデル確定 → 解釈・診断へ"]
    style A fill:#4682b4,color:#fff
    style B fill:#5f9ea0,color:#fff
    style C fill:#6b8e23,color:#fff
    style D fill:#cd853f,color:#fff
```

**ΔAIC の目安**（全ステップ共通）：

| ΔAIC | 解釈 |
|------|------|
| 0–2 | ほぼ同等。パラメータの少ないモデルを選ぶのが倹約的 |
| 4–7 | 中程度の証拠。ΔAIC が大きい側のモデルは劣る可能性 |
| 10+ | 強い証拠。ΔAIC が大きい側のモデルは実質的に支持されない |

### ① ランダム効果構造の比較

::: {.callout-important}
#### モデル比較の鉄則：一度に動かすのは一つだけ

ランダム効果構造と固定効果を同時に変えると、何が改善に寄与したか分からなくなります。手順は：

1. **固定効果を固定** → ランダム効果構造を比較（①）
2. **ランダム構造を固定** → リンク関数を比較（②）
3. **ランダム構造・リンクを固定** → 固定効果を比較（③）
:::

まず sender / receiver の組み合わせを変えた候補を推定し、GLM（ランダム効果なし）とステップ 2–3 で推定済みの `m_glmm_ri`・`m_glmm_rs` を含めて一括比較します。

```{r}
#| label: fit-random-structure-candidates

# 候補1: sender のランダム切片のみ
m_glmm_sender <- glmer(
  tie ~ same_class + prof_gap + score_sender_sc + score_receiver_sc + score_gap +
    (1 | sender),
  family = binomial(link = "logit"),
  data = dat_bin,
  control = glmerControl(optimizer = "bobyqa", optCtrl = list(maxfun = 2e5))
)

# 候補2: receiver のランダム切片のみ
m_glmm_receiver <- glmer(
  tie ~ same_class + prof_gap + score_sender_sc + score_receiver_sc + score_gap +
    (1 | receiver),
  family = binomial(link = "logit"),
  data = dat_bin,
  control = glmerControl(optimizer = "bobyqa", optCtrl = list(maxfun = 2e5))
)
# 候補3: m_glmm_ri（sender + receiver）と候補4: m_glmm_rs（ランダム傾き追加）は既に推定済み

# ── 全候補の AIC/BIC を一括比較 ──
make_ic_table(
  "GLM（ランダム効果なし）"       = m_glm,
  "RI: sender のみ"              = m_glmm_sender,
  "RI: receiver のみ"            = m_glmm_receiver,
  "RI: sender + receiver"        = m_glmm_ri,
  "RS: sender 傾き + receiver RI" = m_glmm_rs
) |>
  gt() |>
  tab_header(title = "① ランダム効果構造の候補比較")
```

**この表の読み方**：

1. **GLM vs GLMM**: ΔAIC が 10 以上なら、ランダム効果の追加が適合を大幅に改善している証拠。
2. **sender のみ vs receiver のみ vs 両方**: 両方入れることで AIC が改善しているかを確認。
3. **ランダム傾き**: AIC が改善していても特異適合（`isSingular = TRUE`）ならば採用は避ける。

`compare_performance()` で AIC/BIC/R² を同時に確認することもできます[@performance_compare]。

```{r}
#| label: compare-random-structure-performance

# AIC/BIC/R² を一括比較（conditional R² が marginal を大きく上回るほどランダム効果が重要）
performance::compare_performance(
  m_glm, m_glmm_sender, m_glmm_receiver, m_glmm_ri, m_glmm_rs
) |>
  gt() |>
  tab_header(title = "compare_performance(): ランダム効果構造の比較")
```

次に LRT（尤度比検定）でランダム効果構造を正式に比較します。LRT は **入れ子関係にあるモデル同士** でのみ有効です。

```{r}
#| label: lrt-random-structure

# 同一固定効果でのランダム構造比較（LRT）
anova(m_glmm_sender, m_glmm_ri, test = "Chisq")
anova(m_glmm_receiver, m_glmm_ri, test = "Chisq")
anova(m_glmm_ri, m_glmm_rs, test = "Chisq")
```

**LRT 出力の読み方**：

| 出力項目 | 読み方 |
|---------|--------|
| `Chisq` | カイ二乗検定統計量。大きいほど 2 モデル間の適合差が大きい |
| `Df` | パラメータ数の差（追加されたランダム効果の数）|
| `Pr(>Chisq)` | p 値。有意なら、より複雑なモデルが有意に適合を改善 |

::: {.callout-note}
#### 境界問題に注意

ランダム効果の分散はゼロ以上の制約があるため、帰無仮説（分散 = 0）はパラメータ空間の境界に位置します。このため LRT の p 値は**保守的**（やや大きめ）になる傾向があります[@self1987; @stram1994]。実務的には、LRT が有意でなくても AIC/BIC が改善していれば、ランダム効果の追加を検討する余地があります。
:::

次に特異適合の正式チェックと、ランダム効果の主成分分析（`rePCA()`）を行います。`rePCA()` はランダム効果の分散共分散行列を主成分分解し、各成分がどれだけ情報を持つかを示します。

```{r}
#| label: singularity-repca

# 特異適合チェック（FALSE = 安全、TRUE = 過剰パラメータ化）
isSingular(m_glmm_ri)  # ランダム切片モデル
isSingular(m_glmm_rs)  # ランダム傾きモデル

# ランダム効果の分散共分散構造を確認（分散が 0 に近い項があれば過剰）
VarCorr(m_glmm_rs)

# ランダム効果の主成分分析（各成分がどれだけ情報を持つか）
# → 1成分で 99%+ なら、ランダム傾きは実質不要
rePCA(m_glmm_rs)
```

`isSingular()` と `rePCA()` は、ランダム効果が過剰でないかを点検する定番です[@lme4issingular; @lme4repca]。

**出力の読み方**：

| 関数 | 結果 | 意味 |
|------|------|------|
| `isSingular()` | `FALSE` | 安全。ランダム効果構造は推定可能 |
| `isSingular()` | `TRUE` | 特異適合。分散成分の一部が 0 に収束 |
| `VarCorr()` | 分散 ≈ 0 の項あり | その項のランダム効果は過剰 |
| `rePCA()` | 1成分で 99%+ | ランダム傾きは実質的に不要 |

`m_glmm_rs` が特異適合かつ適合改善が小さい場合は、
ランダム切片モデルを採用するのが実務上の安全策です。

#### ランダム効果の可視化

```{r}
#| label: sjplot-random-effects

# ランダム効果の個人ごとの推定値を可視化
# → 各点が 1 人の sender（または receiver）
sjPlot::plot_model(m_glmm_ri, type = "re")
```

`plot_model(type = "re")` で sender / receiver のランダム効果分布を可視化します。

**図の読み方**：

- 各点が 1 人の sender（または receiver）のランダム効果推定値。
- **信頼区間が 0 をまたぐ**個人は、全体平均と有意に異ならない。
- 信頼区間が 0 をまたがない個人は、特に助言をしやすい（正）または しにくい（負）。
- sender と receiver のプロットを比べ、**どちらの個人差が大きいか**を視覚的に判断できる。散布の幅が大きいほど個人差が大きい。

### ② リンク関数の選択

ランダム効果構造が決まったら、次はリンク関数を選択します。5 章で学んだように、ロジスティック GLMM でも `logit`（標準）、`probit`（潜在正規尺度）、`cloglog`（非対称）の 3 つを選べます。

```{r}
#| label: link-compare-glmm

# リンク関数だけ変えて比較（固定効果・ランダム構造は同一）
m_glmm_probit <- glmer(
  tie ~ same_class + prof_gap + score_sender_sc + score_receiver_sc + score_gap +
    (1 | sender) + (1 | receiver),
  family = binomial(link = "probit"),   # 潜在正規尺度
  data = dat_bin,
  control = glmerControl(optimizer = "bobyqa", optCtrl = list(maxfun = 2e5))
)

m_glmm_cloglog <- glmer(
  tie ~ same_class + prof_gap + score_sender_sc + score_receiver_sc + score_gap +
    (1 | sender) + (1 | receiver),
  family = binomial(link = "cloglog"),  # 非対称リンク
  data = dat_bin,
  control = glmerControl(optimizer = "bobyqa", optCtrl = list(maxfun = 2e5))
)

# 3 リンク関数を AIC/BIC で比較
performance::compare_performance(m_glmm_ri, m_glmm_probit, m_glmm_cloglog) |>
  gt() |>
  tab_header(title = "② リンク関数の比較: logit vs probit vs cloglog")
```

AIC がほぼ同等であれば、**logit を採用するのが実務上の推奨**です。理由は（1）OR（オッズ比）として解釈可能、（2）読者が最も慣れている、（3）先行研究との比較が容易、の 3 点です。

#### GLM vs GLMM の係数比較（中間確認）

ここまでの選択を確認するため、`sjPlot::tab_model()` で GLM と GLMM の係数を並べて表示します。**SE の変化**に注目してください。

```{r}
#| label: sjplot-model-compare

# GLM と GLMM の係数を並べて比較する表
# → SE の変化と有意性の変化に注目
sjPlot::tab_model(
  m_glm, m_glmm_ri,
  show.aic = TRUE,   # AIC を表示
  show.se  = TRUE,   # 標準誤差を表示
  dv.labels = c("GLM（基準）", "GLMM（ランダム切片）")
)
```

**注目すべき点**：

- **SE の変化**: GLMM の SE が GLM より大きくなっていれば、GLM が SE を過小評価していた証拠。
- **有意性の変化**: GLM では有意だった予測子が GLMM では非有意になることがある。これは「見かけの有意性」が依存構造の無視によるものだったことを示す。
- **ICC / R²**: 表の下部に表示される。conditional R² が marginal R² より大きければ、ランダム効果が重要。

### ③ 固定効果の比較（ランダム構造は固定）

最後に、採用したランダム構造（ここでは `m_glmm_ri`）を固定したまま、固定効果を比較します[@lme4pvalues; @meteyard2020]。

以下では `update()` を使い、`m_glmm_ri` から各予測子を 1 つずつ落としたモデルを作成します。これにより「その予測子を入れることで適合がどれだけ改善するか」を個別に検証できます。

```{r}
#| label: fit-fixed-effect-candidates

# update() で「フルモデルから 1 予測子だけ落としたモデル」を作成
# → 各予測子の寄与を個別に検証するため
# 注: score_gap は score_sender_sc と score_receiver_sc から算出した派生変数なので、
#     ここでは主要な 4 予測子のみを個別に検証する
m_glmm_drop_same     <- update(m_glmm_ri, . ~ . - same_class)
m_glmm_drop_prof     <- update(m_glmm_ri, . ~ . - prof_gap)
m_glmm_drop_sender   <- update(m_glmm_ri, . ~ . - score_sender_sc)
m_glmm_drop_receiver <- update(m_glmm_ri, . ~ . - score_receiver_sc)
```

次に LRT で各予測子の寄与を正式に検定します。`drop1()` は「フルモデルから各予測子を 1 つずつ落とし、適合が有意に悪化するかを一括検定する」便利関数です。

```{r}
#| label: lrt-fixed-effects

# drop1(): 各予測子の LRT を一括実行
# → 「この予測子を落とすと適合がどれだけ悪化するか」を検定
drop1(m_glmm_ri, test = "Chisq")
```

**`drop1()` 出力の読み方**：

| 出力項目 | 読み方 |
|---------|--------|
| `Df` | その予測子のパラメータ数（通常 1）|
| `AIC` | その予測子を落とした場合の AIC。フルモデルより大きければ、落とすと悪化する |
| `LRT` | 尤度比統計量。大きいほど、その予測子の寄与が大きい |
| `Pr(Chi)` | p 値。p < .05 なら、その予測子はモデルの適合に有意に寄与 |

- **非有意な予測子**については、理論的根拠がなければ除外を検討できるが、統制変数として投入した場合は残すのが一般的。

::: {.callout-tip collapse="true"}

## 固定効果の解釈（OR）

logit スケールの係数（β）はそのままでは解釈しにくいため、**オッズ比（OR = exp(β)）** に変換して報告します。OR は「その予測子が 1 単位増加したとき、イベント（助言関係）のオッズが何倍になるか」を示します。

#### 方法 A: `sjPlot::tab_model()` で OR を表示（実務向け）

`tab_model()` は係数を自動的に OR に変換し、95% CI 付きの表を生成します。
**論文やレポートに使う表はこちらで十分**です。

```{r}
#| label: or-sjplot

sjPlot::tab_model(
  m_glmm_ri,
  show.se = TRUE,      # 標準誤差を表示
  show.stat = TRUE,    # z 値を表示
  dv.labels = "GLMM (Random Intercepts)",
  title = "Fixed Effects as Odds Ratios"
)
```

#### 方法 B: 手計算で OR 表を作る（原理理解のため）

`sjPlot::tab_model()` の裏側では `exp(β)` の計算が行われています。
以下のコードでこれを手動で再現します。**OR = exp(logit 係数)**、**CI = exp(係数 ± 1.96 × SE)** です。

```{r}
#| label: or-table

coef_mat <- summary(m_glmm_ri)$coefficients

or_tbl <- tibble(
  term           = rownames(coef_mat),
  estimate_logit = coef_mat[, "Estimate"],
  std_error      = coef_mat[, "Std. Error"],
  z_value        = coef_mat[, "z value"],
  p_value        = coef_mat[, "Pr(>|z|)"],
  OR      = exp(estimate_logit),                       # exp(β)
  OR_low  = exp(estimate_logit - 1.96 * std_error),    # 95% CI 下限
  OR_high = exp(estimate_logit + 1.96 * std_error)     # 95% CI 上限
)

or_tbl |>
  gt() |>
  tab_header(title = "Fixed Effects as Odds Ratios（手計算版）") |>
  fmt_number(columns = c(estimate_logit, std_error, z_value, OR, OR_low, OR_high), decimals = 3) |>
  fmt_scientific(columns = p_value, decimals = 2) |>
  cols_label(
    term = "Term", estimate_logit = "β (logit)", std_error = "SE",
    z_value = "z", p_value = "p", OR = "OR",
    OR_low = "OR 2.5%", OR_high = "OR 97.5%"
  )
```

方法 A と方法 B の OR 値が一致することを確認してください。

**OR 表の全予測子の解釈**（以下は出力例に基づく解釈パターンです。実際の数値で読み替えてください）：

| 予測子 | 解釈の読み方 |
|--------|-------------|
| `(Intercept)` | 基準条件（same_class=0, prof_gap=0, 全スコア=0）でのオッズ。OR < 1 ならベースラインで助言関係は少ない |
| `same_class` | **同クラスのペアは異クラスに比べて OR 倍のオッズで助言関係をもつ**。OR > 1 なら同クラスが助言を促進 |
| `prof_gap` | 熟達度差が 1 段階広がるごとに、助言のオッズが OR 倍に変化。OR < 1 なら差が大きいほど助言が減る |
| `score_sender_sc` | 送り手のスコアが 1 SD 高いごとに、助言のオッズが OR 倍に変化 |
| `score_receiver_sc` | 受け手のスコアが 1 SD 高いごとに、助言のオッズが OR 倍に変化 |
| `score_gap` | スコア差が 1 SD 広がるごとに、助言のオッズが OR 倍に変化 |

::: {.callout-tip}
#### OR 報告の 4 点セット

1. **方向**: OR > 1（促進）か OR < 1（抑制）か
2. **OR と 95% CI**: OR = X.XX, 95% CI [LL, UL]
3. **p 値**: 統計的有意性
4. **確率差への変換**（応答スケール）: 次のセクションで `emmeans` を使って報告

::: {.callout-note}
OR は**確率差そのもの**ではありません。例えば OR = 2.0 は「オッズが 2 倍」であり、確率が 2 倍になるわけではありません。確率差を知るには、次のセクションの応答スケール予測が必要です。
:::
:::

## 応答スケールでの報告（予測確率）

### 群比較（same class）

`emmeans()` は、他の予測子を平均的な値に固定したうえで、特定の条件における推定周辺平均（EMMs）を算出します。`type = "response"` を指定すると、logit スケールではなく**確率スケール**で結果を返します。

```{r}
#| label: emmeans-class

# same_class の各水準での推定確率（他の予測子は平均値に固定）
emm_class <- emmeans(m_glmm_ri, ~ same_class, type = "response")

# 確率スケールでの結果表
as.data.frame(emm_class) |>
  gt() |>
  tab_header(title = "Predicted Probability by Same-Class Condition")

# ペアワイズ比較（OR と p 値）
pairs(emm_class)
```

**EMMs 表の読み方**：

- `prob`: 各条件での推定確率。same_class = 0（異クラス）vs same_class = 1（同クラス）の確率を比較。
- `asymp.LCL` / `asymp.UCL`: 確率の 95% 信頼区間。
- **確率差**: 2 つの `prob` の差が、same_class の実質的効果を示す。例えば 0.15 vs 0.25 なら、同クラスは約 10 ポイント高い。

**`pairs()` の読み方**：

- `odds.ratio`: 2 条件間の OR（OR 表の same_class の OR と一致するはず）。
- `p.value`: ペアワイズ比較の p 値。

## 診断（必須セット）

### marginal R² と conditional R²（GLMMで必須）

混合効果モデルの $R^2$ は、単一の値ではなく
**marginal / conditional** を分けて報告するのが標準です
[@nakagawa2013; @johnson2014; @nakagawa2017]。

$$
R^2_{\mathrm{marginal}}
=
\frac{\mathrm{Var}(X\beta)}
{\mathrm{Var}(X\beta)+\mathrm{Var}(Zu)+\mathrm{Var}_{\mathrm{dist}}}
$$

$$
R^2_{\mathrm{conditional}}
=
\frac{\mathrm{Var}(X\beta)+\mathrm{Var}(Zu)}
{\mathrm{Var}(X\beta)+\mathrm{Var}(Zu)+\mathrm{Var}_{\mathrm{dist}}}
$$

- `marginal R²`: 固定効果だけで説明される割合
- `conditional R²`: 固定 + ランダム効果で説明される割合
- 差分 `conditional - marginal`: ランダム効果が担う説明分の目安

```{r}
#| label: r2-glmm

# Nakagawa の R² を計算（GLMM 専用の分散分解）
r2_glmm <- performance::r2_nakagawa(m_glmm_ri)
r2_df   <- as.data.frame(r2_glmm)

# 見やすい表にまとめる
r2_tbl <- tibble(
  metric = c(
    "Marginal R²（固定効果のみ）",
    "Conditional R²（固定 + ランダム効果）",
    "ランダム効果の寄与（差分）"
  ),
  value = c(
    r2_df$R2_marginal[1],
    r2_df$R2_conditional[1],
    r2_df$R2_conditional[1] - r2_df$R2_marginal[1]  # この差がランダム効果の説明分
  )
)

r2_tbl |>
  gt() |>
  fmt_number(columns = value, decimals = 3) |>
  tab_header(title = "R² 分解（Nakagawa's R²）")
```

`performance::r2_nakagawa()` は、GLMM の分散分解に基づく
R²（Nakagawa系）を実装した関数です[@performance_r2; @performancepkg]。

::: {.callout-note}
#### 読み方（実務）

- marginal R²が小さく、conditional R²が大きい:
  個体差・項目差などのランダム効果が主要因になっている可能性。
- 両者が近い:
  固定効果だけで大部分が説明され、ランダム効果の追加利得は小さい。
- R²の大小だけで採否を決めず、`compare_performance()` と残差診断を併用する。
:::

::: {.panel-tabset}

#### DHARMa 残差診断

DHARMa はシミュレーションベースの残差を生成し、モデルの仮定が満たされているかを視覚的・統計的に検証します。`n = 250` はシミュレーション回数です（推奨は 250–1000）。

```{r}
#| label: diag-dharma

# シミュレーション残差を生成（n = 250 回のシミュレーション）
res_glmm <- DHARMa::simulateResiduals(m_glmm_ri, n = 250)

plot(res_glmm)                      # QQ プロット + 残差 vs 予測値
DHARMa::testDispersion(res_glmm)    # 過分散テスト（p > .05 なら OK）
DHARMa::testUniformity(res_glmm)    # 残差分布テスト（p > .05 なら OK）
```

**読み方**：

- **QQ プロット（左）**: 点が対角線に沿っていれば残差分布は適切。大きく外れていればモデルのミスフィット。
- **残差 vs 予測値（右）**: パターン（U 字、扇形など）がなければ OK。系統的パターンがあればモデル構造の問題。
- `testDispersion`: **p > .05 なら過分散の問題なし**。有意なら過分散（観測データの分散がモデルの想定より大きい）。
- `testUniformity`: **p > .05 なら残差分布に大きな問題なし**。有意なら分布の仮定に懸念。

::: {.callout-important}
#### DHARMa 診断の合格基準

以下の 3 条件がすべて満たされていれば、モデルの当てはまりは実務上問題ない：

1. `testDispersion` が非有意（p > .05）
2. `testUniformity` が非有意（p > .05）
3. QQ プロットと残差プロットに系統的パターンがない
:::

#### 過分散チェック

```{r}
#| label: diag-overdispersion

# 過分散チェック（dispersion ratio ≈ 1.0 なら問題なし）
performance::check_overdispersion(m_glmm_ri)
```

**読み方**：

- `Dispersion ratio` ≈ 1.0 なら過分散なし。1.0 から大きく離れるほど過分散の懸念。
- p > .05 なら過分散は統計的に有意ではない。

::: {.callout-note}
#### 2値 GLMM での過分散について

厳密には、2値アウトカム（0/1）のロジスティックモデルでは過分散は定義上生じません（各観測の分散は $p(1-p)$ で固定）。ただし、**群レベルでの集計データ**や**観測レベルのランダム効果がない場合**は「見かけの過分散」が起こりえます。DHARMa の検定結果と合わせて総合的に判断してください。
:::

#### 共線性チェック

```{r}
#| label: diag-collinearity

# VIF（分散拡大因子）: < 3 なら安全、> 5 なら要注意
performance::check_collinearity(m_glm)      # GLM 版
performance::check_collinearity(m_glmm_ri)  # GLMM 版（両方で確認）
```

**VIF（分散拡大因子）の判断基準**：

| VIF | 判断 |
|-----|------|
| < 3 | 問題なし |
| 3–5 | 注意。解釈に影響がないか確認 |
| > 5 | 共線性の懸念。予測子の除外や合成を検討 |
| > 10 | 深刻な共線性。推定が不安定な可能性が高い |

GLM と GLMM の両方で VIF を確認しているのは、ランダム効果の追加で共線性パターンが変化しないことを確認するためです。

#### check_model（総合診断）

```{r}
#| label: diag-checkmodel

# 総合的な視覚診断（複数パネルを一括表示）
performance::check_model(m_glm)      # GLM 版
performance::check_model(m_glmm_ri)  # GLMM 版
```

`check_model()` はモデル仮定の可視化を一括で確認できるため、
最終モデルの報告前チェックとして有用です[@performance_checkmodel]。

**パネルの読み方**：

- **Posterior predictive check**: 実データ（黒線）とモデルからのシミュレーション（青線）が近いか
- **Linearity**: 残差の平均が 0 付近を推移しているか
- **Homogeneity of variance**: 残差の散布が予測値に依存していないか
- **Influential observations**: Cook の距離が極端に大きい観測がないか
- **VIF**: 共線性の視覚的確認

:::

#### 診断結果の 1 文報告テンプレート

すべての診断が合格した場合、論文には以下の 1 文を記載すれば十分です：

> Diagnostic checks indicated no major violations: DHARMa simulation-based tests for dispersion and uniformity were non-significant, VIF values were below 3, and visual inspection of residual plots revealed no systematic patterns.

## 5章からの接続: GLMM と GAMM の使い分け

- **GLMM**: 非線形よりも、階層・反復構造の扱いが主課題のとき。
- **GAMM**: 階層構造に加えて連続予測子の非線形も重要なとき[@wood2017; @wood2020; @pedersen2019]。

次の判断が有効です。

1. まず GLMM（線形効果）
2. 残差/理論から非線形が必要なら GAMM へ拡張
3. 複雑化で解釈が落ちる場合は GLMM に戻して報告

::: {.callout-note}
#### アウトカムが 2 値以外の場合

本章は 2 値（binomial）ですが、`glmer()` の `family` 引数を変えるだけで同じ枠組みを他のデータ型に適用できます。

| データ型 | family 指定 | 変わる点 | 参考 |
|----------|-----------|---------|------|
| **反応時間**（正の連続量）| `Gamma(link = "log")` | 係数は乗法的（倍率）解釈。変換の問題を回避できる[@lo2015] | 5 章ケース 3 |
| **カウント** | `poisson` or `MASS::negative.binomial` | IRR（発生率比）で報告 | 5 章ケース 2 |
| **多値カテゴリカル** | `mclogit` or `brms` で多項ロジット | 3 値以上の選択データ[@gudmestad2025] | — |
| **学習者コーパス** | データ型に応じて上記を選択 | @gries2021 が完全なワークフローを提示 | — |

ランダム効果の考え方（`(1 | sender) + (1 | receiver)` など）は family が変わっても**まったく同じ**です。
:::

```text
We analyzed directed advice ties among 30 EFL learners using a
binomial generalized linear mixed-effects model (GLMM; Bates et al.,
2015) implemented in R (lme4 package, version 1.1-x). The binary
outcome was whether a given sender–receiver pair had an advice tie
(1) or not (0). Fixed effects included same-class membership
(binary), proficiency gap (ordinal), standardized sender baseline
score, standardized receiver baseline score, and absolute score gap.
To account for the non-independence of dyadic observations, we
specified crossed random intercepts for sender and receiver.

Model selection proceeded in two stages: (a) random-effect structure
was compared via AIC, BIC, and likelihood-ratio tests, with
singularity checks (isSingular; Bates et al., 2015); (b) fixed
effects were evaluated via backward elimination (drop1) with the
random structure held constant. The optimizer was set to bobyqa
(maxfun = 200,000). Diagnostics included DHARMa simulation-based
residual tests (Hartig, 2022), overdispersion checks, and VIF-based
collinearity assessment (performance package; Lüdecke et al., 2021).
```

1. **GLM vs GLMM の根拠**: ICC/DEFF またはモデル比較（ΔAIC）で GLMM の必要性を示したか

## トラブルシューティング（よくあるつまずき）

::: {.callout-warning}
#### 収束警告（convergence warning）が出る

**症状**: `Model failed to converge` や `max|grad| = ...` といった警告メッセージ。

**原因**: 最適化アルゴリズムが最大反復回数内に収束しなかった。変数のスケールが大きく異なる場合や、ランダム効果構造が複雑すぎる場合に起こりやすい。

**段階的な修正手順**:

1. **変数の標準化**: 連続予測子を z スコア化する（`scale()`）。これが最も効果的。
2. **オプティマイザ変更**: `glmerControl(optimizer = "bobyqa")` に変更。
3. **反復回数増加**: `optCtrl = list(maxfun = 2e5)` または `5e5`。
4. **別のオプティマイザ**: `nloptwrap` や `optimx` パッケージの `nlminb` を試す。
5. **ランダム構造簡素化**: ランダム傾きを落とす。

```r
# 段階的対処の実装例
control_step1 <- glmerControl(optimizer = "bobyqa", optCtrl = list(maxfun = 2e5))
control_step2 <- glmerControl(optimizer = "bobyqa", optCtrl = list(maxfun = 5e5))
```
:::

::: {.callout-warning}
#### 特異適合（singular fit）

**症状**: `isSingular(model)` が `TRUE` を返す。`boundary (singular) fit` という警告。

**原因**: ランダム効果の分散成分が 0（またはその境界）に推定された。モデルの複雑さに対してデータが不足している。

**修正手順**:

1. `VarCorr(model)` で分散が 0 に近い項を特定。
2. その項を含むランダム効果を除外してモデルを再推定。
3. 論文には「ランダム傾きを試みたが特異適合が生じたため、ランダム切片モデルを採用した」と記載。
:::

::: {.callout-warning}
#### OR の誤解

**症状**: OR = 2.0 を「確率が 2 倍」と誤って解釈。

**原因**: OR（オッズ比）と確率比（リスク比）の混同。

**正しい理解**: OR = 2.0 は「オッズが 2 倍」。ベースライン確率が低い場合（例：p = .05）は確率もほぼ 2 倍に近いが、ベースラインが高い場合（例：p = .50）はオッズ 2 倍でも確率は .50 → .67 程度の変化にとどまる。

**対処**: OR だけでなく、`emmeans()` による応答スケール予測（確率差）を必ず併記する。
:::

::: {.callout-warning}
#### `re.form` の混同

**症状**: 予測値が観測値と大きくずれる、または個人ごとの予測が全員同じになる。

**原因**: `predict()` の `re.form` 引数の誤用。

**正しい使い分け**:

| 引数 | 意味 | 用途 |
|------|------|------|
| `re.form = NA` | ランダム効果を含まない（母平均的予測）| 論文の主要結果図、一般的な効果の報告 |
| `re.form = NULL` | ランダム効果を含む（条件付き予測）| 個人レベルの予測、モデルの適合度確認 |
| `re.form = ~0` | `NA` と同じ | 同上 |
:::

::: {.callout-warning}
#### DHARMa の警告が出た

**症状**: `testDispersion` や `testUniformity` が有意（p < .05）。

**原因と対処**:

- **testDispersion が有意**: 過分散の可能性。2値データでは観測レベルのランダム効果（`(1 | obs_id)`）を追加するか、`MASS::glmmPQL` を検討。ただし、2値単独観測では過分散は定義上起こらないため、群レベルの集計をしていないか確認。
- **testUniformity が有意**: モデル構造のミスフィット。非線形効果の追加（GAMM への拡張）や、外れ値の確認を検討。
- **有意だが視覚的に問題がない場合**: サンプルサイズが大きいと微小な逸脱でも有意になることがあります。QQ プロットと残差プロットを視覚的に確認し、系統的パターンがなければ実務上は問題ないと判断できます。混合モデルは軽度の分布仮定の逸脱に対して頑健であることがシミュレーションで確認されているため[@schielzeth2020]、検定が有意でも**視覚的に問題がなければ過度に心配する必要はありません**。
:::

::: {.callout-warning}
#### 完全分離（complete separation）

**症状**: `glmer` が収束しない、または係数が極端に大きい（|β| > 10）。

**原因**: ある予測子の値によってアウトカムが完全に予測される（例：same_class = 1 のとき tie が常に 1）。

**修正手順**:

1. クロス表で完全分離の有無を確認: `table(dat_bin$same_class, dat_bin$tie)`
2. 完全分離がある場合、`blme::bglmer()` で弱い事前分布を追加するか、Firth 補正を検討。
3. カテゴリの統合や、問題のある予測子の除外も選択肢。
:::

::: {.callout-warning}
#### グループ数が少ない

**症状**: ランダム効果の分散推定が不安定。推定値が極端に大きいまたは小さい。

**原因**: ランダム効果のグループ数（例：sender 数）が少ない（目安として < 10）と、分散の推定に十分な情報がない。

**対処**:

- グループ数が 5 未満の場合、ランダム効果ではなく固定効果として投入することを検討。
- グループ数が 10–20 の場合、ランダム切片のみに留め、ランダム傾きは避ける。
- ベイズ推定（`brms` パッケージ）は少グループでもより安定した推定を行える。
:::

#### 基礎を固めたい

本章を読んで「ランダム効果って何をしているの？」がまだ曖昧な場合は、以下の順で読み進めると効率的です。

1. @winter2013 — 言語学の例題で、線形モデルから混合モデルへの拡張を段階的に解説。数式よりコードで理解したい人向け。
2. @brown2021 — `lme4` で線形混合モデルを一から実装する R チュートリアル。心理学データを使用。
3. @debruine2021 — 「自分でデータを生成して、ランダム効果が何をしているか体感する」アプローチ。式の意味が腑に落ちます。

#### SLA の文脈で学びたい

SLA・応用言語学に特化した GLMM ガイドです。論文の方法欄を書くときにも引用として役立ちます。

- @cunnings2012 — SLA 研究者を対象に、混合モデルの基礎と利点を体系的に解説した先駆的論文。「なぜ ANOVA ではダメか」の議論も含む。
- @linck2015 — ランダム効果構造の選択やモデル比較の手順を SLA データで実演。本章のステップ 2-3 と直接対応する内容。
- @gries2021 — 学習者コーパスデータで GLMM の完全なワークフローを提示。カテゴリカル応答変数の扱いに重点を置いた実践例。
- @gudmestad2025 — 3 値以上のカテゴリカル従属変数（例：正用・誤用・無反応）を扱う多項ロジスティック混合モデルの実装ガイド。

#### アウトカムが 2 値以外のとき

本章は 2 値（binomial）でしたが、同じ GLMM の枠組みを他のデータ型にも拡張できます。

- @jaeger2008; @quene2008 — 2 値データでの GLMM の基礎。本章の理論的背景。
- @lo2015 — **反応時間データ**に対する「変換してから LMM」vs「GLMM（Gamma/逆ガウス）で直接扱う」を比較。変換の問題点を具体的に示した重要論文。
- @brysbaert2018 — 混合モデルでのエフェクトサイズとパワー分析。サンプルサイズ計画に不可欠。

#### ランダム効果構造で悩んだら

「最大構造にすべき」vs「節約的にすべき」という論争は、GLMM の最も実務的な難所です。

- @barr2013 — 「Keep it maximal」：確認的仮説検定ではランダム効果構造を最大にすべきという主張。
- @bates2015parsimonious; @matuschek2017 — 「節約的モデリング」：過剰パラメータ化を避け、データが支持する範囲に留めるべきという反論。
- @schielzeth2020 — 混合モデルが分布仮定の逸脱にどこまで耐えるかを大規模シミュレーションで検証。「多少の違反は大丈夫」の根拠。

実務的な判断基準は @meteyard2020 が最もバランスよくまとめています。
