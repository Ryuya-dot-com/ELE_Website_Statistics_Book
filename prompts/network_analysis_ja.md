# ネットワーク分析：プロンプトとステップ

このドキュメントは、SLA/教育研究のユースケースを想定した「うまく動く」プロンプトと、実務での進め方チェックリストです。必要に応じてデータ仕様に差し替えて使ってください。

## まず確認するヒアリング項目

- 目的：説明・探索・比較・予測のどれか？
- ネットワークのタイプ：一模（例：学習者同士の関係）/二部（例：学習者×課題）/重み付き/有向？
- データ形式：エッジリスト/隣接行列/イベントログ？
- ノード属性：グループ/熟達度/役割/時間情報など
- 重要な問い：中心人物は？クラスタは？構造変化は？
- 再現性：Rのみ/一部Python併用可？出力形式はHTML/PDF？

## コア・プロンプト（テンプレート）

以下をペーストして変数名や条件を埋めてください。

```
あなたはRのネットワーク分析アシスタントです。tidyverse, tidygraph, ggraph, igraph を優先的に使用し、再現可能で読みやすいコードを提案してください。

要件:
- 目的: <例: 学習者間の協働ネットワーク構造の可視化と中心性分析>
- データ: エッジリスト: <edges.csv> (from, to, weight), ノード: <nodes.csv> (id, group, label)
- ネットワーク: <無向/有向>, 重み: <あり/なし>
- 出力: 図 (静的), 指標表 (degree/betweenness/closeness/eigen)

期待するステップ:
1) データ読み込みと検査
2) tbl_graphの作成
3) 可視化 (FRレイアウト, ラベル/色/太さの調整)
4) 中心性とコミュニティ検出 (Louvain/Leiden)
5) 主要指標の表を作成し、上位ノードを解釈

制約:
- パッケージのインストールはコメントで示し、コードは eval: false でも実行可能な形
- set.seed を指定して再現性を担保
```

## ステップ・チェックリスト（実務）

- データのカーディナリティ・欠損の確認（孤立ノードの扱い）
- 無向/有向・重みの扱い・二部グラフの投影の妥当性
- レイアウトの安定化（seed固定、座標保存）
- 指標のスケーリング（比較性を確保）
- コミュニティの解釈（外的基準との整合）
- 図の凡例・注記・再現コードの同梱

## 追加プロンプト（状況別）

- 「二部ネットワーク（学習者×課題）から学習者同士の一模ネットワークを投影し、重みでしきい値0.2以上を保持して可視化してください」
- 「時間t1/t2でネットワーク構造の差（密度/平均距離/モジュラリティ）を比較し、図を2つ並べて出力してください」
- 「変数間の相関ネットワーク（qgraph）を作成し、閾値0.3、LASSO推定（EBICglasso）でスパース化してください」
