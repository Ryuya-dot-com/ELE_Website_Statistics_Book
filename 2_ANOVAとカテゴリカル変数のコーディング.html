4  ANOVAとカテゴリカル変数コーディング

4.1 はじめに

英語教育研究では、次のような問いがしばしば立てられます。

  • 「注釈付き読解（Glossing）は、作文練習（Writing）より語彙学習に効果的か？」
  • 「テスト時点（直後 vs 遅延）によって、学習の保持は異なるか？」
  • 「指導法の効果は、テスト時点によって変わるのか（交互作用はあるか）？」

これらはすべてANOVA（分散分析）の問いです。しかし、多くの教科書ではANOVAと回帰分析を別章で扱うため、両者の関連が見えにくくなっています。

本章の核心的なメッセージは、ANOVAは回帰モデルの一形態であるということです。カテゴリカル変数（指導法の種類、テスト時点など）を数値に変換（コーディング）して回帰モデルに投入する——その変換方法の選択によって、回帰係数が「何との比較」を意味するかが変わります。この統一的な理解があれば、研究デザインに応じた分析の選択と結果解釈が格段に明瞭になります。

前提知識

本章では、回帰分析の基礎（係数・p値・R²の意味）を理解していることを前提とします。t検定の経験があれば、本章の内容は自然に接続します。

本章の到達目標:

  1. ANOVAを線形モデル \(\mathbf{y} = \mathbf{X}\boldsymbol{\beta} + \boldsymbol{\varepsilon}\) として理解し、設計行列 \(\mathbf{X}\) の役割を説明できる
  2. 被験者間・被験者内・混合計画の違いを理解し、各仮定（特に球面性）を正しく確認できる
  3. afex と emmeans を使って、混合ANOVAの主効果・交互作用・事後比較を再現可能に実行・報告できる
  4. Treatment / Sum / Helmert / Polynomial codingの違いを理解し、研究質問に適したコーディングを選択できる

本章の流れ:

本章は、(1) ANOVA を回帰モデルとして捉える統一的理解から始め、(2) 研究計画と仮定を整理し、(3) 英語教育データの生成を行ったうえで、(4) ANOVA の実行・事後比較・効果量の算出へ進みます。続いて (5) 正規性・等分散性・球面性の診断を実施し、(6) Treatment / Sum / Helmert / Polynomial の4種類のコントラストコーディングを比較します。最後に (7)-(8) として、再現可能な報告テンプレートまで一気通貫で扱います。

4.2 1. ANOVAを回帰モデルとして捉える

4.2.1 共通の枠組み

ANOVAも回帰も、基本式は同じです。

\[ \mathbf{y} = \mathbf{X}\boldsymbol{\beta} + \boldsymbol{\varepsilon}, \quad \boldsymbol{\varepsilon} \sim N(0, \sigma^2\mathbf{I}) \]

  • \(\mathbf{y}\): 従属変数（例: テスト得点）
  • \(\mathbf{X}\): 設計行列（カテゴリカル変数はコーディングされる）
  • \(\boldsymbol{\beta}\): 係数
  • \(\boldsymbol{\varepsilon}\): 誤差

この数式で鍵となるのが設計行列 \(\mathbf{X}\) です。回帰分析では \(\mathbf{X}\) に連続変数（英語力スコアなど）がそのまま入りますが、ANOVAではカテゴリカル変数を数値に変換して \(\mathbf{X}\) に入れます。この変換が「コントラストコーディング」の正体です。

具体例: 3群（Control, Glossing, Keyword）各2名のデータで、設計行列は次のようになります（Treatment codingの場合）。

参加者 群 切片列 Glossing列 Keyword列
1 Control 1 0 0
2 Control 1 0 0
3 Glossing 1 1 0
4 Glossing 1 1 0
5 Keyword 1 0 1
6 Keyword 1 0 1

カテゴリカル予測子が \(k\) 水準なら、通常 \(k-1\) 本のコントラスト列が必要です。この \(\mathbf{X}\) の中身（0/1の配置）がコーディング方式によって変わるのが、セクション6で詳しく学ぶ内容です。

核心原則

どのコーディングを選ぶかで係数 \(\boldsymbol{\beta}\) の意味は変わるが、同じモデル空間なら適合値や全体検定（F値）は本質的に同じです（Schad et al., 2020）。

4.2.2 F検定の意味

ANOVAのF統計量は、

\[ F = \frac{\text{MS}_{\text{effect}}}{\text{MS}_{\text{error}}} \]

であり、直感的にはシグナル（効果による変動）とノイズ（誤差による変動）の比です。F値が大きいほど、「群間の差が偶然の変動では説明しにくい」ことを意味します。

  • F ≈ 1: 効果による変動と誤差変動が同程度 → 群間に体系的な差がない可能性が高い
  • F >> 1: 効果による変動が誤差変動よりはるかに大きい → 群間に体系的な差がある可能性が高い

F値に対応するp値は、「帰無仮説（群間に差がない）のもとでこのF値以上が得られる確率」です。p < .05 なら、偶然でこれほど大きなF値が出る確率は5%未満であり、通常は帰無仮説を棄却します。

なお、2群比較では \(t^2 = F\) が成り立つため、t検定とANOVAは対立概念ではなく同一枠組みの特殊ケースです。3群以上の比較ではt検定を繰り返すと多重検定の問題が生じるため、ANOVAが必要になります。

4.3 2. 研究計画と仮定

4.3.1 英語教育研究での典型デザイン

英語教育研究では、次の3タイプがよく使われます。

デザイン 構造 英語教育での典型例
被験者間計画 各参加者が1条件のみ 指導法A群（n=60）vs 指導法B群（n=60）で語彙テストを比較
被験者内計画 各参加者が全条件を経験 同一学習者の直後テスト・1週間後テスト・1か月後テストを比較
混合計画 被験者間 + 被験者内の組合せ 指導法（被験者間）× テスト時点（被験者内）

研究課題が「群間差の検証」であれば、まず同一参加者を複数条件で測定しているかを確認します。測定していなければ被験者間計画、測定していれば次に「全要因が被験者内か」を判定し、全要因が被験者内なら被験者内計画、被験者間要因も含まれるなら混合計画を選択します。混合計画は英語教育研究で最も頻出し、本章の主要実装例もこの設計に基づきます。

本章の実装例は、英語教育研究で最も頻出する混合計画を中心に扱います。

4.3.2 仮定の整理

ANOVAが妥当な推論を行うためには、いくつかの前提条件（仮定）が必要です。以下の表で概観し、その後で各仮定の実務的な意味を説明します。

仮定 何を要求するか 主な確認方法 注意点
独立性 観測単位間の誤差が独立 研究デザインで担保 統計手法だけでは救済不能
正規性 残差が概ね正規 QQプロット, Shapiro-Wilk検定 大標本（n > 30）では頑健
等分散性 群間で残差分散が類似 Levene検定, 残差プロット 群サイズが等しければ頑健
球面性 反復測定要因（3水準以上）の差得点分散が等しい Mauchly検定 + GG/HF補正 2水準要因では自動的に満たされる

各仮定の実務的な意味:

  • 独立性: 最も重要な仮定です。ある学習者の誤差が別の学習者に影響しないこと。例えば、同じ教室で相互に影響し合う状況では独立性が破れます。この仮定は研究デザインで担保するものであり、統計手法では事後的に救済できません（クラスタ構造がある場合は混合モデル[Ch.6]を検討）
  • 正規性: 残差（予測からのずれ）が正規分布に従うこと。群あたり n > 30 であれば、中心極限定理によりANOVAは軽度の非正規性に対して頑健です。正誤データ（0/1）のように明らかに非正規な場合は、ロジスティック回帰（Ch.5-6）を使います
  • 等分散性: 群間で分散が大きく異ならないこと。群サイズが等しく、最大SD / 最小SD < 3 であれば概ね問題ありません。大きな違反がある場合はWelch ANOVAを検討します
  • 球面性: 反復測定要因が3水準以上の場合に確認が必要です。「各時点ペアの差得点の分散が等しい」という条件で、違反時はGreenhouse-Geisser補正を適用します。2水準の反復測定要因では自動的に満たされます（差の分散が1つしかないため）
球面性と複合対称性の区別

複合対称性（compound symmetry）は球面性より強い仮定です。複合対称性は「全時点ペアの相関が等しく、かつ全時点の分散が等しい」ことを要求しますが、球面性は「差得点の分散が等しい」ことのみを要求します。両者を混同しないようにしてください（詳しくはセクション5の診断で扱います）。

仮定の具体的な検証手順はセクション5で、コードとともに詳しく解説します。

        p <- tryCatch(
          get_plots(fallback_checks),
          error = function(e2) NULL
        )
        if (!is.null(p)) {
          return(p)
        }
      }

      list()
    }
  )

  if (length(diagnostic_plots) == 0) {
    return(invisible(NULL))
  }

  purrr::iwalk(diagnostic_plots, function(p, nm) {

}

id_tbl <- tibble(

4.5 4. 混合要因ANOVAの実行

4.5.1 主効果・交互作用

この混合計画では、ANOVAは以下の3つのF検定を出力します。

  1. Instructionの主効果: 全体として、Glossingの得点はWritingより高いか？
  2. Timeの主効果: 全体として、直後テストの得点は遅延テストより高いか？
  3. Instruction × Time の交互作用: 指導法の効果は時点によって異なるか？
交互作用を先に確認する

交互作用が有意であれば、主効果は条件付きになります。例えば、「Glossingが全体的に優れている」と言っても、実際にはその優位性が直後テストに限定されている可能性があります。交互作用が有意な場合は、必ず単純主効果（各条件内での比較）を確認してください。

以下のコードでは afex::aov_ez() を使います。afexは Type III平方和、偏η²（偏イータ2乗）、球面性補正を統一的に扱えるため、混合計画ANOVAの標準的なツールです。

コード 
fit_aov <- aov_ez(
  id = "Participant",
  dv = "Score",
  between = "Instruction",
  within = "Time",
  covariate = "Proficiency",
  factorize = FALSE,
  data = dat,
  type = 3,
  anova_table = list(es = "pes")
)

nice(fit_aov) |>
  gt() |>
  tab_header(
    title = "Mixed ANOVA (Type III)",
    subtitle = md("Dependent variable: **Score**")

4.5.2 事後比較（推定周辺平均）

ANOVAのF検定は「全体的に差があるか」を教えてくれますが、「どの群ペアが具体的にどの程度異なるか」は教えてくれません。そこで事後比較（post-hoc comparisons）を行います。

emmeansパッケージは以下の2ステップで事後比較を行います。

  1. 推定周辺平均（EMM）の算出: モデルから各条件の調整済み平均を推定する
  2. ペア比較の実行: 各ペアの平均差・SE・p値・信頼区間を算出する
なぜ emmeans を使うのか（係数表ではダメなのか）

summary(model) の係数表は、コーディング方式に依存した特定のペア比較しか示しません。例えば Treatment coding では参照群との差のみが直接読めます。一方 emmeans() は、コーディングに依存しないモデル整合的な推定平均を計算し、任意のペア比較を統一的に行えます。

以下のコードでは、2種類の比較を行います。

  • 第1の比較: 各時点（Immediate / Delayed）内で指導法を比較 → 「直後テストでGlossing > Writingか？ 遅延テストではどうか？」

多重比較補正（Holm法）について

複数のペア比較を同時に行うと、偶然に有意な結果が出る確率が高まります（多重検定の問題）。Holm法は、Bonferroni補正より検出力が高く、かつファミリーワイズの第1種過誤率を制御する方法です。特段の理由がなければ、Holm法が推奨されます。Tukey法は全ペア比較に特化しており、3群以上の場合に適しています。

  • 偏η²（partial eta-squared）: 特定の効果が、他の効果を統制した上で説明する分散の割合
  • 偏ω²（partial omega-squared）: η²の不偏推定量。サンプルサイズの影響を補正するため、η²よりやや小さくなる

Cohen（1988）の効果量解釈基準:

効果量 小 中 大
偏η²（偏イータ2乗） .01 .06 .14
偏ω²（偏オメガ2乗） .01 .06 .14
η² と ω² の使い分け
  • 偏η²: 報告に最も広く使われる。計算が単純で解釈しやすい。ただしサンプルサイズが小さいと過大推定になりやすい
  • 偏ω²: 母集団パラメータの不偏推定量。サンプルサイズが小さい研究（n < 50）では ω² が望ましい
  • 実務的には: 偏η² + 95%信頼区間を主指標とし、ω² を補助的に併記するのが堅実です

R²との関係: R²はモデル全体の説明力、偏η²は個別効果の説明力です。例えば、R² = .45 でも Instructionの偏η² = .07 であれば、「モデル全体は得点の45%を説明するが、そのうち指導法が説明するのは7%程度（残りはProficiencyやTimeが説明）」と解釈できます。

4.6 5. 仮定診断と実務判断

ANOVAの結果を報告する前に、モデルの前提条件が大きく破綻していないかを確認します。ここで重要な考え方を先に述べます。

仮定違反 ≠ 分析不可能

仮定の検定で p < .05 が出ても、それだけで「ANOVAが使えない」わけではありません。ANOVAは多くの仮定違反に対して頑健（robust）です。特に：

  • 正規性: 群あたり n > 30 であれば、中心極限定理により多少の非正規性は問題になりにくい
  • 等分散性: 群間のサンプルサイズが等しければ、分散比が 3:1 程度まで頑健
  • 球面性: 2水準の反復測定要因では自動的に満たされる

重要なのは「仮定が完全に満たされているか」ではなく、「違反の程度がどのくらいか」「結論に影響するか」を総合的に判断することです。

以下では、正規性・等分散性・総合診断・球面性の順に確認します。まず残差の QQ プロットと Shapiro 検定で正規性を評価し、p > .05 あるいは n > 30 であれば大きな問題なしと判断します。次に Levene 検定と残差プロットで等分散性を確認し、分散比が大きい場合は Welch ANOVA やロバスト法を検討します。さらに反復測定要因については、水準が2つなら球面性は自動的に満たされ、水準が3つ以上なら Mauchly 検定の結果に応じて GG/HF 補正を適用します。

以下のコードでは、lm()で当てはめた線形モデルに対して3つの診断を一括実行します。

  1. Shapiro-Wilk検定: 残差が正規分布から有意にずれているかを検定
  2. Levene検定: 群間で残差の分散が等しいかを検定
  3. check_model(): 残差診断を個別プロットとして出力

球面性（Sphericity） は反復測定要因が3水準以上ある場合に確認が必要です。

本章のデータではTimeが2水準（Immediate, Delayed）のみなので、球面性は自動的に満たされます。差の分散は1つしかないため、「差の分散が等しいか」を問う余地がありません。

概念 要求する条件 強さ

球面性（Sphericity） 時点間の差得点の分散が等しい より弱い仮定

複合対称性が成り立つ → 球面性も成り立つ（逆は不成立）。実務では球面性のみを確認すれば十分です。

  1. Mauchly検定で球面性を確認する

    • ε ≥ 0.75 → Huynh-Feldt（HF）補正（やや自由度を回復）

4.7 6. カテゴリカル変数のコーディング

ANOVAを回帰モデルとして扱う場合、カテゴリカル変数（例：指導法の種類）を数値に変換する必要があります。この変換方法がコントラストコーディング（contrast coding）です。

コーディングの核心を一文で表すなら、「どの群とどの群を比較するかを、設計行列のなかで数学的に指定する方法」です。同じ風景を異なる視点から描くように、同じデータを異なるコーディングで表現すると、回帰係数が「何との比較か」が変わります。

コーディング選択の核心原則

コーディングは「係数の意味」を変える。 ただし同じモデル空間なら、適合値・残差・全体的なモデル適合（F値やR²）は本質的に変わらない。

つまり、コーディング選択は「どの比較を回帰係数から直接読み取りたいか」によって決まります。

以下では、同一の3群データに対して4種類のコーディングを順に適用し、係数の意味がどう変わるかを比較します。

4.7.1 比較用データ（3群）

ここでは、語彙学習支援法の3条件を比較するデータを作成します。

  • Control（統制群）: 語彙リストのみ提供
  • Glossing（注釈群）: 読解テキスト中に語義注釈を併記
  • Keyword（記憶術群）: キーワード法による記憶術を指導

設計行列の読み方: 各行の合計が0になるのが”Sum” codingの特徴です。最後の水準（Keyword）は −1, −1 として、他の2水準の「裏返し」として暗黙的に定義されます。

係数表の読み方:

係数 意味 解釈例
(Intercept) 全体平均（grand mean） ≈ 56.7 → 3群全体の平均得点
Group1 Control − 全体平均 ≈ −6.7 → 統制群は全体平均より約6.7点低い
Group2 Glossing − 全体平均 ≈ 1.3 → 注釈群は全体平均より約1.3点高い
(暗黙) Keyword − 全体平均 = −(−6.7 + 1.3) = 5.4 → 記憶術群は全体平均より約5.4点高い
なぜSum codingがANOVAと整合的か

ANOVAのF検定は「全体的に群間に差があるか」を問います。この「全体的」という発想は、各群を全体平均からの偏差として捉えるSum codingと対応します。一方、Treatment codingは「特定の参照群との差」を強調するため、ANOVAのF検定と直感的に結びつきにくい面があります。

いつ使うか: 複数要因のANOVAで主効果を報告する際、または特定の参照群を置かない場合に適しています。

4.7.4 Helmert coding（逐次比較コーディング）

段階的・累積的な比較を行うコーディングです。各水準を、その「後に続く水準の平均」と比較します。

このコーディングは、水準に自然な順序がある場合に有効です。英語教育研究では：

  • 指導の複雑さが段階的に上がる場合（例：Control → Glossing → Keyword）
  • テスト時点が増えていく場合（例：Pre → Post → Delayed）
  • 処遇の強度が段階的に増加する場合

Helmert codingが答える問い:

  1. 第1比較: Control は、Glossing・Keyword の平均と異なるか？（統制群 vs 処遇群全体）
  2. 第2比較: Glossing は Keyword と異なるか？（処遇群間の差）

Rの contr.helmert() はReverse Helmert

Rの contr.helmert() は、各水準を先行する水準群の平均と比較する「逆Helmert」です。教科書によっては「後続水準の平均と比較する」定義が使われることがあります。解釈の方向に注意してください。

係数のスケーリングにも注意が必要です。Rの出力をそのまま「差」として読むには、比較する群数に応じたスケーリング因子を考慮する必要があります。実務的には、具体的な群平均差は emmeans() で確認するほうが確実です。

いつ使うか: 研究仮説が「統制群 vs 処遇群全体 → 処遇群間の差」という階層的な構造をもつ場合に適しています。例えば、まず「何らかの介入をすること自体に効果があるか」を確認し、次に「介入の種類による差があるか」を検討する、という段階的な問いに対応します。

4.7.5 順序尺度にはPolynomial coding

順序因子のトレンド（線形・曲線的なパターン）を検出するためのコーディングです。

CEFRのような言語習熟度レベル（A1 < A2 < B1 < B2）では、「レベルが上がるにつれて得点がどう変化するか」の形状が関心事になります。

  • 得点が均等に伸びている → 線形トレンド（直線的な上昇）
  • 初級段階で伸びが大きく、上級段階で伸びが鈍化する → 二次（曲線的）トレンド（天井効果）
  • さらに複雑なパターン → 三次トレンド

Polynomial codingは、こうしたトレンドの各成分を独立に検定します。

コード 
set.seed(2024)

dat_ord <- tibble(
  CEFR = factor(rep(c("A1", "A2", "B1", "B2"), each = 80),
                levels = c("A1", "A2", "B1", "B2"),
                ordered = TRUE),
  Score = c(
    rnorm(80, 48, 7),
    rnorm(80, 56, 7),
    rnorm(80, 66, 7),
    rnorm(80, 74, 7)
  )
)

contrasts(dat_ord$CEFR) <- contr.poly(4)

fit_poly <- lm(Score ~ CEFR, data = dat_ord)

tidy(fit_poly) |>
  mutate(across(where(is.numeric), ~ round(., 3))) |>
  gt() |>
  tab_header(title = "Polynomial Contrasts for Ordered CEFR Levels")

係数表の読み方:

成分 意味 判断基準
CEFR.L（Linear） 線形成分: レベルが上がると得点が直線的に上昇するか p < .05 → 有意な線形トレンドあり
CEFR.Q（Quadratic） 二次成分: 伸び方に曲線的パターン（加速・減速）があるか p < .05 → 直線からの有意な逸脱
CEFR.C（Cubic） 三次成分: さらに複雑な変動があるか 通常は無視可。有意なら解釈が困難

解釈の手順:

  1. まず .L（線形成分）を確認 → 有意なら「レベルが上がるほど得点が上昇する」
  2. 次に .Q（二次成分）を確認 → 有意なら「その上昇は均等ではなく、曲線的」
  3. .C（三次成分）→ 4水準では通常解釈しない
報告例

「CEFRレベルの主効果は有意であった。Polynomial contrast analysisにより、有意な線形トレンド（.L: t = XX.X, p < .001）が確認され、レベルが上がるほど語彙テスト得点が上昇する傾向が示された。二次成分（.Q）は有意でなく（p = .XX）、この上昇パターンはほぼ直線的であった。」

いつ使うか: 水準に明確な順序がある因子（習熟度レベル、テスト時点、練習回数など）で、「上昇パターンの形状」に関心がある場合に適しています。

4.8 7. 研究質問に基づくコーディング選択

ここまで4つのコーディング方式を見てきました。実際の研究でどれを選ぶかは、あなたのリサーチクエスチョンが何を問うているかによって決まります。

研究質問の例 推奨コーディング 係数が直接示すもの
「Glossing指導はControl群より効果的か？」 Treatment 各処遇群と参照群（Control）の差
「各指導法は全体平均からどの程度ずれるか？」 Sum 各群の全体平均（grand mean）からの偏差
「まず処遇群全体が統制群と異なるか、次に処遇群間の差は？」 Helmert 逐次的・階層的な群比較
「CEFR A1→B2で得点は線形に伸びるか、曲線的か？」 Polynomial 線形・二次・三次のトレンド成分

選択の実務手順は次の通りです。まず統制群のような明確な参照群がある場合は Treatment coding を優先します。参照群がない場合は水準に順序があるかを確認し、順序がありトレンド形状（線形・曲線）に関心があるなら Polynomial coding、順序はあるがトレンド形状が主眼でなければ Sum coding を選びます。順序がない場合は、段階的・階層的比較が自然なら Helmert coding、そうでなければ Sum coding が第一候補になります。

迷ったときの3原則
  1. 最も単純で直感的なコーディングを選ぶ: 係数を見て即座に「答え」がわかるものが良いコーディングです
  2. 統制群がある介入研究なら Treatment が第一候補: 英語教育研究の多くはこのパターンに該当します
  3. 複数のコーディングで試しても問題なし: 予測値・F値は変わらないので、「別のコーディングで見たらどう解釈できるか」を確認するのは健全な実践です

4.10 9. トラブルシューティング（実務で詰まりやすい点）

交互作用が有意なのに主効果だけで結論を出す

症状: 「Instructionの主効果が有意なので、Glossingが効果的である」と報告したが、査読者から「交互作用が有意なのに主効果だけで議論するのは不適切」と指摘される。

原因: 交互作用が有意な場合、主効果は条件付き（qualified）です。「全体的にGlossingが高い」ことは事実でも、その優位性がImmediateに限定されている（Delayedでは差がない）可能性があります。主効果だけの報告では、この重要な情報が失われます。

修正: 交互作用が有意なら、必ず emmeans() で各条件内の比較（simple effects）を報告してください。主効果は「補助情報」として扱い、単純主効果を中心に議論します。

球面性と複合対称性の混同

症状: 「Mauchly検定で p > .05 だったので、複合対称性が成り立つ」と記述してしまう。

原因: Mauchly検定は球面性を検定するものであり、複合対称性を検定するものではありません。球面性（差の分散が等しい）は複合対称性（全ペアの相関が等しい + 全分散が等しい）より弱い仮定です。

修正: 「球面性が満たされた」と正しく記述してください。Mauchly検定で p > .05 → 球面性は棄却されない、p < .05 → GG/HF補正を適用。

係数表だけでペア比較を判断する

症状: summary(model) の係数表を見て、「Group2の p < .05 だからGlossing > Control」「Group3の p > .05 だからKeyword = Control」と判断する。しかし、コーディングを変えると異なる結論に見える。

原因: 係数表の p値はコーディング方式に依存した特定の比較の結果です。Treatment codingなら参照群との比較のみ、Sum codingなら全体平均との比較のみが直接読めます。全ペア比較は係数表からは読めません。

修正: ANOVAの全体検定後は、必ず emmeans() で事後比較を行ってください。emmeans の結果はコーディングに依存しない統一的な推定値を提供します。

p値と効果量を混同する

症状: 「p < .001 なので大きな効果がある」または「p = .04 なので効果は小さい」と解釈する。

原因: p値はサンプルサイズに強く依存します。n = 1000 なら些細な差（η² = .005）でも p < .001 になりえます。n = 20 なら大きな差（η² = .15）でも p = .08 になりえます。p値は「効果の存在の証拠」であり、「効果の大きさ」を示すものではありません。

修正: 常に p値と効果量（偏η² + 95% CI）を併記してください。解釈は「統計的有意性」（p値）と「実質的意義」（効果量）を分離して議論します。

コーディング方式を論文に記載しない

症状: コードでは set_sum_contrasts() を使っているが、論文のMethodセクションにコーディング方式の記載がない。査読者から再現性を問われる。

原因: Rのデフォルト（Treatment coding）と afex のデフォルト（Sum coding）は異なります。どのコーディングを使ったかを明記しないと、読者が同じコードを実行しても異なる係数が得られる可能性があります。

修正: Methodセクションに以下のように明記してください。

Sum coding was applied to all categorical variables using
afex::set_sum_contrasts(), ensuring that model coefficients
represent deviations from the grand mean, consistent with
standard ANOVA interpretation.

## はじめに

英語教育研究では、次のような問いがしばしば立てられます。

- 「注釈付き読解（Glossing）は、作文練習（Writing）より語彙学習に効果的か？」
- 「テスト時点（直後 vs 遅延）によって、学習の保持は異なるか？」
- 「指導法の効果は、テスト時点によって変わるのか（交互作用はあるか）？」

これらはすべて**ANOVA（分散分析）**の問いです。しかし、多くの教科書ではANOVAと回帰分析を別章で扱うため、両者の関連が見えにくくなっています。

本章の核心的なメッセージは、**ANOVAは回帰モデルの一形態である**ということです。カテゴリカル変数（指導法の種類、テスト時点など）を数値に変換（コーディング）して回帰モデルに投入する——その変換方法の選択によって、回帰係数が「何との比較」を意味するかが変わります。この統一的な理解があれば、研究デザインに応じた分析の選択と結果解釈が格段に明瞭になります。

::: {.callout-note}
### 前提知識

本章では、回帰分析の基礎（係数・p値・R²の意味）を理解していることを前提とします。t検定の経験があれば、本章の内容は自然に接続します。
:::

**本章の到達目標**:

1. ANOVAを線形モデル $\mathbf{y} = \mathbf{X}\boldsymbol{\beta} + \boldsymbol{\varepsilon}$ として理解し、設計行列 $\mathbf{X}$ の役割を説明できる
2. 被験者間・被験者内・混合計画の違いを理解し、各仮定（特に球面性）を正しく確認できる
3. `afex` と `emmeans` を使って、混合ANOVAの主効果・交互作用・事後比較を再現可能に実行・報告できる
4. Treatment / Sum / Helmert / Polynomial codingの違いを理解し、研究質問に適したコーディングを選択できる

**本章の流れ**:

本章は、(1) ANOVA を回帰モデルとして捉える統一的理解から始め、(2) 研究計画と仮定を整理し、(3) 英語教育データの生成を行ったうえで、(4) ANOVA の実行・事後比較・効果量の算出へ進みます。続いて (5) 正規性・等分散性・球面性の診断を実施し、(6) Treatment / Sum / Helmert / Polynomial の4種類のコントラストコーディングを比較します。最後に (7)-(8) として、再現可能な報告テンプレートまで一気通貫で扱います。

## 1. ANOVAを回帰モデルとして捉える

### 共通の枠組み

ANOVAも回帰も、基本式は同じです。

$$
\mathbf{y} = \mathbf{X}\boldsymbol{\beta} + \boldsymbol{\varepsilon}, \quad \boldsymbol{\varepsilon} \sim N(0, \sigma^2\mathbf{I})
$$

- $\mathbf{y}$: 従属変数（例: テスト得点）
- $\mathbf{X}$: 設計行列（カテゴリカル変数はコーディングされる）
- $\boldsymbol{\beta}$: 係数
- $\boldsymbol{\varepsilon}$: 誤差

この数式で鍵となるのが**設計行列 $\mathbf{X}$** です。回帰分析では $\mathbf{X}$ に連続変数（英語力スコアなど）がそのまま入りますが、ANOVAでは**カテゴリカル変数を数値に変換して** $\mathbf{X}$ に入れます。この変換が「コントラストコーディング」の正体です。

**具体例**: 3群（Control, Glossing, Keyword）各2名のデータで、設計行列は次のようになります（Treatment codingの場合）。

| 参加者 | 群 | 切片列 | Glossing列 | Keyword列 |
|-------|-----|--------|-----------|----------|
| 1 | Control | 1 | 0 | 0 |
| 2 | Control | 1 | 0 | 0 |
| 3 | Glossing | 1 | 1 | 0 |
| 4 | Glossing | 1 | 1 | 0 |
| 5 | Keyword | 1 | 0 | 1 |
| 6 | Keyword | 1 | 0 | 1 |

カテゴリカル予測子が $k$ 水準なら、通常 $k-1$ 本のコントラスト列が必要です。この $\mathbf{X}$ の中身（0/1の配置）が**コーディング方式によって変わる**のが、セクション6で詳しく学ぶ内容です。

::: {.callout-important}
#### 核心原則

**どのコーディングを選ぶかで係数 $\boldsymbol{\beta}$ の意味は変わるが、同じモデル空間なら適合値や全体検定（F値）は本質的に同じ**です（Schad et al., 2020）。
:::

### F検定の意味

ANOVAのF統計量は、

$$
F = \frac{\text{MS}_{\text{effect}}}{\text{MS}_{\text{error}}}
$$

であり、直感的には**シグナル（効果による変動）とノイズ（誤差による変動）の比**です。F値が大きいほど、「群間の差が偶然の変動では説明しにくい」ことを意味します。

- F ≈ 1: 効果による変動と誤差変動が同程度 → 群間に体系的な差がない可能性が高い
- F >> 1: 効果による変動が誤差変動よりはるかに大きい → 群間に体系的な差がある可能性が高い

F値に対応するp値は、「帰無仮説（群間に差がない）のもとでこのF値以上が得られる確率」です。p < .05 なら、偶然でこれほど大きなF値が出る確率は5%未満であり、通常は帰無仮説を棄却します。

なお、2群比較では $t^2 = F$ が成り立つため、t検定とANOVAは対立概念ではなく同一枠組みの特殊ケースです。3群以上の比較ではt検定を繰り返すと多重検定の問題が生じるため、ANOVAが必要になります。

## 2. 研究計画と仮定

### 英語教育研究での典型デザイン

英語教育研究では、次の3タイプがよく使われます。

| デザイン | 構造 | 英語教育での典型例 |
|---------|------|-------------------|
| **被験者間計画** | 各参加者が1条件のみ | 指導法A群（n=60）vs 指導法B群（n=60）で語彙テストを比較 |
| **被験者内計画** | 各参加者が全条件を経験 | 同一学習者の直後テスト・1週間後テスト・1か月後テストを比較 |
| **混合計画** | 被験者間 + 被験者内の組合せ | 指導法（被験者間）× テスト時点（被験者内） |

研究課題が「群間差の検証」であれば、まず同一参加者を複数条件で測定しているかを確認します。測定していなければ被験者間計画、測定していれば次に「全要因が被験者内か」を判定し、全要因が被験者内なら被験者内計画、被験者間要因も含まれるなら混合計画を選択します。混合計画は英語教育研究で最も頻出し、本章の主要実装例もこの設計に基づきます。

本章の実装例は、英語教育研究で最も頻出する**混合計画**を中心に扱います。

### 仮定の整理

ANOVAが妥当な推論を行うためには、いくつかの前提条件（仮定）が必要です。以下の表で概観し、その後で各仮定の実務的な意味を説明します。

| 仮定 | 何を要求するか | 主な確認方法 | 注意点 |
|---|---|---|---|
| 独立性 | 観測単位間の誤差が独立 | 研究デザインで担保 | 統計手法だけでは救済不能 |
| 正規性 | 残差が概ね正規 | QQプロット, Shapiro-Wilk検定 | 大標本（n > 30）では頑健 |
| 等分散性 | 群間で残差分散が類似 | Levene検定, 残差プロット | 群サイズが等しければ頑健 |
| 球面性 | 反復測定要因（3水準以上）の差得点分散が等しい | Mauchly検定 + GG/HF補正 | 2水準要因では自動的に満たされる |

**各仮定の実務的な意味**:

- **独立性**: 最も重要な仮定です。ある学習者の誤差が別の学習者に影響しないこと。例えば、同じ教室で相互に影響し合う状況では独立性が破れます。この仮定は研究デザインで担保するものであり、統計手法では事後的に救済できません（クラスタ構造がある場合は混合モデル[Ch.6]を検討）
- **正規性**: 残差（予測からのずれ）が正規分布に従うこと。群あたり n > 30 であれば、中心極限定理によりANOVAは軽度の非正規性に対して頑健です。正誤データ（0/1）のように明らかに非正規な場合は、ロジスティック回帰（Ch.5-6）を使います
- **等分散性**: 群間で分散が大きく異ならないこと。群サイズが等しく、最大SD / 最小SD < 3 であれば概ね問題ありません。大きな違反がある場合はWelch ANOVAを検討します
- **球面性**: 反復測定要因が3水準以上の場合に確認が必要です。「各時点ペアの差得点の分散が等しい」という条件で、違反時はGreenhouse-Geisser補正を適用します。**2水準の反復測定要因では自動的に満たされます**（差の分散が1つしかないため）

::: {.callout-note}
#### 球面性と複合対称性の区別

**複合対称性（compound symmetry）は球面性より強い仮定**です。複合対称性は「全時点ペアの相関が等しく、かつ全時点の分散が等しい」ことを要求しますが、球面性は「差得点の分散が等しい」ことのみを要求します。両者を混同しないようにしてください（詳しくはセクション5の診断で扱います）。
:::

仮定の具体的な検証手順はセクション5で、コードとともに詳しく解説します。

## 4. 混合要因ANOVAの実行

### 主効果・交互作用

この混合計画では、ANOVAは以下の3つのF検定を出力します。

1. **Instructionの主効果**: 全体として、Glossingの得点はWritingより高いか？
2. **Timeの主効果**: 全体として、直後テストの得点は遅延テストより高いか？
3. **Instruction × Time の交互作用**: 指導法の効果は時点によって異なるか？

::: {.callout-important}
#### 交互作用を先に確認する

交互作用が有意であれば、主効果は**条件付き**になります。例えば、「Glossingが全体的に優れている」と言っても、実際にはその優位性が直後テストに限定されている可能性があります。交互作用が有意な場合は、必ず単純主効果（各条件内での比較）を確認してください。
:::

以下のコードでは `afex::aov_ez()` を使います。`afex`は Type III平方和、偏η²（偏イータ2乗）、球面性補正を統一的に扱えるため、混合計画ANOVAの標準的なツールです。

```{r}
#| label: mixed-anova

fit_aov <- aov_ez(
  id = "Participant",
  dv = "Score",
  between = "Instruction",
  within = "Time",
  covariate = "Proficiency",
  factorize = FALSE,
  data = dat,
  type = 3,
  anova_table = list(es = "pes")
)

nice(fit_aov) |>
  gt() |>
  tab_header(
    title = "Mixed ANOVA (Type III)",
    subtitle = md("Dependent variable: **Score**")
  )
```

**ANOVA出力の読み方**:

| 列名 | 意味 | 読み方 |
|------|------|--------|
| `Effect` | 検定対象の効果 | Instruction（主効果）、Time（主効果）、Instruction:Time（交互作用）、Proficiency（共変量） |
| `df` | 自由度（分子, 分母） | F分布の形状を決定するパラメータ |
| `MSE` | 誤差の平均平方 | 誤差変動の大きさ（F値の分母） |
| `F` | F統計量 | 大きいほど効果が顕著 |
| `pes` | 偏η²（partial eta-squared） | 効果量。0〜1の範囲で、その効果が説明する分散の割合 |
| `p.value` | p値 | < .05 なら「帰無仮説（効果なし）のもとでこのF値以上が得られる確率が5%未満」 |

**解釈の手順**:

1. **交互作用（Instruction:Time）** の p値を最初に確認する
2. 交互作用が有意 → セクション4.2の事後比較に進む
3. 交互作用が非有意 → 主効果（Instruction, Time）の解釈が直接的に可能

### 事後比較（推定周辺平均）

ANOVAのF検定は「全体的に差があるか」を教えてくれますが、「どの群ペアが具体的にどの程度異なるか」は教えてくれません。そこで**事後比較（post-hoc comparisons）**を行います。

`emmeans`パッケージは以下の2ステップで事後比較を行います。

1. **推定周辺平均（EMM）の算出**: モデルから各条件の調整済み平均を推定する
2. **ペア比較の実行**: 各ペアの平均差・SE・p値・信頼区間を算出する

::: {.callout-tip}
#### なぜ `emmeans` を使うのか（係数表ではダメなのか）

`summary(model)` の係数表は、**コーディング方式に依存した特定のペア比較**しか示しません。例えば Treatment coding では参照群との差のみが直接読めます。一方 `emmeans()` は、コーディングに依存しない**モデル整合的な推定平均**を計算し、任意のペア比較を統一的に行えます。
:::

以下のコードでは、2種類の比較を行います。

- **第1の比較**: 各時点（Immediate / Delayed）内で指導法を比較 → 「直後テストでGlossing > Writingか？ 遅延テストではどうか？」
- **第2の比較**: 各指導法（Glossing / Writing）内で時点を比較 → 「Glossing群は直後→遅延で有意に低下するか？ Writing群ではどうか？」

```{r}
#| label: posthoc

emm <- emmeans(fit_aov, ~ Instruction * Time)

# 時点ごとに指導法を比較
pairs(emm, by = "Time", adjust = "holm") |>
  as.data.frame() |>
  mutate(across(where(is.numeric), ~ round(., 3))) |>
  gt() |>
  tab_header(title = "Pairwise Comparisons: Instruction within Time (Holm-adjusted)")

# 指導法ごとに時点を比較
pairs(emm, by = "Instruction", adjust = "holm") |>
  as.data.frame() |>
  mutate(across(where(is.numeric), ~ round(., 3))) |>
  gt() |>
  tab_header(title = "Pairwise Comparisons: Time within Instruction (Holm-adjusted)")
```

**出力の読み方**:

| 列名 | 意味 |
|------|------|
| `contrast` | 比較の内容（例: Writing - Glossing） |
| `estimate` | 推定平均差（正ならば最初の群が高い） |
| `SE` | 標準誤差（推定の不確実性） |
| `df` | 自由度 |
| `t.ratio` | t統計量（estimate / SE） |
| `p.value` | Holm補正後のp値 |

**解釈例**: もし Instruction within Immediate の行で estimate = −5.2, p = .002 であれば、「直後テストにおいて Writing群はGlossing群より5.2点低く、この差は統計的に有意である（Holm補正後 p = .002）」と解釈します。

::: {.callout-note}
#### 多重比較補正（Holm法）について

複数のペア比較を同時に行うと、偶然に有意な結果が出る確率が高まります（多重検定の問題）。Holm法は、**Bonferroni補正より検出力が高く、かつファミリーワイズの第1種過誤率を制御する**方法です。特段の理由がなければ、Holm法が推奨されます。Tukey法は全ペア比較に特化しており、3群以上の場合に適しています。
:::

### 効果量

**なぜp値だけでは不十分なのか**: p値はサンプルサイズに大きく依存します。大標本（n = 1000）では実質的に些細な差でも p < .001 になりえます。逆に小標本（n = 20）では大きな効果でも p > .05 となりえます。効果量は「差の大きさそのもの」を示すため、p値の補完情報として不可欠です。

```{r}
#| label: effect-size

# 全体モデルのR^2
performance::r2(lm(Score ~ Instruction * Time + Proficiency, data = dat))

# ANOVA系効果量
eta_squared(fit_aov, partial = TRUE, ci = 0.95)
omega_squared(fit_aov, partial = TRUE, ci = 0.95)
```

**出力の読み方**:

- **R²**（全体モデル）: 全予測子（Instruction, Time, 交互作用, Proficiency）が従属変数の分散の何%を説明するかを示す
- **偏η²（partial eta-squared）**: 特定の効果が、**他の効果を統制した上で**説明する分散の割合
- **偏ω²（partial omega-squared）**: η²の不偏推定量。サンプルサイズの影響を補正するため、η²よりやや小さくなる

**Cohen（1988）の効果量解釈基準**:

| 効果量 | 小 | 中 | 大 |
|--------|-----|-----|-----|
| 偏η²（偏イータ2乗） | .01 | .06 | .14 |
| 偏ω²（偏オメガ2乗） | .01 | .06 | .14 |

::: {.callout-tip}
#### η² と ω² の使い分け

- **偏η²**: 報告に最も広く使われる。計算が単純で解釈しやすい。ただしサンプルサイズが小さいと過大推定になりやすい
- **偏ω²**: 母集団パラメータの不偏推定量。サンプルサイズが小さい研究（n < 50）では ω² が望ましい
- **実務的には**: 偏η² + 95%信頼区間を主指標とし、ω² を補助的に併記するのが堅実です

**R²との関係**: R²はモデル全体の説明力、偏η²は個別効果の説明力です。例えば、R² = .45 でも Instructionの偏η² = .07 であれば、「モデル全体は得点の45%を説明するが、そのうち指導法が説明するのは7%程度（残りはProficiencyやTimeが説明）」と解釈できます。
:::

## 5. 仮定診断と実務判断

ANOVAの結果を報告する前に、モデルの前提条件が大きく破綻していないかを確認します。ここで重要な考え方を先に述べます。

::: {.callout-important}
### 仮定違反 ≠ 分析不可能

仮定の検定で p < .05 が出ても、それだけで「ANOVAが使えない」わけではありません。ANOVAは多くの仮定違反に対して**頑健（robust）**です。特に：

- **正規性**: 群あたり n > 30 であれば、中心極限定理により多少の非正規性は問題になりにくい
- **等分散性**: 群間のサンプルサイズが等しければ、分散比が 3:1 程度まで頑健
- **球面性**: 2水準の反復測定要因では自動的に満たされる

重要なのは「仮定が完全に満たされているか」ではなく、「違反の程度がどのくらいか」「結論に影響するか」を総合的に判断することです。
:::

以下では、正規性・等分散性・総合診断・球面性の順に確認します。まず残差の QQ プロットと Shapiro 検定で正規性を評価し、`p > .05` あるいは `n > 30` であれば大きな問題なしと判断します。次に Levene 検定と残差プロットで等分散性を確認し、分散比が大きい場合は Welch ANOVA やロバスト法を検討します。さらに反復測定要因については、水準が2つなら球面性は自動的に満たされ、水準が3つ以上なら Mauchly 検定の結果に応じて GG/HF 補正を適用します。

以下のコードでは、`lm()`で当てはめた線形モデルに対して3つの診断を一括実行します。

1. **Shapiro-Wilk検定**: 残差が正規分布から有意にずれているかを検定
2. **Levene検定**: 群間で残差の分散が等しいかを検定
3. **`check_model()`**: 残差診断を個別プロットとして出力

```{r}
#| label: assumptions
#| fig-width: 10
#| fig-height: 4.8

fit_lm <- lm(Score ~ Instruction * Time + Proficiency, data = dat)

# 正規性（残差）
shapiro.test(residuals(fit_lm))

# 等分散性
car::leveneTest(Score ~ Instruction * Time, data = dat)

# 総合診断
plot_check_model_separate(fit_lm, model_name = "fit_lm")
```

::: {.panel-tabset}

### 正規性の読み方

**Shapiro-Wilk検定の出力**:

| 出力項目 | 意味 | 判断基準 |
|---------|------|---------|
| `W` | 検定統計量（1に近いほど正規） | W > 0.95 なら概ね正規 |
| `p-value` | 正規分布からの逸脱の有意性 | p > .05 → 正規性を棄却しない |

**実務的な判断**:

- p > .05 の場合: 残差は正規分布と有意に異ならない → 問題なし
- p < .05 の場合: 統計的には非正規だが、**サンプルサイズが大きいと些細な逸脱でも有意になる**。本データは n = 240（120名×2時点）なので、QQプロットで視覚的に確認するほうが実用的
- **QQプロットの読み方**（`check_model()`左上パネル）:
  - 点が対角線に沿う → 正規性OK
  - 左下で曲がる → 左裾が重い（負の歪み）
  - 右上で曲がる → 右裾が重い（正の歪み）
  - S字カーブ → 尖度（kurtosis）の問題

### 等分散性の読み方

**Levene検定の出力**:

| 出力項目 | 意味 | 判断基準 |
|---------|------|---------|
| `F value` | 群間の分散差の大きさ | 小さいほど等分散 |
| `Pr(>F)` | 等分散性からの逸脱の有意性 | p > .05 → 等分散を棄却しない |

**実務的な判断**:

- p > .05: 群間で分散は有意に異ならない → 等分散性OK
- p < .05 かつ分散比 < 3: 軽微な違反。ANOVAは群サイズが等しければ頑健なので、通常は問題なし
- p < .05 かつ分散比 > 3: 深刻な違反。Welch ANOVAやブートストラップ法を検討

::: {.callout-tip}
### 分散比の確認方法

群ごとの標準偏差を確認し、最大SDと最小SDの比が3未満かどうかを見ます。

```text
dat |> group_by(Instruction, Time) |> summarise(SD = sd(Score))
```

最大SD / 最小SD < 3 であれば、実務的には問題ありません。
:::

### 総合診断プロットの読み方

`check_model()`の診断を個別プロットで出力した場合も、読み方は次の6項目で同じです。

| パネル | 確認事項 | OKの基準 | 注意が必要な場合 |
|-------|---------|---------|---------------|
| **Residuals vs Fitted** | 残差に系統的パターンがないか | 赤い線が水平・ランダムな散布 | 曲線パターン → 非線形性の見落とし |
| **Normal QQ** | 残差の正規性 | 点が対角線に沿う | 裾で大きく外れる → 外れ値の影響 |
| **Scale-Location** | 分散の均一性 | 赤い線が水平 | 右上がり → 不等分散性 |
| **Cook's Distance** | 影響力の大きい観測値 | 全点が 0.5 未満 | 1.0 超の点 → 外れ値の個別検討 |
| **Collinearity (VIF)** | 予測子間の多重共線性 | VIF < 5 | VIF > 10 → 深刻な共線性 |
| **Posterior Predictive Check** | モデルの全体的な適合 | 予測分布と実データが重なる | 乖離 → モデルの再検討 |

### 球面性と補正

**球面性（Sphericity）** は反復測定要因が3水準以上ある場合に確認が必要です。

本章のデータでは`Time`が2水準（Immediate, Delayed）のみなので、**球面性は自動的に満たされます**。差の分散は1つしかないため、「差の分散が等しいか」を問う余地がありません。

::: {.callout-note}
### 球面性と複合対称性の違い

| 概念 | 要求する条件 | 強さ |
|-----|------------|-----|
| **複合対称性（Compound Symmetry）** | 全時点ペアの相関が等しく、全時点の分散が等しい | より強い仮定 |
| **球面性（Sphericity）** | 時点間の差得点の分散が等しい | より弱い仮定 |

複合対称性が成り立つ → 球面性も成り立つ（逆は不成立）。実務では球面性のみを確認すれば十分です。
:::

**3水準以上の反復測定要因がある場合の手順**:

1. **Mauchly検定**で球面性を確認する
2. p < .05 で球面性が棄却された場合、**自由度補正**を適用する:
   - ε（イプシロン）< 0.75 → **Greenhouse-Geisser（GG）補正**（保守的）
   - ε ≥ 0.75 → **Huynh-Feldt（HF）補正**（やや自由度を回復）
3. `afex`では `anova_table = list(correction = "GG")` で自動補正される

以下は3時点以上のデータに対する球面性検定のコード例です（本章のデータには該当しないため `eval: false` としています）。

:::

## 6. カテゴリカル変数のコーディング

ANOVAを回帰モデルとして扱う場合、カテゴリカル変数（例：指導法の種類）を数値に変換する必要があります。この変換方法が**コントラストコーディング（contrast coding）**です。

コーディングの核心を一文で表すなら、「どの群とどの群を比較するかを、設計行列のなかで数学的に指定する方法」です。同じ風景を異なる視点から描くように、同じデータを異なるコーディングで表現すると、回帰係数が「何との比較か」が変わります。

::: {.callout-important}
### コーディング選択の核心原則

**コーディングは「係数の意味」を変える。**
ただし同じモデル空間なら、適合値・残差・全体的なモデル適合（F値やR²）は本質的に変わらない。

つまり、コーディング選択は「どの比較を回帰係数から直接読み取りたいか」によって決まります。
:::

以下では、同一の3群データに対して4種類のコーディングを順に適用し、係数の意味がどう変わるかを比較します。

### 比較用データ（3群）

ここでは、語彙学習支援法の3条件を比較するデータを作成します。

- **Control**（統制群）: 語彙リストのみ提供
- **Glossing**（注釈群）: 読解テキスト中に語義注釈を併記
- **Keyword**（記憶術群）: キーワード法による記憶術を指導

各群100名、単語テスト得点（Score）を測定します。

```{r}
#| label: contrast-data

set.seed(2024)

dat_c <- tibble(
  Group = factor(rep(c("Control", "Glossing", "Keyword"), each = 100),
                 levels = c("Control", "Glossing", "Keyword")),
  Score = c(
    rnorm(100, mean = 50, sd = 8),
    rnorm(100, mean = 58, sd = 8),
    rnorm(100, mean = 62, sd = 8)
  )
)

dat_c |>
  group_by(Group) |>
  summarise(Mean = mean(Score), SD = sd(Score), .groups = "drop") |>
  gt() |>
  tab_header(title = "Descriptive Statistics for Contrast Coding")
```

**読み方のポイント**: 記述統計表の群平均を確認してください。Control < Glossing < Keyword の順で得点が上昇しています。この「Controlからの差がどの程度か」「全体平均からの偏差はどの程度か」「段階的な伸びがあるか」——これらを4通りのコーディングで異なる角度から数値化していきます。

### Treatment coding（ダミーコーディング）

**最も直感的で、介入研究において最も広く使われるコーディングです。**参照群（reference group）を基準に、他の群がそこからどれだけ離れているかを回帰係数として推定します。

英語教育研究では、統制群（Control）と処遇群（Glossing, Keyword）を比較する場面で自然に使えます。

```{r}
#| label: treatment-coding

dat_treat <- dat_c
contrasts(dat_treat$Group) <- contr.treatment(3)

contrasts(dat_treat$Group) |>
  as.data.frame() |>
  tibble::rownames_to_column("Level") |>
  gt() |>
  tab_header(title = "Design Matrix: Treatment Coding")

fit_treat <- lm(Score ~ Group, data = dat_treat)

tidy(fit_treat) |>
  mutate(across(where(is.numeric), ~ round(., 3))) |>
  gt() |>
  tab_header(title = "Coefficient Table: Treatment Coding")
```

**設計行列の読み方**:

| Level | 列2 | 列3 | 意味 |
|-------|-----|-----|------|
| Control | 0 | 0 | 参照群。切片のみで表現 |
| Glossing | 1 | 0 | 切片 + β₁ で予測 |
| Keyword | 0 | 1 | 切片 + β₂ で予測 |

**係数表の読み方**:

| 係数 | 意味 | 解釈例 |
|-----|------|--------|
| `(Intercept)` | **Control群の平均得点** | ≈ 50 → 統制群の語彙テスト平均 |
| `Group2` | **Glossing − Control の差** | ≈ 8 → 注釈群は統制群より約8点高い |
| `Group3` | **Keyword − Control の差** | ≈ 12 → 記憶術群は統制群より約12点高い |

::: {.callout-tip}
#### Treatment coding の利点と限界

**利点**: 係数を読むだけで「各処遇群が統制群よりどれだけ効果的か」が直接わかります。

**限界**: Glossing vs Keyword の直接比較はこの表から読めません（差は計算可能ですがSEの算出が複雑）。全ペアの比較が必要な場合は `emmeans()` を使いましょう（セクション4.2参照）。
:::

### Sum coding（偏差コーディング）

**ANOVAの伝統的な主効果解釈と最も整合的なコーディングです。**全体平均（grand mean）を基準に、各群がそこからどれだけずれるかを推定します。

Treatment codingが「特定の参照群との差」を見るのに対し、Sum codingは「全体の傾向からの偏差」を見ます。なお、`afex`パッケージの `set_sum_contrasts()` はこのコーディングを全因子に適用する関数です（本章の冒頭で実行済み）。

```{r}
#| label: sum-coding

dat_sum <- dat_c
contrasts(dat_sum$Group) <- contr.sum(3)

contrasts(dat_sum$Group) |>
  as.data.frame() |>
  tibble::rownames_to_column("Level") |>
  gt() |>
  tab_header(title = "Design Matrix: Sum Coding")

fit_sum <- lm(Score ~ Group, data = dat_sum)

tidy(fit_sum) |>
  mutate(across(where(is.numeric), ~ round(., 3))) |>
  gt() |>
  tab_header(title = "Coefficient Table: Sum Coding")
```

**設計行列の読み方**: 各行の合計が0になるのが"Sum" codingの特徴です。最後の水準（Keyword）は −1, −1 として、他の2水準の「裏返し」として暗黙的に定義されます。

**係数表の読み方**:

| 係数 | 意味 | 解釈例 |
|-----|------|--------|
| `(Intercept)` | **全体平均（grand mean）** | ≈ 56.7 → 3群全体の平均得点 |
| `Group1` | **Control − 全体平均** | ≈ −6.7 → 統制群は全体平均より約6.7点低い |
| `Group2` | **Glossing − 全体平均** | ≈ 1.3 → 注釈群は全体平均より約1.3点高い |
| *(暗黙)* | **Keyword − 全体平均** | = −(−6.7 + 1.3) = 5.4 → 記憶術群は全体平均より約5.4点高い |

::: {.callout-note}
#### なぜSum codingがANOVAと整合的か

ANOVAのF検定は「全体的に群間に差があるか」を問います。この「全体的」という発想は、各群を**全体平均からの偏差**として捉えるSum codingと対応します。一方、Treatment codingは「特定の参照群との差」を強調するため、ANOVAのF検定と直感的に結びつきにくい面があります。

**いつ使うか**: 複数要因のANOVAで主効果を報告する際、または特定の参照群を置かない場合に適しています。
:::

### Helmert coding（逐次比較コーディング）

**段階的・累積的な比較を行うコーディングです。**各水準を、その「後に続く水準の平均」と比較します。

このコーディングは、水準に自然な順序がある場合に有効です。英語教育研究では：

- 指導の複雑さが段階的に上がる場合（例：Control → Glossing → Keyword）
- テスト時点が増えていく場合（例：Pre → Post → Delayed）
- 処遇の強度が段階的に増加する場合

**Helmert codingが答える問い**:

1. **第1比較**: Control は、Glossing・Keyword の平均と異なるか？（統制群 vs 処遇群全体）
2. **第2比較**: Glossing は Keyword と異なるか？（処遇群間の差）

```{r}
#| label: helmert-coding

dat_helm <- dat_c
contrasts(dat_helm$Group) <- contr.helmert(3)

contrasts(dat_helm$Group) |>
  as.data.frame() |>
  tibble::rownames_to_column("Level") |>
  gt() |>
  tab_header(title = "Design Matrix: Helmert Coding")

fit_helm <- lm(Score ~ Group, data = dat_helm)

tidy(fit_helm) |>
  mutate(across(where(is.numeric), ~ round(., 3))) |>
  gt() |>
  tab_header(title = "Coefficient Table: Helmert Coding")
```

**係数表の読み方**:

| 係数 | 比較内容 | 解釈例 |
|-----|---------|--------|
| `(Intercept)` | 全体平均 | ≈ 56.7（Sum codingの切片と同じ） |
| `Group1` | Control vs (Glossing + Keyword) の平均 | 統制群と処遇群全体の差（スケーリング済み）|
| `Group2` | Glossing vs Keyword | 2つの処遇群間の差（スケーリング済み）|

::: {.callout-warning}
#### Rの `contr.helmert()` はReverse Helmert

Rの `contr.helmert()` は、各水準を**先行する水準群の平均**と比較する「逆Helmert」です。教科書によっては「後続水準の平均と比較する」定義が使われることがあります。解釈の方向に注意してください。

係数のスケーリングにも注意が必要です。Rの出力をそのまま「差」として読むには、比較する群数に応じたスケーリング因子を考慮する必要があります。実務的には、具体的な群平均差は `emmeans()` で確認するほうが確実です。
:::

**いつ使うか**: 研究仮説が「統制群 vs 処遇群全体 → 処遇群間の差」という階層的な構造をもつ場合に適しています。例えば、まず「何らかの介入をすること自体に効果があるか」を確認し、次に「介入の種類による差があるか」を検討する、という段階的な問いに対応します。

### 順序尺度にはPolynomial coding

**順序因子のトレンド（線形・曲線的なパターン）を検出するためのコーディングです。**

CEFRのような言語習熟度レベル（A1 < A2 < B1 < B2）では、「レベルが上がるにつれて得点がどう変化するか」の**形状**が関心事になります。

- 得点が均等に伸びている → **線形トレンド**（直線的な上昇）
- 初級段階で伸びが大きく、上級段階で伸びが鈍化する → **二次（曲線的）トレンド**（天井効果）
- さらに複雑なパターン → **三次トレンド**

Polynomial codingは、こうしたトレンドの各成分を独立に検定します。

```{r}
#| label: polynomial-coding

set.seed(2024)

dat_ord <- tibble(
  CEFR = factor(rep(c("A1", "A2", "B1", "B2"), each = 80),
                levels = c("A1", "A2", "B1", "B2"),
                ordered = TRUE),
  Score = c(
    rnorm(80, 48, 7),
    rnorm(80, 56, 7),
    rnorm(80, 66, 7),
    rnorm(80, 74, 7)
  )
)

contrasts(dat_ord$CEFR) <- contr.poly(4)

fit_poly <- lm(Score ~ CEFR, data = dat_ord)

tidy(fit_poly) |>
  mutate(across(where(is.numeric), ~ round(., 3))) |>
  gt() |>
  tab_header(title = "Polynomial Contrasts for Ordered CEFR Levels")
```

**係数表の読み方**:

| 成分 | 意味 | 判断基準 |
|-----|------|---------|
| `CEFR.L`（Linear） | **線形成分**: レベルが上がると得点が直線的に上昇するか | p < .05 → 有意な線形トレンドあり |
| `CEFR.Q`（Quadratic） | **二次成分**: 伸び方に曲線的パターン（加速・減速）があるか | p < .05 → 直線からの有意な逸脱 |
| `CEFR.C`（Cubic） | **三次成分**: さらに複雑な変動があるか | 通常は無視可。有意なら解釈が困難 |

**解釈の手順**:

1. まず `.L`（線形成分）を確認 → 有意なら「レベルが上がるほど得点が上昇する」
2. 次に `.Q`（二次成分）を確認 → 有意なら「その上昇は均等ではなく、曲線的」
3. `.C`（三次成分）→ 4水準では通常解釈しない

::: {.callout-tip}
#### 報告例

「CEFRレベルの主効果は有意であった。Polynomial contrast analysisにより、有意な線形トレンド（.L: *t* = XX.X, *p* < .001）が確認され、レベルが上がるほど語彙テスト得点が上昇する傾向が示された。二次成分（.Q）は有意でなく（*p* = .XX）、この上昇パターンはほぼ直線的であった。」
:::

**いつ使うか**: 水準に明確な順序がある因子（習熟度レベル、テスト時点、練習回数など）で、「上昇パターンの形状」に関心がある場合に適しています。

### コーディング間の不変性チェック

```{r}
#| label: invariance-check

anova(fit_treat)
anova(fit_sum)
anova(fit_helm)

all.equal(unname(fitted(fit_treat)), unname(fitted(fit_sum)))
all.equal(unname(fitted(fit_treat)), unname(fitted(fit_helm)))
```

**出力の確認ポイント**:

1. **ANOVA表を比較**: `anova(fit_treat)`, `anova(fit_sum)`, `anova(fit_helm)` のF値・p値はすべて同一です。これは、コーディングが変わっても「群間に差があるか」という全体的な検定結果は不変であることを意味します。
2. **予測値を比較**: `all.equal()` が `TRUE` を返します。すべての観測値に対する予測得点が、コーディング方式を問わず完全に一致します。

**実務的な含意**: コーディング選択に「正解」や「不正解」はありません。係数の意味が変わるだけで、モデルの適合そのものは同じです。したがって、**研究質問に対して最も直接的に答えを与えるコーディングを選ぶ**ことが合理的です。

## 7. 研究質問に基づくコーディング選択

ここまで4つのコーディング方式を見てきました。実際の研究でどれを選ぶかは、**あなたのリサーチクエスチョンが何を問うているか**によって決まります。

| 研究質問の例 | 推奨コーディング | 係数が直接示すもの |
|---|---|---|
| 「Glossing指導はControl群より効果的か？」 | **Treatment** | 各処遇群と参照群（Control）の差 |
| 「各指導法は全体平均からどの程度ずれるか？」 | **Sum** | 各群の全体平均（grand mean）からの偏差 |
| 「まず処遇群全体が統制群と異なるか、次に処遇群間の差は？」 | **Helmert** | 逐次的・階層的な群比較 |
| 「CEFR A1→B2で得点は線形に伸びるか、曲線的か？」 | **Polynomial** | 線形・二次・三次のトレンド成分 |

選択の実務手順は次の通りです。まず統制群のような明確な参照群がある場合は Treatment coding を優先します。参照群がない場合は水準に順序があるかを確認し、順序がありトレンド形状（線形・曲線）に関心があるなら Polynomial coding、順序はあるがトレンド形状が主眼でなければ Sum coding を選びます。順序がない場合は、段階的・階層的比較が自然なら Helmert coding、そうでなければ Sum coding が第一候補になります。

::: {.callout-tip}
### 迷ったときの3原則

1. **最も単純で直感的なコーディングを選ぶ**: 係数を見て即座に「答え」がわかるものが良いコーディングです
2. **統制群がある介入研究なら Treatment が第一候補**: 英語教育研究の多くはこのパターンに該当します
3. **複数のコーディングで試しても問題なし**: 予測値・F値は変わらないので、「別のコーディングで見たらどう解釈できるか」を確認するのは健全な実践です
:::

## 9. トラブルシューティング（実務で詰まりやすい点）

::: {.callout-warning}
### 交互作用が有意なのに主効果だけで結論を出す

**症状**: 「Instructionの主効果が有意なので、Glossingが効果的である」と報告したが、査読者から「交互作用が有意なのに主効果だけで議論するのは不適切」と指摘される。

**原因**: 交互作用が有意な場合、主効果は**条件付き（qualified）**です。「全体的にGlossingが高い」ことは事実でも、その優位性がImmediateに限定されている（Delayedでは差がない）可能性があります。主効果だけの報告では、この重要な情報が失われます。

**修正**: 交互作用が有意なら、必ず `emmeans()` で各条件内の比較（simple effects）を報告してください。主効果は「補助情報」として扱い、単純主効果を中心に議論します。
:::

::: {.callout-warning}
### 球面性と複合対称性の混同

**症状**: 「Mauchly検定で p > .05 だったので、複合対称性が成り立つ」と記述してしまう。

**原因**: Mauchly検定は**球面性**を検定するものであり、複合対称性を検定するものではありません。球面性（差の分散が等しい）は複合対称性（全ペアの相関が等しい + 全分散が等しい）より弱い仮定です。

**修正**: 「球面性が満たされた」と正しく記述してください。Mauchly検定で p > .05 → 球面性は棄却されない、p < .05 → GG/HF補正を適用。
:::

::: {.callout-warning}
### 係数表だけでペア比較を判断する

**症状**: `summary(model)` の係数表を見て、「Group2の p < .05 だからGlossing > Control」「Group3の p > .05 だからKeyword = Control」と判断する。しかし、コーディングを変えると異なる結論に見える。

**原因**: 係数表の p値は**コーディング方式に依存した特定の比較**の結果です。Treatment codingなら参照群との比較のみ、Sum codingなら全体平均との比較のみが直接読めます。全ペア比較は係数表からは読めません。

**修正**: ANOVAの全体検定後は、必ず `emmeans()` で事後比較を行ってください。`emmeans` の結果はコーディングに依存しない統一的な推定値を提供します。
:::

::: {.callout-warning}
### p値と効果量を混同する

**症状**: 「p < .001 なので大きな効果がある」または「p = .04 なので効果は小さい」と解釈する。

**原因**: p値はサンプルサイズに強く依存します。n = 1000 なら些細な差（η² = .005）でも p < .001 になりえます。n = 20 なら大きな差（η² = .15）でも p = .08 になりえます。p値は「効果の存在の証拠」であり、「効果の大きさ」を示すものではありません。

**修正**: 常に p値と効果量（偏η² + 95% CI）を併記してください。解釈は「統計的有意性」（p値）と「実質的意義」（効果量）を分離して議論します。
:::

::: {.callout-warning}
### コーディング方式を論文に記載しない

**症状**: コードでは `set_sum_contrasts()` を使っているが、論文のMethodセクションにコーディング方式の記載がない。査読者から再現性を問われる。

**原因**: Rのデフォルト（Treatment coding）と `afex` のデフォルト（Sum coding）は異なります。どのコーディングを使ったかを明記しないと、読者が同じコードを実行しても異なる係数が得られる可能性があります。

**修正**: Methodセクションに以下のように明記してください。

```text
Sum coding was applied to all categorical variables using
afex::set_sum_contrasts(), ensuring that model coefficients
represent deviations from the grand mean, consistent with
standard ANOVA interpretation.
```
:::
