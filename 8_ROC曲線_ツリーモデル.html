11 はじめに

5章・6章で扱ったロジスティック回帰では、「ある説明変数が正答確率にどう影響するか」を1本のグローバルな式でモデル化しました。しかし英語教育データでは、学習者要因と語彙項目要因の複雑な組み合わせパターンが頻出します。たとえば、習熟度が高い学習者は retrieval practice 条件で特に効果が大きいが、低習熟者ではその差が消える、といったパターンです。回帰分析でこうしたパターンを捉えるには交互作用項を手動で指定する必要がありますが、どの変数の組み合わせが重要かを事前に知ることは難しいのが実情です。

ツリーモデル（決定木・ランダムフォレスト・XGBoost）は、こうした非線形関係や相互作用を、分析者が事前に指定しなくても自動的に発見できる手法群です。本章では、これら3つのツリーモデルを同じ教育データで比較し、モデルの性能評価にROC曲線とAUCを用います。

英語教育研究で想定される予測課題の例:

  • 語彙テスト項目の正誤（正解/不正解）を予測する
  • 学習者が次の小テストで基準を超えるかを予測する
  • CEFR到達判定の補助指標を構築する

本章を通して、「どの学習者が、どの語彙項目を、どの条件で正答するか」を予測し、その予測を教育判断に変換する一連の流れを体系的に学びます。

11.1 最初にここだけ（はじめに）

  • ツリーモデル = データを「もし proficiency > 0.4 ならば…」のような条件分岐で繰り返し分割し、局所ごとに予測を出す方法。回帰分析が1本の式で全データを説明するのに対し、ツリーモデルはデータの「区画」ごとに異なる予測を出す。
  • ROC-AUC = ランダムに正答者と誤答者を1人ずつ選んだとき、モデルが正答者に高い確率を正しく与える割合。1.0が完璧、0.5がランダム。
読む順序（迷ったとき）
  • 最短ルート（20分）: はじめに → ケース設定 → 決定木 → 3モデル比較 → まとめ
  • 標準ルート（60分）: 上記 + RF + XGBoost + 閾値 + キャリブレーション
  • 拡張ルート（90分+）: 上記 + 回帰→MLの橋渡し + SHAP + ハイパーパラメータ詳細 + 重要度比較
この章の到達目標
  1. ロジスティック回帰とツリーモデルの違いを、「的のずれ」と「散らばり」（バイアスとバリアンス）の観点から説明できる。
  2. 決定木・ランダムフォレスト・XGBoostの3モデルを同じデータで比較できる。
  3. ROC-AUCだけでなく、複数の評価指標（PR-AUC・閾値依存指標・予測確率の信頼性）も確認し、報告できる。
  4. 変数の相対的重要性を測る複数の方法（SHAP、dominance analysis、permutation importance）の違いを理解し、使い分けられる。
  5. 「精度が高い」で終わらず、教育実務で使える判断に落とし込める。

11.2 回帰分析から機械学習への橋渡し

読者の多くは、5章・6章でロジスティック回帰を学んでいるはずです。ツリーモデルに進む前に、「なぜ回帰だけでは不十分な場面があるのか」「ツリーモデルは何が本質的に異なるのか」を整理します。この節を理解すると、以降の決定木・RF・XGBoostの位置づけがはっきりします。

11.2.1 回帰分析の限界を直感で理解する

ロジスティック回帰は、すべてのデータに対して1本のグローバルな式をあてはめます。

\[ \log\frac{P(\text{correct})}{1-P(\text{correct})} = \beta_0 + \beta_1 x_1 + \beta_2 x_2 + \cdots \]

この式は、\(x_1\)（たとえば習熟度）が1単位増えると対数オッズが \(\beta_1\) だけ変化する、という全データに共通のルールを仮定しています。しかし教育データでは、「習熟度が高い層では retrieval practice が特に効く」「低頻度語では語長の影響が大きくなる」のように、ある条件下でだけ効く変数が少なくありません。

回帰分析でも交互作用項（proficiency:condition）を手動で追加すれば対応可能ですが、どの交互作用を入れるかは分析者が事前に決める必要があります。変数が10個あれば交互作用の候補は45通り、3次の交互作用まで含めればさらに膨大になります。

ツリーモデルは、この交互作用の発見をデータから自動的に行います。

以下の図は、同じデータに対して「回帰の線形フィット」と「ツリーの矩形分割」がどう異なるかを示しています。コードでは (1) 非線形な真の関係を持つデータを生成し、(2) ロジスティック回帰でフィットし、(3) 決定木でフィットして、3パネルの比較図を作成します。

  • 回帰: 全データに1本の式（グローバル）。係数を読めば「何が効くか」がわかる。
  • ツリー: データを条件分岐で分割し、各領域で別の予測を出す（ローカル）。分岐規則を読めば「どの条件で効くか」がわかる。

11.2.2 バイアスとバリアンスのトレードオフ

モデル選択を理解するうえで最も重要な概念が、バイアス・バリアンスのトレードオフです(Hastie ほか, 2009; James ほか, 2021)。

  • バイアス（bias）: モデルが現実を単純化しすぎることによる系統的な誤差。ロジスティック回帰が非線形な関係を直線的に近似するとき、これが生じる。
  • バリアンス（variance）: データが変わるとモデルの予測が大きく変わる不安定さ。深い決定木はデータの細部まで追いかけるため、別のサンプルでは異なる構造になりやすい。
コード 
bv_dat <- tibble(
  complexity = seq(0.5, 10, length.out = 100),
  train_error = 0.45 * exp(-0.5 * complexity) + 0.02,
  test_error  = 0.45 * exp(-0.5 * complexity) + 0.02 + 0.005 * complexity^1.8,
  bias2       = 0.45 * exp(-0.5 * complexity),
  variance    = 0.005 * complexity^1.8
)

この図の読み方:

  • 左側（低複雑度）: モデルが単純すぎてデータのパターンを拾えない（未学習、バイアスが大きい）。
  • 右側（高複雑度）: モデルがデータのノイズまで拾ってしまう（過学習、バリアンスが大きい）。
  • 最適点: テスト誤差が最小になる複雑度。この点を交差検証（後述）で探すのが本章の基本戦略。

数値で実感する: 仮に語彙テストの正答率予測で以下のような結果が出たとします。

モデル 訓練データ正答率 テストデータ正答率 症状
浅い決定木（深さ2） 67% 65% バイアス大（両方低い＝パターンを捉えきれていない）
深い決定木（深さ10） 95% 60% バリアンス大（訓練とテストの差＝ノイズを学習）

訓練データとテストデータの正答率が両方とも低いならバイアスの問題、訓練だけ高くテストが低いならバリアンスの問題です。

本章の3つのツリーモデルは、このトレードオフの異なる位置にいます:

  • 決定木: 深くすれば表現力は高いが、バリアンスが大きい（データの揺れに弱い）。
  • ランダムフォレスト: 多数の木の多数決（バギング）でバリアンスを抑制する。
  • XGBoost: 前の木の誤差を次の木が修正（ブースティング）し、バイアスを段階的に減らす。

11.2.3 ツリーモデルの分岐ロジック：再帰分割

決定木がデータを分ける仕組みを、小さな例で追ってみましょう。

6人の学習者のデータがあるとします。目的変数は正答/誤答、説明変数は proficiency（習熟度）です。

木は「どこで分けると、分けた後のグループが最も”純粋”になるか」を探します。Gini不純度で純粋さを測ります:

\[ G = 1 - \sum_{k=1}^{K} p_k^2 \]

ここで \(p_k\) はノード内のクラス \(k\) の割合です。\(G = 0\)（完全に1クラス）が最も純粋で、\(G = 0.5\)（2クラスが半々）が最も不純です。

コード 
gini <- function(labels) {
  p <- table(labels) / length(labels)
  1 - sum(p^2)
}

# 分割前のGini
gini_before <- gini(mini_split$correct)

# 候補分割点: proficiency = 0.25（CとDの間）
left  <- mini_split |> filter(proficiency <= 0.25)
right <- mini_split |> filter(proficiency > 0.25)
gini_after <- (nrow(left) / 6) * gini(left$correct) +
              (nrow(right) / 6) * gini(right$correct)

tibble(
  Step = c("Before split (all 6)", "Left (prof ≤ 0.25)", "Right (prof > 0.25)", "Weighted Gini after split"),
  Gini = c(gini_before, gini(left$correct), gini(right$correct), gini_after)
) |>
  mutate(Gini = round(Gini, 3)) |>
  gt() |>
  tab_header(
    title = "Gini Impurity Calculation",
    subtitle = "Split at proficiency = 0.25"
  )

分割前のGiniは 0.500（2クラスが半々）。proficiency = 0.25 で分けると、左は全員 incorrect（\(G = 0\)）、右は全員 correct（\(G = 0\)）となり、加重Giniは 0.000 です。これが最も純粋な分割です。

実際の決定木アルゴリズム（CART）は、すべての変数×すべての分割点でこのGini改善量を計算し、最大の改善をもたらす分割を選びます。これを再帰的に繰り返し、停止条件（最小サンプル数、最大深さなど）に達するまで分岐を続けます(Breiman ほか, 1984)。

回帰との対比
  • ロジスティック回帰: 全データに対して \(\beta\) を最適化する（1つのグローバルな最適化問題）。
  • 決定木: 各ノードで局所的に最良の分割を選ぶ（貪欲な局所最適化の繰り返し）。

11.2.4 交差検証の直感的理解

ツリーモデルのハイパーパラメータ（木の深さ、学習率など）を決めるために、本章では交差検証（Cross-Validation; CV）を使います。CVの仕組みを先に押さえておきましょう。

CVの手順:

  1. データをK個に分割する（本章ではK=3）。
  2. 1つ目の分割をテスト用に取っておき、残りで学習する。
  3. 2つ目、3つ目でも同様に繰り返す。
  4. K回の結果を平均して、モデルの汎化性能を推定する。

この方法の利点は、テストデータを「使い切る前に」汎化性能を安定的に推定できる点です。ハイパーパラメータのチューニング（後述の tune_grid）はすべてこのCV上で行い、最終評価は別に確保したテストデータで行います。

なぜ3-foldなのか

本章はチュートリアルのため計算時間を短縮する目的で3-foldを採用しています。実際の研究では 5-fold または10-fold が一般的です(James ほか, 2021)。

evaluate_threshold <- function(truth, prob, threshold) {
  pred <- factor(
    if_else(prob >= threshold, "correct", "incorrect"),
    levels = levels(truth)
  )

  tp <- sum(pred == "correct" & truth == "correct")
  fp <- sum(pred == "correct" & truth == "incorrect")
  tn <- sum(pred == "incorrect" & truth == "incorrect")
  fn <- sum(pred == "incorrect" & truth == "correct")

  tibble(
    threshold = threshold,
    TP = tp, FP = fp, TN = tn, FN = fn,
    Sensitivity = tp / (tp + fn),
    Specificity = tn / (tn + fp),
    Precision = tp / (tp + fp)
  )
}

15.1 補助解析：Dominance Analysis（GLM）

係数表は「方向と有意性」を示すのに有用ですが、変数間でスケールが異なるため「どの変数が最も重要か」の比較は直接できません。標準化係数を使う方法もありますが、変数間の相関が強い場合は不安定です。Dominance analysisは、すべてのサブモデルの組み合わせを比較することでこの問題を解決します(Azen & Budescu, 2003; Azen & Traxel, 2009; Budescu, 1993)。

文献での活用ポイント（具体）:

  • Azen & Traxel (2009) は、ロジスティック回帰で dominance analysis を使うために、 R2類似指標の選定とブートストラップ推論を組み合わせて、重要度の再現性を検討しました。
  • Mizumoto (2022) は、応用言語学データの再解析で dominance analysis を併用し、 「標準化βだけでは見えにくい相対的重要性」を示しました。
  • 本章での具体例としては、proficiency と condition の relative share を比較し、
    「学習者要因と授業デザイン要因のどちらが予測改善に効いているか」を同じ尺度で読む、という使い方になります。

以下のコードでは、(1) 通常のglmオブジェクトを作成し、(2) dominanceanalysisパッケージで全てのサブモデルの組み合わせにおける各変数の平均的な改善量を計算し、(3) Nagelkerke R2ベースの結果を表にします。

16 決定木（Decision Tree）

最初にここだけ（決定木）

決定木は「もし proficiency > 0.4 ならば…」のようなif-thenルールの積み重ね。1本見れば予測根拠が明確にわかる。ただし1本の木はデータの揺れに弱い（高バリアンス）。

この節の目的は、解釈可能性の高い木モデルを構築し、分岐規則を確認することです。

決定木の基本構造
  • 決定木は、データを「条件分岐」で繰り返し分けるモデルです。
  • 分岐点（ノード）では、例: proficiency > 0.4 のような質問をします。
  • 最後の葉（leaf）で、正答確率やクラス（correct / incorrect）を出します。
  • 良い点: ルールが可視化でき、教育的解釈がしやすい。
  • 注意点: 1本の木はデータの小さな揺れに影響されやすい。

英語教育の文脈では、「どの条件の学習者が正答しやすいか」をif-thenルールで説明できる点が、決定木の大きな利点です。分岐ロジックの詳細（Gini不純度による分割選択）は、先の「回帰から機械学習への橋渡し」で解説しました。ここではその仕組みを実際の教育データに適用し、ハイパーパラメータのチューニングから可視化までを行います。

16.4 解釈ポイント

決定木の最大の強みは透明性です。上の図を見れば、「proficiency が高い → さらに word_freq が大きい → 正答確率が高い」のように、予測の根拠をif-thenルールとして直接読み取れます。これは教育現場で「なぜこの学習者が正答/誤答と判定されたのか」を説明する際に非常に有用です。

一方、単木の限界も明確です:

  • 高バリアンス: データの一部を変えるだけで、木の構造が大きく変わることがある。

したがって、決定木は「予測の仕組みを説明するモデル」として使い、精度を求める場合は次節以降のアンサンブル手法を検討します。

17 ランダムフォレスト（Random Forest）

最初にここだけ（ランダムフォレスト）

不安定な単木をたくさん作って多数決する。各木は別々のデータ（ブートストラップ）と別々の変数（ランダム選択）で育てるので、木どうしが似すぎない。結果としてバリアンスが大幅に下がる。

この節の目的は、単木の弱点（不安定性）をアンサンブルで補うことです(Breiman, 2001)。

17.1 バギングの原理

ランダムフォレストの土台はバギング（Bootstrap Aggregating）です。手順は以下の通りです:

  1. 元データからブートストラップ標本（復元抽出）を作る。
  2. その標本で1本の決定木を育てる。
  3. 1-2をB回繰り返す（本章ではB = 300）。
  4. 全B本の木の予測を多数決（分類の場合）または平均（回帰の場合）で統合する。

比喩で言えば、「多数の専門家に別々の資料を渡し、独立に判断してもらって多数決をとる」イメージです。1人の判断は偏りやすくても、独立した多数の専門家の多数決は安定します。本章では300本の木（trees = 300）を使用します。

ランダムフォレストはバギングに加えて、各分岐で変数のランダム選択（mtry個だけ候補にする）を行います。これにより、強い変数（例: proficiency）がすべての木で最初に選ばれる状況を防ぎ、木どうしの相関を下げてバリアンス削減効果を高めます(Breiman, 2001)。

18 XGBoost

最初にここだけ（XGBoost）

XGBoostは「前の木が間違えた部分を、次の木が重点的に修正する」逐次学習法。ランダムフォレストが木を並列に育てるのに対し、XGBoostは直列に育てて誤差を段階的に減らす。強力だが設定項目が多い。

この節の目的は、高精度モデルを構築しつつ、重要特徴を解釈可能な形で確認することです(Chen & Guestrin, 2016)。

18.1 ブースティングの原理：バギングとの対比

ランダムフォレスト（バギング）とXGBoost（ブースティング）は、アンサンブルの組み立て方が根本的に異なります。

Bagging (RF) vs. Boosting (XGBoost)
Two fundamentally different ensemble strategies
観点 バギング（RF） ブースティング（XGBoost）
木の育て方 各木を独立に並列で育てる 前の木の誤差を踏まえて逐次に育てる
学習データ ブートストラップ標本（復元抽出） 全データ（ただし誤差が大きい観測に重点）
予測の統合 多数決 or 平均 加重和（学習率で縮小しながら積み上げ）
主な効果 バリアンスの削減 バイアスの段階的削減
各木の深さ 通常は深い木 通常は浅い木（stump〜数段）
過学習リスク 低い（木の独立性が高い） 反復数が多いと過学習しやすい
チューニング難度 比較的容易（mtryが主要） やや高い（学習率・反復数・深さなど）

ブースティングの手順を、比喩を使って説明します:

  1. 1回目の家庭教師が全体を見て、「習熟度が低い学習者は苦手」と大まかに判断する（浅い木）。
  2. 2回目の家庭教師は、1回目が見逃した部分（例: 同根語なのに間違えた高習熟者）を重点的に指導する。
  3. 3回目はさらに細かい見逃しを拾う。
  4. 最終予測は、全家庭教師の判断を学習率（learn_rate）で縮小しながら足し合わせたもの。

数式で書くと、\(m\)回目までのモデルの予測は:

\[ \hat{f}_m(x) = \hat{f}_{m-1}(x) + \eta \cdot h_m(x) \]

ここで \(h_m(x)\) は \(m\)回目の木の予測、\(\eta\) は学習率（0〜1）です。学習率を小さくすると各木の寄与が控えめになり、より多くの反復が必要になりますが、過学習しにくくなります(Friedman, 2001)。

XGBoostの「勾配」とは

XGBoostの正式名称は eXtreme Gradient Boosting です。「勾配」という言葉は、回帰分析で傾き（slope）を意味するのと同じ数学的概念です。回帰では損失関数を微分して最適な係数を求めましたが、XGBoostでも同様に損失関数の勾配を使って「次の木がどの方向に改善すべきか」を計算します。残差ではなく勾配を使うことで、対数損失（分類）や二乗誤差（回帰）など任意の損失関数に対応できます。

XGBoostは設定項目が多いため、各パラメータの意味を整理しておきます。

18.4 SHAP（全体と個別の解釈）

18.4.1 SHAPとは何か

ロジスティック回帰では、係数（\(\beta\)）が「各変数の寄与」を表していました。たとえば \(\beta_\text{proficiency} = 1.2\) なら、「習熟度が1単位上がると対数オッズが1.2増える」と読めます。しかしツリーモデルには係数がありません。SHAPはツリーモデルでも「この予測に各変数がいくらずつ貢献したか」を分解する手法です(Lundberg ほか, 2020; Lundberg & Lee, 2017)。

考え方の核は、ゲーム理論のShapley値に基づいています。「チーム（全特徴）の成果（予測）を、各メンバー（特徴）に公平に分配するにはどうすればよいか」を計算します。具体例で説明しましょう:

  • テストデータ全体の平均正答確率が 0.55 とする。
  • ある学習者（Case A）の予測が 0.85 なら、0.30 の「上積み」がある。
  • SHAPはこの 0.30 を「proficiency が +0.15、word_freq が +0.08、anxiety が +0.05、…」のように各特徴に分配する。

この分解により、「なぜこの学習者は正答と予測されたのか」を特徴ごとに説明できます。

18.4.2 論文での活用例

先行研究マップの文献を、本章の実装に対応づけます:

  • Dominance（Budescu (1993); Azen & Budescu (2003) 系）: GLMベースラインで使用済み。全サブモデル比較で変数の相対的重要性を算出。本章の glm-dominance-analysis に対応。
  • SHAP local → global（Lundberg & Lee (2017) 系）: 1件の予測をベースライン＋寄与に分解し、多数集約して全体理解へ進む。Lundberg ほか (2020) は医療データで希少だが高リスクな要因を抽出。本章の xgb-shap-local → xgb-shap-global に対応。
  • 教育データでの読み替え: たとえばある学習者のlocal SHAPで test_time_delayed が負、proficiency が正なら、「遅延テスト条件は不利だが、習熟度がそれを一部補っている」と解釈できます。

19.3 報告の指針

4手法は「異なる質問に答える異なる道具」です。報告時は次のように使い分けます:

  • GLMを主モデルとして使う場合: Dominance analysisを主報告。係数表と組み合わせて相対的重要性を議論する。
  • ツリーモデルを主モデルとして使う場合: SHAPを主報告（理論的公平性が高い）。Permutation importanceを補助として併記し、Gain importanceは参考に留める(Strobl ほか, 2007)。
  • 複数手法の合意: 全手法で上位に来る変数（例: proficiency, word_freq）は頑健に重要と判断できる。手法間で順位が異なる変数は、「どの手法を信頼するか」の根拠を明記する。

20.5 ROC曲線の直感

ROC曲線がどうやって作られるかを直感的に理解しておきましょう。

  1. 閾値を 1.0（全員を incorrect と判定）に設定する → TPR = 0, FPR = 0（左下の点）。
  2. 閾値を少しずつ下げる → correct と判定される人が増え、TPR（感度）が上がる。同時にFPR（偽陽性率）も上がる。
  3. 閾値を 0.0（全員を correct と判定）にする → TPR = 1, FPR = 1（右上の点）。
  4. 各閾値で得られた (FPR, TPR) の点をつないだ曲線がROC曲線です。

AUCの直感的意味: ランダムに正答者1人と誤答者1人を選んだとき、モデルが正答者に高い確率を与える割合。AUC = 1.0 なら完璧な識別、AUC = 0.5 ならランダムと同等です(Fawcett, 2006)。

21 閾値をどう決めるか（ROCを運用に接続）

最初にここだけ（閾値）

AUCは閾値なしの比較指標。実務では1つの閾値を決めて「介入する/しない」を判断する。閾値の選び方は目的次第: バランス型（Youden）、見逃し回避型（低閾値）、誤警報回避型（高閾値）。

AUCは「ランク付けの上手さ」を示しますが、現場では最終的に閾値が必要です。ここが「研究結果を実務判断へ変換する」一番重要な節です。

21.1 閾値選択の3つのアプローチ

閾値の決め方は目的によって異なります。以下の3つが代表的です。

Three Approaches to Threshold Selection
Choose based on your educational purpose
アプローチ 計算方法 適する場面 教育での例
Youden's J（バランス型） Sensitivity + Specificity − 1 を最大化 感度と特異度のバランスが重要な場面 研究報告のデフォルト
コストベース FNコスト × FN + FPコスト × FP を最小化 見逃しと誤警報のコストが非対称な場面 語彙スクリーニング（見逃しが高コスト）

21.4 教育場面での閾値判断の具体例

  • 語彙習得スクリーニング（学習困難リスクの早期検出）: 見逃し（FN）のコストが高い。学習困難な生徒を見逃すと支援が遅れる。→ 低閾値でSensitivityを優先。FP（過剰検出）は追加テストで確認すればよい。
  • 進級判定（合格基準の設定）: 誤判定（FP）で基準未達の学習者を通過させるコストが高い。→ 高閾値でSpecificity/Precisionを優先。
  • 研究報告（一般的なモデル比較）: 特定のコスト構造を仮定しない。→ Youden’s J でバランスをとり、他の閾値候補も感度分析として併記する。

先ほどの3ケースで考えると、Case A（高習熟+有利条件）とCase B（低習熟+不利条件）は閾値の影響を受けにくい（どちらもほぼ確実に一方に分類される）。Case C（中習熟+中間条件）がボーダーラインであり、閾値の選択によって判定が変わる典型的な学習者です。

22 キャリブレーション確認（確率をそのまま信じてよいか）

最初にここだけ（キャリブレーション）

予測確率が0.80なら、実際に80%の人が正答していてほしい。この一致度がキャリブレーション。AUCが高くてもキャリブレーションが悪いことがある。

高いAUCでも、確率が歪んでいる場合があります。教育現場で「0.8の確率で正答」のように使うなら、校正の確認が必要です。

なぜキャリブレーションが重要か: AUCは「順位の正しさ」（正答者に高い確率を与えるか）を測りますが、「確率そのものの正しさ」は保証しません。天気予報の比喩で言えば、「降水確率70%」と言った日のうち実際に雨が降ったのが70%なら、キャリブレーションが良いと言えます。教育現場で「この生徒は80%の確率で基準を超える」と伝えるなら、その80%が信頼できるかの確認が不可欠です。

特に、学習支援の優先順位づけに確率を使う場合は、この確認を省略しないのが安全です。

22.1 Brier scoreの解釈

Brier scoreは予測確率の精度を定量化する指標です:

\[ \text{Brier} = \frac{1}{N}\sum_{i=1}^{N}(p_i - y_i)^2 \]

ここで \(p_i\) は予測確率、\(y_i\) は実際の結果（0 or 1）です。0が完璧（予測確率と実際が完全一致）で、ベースレートが50%のときのランダム予測は0.25です。先ほどの指標テーブルでBrierが低いモデルほどキャリブレーションが良い傾向にあります。

キャリブレーション改善の選択肢

本章ではキャリブレーションの確認までを行いますが、系統的なずれがある場合の改善手法も存在します。代表的なものとしてPlatt scaling（ロジスティック変換による後処理）やisotonic regression（単調回帰による後処理）があり、Rでは probably パッケージで実装可能です。詳細は本章の範囲外としますが、確率値を直接使う運用では検討に値します(Steyerberg ほか, 2010)。

23.3 限界と今後の課題

  • 外的妥当性: 本章のデータは架空の生成データであり、実データでの再現が必要。
  • 多水準構造の簡略化: 本章では learner_id と item_id の変量効果を含めていない。実際のデータでは、6章で扱ったGLMMとの比較が必要になる場合がある。
  • 公平性: 学習者の背景（L1, 性別等）によって予測精度が異なる可能性がある。サブグループ別の性能評価を行うことが望ましい。
  • 因果推論との区別: ツリーモデルは予測に優れるが、「retrieval conditionが正答率を因果的に高める」とは直接言えない。因果推論には別の枠組み（例: 傾向スコア、差の差）が必要。

24 方法論上の留意点

留意点1: AUC単独でモデル選択を行わない
  • 対応: PR-AUC、閾値依存指標（Sens/Spec/Precision）も併記する。
留意点2: 目的変数のイベント水準を明示する
  • 対応: event_level = "second" などを明示し、再現可能性を確保する。
留意点3: 前処理リークを防止する
  • 対応: recipeの学習は学習データのみに限定し、workflowで一体化する。
留意点4: 複雑モデルを無条件に採用しない
  • 対応: 単純モデル（GLM/単木）を必ず比較対象に含める。
留意点5: カテゴリカル変数の重要度に注意する
  • 対応: 水準数が多いカテゴリカル変数は、Gini/Gainベースの重要度で過大評価されやすい(Gries, 2020)。Permutation importanceやSHAPと併用する。
留意点6: Gain重要度のカーディナリティバイアス
  • 対応: XGBoostのGain重要度は分岐回数に依存するため、連続変数を優遇しやすい。SHAPと併記して頑健性を確認する(Strobl ほか, 2007)。
留意点7: サンプルサイズとモデル複雑度のバランス
  • 対応: サンプルサイズが小さい場合（例: N < 500）、ツリーモデルよりGLMの方が安定することがある。データ量に見合ったモデル複雑度を常に検討する。

25 まとめ

最初にここだけ（まとめ）

回帰→決定木→RF→XGBoostの順にモデルの複雑度を上げ、ROC/PR/閾値/キャリブレーションで評価した。ツリーモデルは非線形・相互作用に強いが、ROC-AUC単独ではなく複数指標で判断し、閾値は教育目的に合わせて選ぶ。

本章の主要な結論は次の通りである。

  1. ツリーモデルは非線形・相互作用を自動的に捉える: ロジスティック回帰では手動で交互作用項を指定する必要があるが、ツリーモデルはデータ駆動で最適な分割を発見する。英語教育データのように、学習者要因と項目要因が複雑に絡む場面で特に有用である。

  2. ROCは有用だが、ROCだけでは不十分: PR-AUC（陽性クラスが少ないとき特に重要）、Brier score（確率の信頼性）、閾値依存指標（Sensitivity/Specificity）を組み合わせて初めて、モデルの実力を多面的に評価できる。

  3. 実務判断には閾値とキャリブレーションが必要: AUCは「モデルの潜在能力」を示すが、教育現場で「この学習者に介入する/しない」を決めるには閾値が必要。その閾値は教育目的（見逃し回避 vs 誤警報回避）に合わせて選ぶ。確率値を直接使う場合はキャリブレーションの確認が不可欠。

  4. モデル選択は目的に依存する: 精度だけでなく、解釈可能性、計算コスト、チューニング負荷を考慮する。以下のフローチャートを参考にされたい:

  • データが少ない（< 500行）→ GLMで十分な場合が多い。
  • 非線形・相互作用が複雑 → RF（安定性重視）or XGBoost（精度重視）。
  • 予測根拠の透明性が最優先 → 決定木。
  • 最高精度を追求し、解釈にはSHAPを使う → XGBoost + SHAP。

# はじめに

5章・6章で扱ったロジスティック回帰では、「ある説明変数が正答確率にどう影響するか」を1本のグローバルな式でモデル化しました。しかし英語教育データでは、学習者要因と語彙項目要因の**複雑な組み合わせパターン**が頻出します。たとえば、習熟度が高い学習者は retrieval practice 条件で特に効果が大きいが、低習熟者ではその差が消える、といったパターンです。回帰分析でこうしたパターンを捉えるには交互作用項を手動で指定する必要がありますが、どの変数の組み合わせが重要かを事前に知ることは難しいのが実情です。

**ツリーモデル**（決定木・ランダムフォレスト・XGBoost）は、こうした非線形関係や相互作用を、分析者が事前に指定しなくても自動的に発見できる手法群です。本章では、これら3つのツリーモデルを同じ教育データで比較し、モデルの性能評価に**ROC曲線とAUC**を用います。

英語教育研究で想定される予測課題の例:

- 語彙テスト項目の正誤（正解/不正解）を予測する
- 学習者が次の小テストで基準を超えるかを予測する
- CEFR到達判定の補助指標を構築する

本章を通して、「**どの学習者が、どの語彙項目を、どの条件で正答するか**」を予測し、その予測を教育判断に変換する一連の流れを体系的に学びます。

- **ツリーモデル** = データを「もし proficiency > 0.4 ならば…」のような条件分岐で繰り返し分割し、局所ごとに予測を出す方法。回帰分析が1本の式で全データを説明するのに対し、ツリーモデルはデータの「区画」ごとに異なる予測を出す。
- **ROC-AUC** = ランダムに正答者と誤答者を1人ずつ選んだとき、モデルが正答者に高い確率を正しく与える割合。1.0が完璧、0.5がランダム。

## 回帰分析から機械学習への橋渡し

読者の多くは、5章・6章でロジスティック回帰を学んでいるはずです。ツリーモデルに進む前に、「なぜ回帰だけでは不十分な場面があるのか」「ツリーモデルは何が本質的に異なるのか」を整理します。この節を理解すると、以降の決定木・RF・XGBoostの位置づけがはっきりします。

### 回帰分析の限界を直感で理解する

ロジスティック回帰は、すべてのデータに対して**1本のグローバルな式**をあてはめます。

$$
\log\frac{P(\text{correct})}{1-P(\text{correct})} = \beta_0 + \beta_1 x_1 + \beta_2 x_2 + \cdots
$$

この式は、$x_1$（たとえば習熟度）が1単位増えると対数オッズが $\beta_1$ だけ変化する、という**全データに共通のルール**を仮定しています。しかし教育データでは、「習熟度が高い層では retrieval practice が特に効く」「低頻度語では語長の影響が大きくなる」のように、**ある条件下でだけ効く変数**が少なくありません。

回帰分析でも交互作用項（`proficiency:condition`）を手動で追加すれば対応可能ですが、どの交互作用を入れるかは分析者が事前に決める必要があります。変数が10個あれば交互作用の候補は45通り、3次の交互作用まで含めればさらに膨大になります。

ツリーモデルは、この交互作用の発見を**データから自動的に**行います。

以下の図は、同じデータに対して「回帰の線形フィット」と「ツリーの矩形分割」がどう異なるかを示しています。コードでは (1) 非線形な真の関係を持つデータを生成し、(2) ロジスティック回帰でフィットし、(3) 決定木でフィットして、3パネルの比較図を作成します。

```{r}
#| label: regression-vs-tree-intuition
#| fig-width: 10
#| fig-height: 4
#| fig-cap: "Regression fits a single global curve; a tree partitions the feature space into local regions"

set.seed(42)
demo_dat <- tibble(
  x = seq(-3, 3, length.out = 200),
  y_true = plogis(1.5 * sin(2 * x) + 0.3 * x),
  y_obs = rbinom(200, 1, y_true)
)

# (A) 真の非線形関係
p1 <- demo_dat |>
  ggplot(aes(x, y_true)) +
  geom_line(linewidth = 1.2, color = "#2C7FB8") +
  geom_point(aes(y = y_obs), alpha = 0.15, size = 0.8) +
  labs(title = "(A) True relationship", x = "x", y = "P(correct)") +
  ylim(0, 1)

# (B) ロジスティック回帰（線形）
glm_demo <- glm(y_obs ~ x, data = demo_dat, family = binomial())
demo_dat$glm_pred <- predict(glm_demo, type = "response")

p2 <- demo_dat |>
  ggplot(aes(x, y_true)) +
  geom_line(linewidth = 0.6, color = "gray60", linetype = "dashed") +
  geom_line(aes(y = glm_pred), linewidth = 1.2, color = "#D95F02") +
  geom_point(aes(y = y_obs), alpha = 0.15, size = 0.8) +
  labs(title = "(B) Logistic regression", x = "x", y = "P(correct)") +
  ylim(0, 1)

# (C) ツリー分割
tree_demo <- rpart::rpart(y_obs ~ x, data = demo_dat, method = "class",
                          control = rpart::rpart.control(maxdepth = 3, cp = 0.01))
demo_dat$tree_pred <- predict(tree_demo, type = "prob")[, "1"]

p3 <- demo_dat |>
  ggplot(aes(x, y_true)) +
  geom_line(linewidth = 0.6, color = "gray60", linetype = "dashed") +
  geom_step(aes(y = tree_pred), linewidth = 1.2, color = "#1B9E77") +
  geom_point(aes(y = y_obs), alpha = 0.15, size = 0.8) +
  labs(title = "(C) Decision tree", x = "x", y = "P(correct)") +
  ylim(0, 1)

library(patchwork)
p1 + p2 + p3
```

- **(A)** 真の関係は非線形のS字曲線。
- **(B)** ロジスティック回帰は1本の単調なシグモイドで近似するため、波形を捉えきれない。
- **(C)** 決定木は区間ごとに異なる確率を割り当て、局所的なパターンにフィットする。

::: {.callout-tip}
## 回帰とツリーの本質的な違い
- **回帰**: 全データに1本の式（グローバル）。係数を読めば「何が効くか」がわかる。
- **ツリー**: データを条件分岐で分割し、各領域で別の予測を出す（ローカル）。分岐規則を読めば「どの条件で効くか」がわかる。
:::

### バイアスとバリアンスのトレードオフ

モデル選択を理解するうえで最も重要な概念が、**バイアス・バリアンスのトレードオフ**です[@hastie2009; @james2021]。

- **バイアス（bias）**: モデルが現実を単純化しすぎることによる系統的な誤差。ロジスティック回帰が非線形な関係を直線的に近似するとき、これが生じる。
- **バリアンス（variance）**: データが変わるとモデルの予測が大きく変わる不安定さ。深い決定木はデータの細部まで追いかけるため、別のサンプルでは異なる構造になりやすい。

```{r}
#| label: bias-variance-tradeoff
#| fig-width: 8
#| fig-height: 5
#| fig-cap: "Bias-variance tradeoff: as model complexity increases, training error decreases monotonically, but test error first decreases then increases"

bv_dat <- tibble(
  complexity = seq(0.5, 10, length.out = 100),
  train_error = 0.45 * exp(-0.5 * complexity) + 0.02,
  test_error  = 0.45 * exp(-0.5 * complexity) + 0.02 + 0.005 * complexity^1.8,
  bias2       = 0.45 * exp(-0.5 * complexity),
  variance    = 0.005 * complexity^1.8
)

bv_dat |>
  ggplot(aes(x = complexity)) +
  geom_line(aes(y = train_error, color = "Training error"), linewidth = 1) +
  geom_line(aes(y = test_error, color = "Test error"), linewidth = 1) +
  geom_line(aes(y = bias2, color = "Bias²"), linewidth = 0.8, linetype = "dashed") +
  geom_line(aes(y = variance, color = "Variance"), linewidth = 0.8, linetype = "dashed") +
  geom_vline(xintercept = 3.5, linetype = "dotted", color = "gray40") +
  annotate("text", x = 3.7, y = 0.42, label = "Optimal complexity", hjust = 0, size = 3.5) +
  scale_color_manual(values = c(
    "Training error" = "#2C7FB8",
    "Test error"     = "#D95F02",
    "Bias²"          = "#7570B3",
    "Variance"       = "#E7298A"
  )) +
  labs(
    title = "Bias-Variance Tradeoff",
    subtitle = "Underfitting (left) vs. Overfitting (right)",
    x = "Model complexity",
    y = "Error",
    color = NULL
  ) +
  theme(legend.position = "bottom")
```

この図の読み方:

- **左側（低複雑度）**: モデルが単純すぎてデータのパターンを拾えない（**未学習、バイアスが大きい**）。
- **右側（高複雑度）**: モデルがデータのノイズまで拾ってしまう（**過学習、バリアンスが大きい**）。
- **最適点**: テスト誤差が最小になる複雑度。この点を交差検証（後述）で探すのが本章の基本戦略。

**数値で実感する**: 仮に語彙テストの正答率予測で以下のような結果が出たとします。

| モデル | 訓練データ正答率 | テストデータ正答率 | 症状 |
|--------|:---:|:---:|------|
| 浅い決定木（深さ2） | 67% | 65% | バイアス大（両方低い＝パターンを捉えきれていない） |
| 深い決定木（深さ10） | 95% | 60% | バリアンス大（訓練とテストの差＝ノイズを学習） |
| ランダムフォレスト | 78% | 73% | バギングでバリアンスを抑制 |
| XGBoost | 80% | 75% | ブースティングでバイアスも改善 |

訓練データとテストデータの正答率が**両方とも低い**ならバイアスの問題、**訓練だけ高くテストが低い**ならバリアンスの問題です。

本章の3つのツリーモデルは、このトレードオフの異なる位置にいます:

- **決定木**: 深くすれば表現力は高いが、バリアンスが大きい（データの揺れに弱い）。
- **ランダムフォレスト**: 多数の木の多数決（バギング）でバリアンスを抑制する。
- **XGBoost**: 前の木の誤差を次の木が修正（ブースティング）し、バイアスを段階的に減らす。

::: {.callout-important}
## 3モデルの位置づけ（バイアス・バリアンスの視点）
| モデル | 主な戦略 | 効果 |
|--------|----------|------|
| 決定木 | 1本の木で分割 | 解釈しやすいが不安定 |
| ランダムフォレスト | 独立な木の多数決（バギング） | バリアンスを削減 |
| XGBoost | 前段の残差を逐次修正（ブースティング） | バイアスを削減 |
:::

### ツリーモデルの分岐ロジック：再帰分割

決定木がデータを分ける仕組みを、小さな例で追ってみましょう。

6人の学習者のデータがあるとします。目的変数は正答/誤答、説明変数は `proficiency`（習熟度）です。

```{r}
#| label: manual-split-example

mini_split <- tibble(
  learner = LETTERS[1:6],
  proficiency = c(-1.2, -0.5, 0.1, 0.4, 0.9, 1.5),
  correct = factor(c("incorrect", "incorrect", "incorrect", "correct", "correct", "correct"),
                   levels = c("incorrect", "correct"))
)

mini_split |>
  gt() |>
  tab_header(
    title = "Mini Example: 6 Learners",
    subtitle = "Where should the tree split proficiency?"
  )
```

木は「どこで分けると、分けた後のグループが最も"純粋"になるか」を探します。**Gini不純度**で純粋さを測ります:

$$
G = 1 - \sum_{k=1}^{K} p_k^2
$$

ここで $p_k$ はノード内のクラス $k$ の割合です。$G = 0$（完全に1クラス）が最も純粋で、$G = 0.5$（2クラスが半々）が最も不純です。

```{r}
#| label: gini-calculation

gini <- function(labels) {
  p <- table(labels) / length(labels)
  1 - sum(p^2)
}

# 分割前のGini
gini_before <- gini(mini_split$correct)

# 候補分割点: proficiency = 0.25（CとDの間）
left  <- mini_split |> filter(proficiency <= 0.25)
right <- mini_split |> filter(proficiency > 0.25)
gini_after <- (nrow(left) / 6) * gini(left$correct) +
              (nrow(right) / 6) * gini(right$correct)

tibble(
  Step = c("Before split (all 6)", "Left (prof ≤ 0.25)", "Right (prof > 0.25)", "Weighted Gini after split"),
  Gini = c(gini_before, gini(left$correct), gini(right$correct), gini_after)
) |>
  mutate(Gini = round(Gini, 3)) |>
  gt() |>
  tab_header(
    title = "Gini Impurity Calculation",
    subtitle = "Split at proficiency = 0.25"
  )
```

分割前のGiniは 0.500（2クラスが半々）。`proficiency = 0.25` で分けると、左は全員 incorrect（$G = 0$）、右は全員 correct（$G = 0$）となり、加重Giniは 0.000 です。これが**最も純粋な分割**です。

実際の決定木アルゴリズム（CART）は、すべての変数×すべての分割点でこのGini改善量を計算し、最大の改善をもたらす分割を選びます。これを再帰的に繰り返し、停止条件（最小サンプル数、最大深さなど）に達するまで分岐を続けます[@breiman1984]。

::: {.callout-note}
## 回帰との対比
- **ロジスティック回帰**: 全データに対して $\beta$ を最適化する（1つのグローバルな最適化問題）。
- **決定木**: 各ノードで局所的に最良の分割を選ぶ（貪欲な局所最適化の繰り返し）。
:::

### 交差検証の直感的理解

ツリーモデルのハイパーパラメータ（木の深さ、学習率など）を決めるために、本章では**交差検証（Cross-Validation; CV）**を使います。CVの仕組みを先に押さえておきましょう。

```{r}
#| label: cv-schematic
#| fig-width: 8
#| fig-height: 4
#| fig-cap: "K-fold cross-validation schematic (K=3)"

cv_dat <- expand_grid(
  fold = paste0("Iteration ", 1:3),
  block = 1:3
) |>
  mutate(
    role = case_when(
      fold == "Iteration 1" & block == 1 ~ "Test",
      fold == "Iteration 2" & block == 2 ~ "Test",
      fold == "Iteration 3" & block == 3 ~ "Test",
      TRUE ~ "Train"
    ),
    label = paste0("Fold ", block)
  )

cv_dat |>
  ggplot(aes(x = label, y = fold, fill = role)) +
  geom_tile(color = "white", linewidth = 1.5) +
  geom_text(aes(label = role), size = 4, fontface = "bold") +
  scale_fill_manual(values = c("Train" = "#2C7FB8", "Test" = "#D95F02")) +
  labs(
    title = "3-Fold Cross-Validation",
    subtitle = "Each fold takes a turn as the test set; results are averaged",
    x = NULL, y = NULL, fill = NULL
  ) +
  theme(
    legend.position = "bottom",
    panel.grid = element_blank()
  )
```

CVの手順:

1. データをK個に分割する（本章ではK=3）。
2. 1つ目の分割をテスト用に取っておき、残りで学習する。
3. 2つ目、3つ目でも同様に繰り返す。
4. K回の結果を平均して、モデルの汎化性能を推定する。

この方法の利点は、テストデータを「使い切る前に」汎化性能を安定的に推定できる点です。ハイパーパラメータのチューニング（後述の `tune_grid`）はすべてこのCV上で行い、最終評価は別に確保したテストデータで行います。

::: {.callout-tip}
## なぜ3-foldなのか
本章はチュートリアルのため計算時間を短縮する目的で3-foldを採用しています。実際の研究では **5-fold または10-fold** が一般的です[@james2021]。
:::

この章では、次のように役割分担して読みます。

:::

## 混同行列の確認（最小例）

ここで一度だけ、ROCの土台になる混同行列を小さな例で確認します。

```{r}
#| label: mini-confusion-threshold

# 以降の章でも使える、閾値ごとの基本指標関数
evaluate_threshold <- function(truth, prob, threshold) {
  pred <- factor(
    if_else(prob >= threshold, "correct", "incorrect"),
    levels = levels(truth)
  )

  tp <- sum(pred == "correct" & truth == "correct")
  fp <- sum(pred == "correct" & truth == "incorrect")
  tn <- sum(pred == "incorrect" & truth == "incorrect")
  fn <- sum(pred == "incorrect" & truth == "correct")

  tibble(
    threshold = threshold,
    TP = tp, FP = fp, TN = tn, FN = fn,
    Sensitivity = tp / (tp + fn),
    Specificity = tn / (tn + fp),
    Precision = tp / (tp + fp)
  )
}

mini_dat <- tibble(
  truth = factor(
    c("correct", "incorrect", "correct", "incorrect", "correct", "incorrect"),
    levels = c("incorrect", "correct")
  ),
  pred_prob = c(0.90, 0.80, 0.63, 0.56, 0.42, 0.10)
)

bind_rows(
  evaluate_threshold(mini_dat$truth, mini_dat$pred_prob, 0.40),
  evaluate_threshold(mini_dat$truth, mini_dat$pred_prob, 0.60)
) |>
  mutate(across(where(is.numeric), ~ round(., 3))) |>
  gt() |>
  tab_header(
    title = "Mini Threshold Example",
    subtitle = "Lower threshold often improves sensitivity but increases false positives"
  )
```

この表の読み方:

- 閾値を下げる（0.60 → 0.40）と、拾える正解者（Sensitivity）は上がりやすい。
- その代わり、誤って正解判定する人数（FP）が増えることがある。

## 補助解析：Dominance Analysis（GLM）

係数表は「方向と有意性」を示すのに有用ですが、変数間でスケールが異なるため「どの変数が最も重要か」の比較は直接できません。標準化係数を使う方法もありますが、変数間の相関が強い場合は不安定です。Dominance analysisは、すべてのサブモデルの組み合わせを比較することでこの問題を解決します[@budescu1993; @azen2003; @azen2009]。

文献での活用ポイント（具体）:

- @azen2009 は、ロジスティック回帰で dominance analysis を使うために、
  R2類似指標の選定とブートストラップ推論を組み合わせて、重要度の再現性を検討しました。
- @mizumoto2022 は、応用言語学データの再解析で dominance analysis を併用し、
  「標準化βだけでは見えにくい相対的重要性」を示しました。  
- 本章での具体例としては、`proficiency` と `condition` の relative share を比較し、  
  「学習者要因と授業デザイン要因のどちらが予測改善に効いているか」を同じ尺度で読む、という使い方になります。  

以下のコードでは、(1) 通常のglmオブジェクトを作成し、(2) `dominanceanalysis`パッケージで全てのサブモデルの組み合わせにおける各変数の平均的な改善量を計算し、(3) Nagelkerke R2ベースの結果を表にします。

# 決定木（Decision Tree）

::: {.callout-tip}
## 最初にここだけ（決定木）
決定木は「もし proficiency > 0.4 ならば…」のようなif-thenルールの積み重ね。1本見れば予測根拠が明確にわかる。ただし1本の木はデータの揺れに弱い（高バリアンス）。
:::

この節の目的は、解釈可能性の高い木モデルを構築し、分岐規則を確認することです。

::: {.callout-note}
## 決定木の基本構造
- 決定木は、データを「条件分岐」で繰り返し分けるモデルです。  
- 分岐点（ノード）では、例: `proficiency > 0.4` のような質問をします。  
- 最後の葉（leaf）で、正答確率やクラス（correct / incorrect）を出します。  
- 良い点: ルールが可視化でき、教育的解釈がしやすい。  
- 注意点: 1本の木はデータの小さな揺れに影響されやすい。  
:::

英語教育の文脈では、「どの条件の学習者が正答しやすいか」をif-thenルールで説明できる点が、決定木の大きな利点です。分岐ロジックの詳細（Gini不純度による分割選択）は、先の「回帰から機械学習への橋渡し」で解説しました。ここではその仕組みを実際の教育データに適用し、ハイパーパラメータのチューニングから可視化までを行います。

## モデルとチューニング

まずは「深すぎる木を避ける」ことを意識して、単純な探索から始めます。

`grid_regular`は各ハイパーパラメータの範囲を等間隔に区切ってグリッドを作ります。`levels = c(3, 2, 2)`は、cost_complexityを3段階、tree_depthを2段階、min_nを2段階に設定し、計3x2x2=12通りの組み合わせを試すという意味です。

```{r}
#| label: tune-decision-tree

  cost_complexity = tune(),
```

決定木の最大の強みは**透明性**です。上の図を見れば、「proficiency が高い → さらに word_freq が大きい → 正答確率が高い」のように、予測の根拠をif-thenルールとして直接読み取れます。これは教育現場で「なぜこの学習者が正答/誤答と判定されたのか」を説明する際に非常に有用です。

一方、単木の限界も明確です:

- **高バリアンス**: データの一部を変えるだけで、木の構造が大きく変わることがある。

したがって、決定木は「予測の仕組みを説明するモデル」として使い、精度を求める場合は次節以降のアンサンブル手法を検討します。

# ランダムフォレスト（Random Forest）

::: {.callout-tip}
## 最初にここだけ（ランダムフォレスト）
不安定な単木をたくさん作って多数決する。各木は別々のデータ（ブートストラップ）と別々の変数（ランダム選択）で育てるので、木どうしが似すぎない。結果としてバリアンスが大幅に下がる。
:::

この節の目的は、単木の弱点（不安定性）をアンサンブルで補うことです[@breiman2001]。

## バギングの原理

ランダムフォレストの土台は**バギング（Bootstrap Aggregating）**です。手順は以下の通りです:

1. 元データからブートストラップ標本（復元抽出）を作る。
2. その標本で1本の決定木を育てる。
3. 1-2をB回繰り返す（本章ではB = 300）。
4. 全B本の木の予測を**多数決**（分類の場合）または**平均**（回帰の場合）で統合する。

比喩で言えば、「多数の専門家に別々の資料を渡し、独立に判断してもらって多数決をとる」イメージです。1人の判断は偏りやすくても、独立した多数の専門家の多数決は安定します。本章では300本の木（`trees = 300`）を使用します。

ランダムフォレストはバギングに加えて、各分岐で**変数のランダム選択**（mtry個だけ候補にする）を行います。これにより、強い変数（例: proficiency）がすべての木で最初に選ばれる状況を防ぎ、木どうしの**相関を下げて**バリアンス削減効果を高めます[@breiman2001]。

## 変数重要度

ここは「どの要因が予測に効いていたか」を説明する節です。ランダムフォレストでは**Permutation importance**（並べ替え重要度）を使います。原理は次の通りです:

1. テストデータ（またはOOBデータ）で元の予測精度を記録する。
2. ある変数の値をランダムにシャッフルして、予測精度を再計算する。
3. 精度の**低下幅**がその変数の重要度になる。

大きく低下する変数は「その変数がないと予測が壊れる」＝重要な変数です。Gini不純度ベースの重要度（impurity importance）とは異なり、permutation importanceはカテゴリカル変数のカーディナリティに対するバイアスが小さいとされています[@strobl2007]。

# XGBoost

::: {.callout-tip}
## 最初にここだけ（XGBoost）
XGBoostは「前の木が間違えた部分を、次の木が重点的に修正する」逐次学習法。ランダムフォレストが木を並列に育てるのに対し、XGBoostは**直列に**育てて誤差を段階的に減らす。強力だが設定項目が多い。
:::

この節の目的は、高精度モデルを構築しつつ、重要特徴を解釈可能な形で確認することです[@chen2016]。

## ブースティングの原理：バギングとの対比

ランダムフォレスト（バギング）とXGBoost（ブースティング）は、アンサンブルの組み立て方が根本的に異なります。

ブースティングの手順を、比喩を使って説明します:

1. **1回目の家庭教師**が全体を見て、「習熟度が低い学習者は苦手」と大まかに判断する（浅い木）。
2. **2回目の家庭教師**は、1回目が見逃した部分（例: 同根語なのに間違えた高習熟者）を重点的に指導する。
3. **3回目**はさらに細かい見逃しを拾う。
4. 最終予測は、全家庭教師の判断を**学習率（learn_rate）で縮小しながら足し合わせた**もの。

数式で書くと、$m$回目までのモデルの予測は:

$$
\hat{f}_m(x) = \hat{f}_{m-1}(x) + \eta \cdot h_m(x)
$$

ここで $h_m(x)$ は $m$回目の木の予測、$\eta$ は学習率（0〜1）です。学習率を小さくすると各木の寄与が控えめになり、より多くの反復が必要になりますが、過学習しにくくなります[@friedman2001]。

::: {.callout-note}
## XGBoostの「勾配」とは
XGBoostの正式名称は eXtreme Gradient Boosting です。「勾配」という言葉は、回帰分析で傾き（slope）を意味するのと同じ数学的概念です。回帰では損失関数を微分して最適な係数を求めましたが、XGBoostでも同様に損失関数の勾配を使って「次の木がどの方向に改善すべきか」を計算します。残差ではなく勾配を使うことで、対数損失（分類）や二乗誤差（回帰）など任意の損失関数に対応できます。
:::

## SHAP（全体と個別の解釈）

### SHAPとは何か

ロジスティック回帰では、係数（$\beta$）が「各変数の寄与」を表していました。たとえば $\beta_\text{proficiency} = 1.2$ なら、「習熟度が1単位上がると対数オッズが1.2増える」と読めます。しかしツリーモデルには係数がありません。**SHAP**はツリーモデルでも「この予測に各変数がいくらずつ貢献したか」を分解する手法です[@lundberg2017; @lundberg2020]。

考え方の核は、ゲーム理論のShapley値に基づいています。「チーム（全特徴）の成果（予測）を、各メンバー（特徴）に公平に分配するにはどうすればよいか」を計算します。具体例で説明しましょう:

- テストデータ全体の平均正答確率が 0.55 とする。
- ある学習者（Case A）の予測が 0.85 なら、0.30 の「上積み」がある。
- SHAPはこの 0.30 を「proficiency が +0.15、word_freq が +0.08、anxiety が +0.05、…」のように各特徴に分配する。

この分解により、「**なぜ**この学習者は正答と予測されたのか」を特徴ごとに説明できます。

### 論文での活用例

先行研究マップの文献を、本章の実装に対応づけます:

- **Dominance**（@budescu1993; @azen2003 系）: GLMベースラインで使用済み。全サブモデル比較で変数の相対的重要性を算出。本章の `glm-dominance-analysis` に対応。
- **SHAP local → global**（@lundberg2017 系）: 1件の予測をベースライン＋寄与に分解し、多数集約して全体理解へ進む。@lundberg2020 は医療データで希少だが高リスクな要因を抽出。本章の `xgb-shap-local` → `xgb-shap-global` に対応。
- **教育データでの読み替え**: たとえばある学習者のlocal SHAPで `test_time_delayed` が負、`proficiency` が正なら、「遅延テスト条件は不利だが、習熟度がそれを一部補っている」と解釈できます。

### 全体: どの特徴が平均的に効いたか

SHAPの計算は3ステップで進みます: (1) テストデータをXGBoostが受け取れる行列形式に変換する、(2) `predcontrib = TRUE`オプションで各観測×各変数のSHAP値を取得する、(3) 全観測にわたる絶対値の平均（Mean |SHAP|）を計算して全体的な重要度を算出する。

```{r}

#| fig-width: 9
#| fig-height: 6
#| fig-cap: "Global SHAP importance for XGBoost"
```

## ROC曲線の直感

ROC曲線がどうやって作られるかを直感的に理解しておきましょう。

1. 閾値を **1.0**（全員を incorrect と判定）に設定する → TPR = 0, FPR = 0（左下の点）。
2. 閾値を少しずつ**下げる** → correct と判定される人が増え、TPR（感度）が上がる。同時にFPR（偽陽性率）も上がる。
3. 閾値を **0.0**（全員を correct と判定）にする → TPR = 1, FPR = 1（右上の点）。
4. 各閾値で得られた (FPR, TPR) の点をつないだ曲線がROC曲線です。

**AUCの直感的意味**: ランダムに正答者1人と誤答者1人を選んだとき、モデルが**正答者に高い確率を与える割合**。AUC = 1.0 なら完璧な識別、AUC = 0.5 ならランダムと同等です[@fawcett2006]。

# 閾値をどう決めるか（ROCを運用に接続）

::: {.callout-tip}
## 最初にここだけ（閾値）

:::

AUCは「ランク付けの上手さ」を示しますが、現場では最終的に**閾値**が必要です。ここが「研究結果を実務判断へ変換する」一番重要な節です。

::: {.callout-tip}
## 実務ではコストベースで閾値を決める
- 早期介入モデルなら「見逃し（FN）コスト」が高い。
- その場合、Youden最適より低めの閾値にしてRecallを優先する設計がありえます。

## 教育場面での閾値判断の具体例

- **語彙習得スクリーニング**（学習困難リスクの早期検出）: 見逃し（FN）のコストが高い。学習困難な生徒を見逃すと支援が遅れる。→ **低閾値**でSensitivityを優先。FP（過剰検出）は追加テストで確認すればよい。
- **進級判定**（合格基準の設定）: 誤判定（FP）で基準未達の学習者を通過させるコストが高い。→ **高閾値**でSpecificity/Precisionを優先。
- **研究報告**（一般的なモデル比較）: 特定のコスト構造を仮定しない。→ **Youden's J** でバランスをとり、他の閾値候補も感度分析として併記する。

先ほどの3ケースで考えると、Case A（高習熟+有利条件）とCase B（低習熟+不利条件）は閾値の影響を受けにくい（どちらもほぼ確実に一方に分類される）。**Case C（中習熟+中間条件）がボーダーライン**であり、閾値の選択によって判定が変わる典型的な学習者です。

# キャリブレーション確認（確率をそのまま信じてよいか）

::: {.callout-tip}
## 最初にここだけ（キャリブレーション）
予測確率が0.80なら、実際に80%の人が正答していてほしい。この一致度がキャリブレーション。AUCが高くてもキャリブレーションが悪いことがある。
:::

高いAUCでも、確率が歪んでいる場合があります。教育現場で「0.8の確率で正答」のように使うなら、校正の確認が必要です。

**なぜキャリブレーションが重要か**: AUCは「順位の正しさ」（正答者に高い確率を与えるか）を測りますが、「確率そのものの正しさ」は保証しません。天気予報の比喩で言えば、「降水確率70%」と言った日のうち実際に雨が降ったのが70%なら、キャリブレーションが良いと言えます。教育現場で「この生徒は80%の確率で基準を超える」と伝えるなら、その80%が信頼できるかの確認が不可欠です。

特に、学習支援の優先順位づけに確率を使う場合は、この確認を省略しないのが安全です。

```{r}
#| label: calibration-plot
#| fig-width: 8
#| fig-height: 6
#| fig-cap: "Calibration plot for XGBoost (test set)"

xgb_calib <- xgb_pred |>
  mutate(bin = ntile(.pred_correct, 10L)) |>
  group_by(bin) |>
  summarise(
    pred_mean = mean(.pred_correct),
    obs_rate = mean(truth == "correct"),
    n = n(),
    .groups = "drop"
  )

xgb_calib |>
  ggplot(aes(x = pred_mean, y = obs_rate, size = n)) +
  geom_point(color = "#1B9E77", alpha = 0.85) +
  geom_abline(intercept = 0, slope = 1, linetype = "dashed", color = "gray40") +
  scale_size_continuous(range = c(2, 7)) +
  coord_equal() +
  labs(
    title = "Calibration Check",
    subtitle = "Dashed line = perfect calibration",

```

## キャリブレーション図の見方（簡易）
- 点が対角線より**上**: 実際の正答率が予測より高い（過小評価気味）。
- 点が対角線より**下**: 実際の正答率が予測より低い（過大評価気味）。
:::

## Brier scoreの解釈

Brier scoreは予測確率の精度を定量化する指標です:

$$
\text{Brier} = \frac{1}{N}\sum_{i=1}^{N}(p_i - y_i)^2
$$

ここで $p_i$ は予測確率、$y_i$ は実際の結果（0 or 1）です。**0が完璧**（予測確率と実際が完全一致）で、ベースレートが50%のときの**ランダム予測は0.25**です。先ほどの指標テーブルでBrierが低いモデルほどキャリブレーションが良い傾向にあります。

::: {.callout-note}
## キャリブレーション改善の選択肢
本章ではキャリブレーションの**確認**までを行いますが、系統的なずれがある場合の**改善手法**も存在します。代表的なものとしてPlatt scaling（ロジスティック変換による後処理）やisotonic regression（単調回帰による後処理）があり、Rでは `probably` パッケージで実装可能です。詳細は本章の範囲外としますが、確率値を直接使う運用では検討に値します[@steyerberg2010]。
:::

## 限界と今後の課題

- **外的妥当性**: 本章のデータは架空の生成データであり、実データでの再現が必要。

# まとめ

::: {.callout-tip}
## 最初にここだけ（まとめ）
回帰→決定木→RF→XGBoostの順にモデルの複雑度を上げ、ROC/PR/閾値/キャリブレーションで評価した。ツリーモデルは非線形・相互作用に強いが、ROC-AUC単独ではなく複数指標で判断し、閾値は教育目的に合わせて選ぶ。
:::

本章の主要な結論は次の通りである。

1. **ツリーモデルは非線形・相互作用を自動的に捉える**: ロジスティック回帰では手動で交互作用項を指定する必要があるが、ツリーモデルはデータ駆動で最適な分割を発見する。英語教育データのように、学習者要因と項目要因が複雑に絡む場面で特に有用である。

2. **ROCは有用だが、ROCだけでは不十分**: PR-AUC（陽性クラスが少ないとき特に重要）、Brier score（確率の信頼性）、閾値依存指標（Sensitivity/Specificity）を組み合わせて初めて、モデルの実力を多面的に評価できる。

3. **実務判断には閾値とキャリブレーションが必要**: AUCは「モデルの潜在能力」を示すが、教育現場で「この学習者に介入する/しない」を決めるには閾値が必要。その閾値は教育目的（見逃し回避 vs 誤警報回避）に合わせて選ぶ。確率値を直接使う場合はキャリブレーションの確認が不可欠。

4. **モデル選択は目的に依存する**: 精度だけでなく、解釈可能性、計算コスト、チューニング負荷を考慮する。以下のフローチャートを参考にされたい:

- データが少ない（< 500行）→ **GLM**で十分な場合が多い。
- 非線形・相互作用が複雑 → **RF**（安定性重視）or **XGBoost**（精度重視）。
- 予測根拠の透明性が最優先 → **決定木**。
- 最高精度を追求し、解釈にはSHAPを使う → **XGBoost + SHAP**。
