13.1 はじめに

5章・6章で扱ったロジスティック回帰では、「ある説明変数が正答確率にどう影響するか」をモデル化しました。しかし教育テストでは、同じ受験者が複数の項目に回答するため、「どの項目が難しいか」と「誰が高能力か」を同時に推定したい場面が頻出します。

項目応答理論（Item Response Theory; IRT）は、この要請に直接応える枠組みです。受験者の潜在能力（()）と項目特性（困難度・識別力）を同時にモデル化し、「誰にとって、どの項目が、どれくらい難しいか」を推定します(Cai et al., 2016)。特にラッシュモデル（1PL）は、測定モデルとしての単純さ、項目母数の不変性、教育評価における解釈しやすさのため、言語テスト研究で広く利用されています(Andersen, 1973; Boone et al., 2017; Rasch, 1980)。

13.1.1 最初にここだけ（はじめに）

Step 1: 問題を1文で捉える

「同じ合計点の受験者が、本当に同じ能力なのかは、どの項目を正解したかを考慮しないと判断できない。」

Step 2: 覚える式は1つだけ

\[ P(X_{ij}=1\mid \theta_i, b_j)=\frac{\exp(\theta_i-b_j)}{1+\exp(\theta_i-b_j)} \]

正答確率は、受験者能力 (_i) と項目困難度 (b_j) の差だけで決まります。

Step 3: この章でやること

  1. ロジスティック回帰から Rasch モデルへの接続を確認する。
  2. eRm で Rasch 推定 → 適合度 → Wright Map まで一気に実行する。
  3. DIF 分析で公平性を検証し、論文報告形式にまとめる。

Step 4: どこまで読めば実務に使えるか

  • 最低限: 「はじめに」→「glm()からRaschへ」→「Rasch推定」→「適合度」→「まとめ」で研究に使える。
  • 標準: 上記 + 「前提チェック」+「DIF分析」まで進めると査読対応力が上がる。
  • 拡張: 上記 + 「指数型分布族」+「CML導出」+「パッケージ比較」で理論的基盤を固める。第12章で順序データ・多次元・ノンパラメトリックに進む。
読む順序（迷ったとき）
  • 最短ルート（30分）: はじめに → glm()からRaschへ → Rasch推定 → Wright Map → まとめ
  • 標準ルート（60分）: 上記 + 適合度 + 前提チェック + DIF
  • 拡張ルート（90分+）: 上記 + 指数型分布族 + CML導出 + パッケージ比較
用語ミニ辞典
  • ()（シータ）: 受験者の潜在能力。ロジット尺度で表す。
  • (b)（項目困難度）: 正答確率が50%になる () の値。大きいほど難しい。
  • ICC（項目特性曲線）: () に対する正答確率のS字カーブ。ロジスティック曲線そのもの。
  • Wright Map（Person-Item Map）: () と (b) を同一尺度上に並べた図。IRT の中核可視化。
  • Infit / Outfit: モデル期待からの逸脱を測る適合統計量。1.0が期待値。
  • DIF（項目機能差異）: 同能力の受験者でも群間で項目反応が異なる現象。
この章の到達目標
  • ロジスティック回帰との連続性で IRT を説明できる。
  • Rasch モデルの識別可能性と条件付き尤度を数式で説明できる。
  • JML/CML/MML の違いと実務上の含意を説明できる。
  • eRm を使って Rasch モデルを推定し、項目困難度・適合度を解釈できる。
  • 一次元性・局所独立性を最低限チェックできる。
  • Rasch（1PL）, 2PL, 3PL の使い分けを、研究目的に沿って判断できる。
  • DIF 検出とバイアス判断の違いを区別して報告できる。
  • 英語教育研究の論文で報告できる形式に結果を整理できる。

13.3 CTT から IRT へ：何が変わるのか

ここで具体的な場面を考えてみましょう。500人の EFL 学習者に語彙テスト（20項目）を実施したとします。受験者 A と受験者 B はどちらも合計12点でした。CTT ではこの2名は「同じ得点」として扱われます。しかし、A は易しい項目ばかり正解し、B は難しい項目を多く正解していたとしたらどうでしょうか。実際の語彙力に差がある可能性が高いのに、合計点だけではそれが見えません。

IRT はまさにこの問題を解決します。「どの項目を正解したか」というパターン情報を使い、受験者能力と項目困難度を分離して推定します(Cai et al., 2016; Mair & Hatzinger, 2007; Rasch, 1980)。

13.4 Rasch モデルをロジスティック回帰として理解する

前節で CTT と IRT の違いを確認しました。ここでは、5章・6章で馴染みのあるロジスティック回帰から、Rasch モデルへ1ステップで橋渡しします。

13.4.1 glm() から Rasch へ：見慣れた回帰の延長線上

まず、テストデータを「受験者×項目」のロング形式に変換し、項目をダミー変数として通常のロジスティック回帰を当ててみましょう。これは「項目ごとに正答しやすさが違う」ことをモデル化する、最も素朴なアプローチです。

コード 
# 後のセクションで使うシミュレーションデータを先に作成
n_person_pre <- 500
n_item_pre <- 20
b_pre <- seq(-2, 2, length.out = n_item_pre)
theta_pre <- rnorm(n_person_pre, mean = 0, sd = 1)
eta_pre <- outer(theta_pre, b_pre, "-")
prob_pre <- plogis(eta_pre)
resp_pre <- matrix(rbinom(n_person_pre * n_item_pre, 1, prob_pre),
                   nrow = n_person_pre, ncol = n_item_pre)
colnames(resp_pre) <- paste0("Item", sprintf("%02d", 1:n_item_pre))
resp_df_pre <- as.data.frame(resp_pre)

  mutate(person = row_number()) |>
  pivot_longer(-person, names_to = "item", values_to = "response")

glm_fit <- glm(response ~ 0 + item, family = binomial, data = long_df)

glm() と Rasch の違いは何か

上表で glm() の log-odds と Rasch の困難度推定は近い値になります。これは偶然ではありません。

  • glm(response ~ 0 + item) は「全受験者が同じ能力」と暗に仮定し、項目の正答しやすさだけを推定します。
  • Rasch モデルは「受験者ごとに異なる潜在能力 (_i) がある」ことを明示的にモデル化し、その上で項目困難度 (b_j) を推定します。

つまり Rasch は、ロジスティック回帰に受験者ごとの潜在能力を加えた拡張です。数式で書けば、glm() の logit(P) = β_j（項目効果のみ）を logit(P) = θ_i - b_j（受験者効果 + 項目効果）に拡張しただけです。

13.4.2 Rasch モデルの数理的定義

2値 Rasch モデルは次式で表されます(Andersen, 1973; Rasch, 1980)。

\[ P(X_{ij}=1\mid \theta_i, b_j)=\frac{\exp(\theta_i-b_j)}{1+\exp(\theta_i-b_j)} \]

ここで、

  • (X_{ij}): 受験者 (i) が項目 (j) に正答したか（1/0）
  • (_i): 受験者能力（ロジット尺度）
  • (b_j): 項目困難度（正答確率50%になる () の値）

です。言い換えれば、正答確率は受験者能力と項目困難度の差だけで決まるロジスティック関数です。

なぜ Rasch モデルを「処方」するのか

IRT には2つの立場があります(Boone et al., 2017)。

  • 記述的アプローチ（IRT一般）: データに最も当てはまるモデル（1PL/2PL/3PL）を選ぶ。
  • 規範的アプローチ（Rasch学派）: 公平な測定の条件を満たすモデルを先に決め、データがそれに適合するか検証する。

Rasch モデルでは、識別力を等しいと仮定する（1PL）ことで、項目母数の不変性（標本によらない推定）と合計点の十分統計量性が保証されます。英語教育テストでは、異なる受験者集団間でも項目パラメータが安定していることが実務上重要であり、この規範的アプローチが広く採用されています。

Rasch では識別力を等しいと仮定する点が、2PL と異なります(Birnbaum, 1968)。2PL/3PL への拡張は本章後半で扱います。

13.4.3 指数型分布族としての表現

Rasch モデルの尤度は、受験者と項目の全反応行列 (X=(x_{ij})) について

\[ L(\boldsymbol{\theta}, \mathbf{b}\mid X) =\prod_{i=1}^{N}\prod_{j=1}^{J} \frac{\exp\{x_{ij}(\theta_i-b_j)\}}{1+\exp(\theta_i-b_j)} \]

です。これを整理すると

\[ L(\boldsymbol{\theta}, \mathbf{b}\mid X) = \exp\left( \sum_{i=1}^{N}\theta_i r_i-\sum_{j=1}^{J}b_j s_j \right) \prod_{i=1}^{N}\prod_{j=1}^{J}\left\{1+\exp(\theta_i-b_j)\right\}^{-1}, \]

ただし

\[ r_i=\sum_{j=1}^{J}x_{ij},\quad s_j=\sum_{i=1}^{N}x_{ij} \]

です。(r_i) は受験者の素点、(s_j) は項目得点（通過者数）です。

この形は、Rasch が「合計点に基づく推定」と整合的である理由を明示します(Andersen, 1973; Rasch, 1980)。

13.4.4 識別可能性（identifiability）の確認

Rasch 確率は ((_i-b_j)) の差だけで決まるため、任意の定数 (c) に対して

\[ (\theta_i, b_j)\mapsto (\theta_i+c, b_j+c) \]

としても予測確率は不変です。したがって、そのままでは母数が一意に定まりません。 このため実務では次のような制約を置きます。

  • (_{j=1}^{J}b_j=0)（項目困難度の平均を0に固定）
  • あるいは (_{i=1}^{N}_i=0)

eRm::RM() はこの識別制約のもとで推定を行います(Mair & Hatzinger, 2007)。

コード 
# 位置不変性の数値確認: theta,b を同じだけ平行移動しても確率は同一

13.4.5 条件付き尤度（CML）の導出スケッチ

Rasch 推定の理論的核は、(_i) を「邪魔母数（nuisance parameters）」として消去できる点です。 受験者 (i) の反応ベクトルを (_i)、素点を (R_i=r_i) とすると、

\[ P(\mathbf{X}_i=\mathbf{x}_i \mid R_i=r_i, \mathbf{b}) = \frac{\exp\left(-\sum_{j=1}^{J} b_j x_{ij}\right)} {\sum_{\mathbf{z}\in\mathcal{A}(r_i)} \exp\left(-\sum_{j=1}^{J} b_j z_j\right)}, \]

ここで ((r_i)) は「合計が (r_i) になる 0/1 ベクトル全体」です。

重要なのは、右辺に (_i) が現れないことです。
つまり (R_i) で条件付けると、能力母数を積分せずに消去でき、 項目母数 () だけの尤度最大化（CML）が可能になります(Andersen, 1973; Mair & Hatzinger, 2007)。

コード 
# 受験者1名・3項目の小例で theta が消えることを確認
b_small <- c(-1.0, 0.2, 1.1)
x_obs <- c(1, 0, 1) # 合計 r = 2

patterns_r2 <- rbind(
  c(1, 1, 0),
  c(1, 0, 1),
  c(0, 1, 1)
)

num <- exp(-sum(b_small * x_obs))
den <- sum(apply(patterns_r2, 1, function(z) exp(-sum(b_small * z))))
cond_prob <- num / den

tibble(
  conditional_probability = cond_prob
) |>
  gt() |>
  tab_header(
    title = "Conditional Probability Given Raw Score",
    subtitle = "Expression contains b only (θ is canceled)"
  ) |>
  fmt_number(columns = conditional_probability, decimals = 6)
Conditional Probability Given Raw Score

conditional_probability
0.265901

この条件付き確率の計算に \(\theta\) が一切含まれていないことが、CML の核心です。素点 \(R_i = r\) で条件付けることで受験者能力が消去され、項目母数 \(\mathbf{b}\) だけの関数として推定が可能になります。

13.4.6 Andersen の LR 検定の考え方

Andersen 検定は、項目母数の群間不変性を尤度比で検証します(Andersen, 1973)。
例として、受験者を総得点中央値で2群に分けると、

\[ LR=-2\left\{\ell_{\mathrm{pooled}}-\left(\ell_{g1}+\ell_{g2}\right)\right\} \overset{approx.}{\sim}\chi^2_{\nu} \]

を用いて、群ごとに推定された項目困難度が統計的に同一かを確認します。
この検定は Rasch の「項目母数の不変性」という中核要請に直接対応します。

13.6.2 ICC と Person-Item Map

項目特性曲線（ICC）は、各項目について「能力 () が上がるにつれて正答確率がどう変化するか」を示します。Rasch モデルでは全項目の ICC は同じ傾き（識別力）を持ち、位置（困難度）だけが異なります。

コード 
plotICC(rasch_fit, item.subset = "all", empICC = list("raw"), legend = FALSE)

ICC の読み方: 各曲線の変曲点（正答確率 = 0.5 の位置）が、その項目の困難度 (b_j) に対応します。経験的 ICC（点）が理論曲線から大きくずれている項目があれば、モデル不適合の兆候です。

Wright Map（Person-Item Map）

Wright Map は IRT の中核可視化です。受験者能力（()）と項目困難度（(b)）を同一ロジット尺度上に並べ、テストがどの能力帯をカバーしているかを一目で把握できます。

コード 
plotPImap(rasch_fit)
Wright Map の読み方（3つのポイント）
  1. 受験者分布と項目位置の対応: 受験者が集中している能力帯に項目が十分配置されていれば、そのテストはその能力帯で精密に測定できています。逆に、受験者が多いのに項目が少ない帯域は「測定の穴」です。
  2. 天井効果・床効果の確認: 最も易しい項目より下に多くの受験者がいれば床効果、最も難しい項目より上に多くの受験者がいれば天井効果です。
  3. 項目の等間隔性: Rasch の利点は、項目困難度がロジット尺度上で等間隔に解釈できる点です。Wright Map 上で項目が一部に偏っていれば、テスト改良の指針になります。

13.7 適合度の確認

Wright Map で人-項目関係の全体像を把握したところで、次は個々の項目がモデルの期待通りに機能しているかを検証します。

最初にここだけ（適合度）
  • Infit: 能力が項目困難度に近い受験者の反応パターンに重点を置く適合統計量。
  • Outfit: 全受験者を等重みで評価する適合統計量（外れ値の影響を受けやすい）。

13.9 2PLM・3PLM はどう理解すべきか

前節までの分析は、すべて Rasch（1PL）モデルに基づいていました。ここでは、識別力パラメータ (a_j) を項目ごとに自由推定する 2PL、さらに当て推量パラメータ (c_j) を加える 3PL への拡張を確認します。

Rasch（1PL）から 2PL, 3PL への拡張は次式で表せます(Ayala, 2009; Birnbaum, 1968)。

\[ \text{1PL: }P(X_{ij}=1\mid\theta_i)=\operatorname{logit}^{-1}(\theta_i-b_j) \]

\[ \text{2PL: }P(X_{ij}=1\mid\theta_i)=\operatorname{logit}^{-1}\left(a_j(\theta_i-b_j)\right) \]

\[ \text{3PL: }P(X_{ij}=1\mid\theta_i)=c_j+(1-c_j)\operatorname{logit}^{-1}\left(a_j(\theta_i-b_j)\right) \]

ここで - (a_j): 識別力（ICCの傾き） - (b_j): 困難度（正答確率の位置） - (c_j): 下方漸近線（guessing）

です。
3PL は多肢選択式テストでは有用ですが、標本が小さい場合や極端反応が多い場合に (c_j) が不安定になりやすいため、推定拘束や事前分布の工夫が必要です(Ayala, 2009; Baker & Kim, 2004)。

研究実務での使い分け
  • 1PL (Rasch): 測定理論上の厳密さと解釈の透明性を重視するとき。
  • 2PL: 項目間で識別力差が大きく、予測精度を上げたいとき。
  • 3PL: 猜測が構造的に想定される多肢選択で、十分な標本があるとき。

13.11 DIF とバイアス分析

最初にここだけ（DIF 分析）
  • DIF（Differential Item Functioning）: 同じ能力水準の受験者でも、所属する群（L1背景、性別、受験形式など）によって特定項目の正答確率が系統的に異なる現象。
  • なぜ重要か: DIF がある項目は、測定したい構成概念とは無関係な要因で正答率が変わるため、テストの公平性を脅かす。
  • DIF ≠ バイアス: DIF は統計的検出であり、バイアスは「その差が不公平か」の実質的判断。DIF が見つかっても、内容検討の結果「正当な差」であれば除外不要。

英語教育テストでは、L1背景・専攻・受験形式などで項目機能が変わらないかを確認する必要があります(Runnels, 2013; Tan et al., 2024)。

ここではシミュレーション上、Item05 と Item17 に DIF を埋め込んでいます。 実務では、アンカー項目選定と反復的な浄化手続きを行ってください(Runnels, 2013)。

13.11.2 Uniform DIF と Non-uniform DIF の分離

DIF にはさらに2つの種類があります。

13.11.3 DIF とバイアスは同義ではない

DIF は統計的非不変性、バイアスは妥当性判断（解釈上の不公平）です。
したがって DIF 検出は必要条件ですが、内容妥当性・受験プロセス・学習機会差を含む実質的検討が不可欠です(Camilli & Shepard, 1994; Zhao & Hambleton, 2017)。

13.12 英語教育研究での実装上の示唆

ここまでの分析をまとめ、実務でのチェックリストと論文報告に必要な要素を整理します。近年の言語テスト研究では、 - Rasch による構成概念妥当性の検討 - 項目施設値（正答率）だけでは見えない測定上の歪みの同定 - DIF/DTF を含む公平性評価 が重要視されています(Ha, 2021; Runnels, 2013; Tan et al., 2024)。

実務チェックリスト（最小版）
  1. 目的の明確化: 配置判定・到達度評価・研究用尺度化のどれか。
  2. モデル推定: まず Rasch（1PL）で基準線を作る。
  3. 前提確認: 一次元性・局所独立性を確認する。
  4. 適合確認: Infit/Outfit と内容妥当性を併用して判断する。
  5. 比較検討: 必要時のみ 2PL/3PL を検討する。
  6. 公平性確認: DIF/DTF を報告し、影響の実質的解釈を行う。

As assumption checks, we evaluated unidimensionality using tetrachoric-eigenvalue inspection and local independence using Yen’s Q3 residual correlations (Christensen et al., 2017; Yen, 1984). 13.14 まとめ

本章のポイントは次の4点です。

  1. Rasch モデルは、回帰分析の延長で理解できる「測定モデル」である。
  2. 推定法（JML/CML/MML）の違いは理論的前提とバイアス特性に直結する。
  3. 1PL・2PL・3PLは「上位互換」ではなく、研究目的・データ条件で使い分けるべきである。
  4. 英語教育研究では、DIF検出（統計）とバイアス判断（妥当性）を分けて報告して初めて説得力が出る。

## はじめに

5章・6章で扱ったロジスティック回帰では、「ある説明変数が正答確率にどう影響するか」をモデル化しました。しかし教育テストでは、同じ受験者が複数の項目に回答するため、**「どの項目が難しいか」と「誰が高能力か」を同時に推定したい**場面が頻出します。

項目応答理論（Item Response Theory; IRT）は、この要請に直接応える枠組みです。受験者の潜在能力（\(\theta\)）と項目特性（困難度・識別力）を同時にモデル化し、「誰にとって、どの項目が、どれくらい難しいか」を推定します[@cai2016]。特にラッシュモデル（1PL）は、測定モデルとしての単純さ、項目母数の不変性、教育評価における解釈しやすさのため、言語テスト研究で広く利用されています[@rasch1980; @andersen1973; @boone2017]。

### 最初にここだけ（はじめに）

#### Step 1: 問題を1文で捉える

「同じ合計点の受験者が、本当に同じ能力なのかは、どの項目を正解したかを考慮しないと判断できない。」

#### Step 2: 覚える式は1つだけ

$$
P(X_{ij}=1\mid \theta_i, b_j)=\frac{\exp(\theta_i-b_j)}{1+\exp(\theta_i-b_j)}
$$

正答確率は、受験者能力 \(\theta_i\) と項目困難度 \(b_j\) の**差**だけで決まります。

#### Step 3: この章でやること

1. ロジスティック回帰から Rasch モデルへの接続を確認する。
2. `eRm` で Rasch 推定 → 適合度 → Wright Map まで一気に実行する。
3. DIF 分析で公平性を検証し、論文報告形式にまとめる。

#### Step 4: どこまで読めば実務に使えるか

- **最低限**: 「はじめに」→「glm()からRaschへ」→「Rasch推定」→「適合度」→「まとめ」で研究に使える。
- **標準**: 上記 + 「前提チェック」+「DIF分析」まで進めると査読対応力が上がる。
- **拡張**: 上記 + 「指数型分布族」+「CML導出」+「パッケージ比較」で理論的基盤を固める。第12章で順序データ・多次元・ノンパラメトリックに進む。

- **\(\theta\)（シータ）**: 受験者の潜在能力。ロジット尺度で表す。
- **\(b\)（項目困難度）**: 正答確率が50%になる \(\theta\) の値。大きいほど難しい。
- **ICC（項目特性曲線）**: \(\theta\) に対する正答確率のS字カーブ。ロジスティック曲線そのもの。

- **DIF（項目機能差異）**: 同能力の受験者でも群間で項目反応が異なる現象。
:::

## CTT から IRT へ：何が変わるのか

ここで具体的な場面を考えてみましょう。500人の EFL 学習者に語彙テスト（20項目）を実施したとします。受験者 A と受験者 B はどちらも合計12点でした。CTT ではこの2名は「同じ得点」として扱われます。しかし、A は易しい項目ばかり正解し、B は難しい項目を多く正解していたとしたらどうでしょうか。**実際の語彙力に差がある可能性**が高いのに、合計点だけではそれが見えません。

IRT はまさにこの問題を解決します。「どの項目を正解したか」というパターン情報を使い、受験者能力と項目困難度を分離して推定します[@cai2016; @rasch1980; @mair2007]。

## Rasch モデルをロジスティック回帰として理解する

前節で CTT と IRT の違いを確認しました。ここでは、5章・6章で馴染みのあるロジスティック回帰から、Rasch モデルへ**1ステップで橋渡し**します。

### glm() から Rasch へ：見慣れた回帰の延長線上

まず、テストデータを「受験者×項目」のロング形式に変換し、項目をダミー変数として通常のロジスティック回帰を当ててみましょう。これは「項目ごとに正答しやすさが違う」ことをモデル化する、最も素朴なアプローチです。

```{r}
#| label: glm-bridge
#| message: false

# 後のセクションで使うシミュレーションデータを先に作成
n_person_pre <- 500
n_item_pre <- 20
b_pre <- seq(-2, 2, length.out = n_item_pre)
theta_pre <- rnorm(n_person_pre, mean = 0, sd = 1)
eta_pre <- outer(theta_pre, b_pre, "-")
prob_pre <- plogis(eta_pre)
resp_pre <- matrix(rbinom(n_person_pre * n_item_pre, 1, prob_pre),
                   nrow = n_person_pre, ncol = n_item_pre)
colnames(resp_pre) <- paste0("Item", sprintf("%02d", 1:n_item_pre))
resp_df_pre <- as.data.frame(resp_pre)

# ロング形式に変換
long_df <- resp_df_pre |>
  mutate(person = row_number()) |>
  pivot_longer(-person, names_to = "item", values_to = "response")

# 素朴なロジスティック回帰（項目効果のみ）
glm_fit <- glm(response ~ 0 + item, family = binomial, data = long_df)

# Rasch推定（eRm）
rasch_fit_pre <- eRm::RM(resp_df_pre)

# 係数を比較
bridge_tbl <- tibble(
  item = colnames(resp_df_pre),
  glm_logit = coef(glm_fit)[paste0("item", colnames(resp_df_pre))],
  rasch_beta = -as.numeric(coef(rasch_fit_pre)),
  true_b = b_pre
) |>
  mutate(
    glm_logit = round(glm_logit, 3),
    rasch_beta = round(rasch_beta, 3),
    true_b = round(true_b, 3)
  )

bridge_tbl |>
  gt() |>
  tab_header(
    title = "glm() Coefficients vs Rasch Difficulty",
    subtitle = "The logistic regression bridge to IRT"
  ) |>
  cols_label(
    item = "Item",
    glm_logit = "glm() log-odds",
    rasch_beta = "Rasch difficulty (b)",
    true_b = "True difficulty"
  )
```

::: {.callout-tip}
#### glm() と Rasch の違いは何か

上表で `glm()` の log-odds と Rasch の困難度推定は**近い値**になります。これは偶然ではありません。

- `glm(response ~ 0 + item)` は「全受験者が同じ能力」と暗に仮定し、項目の正答しやすさだけを推定します。
- Rasch モデルは「受験者ごとに異なる潜在能力 \(\theta_i\) がある」ことを明示的にモデル化し、その上で項目困難度 \(b_j\) を推定します。

つまり **Rasch は、ロジスティック回帰に受験者ごとの潜在能力を加えた拡張**です。数式で書けば、`glm()` の `logit(P) = β_j`（項目効果のみ）を `logit(P) = θ_i - b_j`（受験者効果 + 項目効果）に拡張しただけです。
:::

### Rasch モデルの数理的定義

2値 Rasch モデルは次式で表されます[@rasch1980; @andersen1973]。

$$
P(X_{ij}=1\mid \theta_i, b_j)=\frac{\exp(\theta_i-b_j)}{1+\exp(\theta_i-b_j)}
$$

ここで、

- \(X_{ij}\): 受験者 \(i\) が項目 \(j\) に正答したか（1/0）
- \(\theta_i\): 受験者能力（ロジット尺度）
- \(b_j\): 項目困難度（正答確率50%になる \(\theta\) の値）

です。言い換えれば、正答確率は受験者能力と項目困難度の**差**だけで決まるロジスティック関数です。

::: {.callout-tip}
#### なぜ Rasch モデルを「処方」するのか

IRT には2つの立場があります[@boone2017]。

- **記述的アプローチ**（IRT一般）: データに最も当てはまるモデル（1PL/2PL/3PL）を選ぶ。
- **規範的アプローチ**（Rasch学派）: 公平な測定の条件を満たすモデルを**先に決め**、データがそれに適合するか検証する。

Rasch モデルでは、識別力を等しいと仮定する（1PL）ことで、**項目母数の不変性**（標本によらない推定）と**合計点の十分統計量性**が保証されます。英語教育テストでは、異なる受験者集団間でも項目パラメータが安定していることが実務上重要であり、この規範的アプローチが広く採用されています。
:::

Rasch では識別力を等しいと仮定する点が、2PL と異なります[@birnbaum1968]。2PL/3PL への拡張は本章後半で扱います。

### 指数型分布族としての表現

Rasch モデルの尤度は、受験者と項目の全反応行列 \(X=(x_{ij})\) について

$$
L(\boldsymbol{\theta}, \mathbf{b}\mid X)
=\prod_{i=1}^{N}\prod_{j=1}^{J}
\frac{\exp\{x_{ij}(\theta_i-b_j)\}}{1+\exp(\theta_i-b_j)}
$$

です。これを整理すると

$$
L(\boldsymbol{\theta}, \mathbf{b}\mid X)
=
\exp\left(
\sum_{i=1}^{N}\theta_i r_i-\sum_{j=1}^{J}b_j s_j
\right)
\prod_{i=1}^{N}\prod_{j=1}^{J}\left\{1+\exp(\theta_i-b_j)\right\}^{-1},
$$

ただし

$$
r_i=\sum_{j=1}^{J}x_{ij},\quad
s_j=\sum_{i=1}^{N}x_{ij}
$$

です。\(r_i\) は受験者の素点、\(s_j\) は項目得点（通過者数）です。

この形は、Rasch が「合計点に基づく推定」と整合的である理由を明示します[@rasch1980; @andersen1973]。

### 識別可能性（identifiability）の確認

Rasch 確率は \((\theta_i-b_j)\) の差だけで決まるため、任意の定数 \(c\) に対して

$$
(\theta_i, b_j)\mapsto (\theta_i+c, b_j+c)
$$

としても予測確率は不変です。したがって、そのままでは母数が一意に定まりません。
このため実務では次のような制約を置きます。

- \(\sum_{j=1}^{J}b_j=0\)（項目困難度の平均を0に固定）
- あるいは \(\sum_{i=1}^{N}\theta_i=0\)

`eRm::RM()` はこの識別制約のもとで推定を行います[@mair2007]。

```{r}
#| label: location-invariance-demo

# 位置不変性の数値確認: theta,b を同じだけ平行移動しても確率は同一
theta_demo <- c(-1.2, 0.0, 1.1)
b_demo <- c(-0.8, 0.3, 1.4)

p_original <- outer(theta_demo, b_demo, function(th, bj) plogis(th - bj))
p_shifted <- outer(theta_demo + 2.5, b_demo + 2.5, function(th, bj) plogis(th - bj))

max_abs_diff <- max(abs(p_original - p_shifted))
tibble(max_abs_difference = max_abs_diff) |>
  gt() |>
  tab_header(
    title = "Location Invariance Check",
    subtitle = "Probabilities are unchanged by common shift"
  ) |>
  fmt_number(columns = max_abs_difference, decimals = 10)
```

この結果は、$\theta$ と $b$ を同じ定数だけ平行移動しても正答確率が不変であること（位置不変性）を数値的に確認しています。差が実質的にゼロであることから、Rasch モデルが差 $\theta - b$ のみに依存する構造であることがわかります。

### 条件付き尤度（CML）の導出スケッチ

Rasch 推定の理論的核は、\(\theta_i\) を「邪魔母数（nuisance parameters）」として消去できる点です。
受験者 \(i\) の反応ベクトルを \(\mathbf{x}_i\)、素点を \(R_i=r_i\) とすると、

$$
P(\mathbf{X}_i=\mathbf{x}_i \mid R_i=r_i, \mathbf{b})
=
\frac{\exp\left(-\sum_{j=1}^{J} b_j x_{ij}\right)}
{\sum_{\mathbf{z}\in\mathcal{A}(r_i)}
\exp\left(-\sum_{j=1}^{J} b_j z_j\right)},
$$

ここで \(\mathcal{A}(r_i)\) は「合計が \(r_i\) になる 0/1 ベクトル全体」です。

重要なのは、右辺に \(\theta_i\) が現れないことです。  
つまり \(R_i\) で条件付けると、能力母数を積分せずに消去でき、
項目母数 \(\mathbf{b}\) だけの尤度最大化（CML）が可能になります[@andersen1973; @mair2007]。

```{r}
#| label: cml-cancellation-demo

# 受験者1名・3項目の小例で theta が消えることを確認
b_small <- c(-1.0, 0.2, 1.1)
x_obs <- c(1, 0, 1) # 合計 r = 2

patterns_r2 <- rbind(
  c(1, 1, 0),
  c(1, 0, 1),
  c(0, 1, 1)
)

num <- exp(-sum(b_small * x_obs))
den <- sum(apply(patterns_r2, 1, function(z) exp(-sum(b_small * z))))
cond_prob <- num / den

tibble(
  conditional_probability = cond_prob
) |>
  gt() |>
  tab_header(
    title = "Conditional Probability Given Raw Score",
    subtitle = "Expression contains b only (θ is canceled)"
  ) |>
  fmt_number(columns = conditional_probability, decimals = 6)
```

この条件付き確率の計算に $\theta$ が一切含まれていないことが、CML の核心です。素点 $R_i = r$ で条件付けることで受験者能力が消去され、項目母数 $\mathbf{b}$ だけの関数として推定が可能になります。

### Andersen の LR 検定の考え方

Andersen 検定は、項目母数の群間不変性を尤度比で検証します[@andersen1973]。  
例として、受験者を総得点中央値で2群に分けると、

$$
LR=-2\left\{\ell_{\mathrm{pooled}}-\left(\ell_{g1}+\ell_{g2}\right)\right\}
\overset{approx.}{\sim}\chi^2_{\nu}
$$

を用いて、群ごとに推定された項目困難度が統計的に同一かを確認します。  
この検定は Rasch の「項目母数の不変性」という中核要請に直接対応します。

## Rasch モデルの推定（eRm）

得点分布を確認したところで、いよいよ Rasch モデルを推定します。`eRm::RM()` を用いて、条件付き最大尤度法（CML）で項目困難度を推定します[@mair2007]。

::: {.callout-note}
#### 推定法の比較（CML / MML / JML）

| 推定法 | 原理 | 主なRパッケージ | 特徴 |
|--------|------|-----------------|------|
| **CML** | 素点で条件付けて \(\theta\) を消去 | `eRm` | Rasch の十分統計量性を直接利用。理論的に一貫[@andersen1973] |
| **MML** | \(\theta\) に分布仮定を置き積分 | `mirt`, `TAM` | 2PL/3PL に柔軟。標本が大きいほど安定[@bock1981] |
| **JML** | \(\theta\) と \(b\) を同時最適化 | `TAM::tam.jml()` | 直感的だが incidental parameter problem あり[@neyman1948] |

本章では CML（`eRm::RM()`）を主に使います。MML ベースの `mirt` はモデル比較（2PL/3PL）で利用します。論文では**推定法を必ず明記**してください[@deayala2009]。
:::

```{r}
#| label: rasch-fit

  as.data.frame() |>

```

### ICC と Person-Item Map

項目特性曲線（ICC）は、各項目について「能力 \(\theta\) が上がるにつれて正答確率がどう変化するか」を示します。Rasch モデルでは全項目の ICC は同じ傾き（識別力）を持ち、位置（困難度）だけが異なります。

```{r}
#| label: icc-plot
#| fig-width: 9
#| fig-height: 6

plotICC(rasch_fit, item.subset = "all", empICC = list("raw"), legend = FALSE)
```

ICC の読み方: 各曲線の変曲点（正答確率 = 0.5 の位置）が、その項目の困難度 \(b_j\) に対応します。経験的 ICC（点）が理論曲線から大きくずれている項目があれば、モデル不適合の兆候です。

#### Wright Map（Person-Item Map）

Wright Map は IRT の**中核可視化**です。受験者能力（\(\theta\)）と項目困難度（\(b\)）を同一ロジット尺度上に並べ、テストがどの能力帯をカバーしているかを一目で把握できます。

```{r}
#| label: pi-map
#| fig-width: 9
#| fig-height: 6

plotPImap(rasch_fit)
```

::: {.callout-tip}
#### Wright Map の読み方（3つのポイント）

1. **受験者分布と項目位置の対応**: 受験者が集中している能力帯に項目が十分配置されていれば、そのテストはその能力帯で精密に測定できています。逆に、受験者が多いのに項目が少ない帯域は「測定の穴」です。
2. **天井効果・床効果の確認**: 最も易しい項目より下に多くの受験者がいれば床効果、最も難しい項目より上に多くの受験者がいれば天井効果です。
3. **項目の等間隔性**: Rasch の利点は、項目困難度がロジット尺度上で等間隔に解釈できる点です。Wright Map 上で項目が一部に偏っていれば、テスト改良の指針になります。
:::

## 適合度の確認

Wright Map で人-項目関係の全体像を把握したところで、次は個々の項目がモデルの期待通りに機能しているかを検証します。

::: {.callout-tip}
#### 最初にここだけ（適合度）
- **Infit**: 能力が項目困難度に近い受験者の反応パターンに重点を置く適合統計量。
- **Outfit**: 全受験者を等重みで評価する適合統計量（外れ値の影響を受けやすい）。
- **MSQ（Mean Square）**: 期待値は 1.0。1.0 より大きい = 予想外の不規則反応、1.0 より小さい = 予想以上に規則的。
:::

Rasch では、Infit/Outfit を使って「モデル期待からの逸脱」を検討します[@mair2007; @boone2017]。

```{r}
#| label: fit-stat

pp <- person.parameter(rasch_fit)
fit_obj <- eRm::itemfit(pp)

fit_tbl <- tibble(
  item = names(fit_obj$i.infitMSQ),
  infit_msq = as.numeric(fit_obj$i.infitMSQ),
  outfit_msq = as.numeric(fit_obj$i.outfitMSQ),
  infit_z = as.numeric(fit_obj$i.infitZ),
  outfit_z = as.numeric(fit_obj$i.outfitZ)
) |>
  arrange(desc(outfit_msq))

fit_tbl |>
  gt() |>
  tab_header(
    title = "Item Fit Statistics",
    subtitle = "Infit/Outfit for Rasch Model"
  ) |>
  fmt_number(columns = c(infit_msq, outfit_msq, infit_z, outfit_z), decimals = 3)
```

Infit MSQ と Outfit MSQ がおおむね 0.7〜1.3 の範囲に収まっていれば、Rasch モデルの期待に概ね沿った反応パターンと解釈できます。Outfit が大きい項目（例: 1.5 を超える場合）は、能力と不釣り合いな正解・不正解パターン（外れ値的反応）の影響を受けている可能性があります。

::: {.callout-warning}
### Fit 指標の読み方（実務上の注意）

- 一般的な目安（例: 0.5〜1.5）はありますが、閾値は研究領域・目的・標本サイズで変わります[@boone2017]。
- 単一指標で機械的に除外せず、内容妥当性と併せて判断してください。
- モデル不適合は、推定値だけでなく等化・DIF解釈にも影響します[@zhao2017]。
:::

### 「不適合」が何を意味するか：ミニデモ

不適合統計量が何を捉えているかを直感的に理解するため、意図的に「おかしな項目」を1つ作成し、Infit/Outfit がどう反応するかを確認します。

```{r}

resp_misfit[low_ability, "Item10"] <- 1 - resp_misfit[low_ability, "Item10"]
```

## IRT の前提チェック

適合度分析で個々の項目の振る舞いを確認しました。ここでは、Rasch モデルが依拠する2つの統計的前提、すなわち**一次元性**と**局所独立性**を検証します。これらの前提が崩れると、推定値の解釈やDIF分析の妥当性に影響が及びます。

### 1. 一次元性（Unidimensionality）

一次元性は「主要な潜在特性が1つであるか」を問う前提です。2値データに対しては、ピアソン相関よりも tetrachoric 相関に基づく固有値分析が適切です[@cai2016]。

```{r}
#| label: unidimensionality
#| fig-width: 8
#| fig-height: 5

eig <- eigen(tetra_mat)$values

eig_tbl <- tibble(
  factor = seq_along(eig),
  eigenvalue = eig
)

ggplot(eig_tbl, aes(x = factor, y = eigenvalue)) +
  geom_line(color = "#2C7FB8", linewidth = 0.8) +
  geom_point(color = "#2C7FB8", size = 2) +
  geom_hline(yintercept = 1, linetype = 2, color = "#B22222") +
```

## 2PLM・3PLM はどう理解すべきか

前節までの分析は、すべて Rasch（1PL）モデルに基づいていました。ここでは、識別力パラメータ \(a_j\) を項目ごとに自由推定する **2PL**、さらに当て推量パラメータ \(c_j\) を加える **3PL** への拡張を確認します。

Rasch（1PL）から 2PL, 3PL への拡張は次式で表せます[@birnbaum1968; @deayala2009]。

$$
\text{1PL: }P(X_{ij}=1\mid\theta_i)=\operatorname{logit}^{-1}(\theta_i-b_j)
$$

$$
\text{2PL: }P(X_{ij}=1\mid\theta_i)=\operatorname{logit}^{-1}\left(a_j(\theta_i-b_j)\right)
$$

$$
\text{3PL: }P(X_{ij}=1\mid\theta_i)=c_j+(1-c_j)\operatorname{logit}^{-1}\left(a_j(\theta_i-b_j)\right)
$$

ここで
- \(a_j\): 識別力（ICCの傾き）
- \(b_j\): 困難度（正答確率の位置）
- \(c_j\): 下方漸近線（guessing）

です。  
3PL は多肢選択式テストでは有用ですが、標本が小さい場合や極端反応が多い場合に
\(c_j\) が不安定になりやすいため、推定拘束や事前分布の工夫が必要です[@baker2004; @deayala2009]。

::: {.callout-note}
### 研究実務での使い分け

- **1PL (Rasch)**: 測定理論上の厳密さと解釈の透明性を重視するとき。
- **2PL**: 項目間で識別力差が大きく、予測精度を上げたいとき。
- **3PL**: 猜測が構造的に想定される多肢選択で、十分な標本があるとき。
:::

## DIF とバイアス分析

::: {.callout-tip}
#### 最初にここだけ（DIF 分析）
- **DIF（Differential Item Functioning）**: 同じ能力水準の受験者でも、所属する群（L1背景、性別、受験形式など）によって特定項目の正答確率が系統的に異なる現象。
- **なぜ重要か**: DIF がある項目は、測定したい構成概念とは無関係な要因で正答率が変わるため、テストの公平性を脅かす。
- **DIF ≠ バイアス**: DIF は統計的検出であり、バイアスは「その差が不公平か」の実質的判断。DIF が見つかっても、内容検討の結果「正当な差」であれば除外不要。
:::

英語教育テストでは、L1背景・専攻・受験形式などで項目機能が変わらないかを確認する必要があります[@runnels2013; @tan2024]。

ここではシミュレーション上、`Item05` と `Item17` に DIF を埋め込んでいます。
実務では、アンカー項目選定と反復的な浄化手続きを行ってください[@runnels2013]。

### Uniform DIF と Non-uniform DIF の分離

DIF にはさらに2つの種類があります。

- **Uniform DIF**: 群差が主に困難度（\(b\)）に現れる（ICC が平行移動する形）
- **Non-uniform DIF**: 群差が識別力（\(a\)）にも現れ、能力水準によって群差の大きさが変化する（ICC が交差する形）

```{r}
  fmt_number(columns = c(X2, p, adj_p), decimals = 3)
```

Uniform DIF（困難度のみの差）と Non-uniform DIF（識別力にも差）を分離して検定することで、DIF の性質を特定できます。Uniform DIF のみの項目は困難度の調整で対処可能ですが、Non-uniform DIF がある場合は能力帯によって群差が逆転しうるため、より慎重な内容検討が必要です。

### DIF とバイアスは同義ではない

**DIF は統計的非不変性**、**バイアスは妥当性判断（解釈上の不公平）**です。  
したがって DIF 検出は必要条件ですが、内容妥当性・受験プロセス・学習機会差を含む実質的検討が不可欠です[@camilli1994; @zhao2017]。

### バイアス影響の簡易チェック（シミュレーション）

ここではアンカー項目で能力を近似し、同一能力水準で群差が得点に残るかを確認します。

```{r}
#| label: bias-impact

anchor_fit <- eRm::RM(resp_df[, anchor_items])
theta_anchor <- as.numeric(person.parameter(anchor_fit)$theta.table[, "Person Parameter"])

bias_df <- tibble(
  group = sim_data$group,
  theta_anchor = theta_anchor,
  total_score = rowSums(resp_df),
  suspect_score = resp_df$Item05 + resp_df$Item17
) |>
  filter(is.finite(theta_anchor))

bias_model_total <- lm(total_score ~ theta_anchor + group, data = bias_df)
bias_model_suspect <- lm(suspect_score ~ theta_anchor + group, data = bias_df)

extract_group_effect <- function(model, outcome_label) {
  cf <- summary(model)$coefficients
  row_id <- grep("^group", rownames(cf), value = TRUE)
  tibble(
    outcome = outcome_label,
    term = row_id,
    estimate = cf[row_id, "Estimate"],
    std_error = cf[row_id, "Std. Error"],
    statistic = cf[row_id, "t value"],
    p_value = cf[row_id, "Pr(>|t|)"]
  )
}

bias_effect_tbl <- bind_rows(
  extract_group_effect(bias_model_total, "Total score (all items)"),
  extract_group_effect(bias_model_suspect, "Suspect score (Item05 + Item17)")
)

bias_effect_tbl |>
  gt() |>
  tab_header(
    title = "Bias Impact Check (Matched on Anchor Ability)",
    subtitle = "Residual group effect after conditioning on theta_anchor"
  ) |>
  fmt_number(columns = c(estimate, std_error, statistic, p_value), decimals = 3)
```

バイアス影響チェックの結果、アンカー能力で条件付けた後もなお suspect score（DIF 項目の合計）に有意な群差が残っていれば、DIF が実際にスコアに影響を与えている（DTF: Differential Test Functioning）と判断できます。逆に、全項目合計点では群差が吸収される場合もあり、DIF の影響は項目レベルとテストレベルで異なりうる点に注意が必要です。

## まとめ

本章のポイントは次の4点です。

1. Rasch モデルは、回帰分析の延長で理解できる「測定モデル」である。
