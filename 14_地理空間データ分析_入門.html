16.1 はじめに

英語教育の研究データには、しばしば地理的な構造が埋め込まれています。たとえば、以下の場面を考えてみてください。

  1. 全国学力調査の結果を都道府県別に比較したい。どの地域の英語スコアが高く、どの地域が低いのか。単なる順位表ではなく、地図上に可視化すれば、地理的パターン（例: 都市圏に高スコアが集中、あるいは東西で傾向が異なる）が一目でわかる。
  2. 教員の英語力指標（TOEICスコア、英検取得率）の年次変化を追跡したい。7年間の変化は全国で一様なのか、それとも特定の地域だけが急速に改善（または停滞）しているのか。年ごとの地図を並べれば、時空間的なトレンドが浮かび上がる。
  3. 2021年に導入されたALT配置拡充施策の効果を評価したい。施策導入前と導入後で英語スコアはどう変化したか。その変化は施策実施地域に限定されるのか、隣接地域にも波及しているのか。

これらの問いに答えるには、数表やグラフだけでなく地図が不可欠です。そして地図を作るには、都道府県の「形」（境界線のポリゴンデータ）と分析データを結びつける技術が必要です。

本章は、そのための基礎工事です。Rの sf パッケージを使って地理空間データを扱う方法を、都道府県別の英語教育指標データを題材に学びます。具体的には、形状データの取得、属性データとの結合、コロプレス図（塗り分け地図）の作成、パネルデータの整形までを最小限のコードで体験します。

この章の読み方（ロードマップ）

本章は以下の3つのパートで構成されています。

  • Part A（Section 3–6）: 地図を描く — 形状データの取得から、データ結合、コロプレス図、年次比較まで。「都道府県別の英語スコアを地図上に描く」という目標を達成する。
  • Part B（Section 7–8）: パネルを整える — パネルデータ（都道府県×年）の整形、要約統計、品質チェック。次章の空間分析に必要なデータ構造を準備する。
  • Part C（Section 9–10）: 空間パターンを測る — Moran’s I による空間自己相関の初歩的な検定と、施策前後差分の可視化。次章（応用編）への橋渡し。

推奨: 初学者は Part A を丁寧に読んでください。Part B はデータ整形の実務知識、Part C は次章の予習として活用してください。

「形状＋属性」の二重構造

地理空間分析のデータは、常に2つの要素から構成されます。

  1. 形状データ（geometry）: 都道府県の境界線を表すポリゴン。「どこにあるか」の情報。
  2. 属性データ（attributes）: 英語スコア、教員TOEIC、英検取得率などの数値。「何がどのくらいか」の情報。

この2つを left_join() で結合して初めて「地図上に色を塗る」ことができます。通常の回帰分析では属性データだけを扱いますが、空間分析では形状データが常に伴走する点が特徴です。

16.3 形状データの取得

16.3.1 sf オブジェクトとは

sf（Simple Features）パッケージは、地理空間データをRのデータフレームとして扱えるようにする仕組みです。通常のデータフレームに geometry という特殊な列が追加されているだけで、dplyr の関数がそのまま使えます。

sf を普通のデータフレームとの対比で理解する 
# 通常のデータフレーム
tibble(pref = "東京都", score = 72.5)

# sf オブジェクト（+ geometry 列）
tibble(pref = "東京都", score = 72.5, geometry = <POLYGON(...)>)
  • sf は「データフレーム + 地理座標」。既存のデータ操作の知識がそのまま活きる。
  • geometry 列は ggplot2 の geom_sf() が自動的に読み取り、地図を描画する。
  • 空間操作が不要な場面では st_drop_geometry() で geometry を外し、軽量なテーブルに戻せる。

16.3.2 形状データの取得（フォールバック付き）

日本の都道府県境界ポリゴンを取得します。パッケージの利用可能性は環境によって異なるため、3段階のフォールバックを設けています。

  1. jpndistrict（推奨）: 日本専用パッケージ。高品質な都道府県境界を提供。
  2. rnaturalearth: 世界の行政区画データ。日本の都道府県レベル（Admin-1）を取得可能。
  3. geodata（GADM）: 別のグローバルデータソース。
コード 
load_pref_sf <- function(){
  # 1) jpndistrict（推奨）
  jp_try <- try({ jpndistrict::jpn_pref(quality = 40) }, silent = TRUE)
  if (!inherits(jp_try, "try-error") && inherits(jp_try, "sf")) {
    return(jp_try |>
             st_transform(4326) |>
             dplyr::rename(pref_name = prefecture) |>
             dplyr::select(pref_name, geometry) |>
             sf::st_make_valid())
  }

  # 2) rnaturalearth（Admin-1, medium scale 明示）
  ne_try <- try({ rnaturalearth::ne_states(country = "Japan", returnclass = "sf") }, silent = TRUE)
  if (!inherits(ne_try, "try-error") && inherits(ne_try, "sf")) {
    return(ne_try |>
             dplyr::transmute(pref_name = name, geometry = geometry) |>
             st_transform(4326) |>
             sf::st_make_valid())
  }

  # 3) geodata::gadm（GADM level-1：都道府県）
  gd_try <- try({ geodata::gadm(country = "JPN", level = 1, path = tempdir()) }, silent = TRUE)
  if (!inherits(gd_try, "try-error")) {
    gd_sf <- sf::st_as_sf(gd_try)
    # 一般的な列名を合わせる（NAME_1 が都道府県名）
    name_col <- intersect(names(gd_sf), c("NAME_1", "NAME_1_", "VARNAME_1"))
    if (length(name_col) == 0) name_col <- names(gd_sf)[1]
    return(gd_sf |>
             dplyr::transmute(pref_name = .data[[name_col[1]]], geometry = geometry) |>
             sf::st_make_valid() |>
             sf::st_transform(4326))
  }

  stop("都道府県ポリゴンの取得に失敗しました。'jpndistrict' または 'rnaturalearth' の利用可能性をご確認ください。")
}

# 実境界のみを使用（ダミーhexには絶対フォールバックしない）
pref_sf <- load_pref_sf()

CRS（座標参照系）とは

地球は球体ですが、コンピュータの画面は平面です。CRS は「球体上の位置を平面上の座標に変換する方法」を定義する規約です。

  • EPSG:4326（WGS84）: 緯度・経度で位置を表す。GPSや多くのWebサービスが使用。本章の可視化はこれで十分。
  • EPSG:3857（Web メルカトル）: Google Maps や OpenStreetMap が使用する投影法。
  • JGD2011系: 日本の公式測量に使われる投影座標。面積や距離の計算にはこちらが正確。

実務のルール: 可視化は EPSG:4326 で行い、面積・距離の計測が必要な場合は投影座標（例: st_transform(x, 6668)）に変換してから実施します。st_crs() で都度確認する習慣をつけてください。

16.3.3 隣接の定義：Queen と Rook

地図を描くだけなら「隣接」を考える必要はありませんが、次章以降で空間自己相関や空間回帰を行う際に「誰が誰の隣人か」を定義する必要があります。ここでは3つの代表的な隣接定義を紹介します。

  • Queen 型: 辺または頂点を共有していれば隣接とみなす。チェスのクイーンが八方向に動けるイメージ。つながりが多く、空間的影響を広く捉える。
  • Rook 型: 辺（境界線）を共有している場合だけ隣接。チェスのルークが上下左右に動くイメージ。Queen より厳格。
  • k 近傍（kNN）: 各地域の重心から距離が近い順に k 個を隣接とする。島や飛地でも必ず隣接が定義でき、孤立ノードを回避できる。
英語教育研究での選択指針
  • 都道府県レベルの分析: Queen が良い出発点。日本の都道府県は比較的整った形状で、辺/頂点の区別は実質的な差を生みにくい。
  • 島嶼部を含む分析: 沖縄や離島を含む場合は kNN（例: k = 4–6）を併用し、孤立ノードを回避する。
  • 感度分析は必須: 主要な結論が隣接定義の選択で変わらないかを確認する（次章 Section 4 で詳述）。

以下のコードは参照用です。次章で実際に使用しますが、ここでは概念の理解のために掲載します。

コード 
# Queen（既定は queen=TRUE）
nb_queen <- spdep::poly2nb(pref_sf, queen = TRUE)
lw_queen <- spdep::nb2listw(nb_queen, style = 'W')

# Rook（辺共有のみ）
nb_rook  <- spdep::poly2nb(pref_sf, queen = FALSE)
lw_rook  <- spdep::nb2listw(nb_rook,  style = 'W')

# k 近傍（例: k=6）
knn      <- spdep::knearneigh(sf::st_coordinates(sf::st_centroid(pref_sf)), k = 6)
nb_knn   <- spdep::knn2nb(knn)
lw_knn   <- spdep::nb2listw(nb_knn, style = 'W')
nb と listw の違い

上のコードには nb と listw の2つのオブジェクトが登場します。

  • nb（隣接リスト）: 「都道府県Aの隣は B, C, D」という接続情報のみ。重みなし。
  • listw（重み付き隣接）: nb に重みを付加したもの。style = 'W'（行標準化）は、各地域の重みの合計が1になるように調整する。

空間分析の関数（moran.test(), lagsarlm() 等）は listw を入力として要求します。nb をそのまま渡すと「not a listw object」というエラーが出るので注意してください。

16.4 都道府県名の正規化（名寄せ）

形状データの都道府県名と、分析対象の指標データの都道府県名は、そのままでは一致しないことがあります。全角/半角、余分な空白、末尾の「都・道・府・県」の有無といった表記の揺れが原因です。以下の norm_pref() 関数は、この揺れを吸収して名寄せを可能にします。

コード 
# 正規化関数：全角/半角・空白・末尾の都/道/府/県の扱い
norm_pref <- function(x){
  x |>
    stringi::stri_trans_nfkc() |>
    stringr::str_replace_all("\\s+", "") |>
    stringr::str_replace("(東京都|北海道|京都府|大阪府)$", "") |>
    stringr::str_replace("県$", "")
}

# 形状側の正規化キー
jpn_key <- pref_sf |>
  st_drop_geometry() |>
  transmute(pref_name,
            pref_key = norm_pref(pref_name))
コードの読み方（パイプチェーン）

本章のRコードでは |> （ネイティブパイプ）を多用します。これは「左辺の結果を右辺の関数の第1引数に渡す」操作です。

# 以下の2つは同じ意味:
result <- f(g(h(x)))        # 入れ子（内側から読む）
result <- x |> h() |> g() |> f()  # パイプ（左から右へ順に読む）

norm_pref() の中身を読むと: (1) NFKC正規化（全角→半角統一）→ (2) 空白除去 → (3) 「東京都/北海道/京都府/大阪府」の除去 → (4) 末尾の「県」の除去、という一連の処理を上から下へ読めます。

名寄せのポイント
  • 全角/半角や余分な空白、末尾の「都・道・府・県」を揃えると結合ミスが激減する。
  • 実データに旧字体表記（例:「鋏」→「挟」）や略称が混ざる場合は、norm_pref() に辞書置換を追加する。
  • 名寄せの成功は、次のステップの anti_join() でゼロ件になることで検証する。

16.5 ダミーデータの作成（英語教育指標）

16.5.1 分析シナリオ

本章では以下のシナリオに基づくシミュレーションデータを使用します。

推奨: まずは quantile で全体像を把握し、必要に応じて jenks や equal と比較して解釈の頑健性を確認する。

16.8 空間自己相関の初歩（Moran’s I）

16.8.1 なぜ空間自己相関を考える必要があるのか

コロプレス図を眺めて「高スコアの都道府県が集まっているように見える」と感じたとしても、それだけでは科学的な主張にはなりません。ランダムなデータでも、たまたまそう見えるパターンが出現することがあります。Moran’s I は、観察された空間パターンが偶然で説明できるかどうかを統計的に検定する指標です。

直感的には、Moran’s I はPearson の相関係数 \(r\) の空間版です。

  • Pearson の \(r\): 2つの変数 \(X\) と \(Y\) の間の線形関連の強さを測る。
  • Moran’s \(I\): ある地点の値と、その隣接地域の加重平均値との間の線形関連を測る。

\(I > 0\) ならば、似た値が近接に集まる傾向（正の空間自己相関）。\(I < 0\) ならば、異なる値が隣り合う傾向。\(I \approx 0\) ならば、空間的にランダムな分布です。

英語教育研究にとっての意味

空間自己相関が存在する場合、以下の2つの含意があります。

  1. 統計的な問題: 通常のOLS回帰は「観測が独立」を前提とする。空間相関があると標準誤差が過小になり、p値が不当に小さくなる（第一種の過誤率が増える）。次章で扱う空間回帰モデルが必要になる場面。
  2. 実質的な関心: 高い英語力が地理的にクラスタしているなら、何がそのパターンを生んでいるのか。地域経済？教育投資の歴史？教員の流動性？空間パターンの存在自体が、新たな研究課題を提起する。

16.8.4 ローカル Moran（LISA クラスタ図）

グローバル Moran’s I は「全体として空間自己相関があるか」を教えてくれますが、「どの都道府県がクラスタの一部か」は教えてくれません。ローカル Moran’s I（LISA: Local Indicators of Spatial Association）は、各地域を以下の4タイプに分類します。

  • High-High（HH）: 自身も隣接も高い。「ホットスポット」。
  • Low-Low（LL）: 自身も隣接も低い。「コールドスポット」。
  • High-Low（HL）: 自身は高いが隣接は低い。空間的アウトライヤ。
  • Low-High（LH）: 自身は低いが隣接は高い。空間的アウトライヤ。
コード 
# ローカル Moran（spdepベース；両側p値を計算）
loc <- spdep::localmoran(dat_moran$eng_score, lw, zero.policy = TRUE)
dat_moran$Ii <- loc[, 'Ii']
p_two <- 2 * pnorm(-abs(loc[, "Z.Ii"]))

z      <- as.numeric(scale(dat_moran$eng_score))
lag_z  <- spdep::lag.listw(lw, z, zero.policy = TRUE)

cluster <- ifelse(p_two >= 0.05, 'Not Sig',
           ifelse(z>0 & lag_z>0, 'High-High',
           ifelse(z<0 & lag_z<0, 'Low-Low',
           ifelse(z>0 & lag_z<0, 'High-Low', 'Low-High'))))

dat_moran$cluster <- factor(cluster, levels = c('High-High','Low-Low','High-Low','Low-High','Not Sig'))

ggplot(dat_moran) +
  geom_sf(aes(fill = cluster), color = 'white') +
  scale_fill_manual(
    values = c('High-High'='#d73027','Low-Low'='#4575b4','High-Low'='#fdae61','Low-High'='#74add1','Not Sig'='grey90'),
    drop = FALSE
  ) +
  theme_bw() +
  labs(title = "LISA Cluster Map (Queen, p < .05)", fill = 'Cluster')

コード 
policy_year <- 2021

  mutate(period = if_else(year <= policy_year, 'pre', 'post')) |>
  group_by(pref_name, period) |>
  summarise(mean_score = mean(eng_score, na.rm = TRUE), .groups = 'drop') |>
  tidyr::pivot_wider(names_from = period, values_from = mean_score) |>
  mutate(diff_post_pre = post - pre)

map_diff <- pref_sf |>
  left_join(prepost, by = 'pref_name')

ggplot(map_diff) +
  geom_sf(aes(fill = diff_post_pre), color = 'white') +
  scale_fill_gradient2(low = 'blue', mid = 'white', high = 'red', midpoint = 0,
                       name = 'Diff (Post - Pre)') +

読み方: 赤い地域は施策後にスコアが上昇、青い地域は低下した都道府県です。ただし、この地図だけでは施策の効果は判定できません。全国的なトレンド（時間固定効果）や各都道府県固有の特性（都道府県固定効果）を統制した上での比較が必要です。次章で DID の枠組みを使ってこの問題に取り組みます。

前後差分≠因果効果

施策前後の差分は「相関」であって「因果」ではありません。施策の有無に関わらず全国的にスコアが上昇していれば、施策導入地域の差分も正になります。因果効果の推定には、対照群との比較（DID）や、時間/地域の固定効果による交絡の統制が必要です。次章の Section 7 で詳述します。

16.11 トラブルシューティング

16.11.1 データ結合の問題

  • left_join() 後に値が全部 NA: キー列の表記ゆれが原因。norm_pref() で両側を正規化し、anti_join() でミスマッチを検出する。
  • 結合後に行数が増えた: キー列に重複がある。count(pref_name, year) で1行以上のレコードがないか確認する。
  • 47都道府県に満たない: 形状データまたは指標データで一部の都道府県が欠損している。n_distinct(pref_name) で確認する。

16.11.2 形状データと CRS の問題

  • 点とポリゴンの場所が合わない: CRS が揃っていない。st_crs() で両方を確認し、st_transform() で統一する。
  • 面積・距離がおかしい: 測地座標（EPSG:4326）で面積計算をしている。投影座標（例: JGD2011、st_transform(x, 6668)）に変換してから計測する。
  • 無効ジオメトリ（self-intersection）: st_is_valid() で検出し、st_make_valid() で修復する。

16.11.3 可視化の問題

  • 年次比較で色の印象がブレる: limits を固定していない。scale_fill_viridis_c(limits = global_limits) で全パネル共通のスケールを指定する。
  • geom_sf() が重い/描画が遅い: 高解像度の形状データを使用している。試作段階では st_simplify() で簡略化するか、低解像度形状を使う。
  • 日本語が文字化け（豆腐化）: YAML の mainfont や R セッションのロケール（Sys.setlocale()）を見直す。Noto Serif JP / Noto Sans JP を推奨。

16.11.4 パネル整形の問題

  • object 'pref_map' not found: チャンクの実行順序が原因。Quarto で上から順に実行されていることを確認する。
  • Inf や NaN が混入: 伸び率のゼロ割。if_else(prev == 0, NA, ...) でガードするか、対数差分を使う。
  • 欠損年が暗黙に無視されている: complete() で明示的にパネルの格子を作成する。

16.11.5 空間分析の問題

  • regions with no neighbours: 島嶼部で孤立ノードが発生。kNN の k を引き上げるか、距離閾値で接続する。
  • not a listw object: nb を直接渡している。nb2listw(nb, style = 'W') で変換する。
  • sfdep と spdep の関数名の違いで混乱: 本書では spdep を基本パッケージとして使用。sfdep::local_moran_perm 等を使いたい場合は別途インストールが必要。
提出前チェックリスト
  • anti_join() の結果が 0 件か（名寄せ成功の確認）
  • st_crs() が揃っているか（CRS の統一）
  • ファセット比較でカラースケールが共通化されているか
  • st_is_valid() で無効ジオメトリがないか
  • パネルの行数 = 都道府県数 × 年数 であるか

## はじめに

英語教育の研究データには、しばしば**地理的な構造**が埋め込まれています。たとえば、以下の場面を考えてみてください。

1. **全国学力調査の結果を都道府県別に比較したい**。どの地域の英語スコアが高く、どの地域が低いのか。単なる順位表ではなく、地図上に可視化すれば、地理的パターン（例: 都市圏に高スコアが集中、あるいは東西で傾向が異なる）が一目でわかる。
2. **教員の英語力指標（TOEICスコア、英検取得率）の年次変化を追跡したい**。7年間の変化は全国で一様なのか、それとも特定の地域だけが急速に改善（または停滞）しているのか。年ごとの地図を並べれば、時空間的なトレンドが浮かび上がる。
3. **2021年に導入されたALT配置拡充施策の効果を評価したい**。施策導入前と導入後で英語スコアはどう変化したか。その変化は施策実施地域に限定されるのか、隣接地域にも波及しているのか。

これらの問いに答えるには、**数表やグラフだけでなく地図が不可欠**です。そして地図を作るには、都道府県の「形」（境界線のポリゴンデータ）と分析データを結びつける技術が必要です。

本章は、そのための**基礎工事**です。Rの `sf` パッケージを使って地理空間データを扱う方法を、都道府県別の英語教育指標データを題材に学びます。具体的には、形状データの取得、属性データとの結合、コロプレス図（塗り分け地図）の作成、パネルデータの整形までを最小限のコードで体験します。

- **Part A（Section 3--6）: 地図を描く** --- 形状データの取得から、データ結合、コロプレス図、年次比較まで。「都道府県別の英語スコアを地図上に描く」という目標を達成する。

- **Part C（Section 9--10）: 空間パターンを測る** --- Moran's I による空間自己相関の初歩的な検定と、施策前後差分の可視化。次章（応用編）への橋渡し。

#### 「形状＋属性」の二重構造

地理空間分析のデータは、常に2つの要素から構成されます。

1. **形状データ（geometry）**: 都道府県の境界線を表すポリゴン。「どこにあるか」の情報。
2. **属性データ（attributes）**: 英語スコア、教員TOEIC、英検取得率などの数値。「何がどのくらいか」の情報。

この2つを `left_join()` で結合して初めて「地図上に色を塗る」ことができます。通常の回帰分析では属性データだけを扱いますが、空間分析では形状データが常に伴走する点が特徴です。
:::

## 形状データの取得

### sf オブジェクトとは

`sf`（Simple Features）パッケージは、地理空間データをRのデータフレームとして扱えるようにする仕組みです。通常のデータフレームに `geometry` という特殊な列が追加されているだけで、`dplyr` の関数がそのまま使えます。

::: {.callout-tip}
#### sf を普通のデータフレームとの対比で理解する

```r
# 通常のデータフレーム
tibble(pref = "東京都", score = 72.5)

# sf オブジェクト（+ geometry 列）
tibble(pref = "東京都", score = 72.5, geometry = <POLYGON(...)>)
```

- `sf` は「データフレーム + 地理座標」。既存のデータ操作の知識がそのまま活きる。
- `geometry` 列は `ggplot2` の `geom_sf()` が自動的に読み取り、地図を描画する。
- 空間操作が不要な場面では `st_drop_geometry()` で `geometry` を外し、軽量なテーブルに戻せる。
:::

### 形状データの取得（フォールバック付き）

日本の都道府県境界ポリゴンを取得します。パッケージの利用可能性は環境によって異なるため、3段階のフォールバックを設けています。

1. **jpndistrict**（推奨）: 日本専用パッケージ。高品質な都道府県境界を提供。
2. **rnaturalearth**: 世界の行政区画データ。日本の都道府県レベル（Admin-1）を取得可能。
3. **geodata（GADM）**: 別のグローバルデータソース。

```{r}
#| label: load_shape
load_pref_sf <- function(){
  # 1) jpndistrict（推奨）
  jp_try <- try({ jpndistrict::jpn_pref(quality = 40) }, silent = TRUE)
  if (!inherits(jp_try, "try-error") && inherits(jp_try, "sf")) {
    return(jp_try |>
             st_transform(4326) |>
             dplyr::rename(pref_name = prefecture) |>
             dplyr::select(pref_name, geometry) |>
             sf::st_make_valid())
  }

  # 2) rnaturalearth（Admin-1, medium scale 明示）
  ne_try <- try({ rnaturalearth::ne_states(country = "Japan", returnclass = "sf") }, silent = TRUE)
  if (!inherits(ne_try, "try-error") && inherits(ne_try, "sf")) {
    return(ne_try |>
             dplyr::transmute(pref_name = name, geometry = geometry) |>
             st_transform(4326) |>
             sf::st_make_valid())
  }

  # 3) geodata::gadm（GADM level-1：都道府県）
  gd_try <- try({ geodata::gadm(country = "JPN", level = 1, path = tempdir()) }, silent = TRUE)
  if (!inherits(gd_try, "try-error")) {
    gd_sf <- sf::st_as_sf(gd_try)
    # 一般的な列名を合わせる（NAME_1 が都道府県名）
    name_col <- intersect(names(gd_sf), c("NAME_1", "NAME_1_", "VARNAME_1"))
    if (length(name_col) == 0) name_col <- names(gd_sf)[1]
    return(gd_sf |>
             dplyr::transmute(pref_name = .data[[name_col[1]]], geometry = geometry) |>
             sf::st_make_valid() |>
             sf::st_transform(4326))
  }

  stop("都道府県ポリゴンの取得に失敗しました。'jpndistrict' または 'rnaturalearth' の利用可能性をご確認ください。")
}

# 実境界のみを使用（ダミーhexには絶対フォールバックしない）
pref_sf <- load_pref_sf()
```

::: {.callout-note}
#### CRS（座標参照系）とは

地球は球体ですが、コンピュータの画面は平面です。CRS は「球体上の位置を平面上の座標に変換する方法」を定義する規約です。

- **EPSG:4326（WGS84）**: 緯度・経度で位置を表す。GPSや多くのWebサービスが使用。本章の可視化はこれで十分。
- **EPSG:3857（Web メルカトル）**: Google Maps や OpenStreetMap が使用する投影法。
- **JGD2011系**: 日本の公式測量に使われる投影座標。面積や距離の計算にはこちらが正確。

**実務のルール**: 可視化は EPSG:4326 で行い、面積・距離の計測が必要な場合は投影座標（例: `st_transform(x, 6668)`）に変換してから実施します。`st_crs()` で都度確認する習慣をつけてください。
:::

### 隣接の定義：Queen と Rook

地図を描くだけなら「隣接」を考える必要はありませんが、次章以降で空間自己相関や空間回帰を行う際に「誰が誰の隣人か」を定義する必要があります。ここでは3つの代表的な隣接定義を紹介します。

- **Queen 型**: **辺または頂点**を共有していれば隣接とみなす。チェスのクイーンが八方向に動けるイメージ。つながりが多く、空間的影響を広く捉える。
- **Rook 型**: **辺（境界線）を共有**している場合だけ隣接。チェスのルークが上下左右に動くイメージ。Queen より厳格。
- **k 近傍（kNN）**: 各地域の重心から**距離が近い順に k 個**を隣接とする。島や飛地でも必ず隣接が定義でき、孤立ノードを回避できる。

::: {.callout-tip}
#### 英語教育研究での選択指針

- **都道府県レベルの分析**: Queen が良い出発点。日本の都道府県は比較的整った形状で、辺/頂点の区別は実質的な差を生みにくい。
- **島嶼部を含む分析**: 沖縄や離島を含む場合は kNN（例: k = 4--6）を併用し、孤立ノードを回避する。
- **感度分析は必須**: 主要な結論が隣接定義の選択で変わらないかを確認する（次章 Section 4 で詳述）。
:::

以下のコードは参照用です。次章で実際に使用しますが、ここでは概念の理解のために掲載します。

```{r}
#| label: nb_demo
#| eval: false
# Queen（既定は queen=TRUE）
nb_queen <- spdep::poly2nb(pref_sf, queen = TRUE)
lw_queen <- spdep::nb2listw(nb_queen, style = 'W')

# Rook（辺共有のみ）
nb_rook  <- spdep::poly2nb(pref_sf, queen = FALSE)
lw_rook  <- spdep::nb2listw(nb_rook,  style = 'W')

# k 近傍（例: k=6）
knn      <- spdep::knearneigh(sf::st_coordinates(sf::st_centroid(pref_sf)), k = 6)
nb_knn   <- spdep::knn2nb(knn)
lw_knn   <- spdep::nb2listw(nb_knn, style = 'W')
```

::: {.callout-note}
#### `nb` と `listw` の違い

上のコードには `nb` と `listw` の2つのオブジェクトが登場します。

- **`nb`（隣接リスト）**: 「都道府県Aの隣は B, C, D」という**接続情報**のみ。重みなし。
- **`listw`（重み付き隣接）**: `nb` に**重み**を付加したもの。`style = 'W'`（行標準化）は、各地域の重みの合計が1になるように調整する。

空間分析の関数（`moran.test()`, `lagsarlm()` 等）は `listw` を入力として要求します。`nb` をそのまま渡すと「not a listw object」というエラーが出るので注意してください。
:::

## 都道府県名の正規化（名寄せ）

形状データの都道府県名と、分析対象の指標データの都道府県名は、そのままでは一致しないことがあります。全角/半角、余分な空白、末尾の「都・道・府・県」の有無といった表記の揺れが原因です。以下の `norm_pref()` 関数は、この揺れを吸収して名寄せを可能にします。

```{r}
#| label: norm_pref
# 正規化関数：全角/半角・空白・末尾の都/道/府/県の扱い
norm_pref <- function(x){
  x |>
    stringi::stri_trans_nfkc() |>
    stringr::str_replace_all("\\s+", "") |>
    stringr::str_replace("(東京都|北海道|京都府|大阪府)$", "") |>
    stringr::str_replace("県$", "")
}

# 形状側の正規化キー
jpn_key <- pref_sf |>
  st_drop_geometry() |>
  transmute(pref_name,
            pref_key = norm_pref(pref_name))
```

::: {.callout-tip}
#### コードの読み方（パイプチェーン）

本章のRコードでは `|>` （ネイティブパイプ）を多用します。これは「左辺の結果を右辺の関数の第1引数に渡す」操作です。

```r
# 以下の2つは同じ意味:
result <- f(g(h(x)))        # 入れ子（内側から読む）
result <- x |> h() |> g() |> f()  # パイプ（左から右へ順に読む）
```

`norm_pref()` の中身を読むと: (1) NFKC正規化（全角→半角統一）→ (2) 空白除去 → (3) 「東京都/北海道/京都府/大阪府」の除去 → (4) 末尾の「県」の除去、という一連の処理を上から下へ読めます。
:::

::: {.callout-note}
#### 名寄せのポイント

- 全角/半角や余分な空白、末尾の「都・道・府・県」を揃えると結合ミスが激減する。
- 実データに旧字体表記（例:「鋏」→「挟」）や略称が混ざる場合は、`norm_pref()` に辞書置換を追加する。
- 名寄せの成功は、次のステップの `anti_join()` でゼロ件になることで検証する。
:::

## ダミーデータの作成（英語教育指標）

### 分析シナリオ

本章では以下のシナリオに基づくシミュレーションデータを使用します。

> **シナリオ**: 2018--2024年の7年間、47都道府県について3つの英語教育指標を追跡する。`eng_score`（全国英語学力調査の複合スコア）、`teacher_toeic`（公立学校教員の平均TOEICスコア）、`eiken_rate`（中学3年生の英検準2級以上取得率）。分析の目的は、これらの指標の地理的分布パターンと年次変化を把握すること。

::: {.callout-note}
#### なぜシミュレーションデータなのか

シミュレーションデータでは**真のパラメータ**（例: トレンドの傾き、地域差の大きさ）が研究者に既知です。分析手法が意図通りに機能しているかを、推定値と真値の比較で検証できます。実データで「おかしな結果」が出たとき、それが手法の誤りなのかデータの特性なのかを判別する力を養うためにも、まずシミュレーションで基本動作を確認することが重要です。
:::

```{r}
#| label: dummy_data
set.seed(2024)
years <- 2018:2024
pref_panel <- expand.grid(pref_name = jpn_key$pref_name, year = years) |>
  as_tibble() |>
  mutate(
    eng_score = round(rnorm(n(), mean = 55 + as.numeric(factor(pref_name))%%10, sd = 7) + (year-2018)*0.6, 1),
    teacher_toeic = round(rnorm(n(), mean = 650 + as.numeric(factor(pref_name))%%15, sd = 45) + (year-2018)*5, 0),
    eiken_rate = pmin(pmax(round(runif(n(), .15, .65) + (year-2018)*.01, 2), 0), 1)
  )

pref_latest <- pref_panel |> filter(year == max(year))
```

### 結合前のミスマッチ検出

形状データと指標データを結合する前に、**名寄せの成功を検証**します。`anti_join()` は「一方にあるが他方にない」レコードを抽出する関数で、ミスマッチの発見に威力を発揮します。

```{r}
#| label: mismatch_check
# 両側の正規化キーでミスマッチを検出
pref_panel_keyed <- pref_panel |> mutate(pref_key = norm_pref(pref_name))
anti_data_vs_shape <- dplyr::anti_join(pref_panel_keyed, jpn_key, by = 'pref_key')
anti_shape_vs_data <- dplyr::anti_join(jpn_key, pref_panel_keyed, by = 'pref_key')

cat("指標データにあるが形状にない:", nrow(anti_data_vs_shape), "件\n")
cat("形状にあるが指標データにない:", nrow(anti_shape_vs_data), "件\n")
```

**確認ポイント**: 両方とも 0 件であれば名寄せ成功です。もし 0 でなければ、`norm_pref()` に辞書置換を追加するか、元データの都道府県名を修正してください。

#### `left_join()` の役割

`left_join(A, B, by = 'key')` は「Aのすべての行を保持し、Bの中でキーが一致する列を追加する」操作です。ここでは形状データ（`pref_sf`）を左側に置くことで、**47都道府県のポリゴンがすべて保持**されます。もし指標データ側に該当する都道府県がなければ `NA` が入ります。

### コロプレス図（塗り分け地図）

コロプレス図は、各地域を数値の大小に応じた色で塗り分ける地図です。英語教育研究では、全国の指標を俯瞰的に把握するための基本ツールとなります。

以下では、`classInt` パッケージの分位点区分（quantile）を用いて5段階に分類し、`ggplot2` で描画します。

```{r}
#| label: choropleth_latest

cuts <- cut(pref_map$eng_score, breaks = cls$brks, include.lowest = TRUE)

ggplot(pref_map) +

```

**地図の読み方**: 濃い青が高スコア、薄い青が低スコアの地域です。高スコアの都道府県が地理的にまとまっているか（クラスタ）、それともランダムに散在しているかを観察してください。もしクラスタが見えるなら、それが次章で検定する**空間自己相関**の視覚的な手がかりです。

::: {.callout-tip}
#### ggplot2 で地図を描くパターン

本章で繰り返し登場する `ggplot2` のパターンを整理します。

```r
ggplot(sf_data) +                         # sf オブジェクトを指定
  geom_sf(aes(fill = variable)) +         # fill で色分け変数を指定
  scale_fill_brewer(palette = 'Blues') +   # カラーパレット（カテゴリ）
  theme_bw() +                       # シンプルなテーマ
  labs(title = '...', fill = '...')       # タイトル・凡例ラベル
```

- **`geom_sf()`**: `sf` オブジェクトの `geometry` 列を自動的に読み取り、地図を描画する。
- **`aes(fill = ...)`**: 塗り分けに使う変数を指定する。連続値なら `scale_fill_viridis_c()`、カテゴリなら `scale_fill_brewer()` や `scale_fill_manual()` を使う。
- **`theme_bw()`**: 背景やグリッドをシンプルにし、データに焦点を当てる。学術論文向き。
:::

::: {.callout-note}
#### 階級区分の選び方

コロプレス図の印象は、階級区分の方法に大きく左右されます。同じデータでも区分を変えると全く異なる印象の地図になり得るため、選択には注意が必要です。

- **quantile（分位点）**: データを均等なグループに分割する。順位比較に向くが、数値的な差の大きさは反映しない。
- **equal（等間隔）**: 値域を等間隔に分割する。直感的だが、外れ値の影響を受けやすい。
- **jenks（自然区分）**: データの「谷間」を自動検出し、自然なまとまりで分割する。探索的分析に有効。

**推奨**: まずは quantile で全体像を把握し、必要に応じて jenks や equal と比較して解釈の頑健性を確認する。
:::

### 年次比較（小さな倍図: small multiples）

複数年のコロプレス図を `facet_wrap()` で並べ、時間的な変化パターンを視覚的に把握します。**カラースケールは全年で統一**することが鉄則です。年ごとにスケールが異なると、色の比較が不可能になります。

```{r}

ggplot(pref_panel_map) +
  geom_sf(aes(fill = eng_score), color = 'white', size = .1) +
  scale_fill_viridis_c(option = 'C', name = 'Score',
                       limits = global_limits, oob = scales::squish) +
  facet_wrap(~ year, ncol = 4) +
  theme_bw(base_size = 10) +
  theme(axis.text = element_blank(), axis.ticks = element_blank()) +

```

::: {.callout-tip}
#### small multiples の読み方

年次比較の小地図（small multiples）を効果的に読むポイント:

- **全体的な色の変化**: 年を追うごとに全体的に暖色化（スコア上昇）しているか。
- **局所的な異変**: 特定の地域だけが急激に変化していないか。これは測定方法の変更や外れ値の兆候かもしれない。
- **空間的パターンの持続性**: 高スコア地域と低スコア地域の位置関係は年を通じて安定しているか、それとも変動しているか。
- **凡例の確認**: `limits = global_limits` により全パネルで共通のスケールになっている。スケールが共通であることを確認してから比較する。
:::

```{r}
  ungroup()
```

```{r}
  select(pref_name, year, eng_score, growth_pct, log_diff_pct) |>

```

- **欠損年の見落とし**: `complete()` で明示的に格子を作らないと、単に「行がない＝0」と誤解される。
- **キーの重複**: 同一都道府県が2行ある場合、`left_join()` で行が膨張する。`count()` で事前に確認する。

## 空間自己相関の初歩（Moran's I）

### なぜ空間自己相関を考える必要があるのか

コロプレス図を眺めて「高スコアの都道府県が集まっているように見える」と感じたとしても、それだけでは科学的な主張にはなりません。ランダムなデータでも、たまたまそう見えるパターンが出現することがあります。**Moran's I** は、観察された空間パターンが偶然で説明できるかどうかを統計的に検定する指標です。

直感的には、Moran's I は**Pearson の相関係数 $r$ の空間版**です。

- Pearson の $r$: 2つの変数 $X$ と $Y$ の間の線形関連の強さを測る。
- Moran's $I$: ある地点の値と、その**隣接地域の加重平均値**との間の線形関連を測る。

$I > 0$ ならば、似た値が近接に集まる傾向（正の空間自己相関）。$I < 0$ ならば、異なる値が隣り合う傾向。$I \approx 0$ ならば、空間的にランダムな分布です。

::: {.callout-note}
#### 英語教育研究にとっての意味

空間自己相関が存在する場合、以下の2つの含意があります。

1. **統計的な問題**: 通常のOLS回帰は「観測が独立」を前提とする。空間相関があると標準誤差が過小になり、p値が不当に小さくなる（第一種の過誤率が増える）。次章で扱う空間回帰モデルが必要になる場面。
2. **実質的な関心**: 高い英語力が地理的にクラスタしているなら、何がそのパターンを生んでいるのか。地域経済？教育投資の歴史？教員の流動性？空間パターンの存在自体が、新たな研究課題を提起する。
:::

### グローバル Moran's I の計算

```{r}
#| label: moran_global
dat_moran <- pref_map |>
  filter(!is.na(eng_score))

# 隣接（まず Queen）。孤立があれば k近傍で補完し、zero.policy を有効化
nb <- spdep::poly2nb(dat_moran, queen = TRUE)
deg <- spdep::card(nb)
augmented <- FALSE
if (any(deg == 0)) {
  coords <- sf::st_coordinates(sf::st_centroid(dat_moran))
  k <- 1
  repeat {
    nb_knn <- spdep::knn2nb(spdep::knearneigh(coords, k = k))
    nb <- spdep::union.nb(nb, nb_knn)
    if (all(spdep::card(nb) > 0) || k >= 8) break
    k <- k + 1
  }
  augmented <- TRUE
}
lw <- spdep::nb2listw(nb, style = 'W', zero.policy = TRUE)

# グローバル Moran（孤立許容）
moran_res <- spdep::moran.test(dat_moran$eng_score, lw, zero.policy = TRUE)
moran_res

if (augmented) message('[注意] 隣接に孤立があったため、k近傍で補完しました。')
```

::: {.callout-tip}
#### 出力の読み方

- **Moran I statistic**: I の値。0 に近ければ空間自己相関なし、正に大きければ正の自己相関。
- **Expectation**: ランダムな配置の場合に期待される I の値（$-1/(n-1)$）。
- **p-value**: 帰無仮説「空間自己相関なし」の下での有意確率。p < .05 ならば空間自己相関が統計的に有意。
:::

### ローカル Moran（LISA クラスタ図）

グローバル Moran's I は「全体として空間自己相関があるか」を教えてくれますが、「**どの都道府県が**クラスタの一部か」は教えてくれません。ローカル Moran's I（LISA: Local Indicators of Spatial Association）は、各地域を以下の4タイプに分類します。

- **High-High（HH）**: 自身も隣接も高い。「ホットスポット」。
- **Low-Low（LL）**: 自身も隣接も低い。「コールドスポット」。
- **High-Low（HL）**: 自身は高いが隣接は低い。空間的アウトライヤ。
- **Low-High（LH）**: 自身は低いが隣接は高い。空間的アウトライヤ。

```{r}
#| label: lisa_map
# ローカル Moran（spdepベース；両側p値を計算）
loc <- spdep::localmoran(dat_moran$eng_score, lw, zero.policy = TRUE)
dat_moran$Ii <- loc[, 'Ii']
p_two <- 2 * pnorm(-abs(loc[, "Z.Ii"]))

z      <- as.numeric(scale(dat_moran$eng_score))
lag_z  <- spdep::lag.listw(lw, z, zero.policy = TRUE)

cluster <- ifelse(p_two >= 0.05, 'Not Sig',
           ifelse(z>0 & lag_z>0, 'High-High',
           ifelse(z<0 & lag_z<0, 'Low-Low',
           ifelse(z>0 & lag_z<0, 'High-Low', 'Low-High'))))

dat_moran$cluster <- factor(cluster, levels = c('High-High','Low-Low','High-Low','Low-High','Not Sig'))

ggplot(dat_moran) +
  geom_sf(aes(fill = cluster), color = 'white') +
  scale_fill_manual(
    values = c('High-High'='#d73027','Low-Low'='#4575b4','High-Low'='#fdae61','Low-High'='#74add1','Not Sig'='grey90'),
    drop = FALSE
  ) +
  theme_bw() +
  labs(title = "LISA Cluster Map (Queen, p < .05)", fill = 'Cluster')
```

**読み方**: 赤い地域（HH）は英語力のホットスポット（高スコアの集積地）、青い地域（LL）はコールドスポット（低スコアの集積地）です。オレンジや水色の地域は空間的アウトライヤで、周囲と異なるパターンを示す都道府県です。灰色は統計的に有意でない地域。

## トラブルシューティング

### データ結合の問題

- **`left_join()` 後に値が全部 NA**: キー列の表記ゆれが原因。`norm_pref()` で両側を正規化し、`anti_join()` でミスマッチを検出する。
- **結合後に行数が増えた**: キー列に重複がある。`count(pref_name, year)` で1行以上のレコードがないか確認する。
- **47都道府県に満たない**: 形状データまたは指標データで一部の都道府県が欠損している。`n_distinct(pref_name)` で確認する。

### 形状データと CRS の問題

- **点とポリゴンの場所が合わない**: CRS が揃っていない。`st_crs()` で両方を確認し、`st_transform()` で統一する。
- **面積・距離がおかしい**: 測地座標（EPSG:4326）で面積計算をしている。投影座標（例: JGD2011、`st_transform(x, 6668)`）に変換してから計測する。
- **無効ジオメトリ（self-intersection）**: `st_is_valid()` で検出し、`st_make_valid()` で修復する。

### 可視化の問題

- **年次比較で色の印象がブレる**: `limits` を固定していない。`scale_fill_viridis_c(limits = global_limits)` で全パネル共通のスケールを指定する。
- **`geom_sf()` が重い/描画が遅い**: 高解像度の形状データを使用している。試作段階では `st_simplify()` で簡略化するか、低解像度形状を使う。
- **日本語が文字化け（豆腐化）**: YAML の `mainfont` や R セッションのロケール（`Sys.setlocale()`）を見直す。Noto Serif JP / Noto Sans JP を推奨。

### パネル整形の問題

- **`object 'pref_map' not found`**: チャンクの実行順序が原因。Quarto で上から順に実行されていることを確認する。
- **`Inf` や `NaN` が混入**: 伸び率のゼロ割。`if_else(prev == 0, NA, ...)` でガードするか、対数差分を使う。
- **欠損年が暗黙に無視されている**: `complete()` で明示的にパネルの格子を作成する。

### 空間分析の問題

- **`regions with no neighbours`**: 島嶼部で孤立ノードが発生。kNN の k を引き上げるか、距離閾値で接続する。
- **`not a listw object`**: `nb` を直接渡している。`nb2listw(nb, style = 'W')` で変換する。
- **`sfdep` と `spdep` の関数名の違いで混乱**: 本書では `spdep` を基本パッケージとして使用。`sfdep::local_moran_perm` 等を使いたい場合は別途インストールが必要。

::: {.callout-tip}
#### 提出前チェックリスト

- `anti_join()` の結果が 0 件か（名寄せ成功の確認）
- `st_crs()` が揃っているか（CRS の統一）
- ファセット比較でカラースケールが共通化されているか
- `st_is_valid()` で無効ジオメトリがないか
- パネルの行数 = 都道府県数 × 年数 であるか
:::
