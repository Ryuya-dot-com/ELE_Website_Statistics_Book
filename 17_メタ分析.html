19.1 メタ分析とは何か――なぜ個別の研究では不十分なのか

「明示的な文法指導は英語ライティングの正確性を向上させるか」という研究課題を考える。先行研究を渉猟すると、効果を報告する研究もあれば、そうでない研究もあり、一見して知見が錯綜している。このような状況において、個別の研究結果をどのように統合し、全体的な傾向を把握すればよいのか――これがメタ分析の出発点である。

メタ分析（meta-analysis）とは、同一の研究課題に取り組んだ複数の実証研究の結果を定量的に統合する統計手法である。各研究を独立した「データポイント」として扱い、統計的に合成することで、個々の研究が抱えるサンプルサイズの制約や結果のばらつきを克服し、より信頼性の高い結論を導くことが可能となる。

ナラティブ・レビュー（先行研究を定性的にまとめる従来型の文献検討）と比較した場合、メタ分析には以下の利点がある。

  • 定量的な統合: 「多くの研究で効果が認められた」という質的判断ではなく、「平均効果は g = 0.42, 95% CI [0.18, 0.66]」と数値で提示できる
  • 異質性の検討: 効果が研究間でどの程度異なるかを定量的に評価できる（後述する \(\tau^2\), \(I^2\) などの指標）
  • モデレーター分析: 「どのような条件下で効果が大きい（あるいは小さい）か」を体系的に探索できる
本章の位置づけ

本章は、メタ分析の実施手順を R の metafor パッケージを中心に解説する実践的なガイドである。効果量の基礎概念については「1_T検定と効果量.qmd」を参照されたい。本章では、複数の研究から得られた効果量を統合する手続きに焦点を当てる。

Plot labels

文字化け（mojibake）を防ぐため、本章のすべての図は英語ラベルで描画している。各図の読み方は本文中で日本語により解説する。

メタ分析は、効果量の算出から異質性の評価、モデレーター分析、出版バイアスの検討まで、段階的に進めていきます。異質性の大きさによって次のステップが分岐します。

実務手順としては、まず研究を収集して効果量を算出し、ランダム効果モデルでプール推定を行います。次に (I2)、(Q)、(2) で異質性を評価し、異質性が大きい場合はサブグループ分析やメタ回帰でモデレーターを探索します。異質性が小さい場合はプール推定値を中心に報告し、どちらの経路でも最終段階でファンネルプロットや Trim-and-Fill による出版バイアス評価を実施します。最後に PRISMA に準拠して報告します。

ノート

英語教育研究のメタ分析では、介入の種類・学習者レベル・アウトカム指標などが異質性の主な要因となります。異質性が大きい場合（\(I^2 > 75\%\)）は、必ずモデレーター分析で要因を探索してください。

rma_to_gt <- function(fit, title = "Meta-Analysis Results") {
  s <- coef(summary(fit))
  tbl <- tibble::tibble(
    Term     = rownames(s),
    Estimate = s[, "estimate"],
    SE       = s[, "se"],
    z        = s[, "zval"],
    p        = s[, "pval"],
    CI_lower = s[, "ci.lb"],
    CI_upper = s[, "ci.ub"]
  )
  tbl |>
    gt() |>
    tab_header(title = title) |>
    fmt_number(columns = c(Estimate, SE, z, CI_lower, CI_upper), decimals = 3) |>
    fmt_number(columns = p, decimals = 4)
}

19.3 PART I: ペアワイズ・メタ分析

本章の前半では、最も基本的な形式である「2群の比較を複数の研究から統合する」ペアワイズ・メタ分析を扱う。学位論文や学術論文において最も頻繁に用いられる分析手法である。

19.3.1 Recipe 1: 効果量の選択――統合の単位を決定する

メタ分析の第一段階は、各研究の結果を共通の尺度、すなわち効果量（effect size）に変換することである。

どの効果量を選択すべきか

選択の基準は、アウトカム（従属変数）の測定尺度によって決まる。

アウトカムの種類 典型例 効果量 escalc() の measure
連続値（同一テスト） TOEFL点数の差 平均差 (MD) "MD"
連続値（異なるテスト） ライティング得点 vs. 文法テスト 標準化平均差 (Hedges g) "SMDH"
二値（成功/失敗） 合格率 対数オッズ比 (log OR) "OR"
相関 動機づけと成績の関連 Fisher z "ZCOR"

英語教育研究では、異なるテストや尺度を用いた研究を統合する場面が多いため、標準化平均差（Hedges g）が最も広く用いられる。

Hedges g の定義

Hedges g は、2群の平均差をプールされた標準偏差で割った値に、小標本バイアスの補正を施した指標である。

\[ g = J(df) \times \frac{\bar{X}_1 - \bar{X}_2}{S_{\text{pooled}}} \]

ここで \(J(df)\) は小標本補正係数であり、サンプルサイズが小さい場合に効果量がやや過大に推定される傾向を修正する。解釈の目安として、\(|g|\) = 0.2（小）、0.5（中）、0.8（大）が広く参照される(Cohen, 1988)。ただし、この基準は分野や研究文脈によって適切性が異なるため、機械的な適用は避けるべきである。応用言語学においては、Plonsky & Oswald (2014) の分野固有の基準（\(d\) = 0.40 で small, 0.70 で medium, 1.00 で large）も参照されたい。

効果量の方向の定義

メタ分析に着手する前に、正の効果量が何を意味するかを明確に定義する必要がある。本章では一貫して「正の値 = 介入（明示的文法指導）が統制群に対して有利」と定義する。この定義をコーディングシートの段階で統一しておくことが、分析全体の整合性を確保するうえで不可欠である。

19.3.2 Recipe 2: データの準備――英語教育研究のシナリオ

分析のシナリオ

以下の仮想的なシナリオに基づき、分析の手順を解説する。

リサーチ・クエスチョン: 明示的な文法指導（Explicit Grammar Instruction）は、EFL/ESLの英語ライティングにおける文法的正確性を向上させるか？

想定する研究の特徴は以下の通りである。

  • 介入群: 明示的文法指導を受けた学習者
  • 統制群: 通常の授業を受けた学習者
  • アウトカム: ライティングテストの得点（連続値）
  • モデレーター候補: 教育段階（中等教育 vs. 高等教育）、介入期間（週数）

以下では 30 件の研究をシミュレーションにより生成する。真のパラメータとして、ベースラインの効果量 g = 0.35、高等教育で +0.10、介入 1 週あたり +0.01 を設定している。

コード 
n_study <- 30
study_id <- sprintf("Study_%02d", 1:n_study)

# モデレーター
level <- sample(c("secondary", "tertiary"), n_study, replace = TRUE, prob = c(0.6, 0.4))
duration <- sample(4:16, n_study, replace = TRUE)
year <- sample(2010:2024, n_study, replace = TRUE)

# 真の効果（ランダム効果モデル）
# ベース 0.35 + 高等教育ボーナス + 週数効果 + 研究間分散 tau^2 = 0.04
tau <- sqrt(0.04)
theta <- 0.35 + ifelse(level == "tertiary", 0.10, 0) +
  0.01 * (duration - 10) + rnorm(n_study, 0, tau)

# 各研究のサンプルサイズと集計統計量
set.seed(2024)
n1 <- sample(25:60, n_study, replace = TRUE)
n2 <- sample(25:60, n_study, replace = TRUE)
sd_pooled <- runif(n_study, 8, 15)
m2 <- rnorm(n_study, 70, 8)
m1 <- m2 + theta * sd_pooled
sd1 <- sd_pooled * runif(n_study, .9, 1.1)
sd2 <- sd_pooled * runif(n_study, .9, 1.1)

raw <- data.frame(study_id, level, duration, year, m1, sd1, n1, m2, sd2, n2)

# Hedges g と分散を計算
es <- escalc(
  measure = "SMDH",
  m1i = m1, sd1i = sd1, n1i = n1,
  m2i = m2, sd2i = sd2, n2i = n2,
  data = raw
)
dat <- cbind(raw, yi = es$yi, vi = es$vi)

escalc() は各研究の集計統計量（平均、SD、サンプルサイズ）から効果量 yi とその分散 vi を算出する。この yi と vi の対がメタ分析の入力データとなる。

コード 
dat |>
  select(study_id, level, duration, yi, vi) |>
  DT::datatable(caption = "最初の10研究の効果量と分散")
実データへの適用

実際のメタ分析では、各論文から平均値・標準偏差・サンプルサイズを抽出し、データフレームに格納する。論文が t 値や F 値のみを報告している場合は、escalc() の引数（ti=, fi= など）や効果量変換パッケージ（esc, effectsize）、オンラインツール（Effect Size Calculator）を用いて変換が可能である。効果量変換の詳細は Recipe 21 で扱う。

19.3.3 Recipe 3: ランダム効果モデルによる統合――分析の中核

固定効果モデルとランダム効果モデルの選択

この選択はデータを観察してから決定するものではなく、研究デザインの性質に基づいて事前に決定する。

  • 固定効果モデル（fixed-effect model）: すべての研究が同一の真の効果を推定していると仮定する。各研究の重みはサンプルサイズ（精度）のみによって決まる。同一の薬剤を同一条件で検証した治験シリーズなどに適している。
  • ランダム効果モデル（random-effects model）: 研究ごとに真の効果が異なりうる（真の効果が分布を持つ）と仮定する。英語教育研究では、学習者の年齢・L1 背景・指導者・テスト形式などが研究間で異なるため、ほぼ常にランダム効果モデルが適切である。

ランダム効果モデルにおける重み \(w_i\) は以下のように定義される。

\[ w_i = \frac{1}{v_i + \hat{\tau}^2} \]

ここで \(v_i\) は各研究の標本分散（サンプリング誤差）、\(\hat{\tau}^2\) は研究間分散の推定値である。固定効果モデル（\(\hat{\tau}^2 = 0\)）と比較すると、小規模な研究に相対的に大きな重みが付与され、信頼区間は広くなる。すなわち、研究間のばらつきに起因する不確実性が推定に反映される。

モデルの適合

コード 
m <- rma(yi, vi, data = dat, method = "REML")
rma_to_gt(m, title = "ランダム効果モデル（REML）の推定結果")
ランダム効果モデル（REML）の推定結果
Term Estimate SE z p CI_lower CI_upper
intrcpt 0.352 0.042 8.395 0.0000 0.270 0.434

出力の解釈

rma() の出力において注目すべきポイントは、大きく 3 点に整理できる。

1. 平均効果量（estimate）

  • 正の値は介入（明示的文法指導）が統制群に対して有利であることを意味する
  • 95% CI が 0 を含まない場合、統計的に有意である
  • ただし、統計的有意性のみならず効果の大きさそのものに注目すべきである。g = 0.10 であっても大規模なメタ分析では有意となりうるが、教育的に意味のある差であるとは限らない

2. 異質性の指標

  • \(\tau^2\)（タウ二乗）: 研究間分散の推定値。\(\sqrt{\tau^2} = \tau\) は効果量と同じ単位で解釈できる。たとえば \(\tau\) = 0.20 であれば、「真の効果量は研究間で SD = 0.20 程度のばらつきを持つ」ことを意味する
  • \(I^2\): 観測されたばらつきのうち、真の異質性（サンプリング誤差に帰属しない部分）が占める割合。概ね 25% = 低、50% = 中、75% = 高とされるが、この基準への過度な依存は避けるべきである
  • \(Q\): 効果量の同質性を検定する統計量。研究数が多い場合にはほぼ常に有意となるため、\(Q\) 検定のみで判断することは適切でない

3. 予測区間（prediction interval）

予測区間は「今後同様の研究を実施した場合、効果量がどの範囲に収まるか」を示す区間である。信頼区間が「平均効果の推定精度」を反映するのに対し、予測区間は「研究間のばらつきを含めた効果の実質的な範囲」を表す。教育実践への含意を論じる際には、予測区間が信頼区間よりも重要な情報を提供する。

コード 
predict(m)

   pred     se  ci.lb  ci.ub  pi.lb  pi.ub 
 0.3926 0.0422 0.3099 0.4753 0.2672 0.5180
解釈における典型的な誤り
  1. p 値のみに基づく結論: 効果量の大きさ・信頼区間・予測区間を必ず併せて報告する
  2. \(I^2\) の過大評価・過小評価: \(I^2\) が低くとも予測区間が広い場合がある。\(I^2\) は「ばらつきの大きさ」ではなく「ばらつきに占める真の異質性の割合」である
  3. 95% CI が 0 を含む = 効果が存在しない: 正確には「本分析においては統計的に有意な効果は検出されなかった」ということであり、効果の大きさ自体の報告には依然として意義がある

19.3.4 Recipe 4: 結果の可視化――フォレストプロットとファンネルプロット

フォレストプロット

フォレストプロットは、メタ分析の結果を報告するうえで最も基本的かつ重要な図である。

コード 
forest(m, slab = dat$study_id, xlab = "Hedges g", refline = 0,
       header = "Study / Effect Size [95% CI]")

図の読み方:

  • 各行が 1 つの研究に対応する。四角形（点推定値）と水平線（95% CI）で構成される
  • 四角形の大きさは当該研究の重み（精度）を反映する。大きい四角形ほどサンプルサイズが大きく精度が高い
  • 垂直参照線（0 の位置）より右に位置する研究は「介入が有利」、左は「統制群が有利」
  • 最下部の菱形（ダイヤモンド）が統合された全体効果であり、菱形の幅が 95% CI に対応する

ファンネルプロット

ファンネルプロットは、出版バイアスの有無を視覚的に検討するための図である。

コード

図の読み方:

  • 横軸が効果量、縦軸が精度（標準誤差；上方ほど精度が高い）
  • バイアスが存在しなければ、大規模研究（上部）は平均効果の近傍に集中し、小規模研究（下部）は広く散布して逆三角形（漏斗型）を形成する
  • 左右非対称が認められる場合、出版バイアスの存在が示唆されるが、非対称性は必ずしも出版バイアスを意味しない。異質性や方法論的質の差異も非対称の原因となりうる

19.3.6 Recipe 6: 異質性の原因の探索――サブグループ分析とメタ回帰

\(I^2\) や \(\tau^2\) が大きい値を示す場合、次の段階として「なぜ研究間で効果にばらつきが生じるのか」を検討する。ここではモデレーター（調整変数）を用いた分析を紹介する。

  • 検出力: サブグループ分析やメタ回帰で実質的な差を検出するには、各群に最低でも 10 件程度の研究が必要とされる。少数の研究に基づいて「差は認められなかった」と結論するのは早計である

19.3.8 Recipe 8: ロバスト分散推定――非独立な効果量への対処

非独立効果量の問題

実際のメタ分析では、1 つの論文が複数のアウトカム（例：語彙テスト、文法テスト、ライティングテスト）について効果量を報告していることが少なくない。これらの効果量は同一の参加者から得られているため統計的に非独立であり、この非独立性を無視すると信頼区間が過度に狭くなり、第一種の過誤率が膨張する。

対処法として主に 2 つのアプローチが存在する。

  1. ロバスト分散推定（RVE）: clubSandwich パッケージの CR2 推定量を用い、クラスタ（= 研究）単位で頑健な標準誤差を算出する方法。研究数が少ない場合にも比較的安定した推定が得られる
  2. 多層モデル（rma.mv()）: metafor の多変量モデルにより、研究内・研究間の分散構造を明示的にモデル化する方法。より柔軟であるが、仮定が増える（Recipe 19 で詳述）

本節のシミュレーションデータでは各研究から 1 効果量のみを抽出しているため両者の差は小さいが、実務への準備として RVE の適用手順を示す。

19.4 PART II: ネットワーク・メタ分析（NMA）

本章の後半では、3 つ以上の介入を同時に比較するネットワーク・メタ分析（Network Meta-Analysis; NMA）を扱う。ペアワイズ・メタ分析が「A vs. B」の 2 群比較を統合するのに対し、NMA は「A vs. B vs. C vs. D」を同時に比較し、直接比較が存在しない介入間の効果も間接推定できる点に特徴がある。

19.4.1 NMA の基本的な考え方

NMA が必要となる状況

たとえば、英語教育研究において以下のような比較研究が存在する場合を考える。

  • Study 1: 統制 vs. グロス指導
  • Study 2: グロス指導 vs. ライティング指導
  • Study 3: 統制 vs. 間隔反復学習 (SRS)
  • Study 4: ライティング指導 vs. SRS

ペアワイズ・メタ分析では各比較を個別に統合するにとどまるが、NMA ではこれらのネットワーク全体を統合し、直接比較が存在しない組み合わせ（例：統制 vs. ライティング指導）の効果も間接推定することが可能となる。加えて、すべての介入のランキングも得られる。

NMA の 2 つの前提

NMA の妥当性は、以下の 2 つの前提に依存する。

  1. 整合性（Consistency）: ある比較について、直接推定と間接推定が統計的に一致すること。たとえば「A vs. B」の直接比較による推定値と、「A vs. C」＋「C vs. B」から間接的に導出された推定値が大きく矛盾しないことを意味する
  2. 推移性（Transitivity）: 異なる比較を行った研究群の参加者特性や研究条件が十分に類似していること。たとえば、グロス vs. 統制の研究が初級学習者のみ、ライティング vs. 統制の研究が上級学習者のみを対象としている場合、推移性の前提が成立しない恐れがある

19.4.2 Recipe 12: NMA のデータ準備

コード

  install.packages("netmeta", repos = "https://cloud.r-project.org")
}

4 つの教授法を比較する 16 件の研究をシミュレーションにより生成する。各研究は 2〜3 群を比較するデザイン（一部が多群試験）である。

コード 
treatments <- c("CTRL", "GLOSS", "WRIT", "SRS")
true_delta <- c(CTRL = 0.00, GLOSS = 0.25, WRIT = 0.45, SRS = 0.60)

make_study <- function(id, arms = sample(2:3, 1)) {
  tr <- sample(treatments, arms, replace = FALSE)
  n  <- sample(30:80, arms, replace = TRUE)
  sd <- runif(arms, 8, 14)
  base <- rnorm(1, 70, 6)
  mu <- sapply(tr, function(t) base + true_delta[t] * mean(sd))
  data.frame(
    study = sprintf("NMA_%02d", id),
    treat = tr, n = n,
    mean = mu + rnorm(arms, 0, 1.0), sd = sd
  )
}

studies <- do.call(rbind, lapply(1:16, make_study))
studies$arm <- ave(studies$treat, studies$study, FUN = seq_along)

wide <- studies |>
  pivot_wider(names_from = arm, values_from = c(treat, n, mean, sd),
              names_sep = "") |>
  arrange(study)
NMA データ整形上の注意
  • 多群試験（3 群以上）は 1 行にまとめる: 各群を別行に分割すると研究内の相関が無視され、誤った推定結果が生じる
  • 介入名の表記統一: 「CTRL」「Control」「統制群」が混在すると、それぞれ異なる介入として処理されてしまう
  • 連結ネットワークの確認: すべての介入が直接的または間接的に比較可能であること（ネットワークが連結していること）が必要条件である

19.4.3 Recipe 13: NMA の適合と結果の解釈

treatments_b <- c("CTRL", "GLOSS", "WRIT", "SRS")
logOR_true <- c(CTRL = 0.00, GLOSS = 0.40, WRIT = 0.70, SRS = 0.90)

make_study_bin <- function(id, arms = sample(2:3, 1)) {
  tr <- sample(treatments_b, arms, replace = FALSE)
  n  <- sample(60:140, arms, replace = TRUE)
  p0 <- plogis(rnorm(1, qlogis(0.45), 0.4))
  p  <- sapply(tr, function(t) plogis(qlogis(p0) + logOR_true[t]))
  ev <- rbinom(arms, size = n, prob = p)
  data.frame(study = sprintf("BIN_%02d", id), treat = tr, n = n, event = ev)
}

Orchard Plot（orchaRd パッケージ）

Orchard plot は、メタ分析の結果を直感的に可視化する比較的新しい手法である(Nakagawa et al., 2021)。各研究の効果量（点）、全体の平均効果（太い線）、95% CI（太い帯）、95% 予測区間（細い線）を 1 つの図に統合して表示する。

コード 
if (requireNamespace("orchaRd", quietly = TRUE)) {
  library(orchaRd)

  # orchaRd は rma.mv オブジェクトを推奨
  # シミュレーションデータで切片のみモデル
  m_orch <- rma.mv(yi, vi,
                   random = ~ 1 | study_id,
                   data = dat,
                   method = "REML")

  orchard_plot(m_orch, mod = "1", group = "study_id", xlab = "Hedges g",
               trunk.size = 3, branch.size = 1.5,
               twig.size = 0.5, k = TRUE)
} else {
  message("orchaRd パッケージが利用できません。GitHub からインストールしてください:\n",
          "remotes::install_github('daniel1noble/orchaRd')")
}

図の読み方:

  • 太い横線: 95% 信頼区間
  • 細い横線: 95% 予測区間（研究間のばらつきを包含）
  • 各点: 個々の研究の効果量（点の大きさは精度を反映）
  • 中央の丸: 全体の平均効果量

予測区間が 0 を含む場合、特定の文脈では効果が存在しないか、あるいは負の方向となる可能性があることを示唆する。

## メタ分析とは何か――なぜ個別の研究では不十分なのか

「明示的な文法指導は英語ライティングの正確性を向上させるか」という研究課題を考える。先行研究を渉猟すると、効果を報告する研究もあれば、そうでない研究もあり、一見して知見が錯綜している。このような状況において、個別の研究結果をどのように統合し、全体的な傾向を把握すればよいのか――これがメタ分析の出発点である。

メタ分析（meta-analysis）とは、同一の研究課題に取り組んだ複数の実証研究の結果を定量的に統合する統計手法である。各研究を独立した「データポイント」として扱い、統計的に合成することで、個々の研究が抱えるサンプルサイズの制約や結果のばらつきを克服し、より信頼性の高い結論を導くことが可能となる。

ナラティブ・レビュー（先行研究を定性的にまとめる従来型の文献検討）と比較した場合、メタ分析には以下の利点がある。

- **定量的な統合**: 「多くの研究で効果が認められた」という質的判断ではなく、「平均効果は *g* = 0.42, 95% CI [0.18, 0.66]」と数値で提示できる
- **異質性の検討**: 効果が研究間でどの程度異なるかを定量的に評価できる（後述する $\tau^2$, $I^2$ などの指標）
- **モデレーター分析**: 「どのような条件下で効果が大きい（あるいは小さい）か」を体系的に探索できる

::: {.callout-note}
### 本章の位置づけ
本章は、メタ分析の実施手順を R の `metafor` パッケージを中心に解説する実践的なガイドである。効果量の基礎概念については「`1_T検定と効果量.qmd`」を参照されたい。本章では、複数の研究から得られた効果量を統合する手続きに焦点を当てる。
:::

:::


メタ分析は、効果量の算出から異質性の評価、モデレーター分析、出版バイアスの検討まで、段階的に進めていきます。異質性の大きさによって次のステップが分岐します。

実務手順としては、まず研究を収集して効果量を算出し、ランダム効果モデルでプール推定を行います。次に \(I^2\)、\(Q\)、\(\tau^2\) で異質性を評価し、異質性が大きい場合はサブグループ分析やメタ回帰でモデレーターを探索します。異質性が小さい場合はプール推定値を中心に報告し、どちらの経路でも最終段階でファンネルプロットや Trim-and-Fill による出版バイアス評価を実施します。最後に PRISMA に準拠して報告します。

::: {.callout-note}
英語教育研究のメタ分析では、介入の種類・学習者レベル・アウトカム指標などが異質性の主な要因となります。異質性が大きい場合（$I^2 > 75\%$）は、必ず**モデレーター分析**で要因を探索してください。
:::

## PART I: ペアワイズ・メタ分析

本章の前半では、最も基本的な形式である「2群の比較を複数の研究から統合する」ペアワイズ・メタ分析を扱う。学位論文や学術論文において最も頻繁に用いられる分析手法である。


### Recipe 1: 効果量の選択――統合の単位を決定する

メタ分析の第一段階は、各研究の結果を共通の尺度、すなわち効果量（effect size）に変換することである。

#### どの効果量を選択すべきか

選択の基準は、アウトカム（従属変数）の測定尺度によって決まる。

| アウトカムの種類 | 典型例 | 効果量 | `escalc()` の `measure` |
|:--|:--|:--|:--|
| 連続値（同一テスト） | TOEFL点数の差 | 平均差 (MD) | `"MD"` |
| 連続値（異なるテスト） | ライティング得点 vs. 文法テスト | 標準化平均差 (Hedges *g*) | `"SMDH"` |
| 二値（成功/失敗） | 合格率 | 対数オッズ比 (log OR) | `"OR"` |
| 相関 | 動機づけと成績の関連 | Fisher *z* | `"ZCOR"` |

英語教育研究では、異なるテストや尺度を用いた研究を統合する場面が多いため、**標準化平均差（Hedges *g*）**が最も広く用いられる。

#### Hedges *g* の定義

Hedges *g* は、2群の平均差をプールされた標準偏差で割った値に、小標本バイアスの補正を施した指標である。

$$
g = J(df) \times \frac{\bar{X}_1 - \bar{X}_2}{S_{\text{pooled}}}
$$

ここで $J(df)$ は小標本補正係数であり、サンプルサイズが小さい場合に効果量がやや過大に推定される傾向を修正する。解釈の目安として、$|g|$ = 0.2（小）、0.5（中）、0.8（大）が広く参照される[@cohen1988_ma]。ただし、この基準は分野や研究文脈によって適切性が異なるため、機械的な適用は避けるべきである。応用言語学においては、@plonsky2014 の分野固有の基準（$d$ = 0.40 で small, 0.70 で medium, 1.00 で large）も参照されたい。

#### 効果量の方向の定義

メタ分析に着手する前に、**正の効果量が何を意味するかを明確に定義する**必要がある。本章では一貫して「正の値 = 介入（明示的文法指導）が統制群に対して有利」と定義する。この定義をコーディングシートの段階で統一しておくことが、分析全体の整合性を確保するうえで不可欠である。


### Recipe 2: データの準備――英語教育研究のシナリオ

#### 分析のシナリオ

以下の仮想的なシナリオに基づき、分析の手順を解説する。

> **リサーチ・クエスチョン**: 明示的な文法指導（Explicit Grammar Instruction）は、EFL/ESLの英語ライティングにおける文法的正確性を向上させるか？

想定する研究の特徴は以下の通りである。

- **介入群**: 明示的文法指導を受けた学習者
- **統制群**: 通常の授業を受けた学習者
- **アウトカム**: ライティングテストの得点（連続値）
- **モデレーター候補**: 教育段階（中等教育 vs. 高等教育）、介入期間（週数）

以下では 30 件の研究をシミュレーションにより生成する。真のパラメータとして、ベースラインの効果量 *g* = 0.35、高等教育で +0.10、介入 1 週あたり +0.01 を設定している。

```{r}
#| label: simulate
n_study <- 30
study_id <- sprintf("Study_%02d", 1:n_study)

# モデレーター
level <- sample(c("secondary", "tertiary"), n_study, replace = TRUE, prob = c(0.6, 0.4))
duration <- sample(4:16, n_study, replace = TRUE)
year <- sample(2010:2024, n_study, replace = TRUE)

# 真の効果（ランダム効果モデル）
# ベース 0.35 + 高等教育ボーナス + 週数効果 + 研究間分散 tau^2 = 0.04
tau <- sqrt(0.04)
theta <- 0.35 + ifelse(level == "tertiary", 0.10, 0) +
  0.01 * (duration - 10) + rnorm(n_study, 0, tau)

# 各研究のサンプルサイズと集計統計量
set.seed(2024)
n1 <- sample(25:60, n_study, replace = TRUE)
n2 <- sample(25:60, n_study, replace = TRUE)
sd_pooled <- runif(n_study, 8, 15)
m2 <- rnorm(n_study, 70, 8)
m1 <- m2 + theta * sd_pooled
sd1 <- sd_pooled * runif(n_study, .9, 1.1)
sd2 <- sd_pooled * runif(n_study, .9, 1.1)

raw <- data.frame(study_id, level, duration, year, m1, sd1, n1, m2, sd2, n2)

# Hedges g と分散を計算
es <- escalc(
  measure = "SMDH",
  m1i = m1, sd1i = sd1, n1i = n1,
  m2i = m2, sd2i = sd2, n2i = n2,
  data = raw
)
dat <- cbind(raw, yi = es$yi, vi = es$vi)
```

`escalc()` は各研究の集計統計量（平均、SD、サンプルサイズ）から効果量 `yi` とその分散 `vi` を算出する。この `yi` と `vi` の対がメタ分析の入力データとなる。

```{r}

  DT::datatable(caption = "最初の10研究の効果量と分散")
```

#### 実データへの適用
実際のメタ分析では、各論文から平均値・標準偏差・サンプルサイズを抽出し、データフレームに格納する。論文が *t* 値や *F* 値のみを報告している場合は、`escalc()` の引数（`ti=`, `fi=` など）や効果量変換パッケージ（`esc`, `effectsize`）、オンラインツール（[Effect Size Calculator](https://www.escal.site/)）を用いて変換が可能である。効果量変換の詳細は Recipe 21 で扱う。

### Recipe 3: ランダム効果モデルによる統合――分析の中核

#### 固定効果モデルとランダム効果モデルの選択

この選択はデータを観察してから決定するものではなく、**研究デザインの性質に基づいて事前に**決定する。

- **固定効果モデル（fixed-effect model）**: すべての研究が同一の真の効果を推定していると仮定する。各研究の重みはサンプルサイズ（精度）のみによって決まる。同一の薬剤を同一条件で検証した治験シリーズなどに適している。
- **ランダム効果モデル（random-effects model）**: 研究ごとに真の効果が異なりうる（真の効果が分布を持つ）と仮定する。英語教育研究では、学習者の年齢・L1 背景・指導者・テスト形式などが研究間で異なるため、**ほぼ常にランダム効果モデルが適切である**。

ランダム効果モデルにおける重み $w_i$ は以下のように定義される。

$$
w_i = \frac{1}{v_i + \hat{\tau}^2}
$$

ここで $v_i$ は各研究の標本分散（サンプリング誤差）、$\hat{\tau}^2$ は研究間分散の推定値である。固定効果モデル（$\hat{\tau}^2 = 0$）と比較すると、小規模な研究に相対的に大きな重みが付与され、信頼区間は広くなる。すなわち、研究間のばらつきに起因する不確実性が推定に反映される。

#### モデルの適合

```{r}
#| label: reml
m <- rma(yi, vi, data = dat, method = "REML")
rma_to_gt(m, title = "ランダム効果モデル（REML）の推定結果")
```

#### 出力の解釈

`rma()` の出力において注目すべきポイントは、大きく 3 点に整理できる。

**1. 平均効果量（estimate）**

- 正の値は介入（明示的文法指導）が統制群に対して有利であることを意味する
- 95% CI が 0 を含まない場合、統計的に有意である
- ただし、**統計的有意性のみならず効果の大きさそのもの**に注目すべきである。*g* = 0.10 であっても大規模なメタ分析では有意となりうるが、教育的に意味のある差であるとは限らない

**2. 異質性の指標**

- **$\tau^2$（タウ二乗）**: 研究間分散の推定値。$\sqrt{\tau^2} = \tau$ は効果量と同じ単位で解釈できる。たとえば $\tau$ = 0.20 であれば、「真の効果量は研究間で *SD* = 0.20 程度のばらつきを持つ」ことを意味する
- **$I^2$**: 観測されたばらつきのうち、真の異質性（サンプリング誤差に帰属しない部分）が占める割合。概ね 25% = 低、50% = 中、75% = 高とされるが、**この基準への過度な依存は避けるべき**である
- **$Q$**: 効果量の同質性を検定する統計量。研究数が多い場合にはほぼ常に有意となるため、$Q$ 検定のみで判断することは適切でない

**3. 予測区間（prediction interval）**

予測区間は「今後同様の研究を実施した場合、効果量がどの範囲に収まるか」を示す区間である。信頼区間が「平均効果の推定精度」を反映するのに対し、予測区間は「研究間のばらつきを含めた効果の実質的な範囲」を表す。教育実践への含意を論じる際には、**予測区間が信頼区間よりも重要な情報を提供する**。

```{r}
#| label: prediction-interval
predict(m)
```

::: {.callout-warning}
#### 解釈における典型的な誤り

1. ***p* 値のみに基づく結論**: 効果量の大きさ・信頼区間・予測区間を必ず併せて報告する
2. **$I^2$ の過大評価・過小評価**: $I^2$ が低くとも予測区間が広い場合がある。$I^2$ は「ばらつきの大きさ」ではなく「ばらつきに占める真の異質性の割合」である
3. **95% CI が 0 を含む = 効果が存在しない**: 正確には「本分析においては統計的に有意な効果は検出されなかった」ということであり、効果の大きさ自体の報告には依然として意義がある
:::

```{r}
       header = "Study / Effect Size [95% CI]")
```

**図の読み方:**

- 各行が 1 つの研究に対応する。**四角形（点推定値）**と**水平線（95% CI）**で構成される
- 四角形の大きさは当該研究の重み（精度）を反映する。大きい四角形ほどサンプルサイズが大きく精度が高い
- **垂直参照線（0 の位置）**より右に位置する研究は「介入が有利」、左は「統制群が有利」
- 最下部の**菱形（ダイヤモンド）**が統合された全体効果であり、菱形の幅が 95% CI に対応する

#### ファンネルプロット

ファンネルプロットは、出版バイアスの有無を視覚的に検討するための図である。

```{r}
#| label: funnel-plot
#| fig-width: 8
#| fig-height: 6
funnel(m, xlab = "Hedges g")
```

**図の読み方:**

- 横軸が効果量、縦軸が精度（標準誤差；上方ほど精度が高い）
- バイアスが存在しなければ、大規模研究（上部）は平均効果の近傍に集中し、小規模研究（下部）は広く散布して**逆三角形（漏斗型）**を形成する
- **左右非対称**が認められる場合、出版バイアスの存在が示唆されるが、**非対称性は必ずしも出版バイアスを意味しない**。異質性や方法論的質の差異も非対称の原因となりうる

#### Egger 検定（ファンネルプロットの非対称性の統計的検定）

```{r}
#| label: egger-test
regtest(m, model = "rma")
```

Egger 検定はファンネルプロットの非対称性を統計的に検定する手法である。*p* < .05 であれば非対称性が示唆されるが、研究数が少ない場合（*k* < 10）は検出力が低いため、結果の解釈には留意が必要である。出版バイアスの評価は、**単一の検定に依拠するのではなく、複数の手法を組み合わせて**行うことが推奨される。

:::

### Recipe 6: 異質性の原因の探索――サブグループ分析とメタ回帰

$I^2$ や $\tau^2$ が大きい値を示す場合、次の段階として「なぜ研究間で効果にばらつきが生じるのか」を検討する。ここではモデレーター（調整変数）を用いた分析を紹介する。

#### サブグループ分析とメタ回帰の使い分け

- **サブグループ分析**: モデレーターが**カテゴリカル変数**（中等教育 vs. 高等教育）の場合に用いる
- **メタ回帰**: モデレーターが**連続変数**（介入期間の週数）の場合に用いる

両者を組み合わせることも可能である。

#### サブグループ分析（教育段階別）

```{r}
#| label: subgroup
m_grp <- rma(yi, vi, mods = ~ factor(level), data = dat, method = "REML")
rma_to_gt(m_grp, title = "サブグループ分析（教育段階別）")
```

`mods = ~ factor(level)` により、教育段階をモデレーターとして投入している。回帰分析の枠組みでは、これは**切片＋ダミー変数の回帰モデル**に相当する。切片が参照カテゴリ（secondary）の平均効果量、回帰係数が tertiary との差分を表す。

各群の平均効果量を直接推定する場合は、切片なしモデルを適合する。

```{r}

rma_to_gt(m_grp0, title = "サブグループ別平均効果量（切片なしモデル）")
```

#### メタ回帰（介入期間）

```{r}
#| label: meta-regression
m_reg <- rma(yi, vi, mods = ~ duration, data = dat, method = "REML")
rma_to_gt(m_reg, title = "メタ回帰（介入期間）")
```

`duration` の回帰係数が正であれば、介入期間が長いほど効果量が大きい傾向を示唆する。

::: {.callout-warning}
#### モデレーター分析における留意事項

- **検出力**: サブグループ分析やメタ回帰で実質的な差を検出するには、各群に最低でも 10 件程度の研究が必要とされる。少数の研究に基づいて「差は認められなかった」と結論するのは早計である
- **生態学的誤謬（ecological fallacy）**: メタ回帰は**研究レベル**の関連を検討している。「介入期間が長い研究ほど効果が大きい」という知見と「個々の学習者が長く学習するほど効果が大きい」という命題は、論理的に等価ではない
- **事前登録**: モデレーター分析は探索的になりやすいため、検討するモデレーターを**事前にプロトコルで登録する**ことが推奨される（PROSPERO 等）
:::

### Recipe 8: ロバスト分散推定――非独立な効果量への対処

#### 非独立効果量の問題

実際のメタ分析では、1 つの論文が複数のアウトカム（例：語彙テスト、文法テスト、ライティングテスト）について効果量を報告していることが少なくない。これらの効果量は同一の参加者から得られているため統計的に非独立であり、この非独立性を無視すると信頼区間が過度に狭くなり、第一種の過誤率が膨張する。

対処法として主に 2 つのアプローチが存在する。

1. **ロバスト分散推定（RVE）**: `clubSandwich` パッケージの CR2 推定量を用い、クラスタ（= 研究）単位で頑健な標準誤差を算出する方法。研究数が少ない場合にも比較的安定した推定が得られる
2. **多層モデル（`rma.mv()`）**: `metafor` の多変量モデルにより、研究内・研究間の分散構造を明示的にモデル化する方法。より柔軟であるが、仮定が増える（Recipe 19 で詳述）

本節のシミュレーションデータでは各研究から 1 効果量のみを抽出しているため両者の差は小さいが、実務への準備として RVE の適用手順を示す。

```{r}
#| label: robust
library(clubSandwich)
cs <- coef_test(m, vcov = "CR2", cluster = dat$study_id)
cs
```

::: {.callout-tip}
#### RVE と通常の推定結果が乖離する場合
RVE を適用すると標準誤差が増大することが多い。これは、非独立性を無視した場合の標準誤差が過度に楽観的であったことを意味する。分析結果の報告においては、RVE による推定値を採用し、通常の推定値との差異について透明性をもって議論することが望ましい。
:::

## PART II: ネットワーク・メタ分析（NMA）

本章の後半では、3 つ以上の介入を同時に比較する**ネットワーク・メタ分析（Network Meta-Analysis; NMA）**を扱う。ペアワイズ・メタ分析が「A vs. B」の 2 群比較を統合するのに対し、NMA は「A vs. B vs. C vs. D」を同時に比較し、直接比較が存在しない介入間の効果も間接推定できる点に特徴がある。


### NMA の基本的な考え方

#### NMA が必要となる状況

たとえば、英語教育研究において以下のような比較研究が存在する場合を考える。

- Study 1: 統制 vs. グロス指導
- Study 2: グロス指導 vs. ライティング指導
- Study 3: 統制 vs. 間隔反復学習 (SRS)
- Study 4: ライティング指導 vs. SRS

ペアワイズ・メタ分析では各比較を個別に統合するにとどまるが、NMA ではこれらのネットワーク全体を統合し、直接比較が存在しない組み合わせ（例：統制 vs. ライティング指導）の効果も**間接推定**することが可能となる。加えて、すべての介入のランキングも得られる。

#### NMA の 2 つの前提

NMA の妥当性は、以下の 2 つの前提に依存する。

1. **整合性（Consistency）**: ある比較について、直接推定と間接推定が統計的に一致すること。たとえば「A vs. B」の直接比較による推定値と、「A vs. C」＋「C vs. B」から間接的に導出された推定値が大きく矛盾しないことを意味する
2. **推移性（Transitivity）**: 異なる比較を行った研究群の参加者特性や研究条件が十分に類似していること。たとえば、グロス vs. 統制の研究が初級学習者のみ、ライティング vs. 統制の研究が上級学習者のみを対象としている場合、推移性の前提が成立しない恐れがある


### Recipe 12: NMA のデータ準備

```{r}
#| label: nma-setup
#| message: false
if (!requireNamespace("netmeta", quietly = TRUE)) {
  install.packages("netmeta", repos = "https://cloud.r-project.org")
}
library(netmeta)
set.seed(2024)
```

4 つの教授法を比較する 16 件の研究をシミュレーションにより生成する。各研究は 2〜3 群を比較するデザイン（一部が多群試験）である。

```{r}
#| label: nma-simulate
treatments <- c("CTRL", "GLOSS", "WRIT", "SRS")
true_delta <- c(CTRL = 0.00, GLOSS = 0.25, WRIT = 0.45, SRS = 0.60)

make_study <- function(id, arms = sample(2:3, 1)) {
  tr <- sample(treatments, arms, replace = FALSE)
  n  <- sample(30:80, arms, replace = TRUE)
  sd <- runif(arms, 8, 14)
  base <- rnorm(1, 70, 6)
  mu <- sapply(tr, function(t) base + true_delta[t] * mean(sd))
  data.frame(
    study = sprintf("NMA_%02d", id),
    treat = tr, n = n,
    mean = mu + rnorm(arms, 0, 1.0), sd = sd
  )
}

studies <- do.call(rbind, lapply(1:16, make_study))
studies$arm <- ave(studies$treat, studies$study, FUN = seq_along)

wide <- studies |>
  pivot_wider(names_from = arm, values_from = c(treat, n, mean, sd),
              names_sep = "") |>
  arrange(study)
```

::: {.callout-warning}
#### NMA データ整形上の注意

- **多群試験（3 群以上）は 1 行にまとめる**: 各群を別行に分割すると研究内の相関が無視され、誤った推定結果が生じる
- **介入名の表記統一**: 「CTRL」「Control」「統制群」が混在すると、それぞれ異なる介入として処理されてしまう
- **連結ネットワークの確認**: すべての介入が直接的または間接的に比較可能であること（ネットワークが連結していること）が必要条件である
:::


### Recipe 13: NMA の適合と結果の解釈

```{r}

               mean  = list(mean1,  mean2,  mean3),
```

### Recipe 16: 二値アウトカムの NMA

二値アウトカム（合格/不合格など）の場合は、オッズ比（OR）や相対リスク（RR）を効果量指標として用いる。

```{r}
#| label: nma-binary
set.seed(2024)
treatments_b <- c("CTRL", "GLOSS", "WRIT", "SRS")
logOR_true <- c(CTRL = 0.00, GLOSS = 0.40, WRIT = 0.70, SRS = 0.90)

make_study_bin <- function(id, arms = sample(2:3, 1)) {
  tr <- sample(treatments_b, arms, replace = FALSE)
  n  <- sample(60:140, arms, replace = TRUE)
  p0 <- plogis(rnorm(1, qlogis(0.45), 0.4))
  p  <- sapply(tr, function(t) plogis(qlogis(p0) + logOR_true[t]))
  ev <- rbinom(arms, size = n, prob = p)
  data.frame(study = sprintf("BIN_%02d", id), treat = tr, n = n, event = ev)
}

studies_b <- do.call(rbind, lapply(1:14, make_study_bin))
studies_b$arm <- ave(studies_b$treat, studies_b$study, FUN = seq_along)

wide_b <- studies_b |>
  pivot_wider(names_from = arm, values_from = c(treat, n, event),
              names_sep = "") |>
  arrange(study)

pw_b <- pairwise(treat = list(treat1, treat2, treat3),
                 event = list(event1, event2, event3),
                 n     = list(n1,     n2,     n3),
                 data = wide_b,
                 studlab = study,
                 sm = "OR")

net_b <- netmeta(TE, seTE, treat1, treat2, studlab, data = pw_b,
                 sm = "OR", method.tau = "REML",
                 reference.group = "CTRL", random = TRUE)

# NMA結果をgt表で表示（vs. CTRL, log OR）
trts_b <- net_b$trts[net_b$trts != net_b$reference.group]
tibble::tibble(
  Treatment = trts_b,
  log_OR    = net_b$TE.random[net_b$reference.group, trts_b],
  CI_lower  = net_b$lower.random[net_b$reference.group, trts_b],
  CI_upper  = net_b$upper.random[net_b$reference.group, trts_b]
) |>
  gt() |>
  tab_header(title = "NMA（二値アウトカム）: ランダム効果推定値（vs. CTRL, log OR）") |>
```

### Recipe 19: 多層モデル（`rma.mv()`）――非独立な効果量への構造的対処

@uchihara2019 のデータでは、同一の研究から複数の効果量が抽出されている（異なるテスト形式、異なるエンカウンター回数など）。このような場合、通常の `rma()` では効果量間の非独立性を適切に処理できない。

多層（マルチレベル）モデル `rma.mv()` を用いることで、**研究内の相関構造**と**研究間のばらつき**を同時にモデル化することが可能となる。

```{r}
#| label: load-uchihara
uchi <- read.csv("data/meta/uchihara2019_clean.csv", stringsAsFactors = FALSE)

# 列名を確認
names(uchi)
```

```{r}
#| label: uchihara-prep
# 効果量（相関 r）を Fisher z に変換
uchi_clean <- uchi |>
  filter(!is.na(ES1..r.) & ES1..r. != "") |>
  mutate(
    r = as.numeric(ES1..r.),
    n = as.numeric(N),
    study_id = Author,
    es_id = row_number()
  ) |>
  filter(!is.na(r) & !is.na(n))

# Fisher z 変換
es_z <- escalc(measure = "ZCOR", ri = uchi_clean$r, ni = uchi_clean$n)
uchi_clean$yi <- as.numeric(es_z$yi)
uchi_clean$vi <- as.numeric(es_z$vi)
```

#### 3水準モデルの適合

```{r}
#| label: multilevel-model
# 3水準モデル: Level 1 = サンプリング誤差, Level 2 = 研究内, Level 3 = 研究間
m_ml <- rma.mv(yi, vi,
               random = ~ 1 | study_id / es_id,
               data = uchi_clean,
               method = "REML")
rma_to_gt(m_ml, title = "3水準多層モデル（Fisher z）")
```

#### 多層モデルの出力の解釈

- **sigma^2.1**（Level 3: 研究間分散）: 異なる研究間での真の効果のばらつき
- **sigma^2.2**（Level 2: 研究内分散）: 同じ研究内の異なる効果量間のばらつき
- **estimate**（全体平均効果）: Fisher z スケール。相関に戻すには `tanh()` を適用

```{r}
#| label: multilevel-interpret
# 全体の平均相関に変換
cat("全体平均 (Fisher z):", round(coef(m_ml), 3), "\n")
cat("全体平均 (r):", round(tanh(coef(m_ml)), 3), "\n")

# 分散成分の割合
total_var <- sum(m_ml$sigma2)
cat("\n分散の内訳:\n")
cat("  研究間:", round(m_ml$sigma2[1] / total_var * 100, 1), "%\n")
cat("  研究内:", round(m_ml$sigma2[2] / total_var * 100, 1), "%\n")
```

#### モデレーター分析（多層モデル）

```{r}
#| label: multilevel-moderator
# テスト形式（Recognition vs. Recall）をモデレーターとして投入

  m_ml_mod <- rma.mv(yi, vi,
                     mods = ~ factor(Test.format),

```

::: {.callout-tip}
#### `rma()` と `rma.mv()` の使い分け

| 状況 | 使うモデル |
|:--|:--|
| 1研究 = 1効果量 | `rma()` で十分 |
| 1研究から複数の効果量 | `rma.mv()` + `random = ~ 1 | study / es_id` |
| 同一研究グループが複数論文 | `rma.mv()` + `random = ~ 1 | lab / study / es_id` |
| 1研究 = 1効果量だが簡易にRVEしたい | `rma()` + `clubSandwich::coef_test()` |

英語教育研究のメタ分析では、1 つの論文から語彙・文法・ライティングなど複数のアウトカムの効果量を抽出することが多いため、**多層モデルの使用が推奨される**。
:::


### Recipe 20: 高度な可視化手法――orchaRd, metaviz, その他

#### Orchard Plot（orchaRd パッケージ）

Orchard plot は、メタ分析の結果を直感的に可視化する比較的新しい手法である[@nakagawa2021]。各研究の効果量（点）、全体の平均効果（太い線）、95% CI（太い帯）、95% 予測区間（細い線）を 1 つの図に統合して表示する。

```{r}
#| label: orchard-plot
#| fig-width: 10
#| fig-height: 6
if (requireNamespace("orchaRd", quietly = TRUE)) {
  library(orchaRd)

  # orchaRd は rma.mv オブジェクトを推奨
  # シミュレーションデータで切片のみモデル
  m_orch <- rma.mv(yi, vi,
                   random = ~ 1 | study_id,
                   data = dat,
                   method = "REML")

  orchard_plot(m_orch, mod = "1", group = "study_id", xlab = "Hedges g",
               trunk.size = 3, branch.size = 1.5,
               twig.size = 0.5, k = TRUE)
} else {
  message("orchaRd パッケージが利用できません。GitHub からインストールしてください:\n",
          "remotes::install_github('daniel1noble/orchaRd')")
}
```

**図の読み方**:

- **太い横線**: 95% 信頼区間
- **細い横線**: 95% 予測区間（研究間のばらつきを包含）
- **各点**: 個々の研究の効果量（点の大きさは精度を反映）
- **中央の丸**: 全体の平均効果量

予測区間が 0 を含む場合、特定の文脈では効果が存在しないか、あるいは負の方向となる可能性があることを示唆する。

#### モデレーター別の Orchard Plot

```{r}

#| fig-width: 10
#| fig-height: 7

  m_orch_mod <- rma.mv(yi, vi,
                       mods = ~ level_f,
                       random = ~ 1 | study_id,
```

### Recipe 21: 効果量の算出と変換――R パッケージの活用

実際のメタ分析では、論文ごとに報告されている統計量の形式が異なることが一般的である。本節では、効果量の算出・変換に特化した R パッケージ群を紹介したうえで、頻出する変換パターンを解説する。

#### 効果量パッケージの比較

メタ分析における効果量の算出には、主に以下の 3 つのパッケージが利用可能である。

| パッケージ | 主な機能 | 入力形式 | 特徴 |
|:--|:--|:--|:--|
| `metafor::escalc()` | 効果量と分散の同時算出 | 集計統計量（*M*, *SD*, *n*, *t*, *F* 等） | メタ分析パイプラインとの直接連携。最も汎用的 |
| `esc` | 多様な統計量からの効果量変換 | *t*, *F*, *r*, *χ²*, *p*, 比率、平均差など | 入力形式ごとに専用関数（`esc_t()`, `esc_f()` 等）を提供。変換の透明性が高い |
| `effectsize` | 既適合モデルからの効果量抽出 | `t.test()`, `aov()`, `lm()` 等のモデルオブジェクト | `easystats` エコシステムの一部。モデルオブジェクトから直接効果量を算出可能 |

以下に各パッケージの使用例を示す。

#### `esc` パッケージによる効果量変換

`esc` パッケージは、論文中に報告された多様な統計量から標準化効果量（Cohen's *d* / Hedges *g*）を算出するための専用関数群を提供する。

```{r}
#| label: esc-examples
library(esc)

# t 値からの変換
esc_t(t = 2.45, totaln = 62, es.type = "g")

# F 値（1自由度）からの変換
esc_f(f = 6.0, totaln = 62, es.type = "g")

# 相関 r からの変換
esc_rpb(r = 0.30, totaln = 80, es.type = "g")

# 平均値・SD からの変換
esc_mean_sd(grp1m = 75.2, grp1sd = 10.1, grp1n = 30,
            grp2m = 70.8, grp2sd = 11.3, grp2n = 32,
            es.type = "g")
```

`esc` パッケージの利点は、各関数が変換の根拠となる数式を明示的に適用する点にある。出力には効果量、標準誤差、信頼区間、サンプルサイズが含まれ、`metafor::escalc()` への入力としてそのまま利用可能である。

#### `effectsize` パッケージによるモデルベースの効果量算出

`effectsize` パッケージは、R の統計モデルオブジェクトから直接効果量を算出する機能を提供する。一次研究の分析において特に有用であるが、メタ分析のための効果量計算にも応用できる。

```{r}
#| label: effectsize-examples
library(effectsize)

# t検定のオブジェクトから効果量を算出
set.seed(2024)
x <- rnorm(30, mean = 75, sd = 10)
y <- rnorm(30, mean = 70, sd = 11)
tt <- t.test(x, y)
hedges_g(x, y)

# Cohen's d と Hedges' g の変換
d_val <- cohens_d(x, y)
as.data.frame(d_val) |>
  gt() |>
  tab_header(title = "Cohen's d の算出結果") |>
  fmt_number(columns = where(is.numeric), decimals = 3)

# r と d の相互変換
d_to_r(d = 0.50)
r_to_d(r = 0.30)
```

::: {.callout-tip}
#### パッケージの使い分けの指針

- **コーディングシートから効果量を一括算出する場合** → `metafor::escalc()` が最適。`measure` 引数でさまざまな効果量指標に対応する
- **論文中の多様な報告形式から個別に変換する場合** → `esc` パッケージの専用関数群が便利。変換過程が透明で再現性が高い
- **自身の一次データの分析で効果量を算出する場合** → `effectsize` パッケージが R のモデルオブジェクトと直接連携できる
- **いずれのパッケージでも対応できない特殊な変換** → オンラインツール（[Effect Size Calculator](https://www.escal.site/)）を参照する
:::

#### `escalc()` による効果量変換の実例

以下では、`metafor::escalc()` を用いた頻出の変換パターンを示す。

##### 相関 *r* から Fisher *z* への変換

```{r}
#| label: es-conversion-r-to-d
# r = 0.30 の場合
r <- 0.30
d_from_r <- 2 * r / sqrt(1 - r^2)
cat("r =", r, "→ d ≈", round(d_from_r, 3), "\n")

# escalc でも可能（Fisher z から逆変換も含む）
es_example <- escalc(measure = "ZCOR", ri = c(0.20, 0.35, 0.10), ni = c(80, 120, 60))

# Fisher z 変換結果
as.data.frame(es_example[, c("yi", "vi")]) |>
  gt() |>
  tab_header(title = "Fisher z 変換結果") |>
  fmt_number(columns = c(yi, vi), decimals = 4)
cat("\n元の r に戻す: tanh(z)\n")
cat("r =", round(tanh(es_example$yi), 3), "\n")
```

##### *t* 値・*F* 値からの効果量算出

```{r}
#| label: es-from-t-f
# t値から d を計算
t_val <- 2.45
n1 <- 30; n2 <- 32
d_from_t <- t_val * sqrt(1/n1 + 1/n2)
cat("t =", t_val, "→ d ≈", round(d_from_t, 3), "\n")

# F値（1自由度）から d を計算
F_val <- 6.0
d_from_F <- sqrt(F_val) * sqrt(1/n1 + 1/n2)
cat("F =", F_val, "→ d ≈", round(d_from_F, 3), "\n")

# escalc で分散を直接計算する方法
# t値から d を計算した後、Hedges' correction で g を求める
J <- 1 - 3 / (4 * (n1 + n2 - 2) - 1)  # Hedges' correction factor
g_from_t <- d_from_t * J
vi_from_t <- (n1 + n2) / (n1 * n2) + g_from_t^2 / (2 * (n1 + n2))
cat("\nHedges g ≈", round(g_from_t, 3),
    ", vi ≈", round(vi_from_t, 4), "\n")
```
